# 4.3.2 RL 기반 에이전트 학습

워크플로우를 탐색하여 좋은 구조를 찾는 방법과는 달리, 강화학습(Reinforcement Learning, RL)은 에이전트 자체의 의사결정 능력을 직접 학습시키는 접근법입니다. 에이전트가 환경과 상호작용하면서 받는 보상 신호를 통해, 어떤 행동이 좋은 결과로 이어지는지를 스스로 학습합니다. 이 방법은 수동으로 설계된 규칙이나 탐색을 통해 발견된 고정 패턴을 사용하는 대신, 에이전트가 다양한 상황에서 유연하게 좋은 계획을 수립하는 능력을 내재화합니다.

GRPO(Group Relative Policy Optimization)는 LLM을 대상으로 한 강화학습 알고리즘 중 하나입니다. 기존의 PPO(Proximal Policy Optimization) 등에 비해 LLM 학습에 더 적합하도록 설계되었습니다. GRPO는 동일한 입력에 대해 여러 샘플을 생성하고, 이 샘플들의 상대적인 품질을 비교하여 정책을 업데이트합니다. 절대적인 보상 값이 아닌 같은 그룹 내의 상대적 순위를 사용하므로, 보상 스케일링 문제가 적고 학습이 더 안정적입니다. Planner 학습에 GRPO를 적용하면, 동일한 목표에 대해 여러 계획을 생성하고 그 중 더 나은 계획을 생성하는 방향으로 학습이 진행됩니다.

자기 진화 온라인 커리큘럼 RL(self-evolving online curriculum RL)은 학습 태스크의 난이도를 에이전트의 현재 수준에 맞게 자동으로 조정하는 방식입니다. 처음에는 간단한 태스크부터 시작하여 에이전트가 기본 능력을 습득하게 하고, 점차 더 복잡한 태스크를 도입하여 능력을 확장합니다. 이때 커리큘럼이 사전에 고정되어 있지 않고, 에이전트의 현재 성능을 실시간으로 평가하여 적절한 난이도의 태스크를 선택합니다. 너무 쉬운 태스크는 학습 효율이 낮고, 너무 어려운 태스크는 보상 신호가 없어 학습이 진행되지 않으므로, 에이전트가 성공과 실패를 적절히 경험할 수 있는 난이도를 유지하는 것이 핵심입니다.

Outcome-Supervised Reward Model(OSRM)은 중간 과정이 아닌 최종 결과만을 기반으로 보상을 제공하는 보상 모델입니다. 과정 기반 보상 모델(Process Reward Model)은 각 단계의 품질을 평가하기 위해 상세한 주석 데이터가 필요하지만, OSRM은 최종 결과의 정확성이나 품질만 평가하므로 학습 데이터를 구하기가 상대적으로 쉽습니다. 에이전트가 어떤 중간 단계를 거쳤든 최종 목표를 달성했는지만 평가하므로, 에이전트는 결과를 달성하기 위해 다양한 경로를 자유롭게 탐색할 수 있습니다. 코드 작성 태스크에서 코드가 모든 테스트 케이스를 통과하는지, 수학 문제에서 최종 답이 맞는지 같은 명확한 성공 기준이 있을 때 OSRM이 잘 작동합니다.

SFT(Supervised Fine-Tuning)→RL 2단계 학습 패턴은 먼저 지도 학습으로 기본 능력을 갖추게 한 후, 강화학습으로 그 능력을 최적화하는 방식입니다. 처음부터 강화학습만으로 학습하면 초기 정책이 너무 무작위적이어서 보상 신호를 거의 받지 못하고 학습이 느립니다. SFT 단계에서 시연 데이터(인간이나 강한 모델이 생성한 좋은 예시)로 학습하면 정책이 합리적인 출발점에서 시작할 수 있습니다. 이후 RL 단계에서는 SFT로 달성한 수준을 초과하여 더 최적화된 행동 패턴을 학습합니다. 이 2단계 접근법은 학습의 안정성과 최종 성능 모두를 높이는 효과적인 전략입니다.

계획-실행 분리 원칙(plan-execute separation principle)은 Planner와 Executor를 별도의 모델로 분리하여 각각 독립적으로 학습시키는 설계입니다. Planner는 목표를 분석하고 실행 계획을 생성하는 역할을 담당하고, Executor는 개별 스텝을 실제로 수행하는 역할을 담당합니다. 이 분리를 통해 계획 수립 능력과 실행 능력을 독립적으로 개선할 수 있고, 같은 Planner에 여러 종류의 Executor를 결합하거나, 반대로 같은 Executor에 다른 Planner를 결합하는 유연한 조합이 가능합니다. 또한 Planner의 RL 학습은 실행 결과를 보상 신호로 사용하므로, Planner가 실행 가능하고 효과적인 계획을 생성하도록 학습됩니다.

도메인 전이(domain transfer)는 한 도메인에서 학습된 계획 능력이 다른 도메인에서도 적용될 수 있는지의 문제입니다. 에이전트가 코드 리뷰 태스크에서 학습한 단계적 분석 능력이 문서 요약 태스크에도 도움이 될 수 있습니다. RL로 학습된 계획 능력의 도메인 전이 가능성은 학습 태스크의 다양성, 학습된 능력의 추상화 수준, 소스 도메인과 타깃 도메인의 유사성에 따라 달라집니다. 충분히 다양한 태스크로 학습하면 특정 도메인에 과적합되지 않고 일반적인 계획 수립 능력을 키울 수 있습니다. 반면 특정 도메인에 집중하여 학습하면 그 도메인에서 더 높은 성능을 얻지만 다른 도메인으로의 전이가 어려울 수 있습니다.

피드백 루프(feedback loop)는 에이전트가 실행 결과로부터 학습하는 순환 구조입니다. 에이전트가 계획을 실행하고, 그 결과가 보상 신호로 변환되어 정책 업데이트에 사용됩니다. 온라인 RL에서 이 루프는 실시간으로 진행되며, 에이전트는 새로운 경험을 축적하면서 지속적으로 능력을 향상시킵니다. 피드백 루프의 품질은 보상 신호의 정확성에 달려 있습니다. 보상이 태스크의 진정한 목표를 반영하지 못하면 에이전트는 보상을 극대화하는 방향으로 학습하지만 실제로는 원하는 행동을 하지 않는 보상 해킹(reward hacking) 현상이 발생할 수 있습니다.

RL 기반 에이전트 학습의 실제 적용에서 주요 도전 과제는 샘플 효율성입니다. LLM을 기반으로 한 에이전트는 한 번의 에피소드가 많은 LLM 추론 호출을 포함하므로, 충분한 학습 데이터를 수집하는 비용이 높습니다. 이를 완화하기 위해 경험 재사용(experience replay), 오프라인 RL 데이터와 온라인 RL 데이터의 혼합, 합성 데이터 활용 등의 전략이 사용됩니다. 또한 보상 신호 설계는 RL의 성공에 결정적인 역할을 하므로, 태스크의 성공 기준을 명확하게 정의하고 이를 측정 가능한 보상 함수로 변환하는 작업이 중요합니다.

RL 기반 학습의 또 다른 중요한 측면은 안전성입니다. 에이전트가 실제 환경에서 RL을 통해 학습할 때, 탐색 과정에서 바람직하지 않은 행동을 할 수 있습니다. 이를 방지하기 위해 시뮬레이션 환경에서 먼저 학습하거나, 학습 중 특정 행동을 제한하는 안전 제약을 적용하거나, 인간의 피드백을 통해 바람직한 행동의 범위를 정의하는 방법이 사용됩니다.

정리하면, RL 기반 에이전트 학습은 GRPO와 같은 알고리즘으로 Planner의 계획 수립 능력을 직접 최적화하며, 자기 진화 커리큘럼은 에이전트의 현재 수준에 맞는 난이도로 학습을 지속합니다. SFT→RL 2단계 패턴은 안정적인 출발점에서 시작하여 성능을 극대화하고, 계획-실행 분리는 두 능력을 독립적으로 개선할 수 있게 합니다. OSRM은 최종 결과 기반의 보상 신호로 다양한 에이전트 학습에 적용 가능하며, 피드백 루프를 통한 지속적 개선이 RL 기반 학습의 핵심 메커니즘입니다.

다음 단원(4.4.1)에서는 에이전트의 구성과 역할을 사람이 설계하는 것이 아니라 진화적 방법으로 자동 생성하는 방식, 즉 진화적 다중 에이전트 생성을 살펴봅니다.

이 단원을 학습한 후에는 GRPO 기반 Planner 학습의 원리를 설명하고, SFT→RL 2단계 학습 패턴을 적용한 에이전트 설계를 이해하며, 자기 진화 커리큘럼과 OSRM을 활용한 학습 시스템을 구성할 수 있습니다.
