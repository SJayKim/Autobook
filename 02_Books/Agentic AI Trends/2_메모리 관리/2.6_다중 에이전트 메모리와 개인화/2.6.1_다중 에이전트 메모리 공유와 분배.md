# 2.6.1 다중 에이전트 메모리 공유와 분배

2.5절에서 메모리가 시간에 따라 진화하는 메커니즘을 살펴보았습니다. 지금까지 다룬 메모리 기법들은 주로 단일 에이전트를 전제로 한 것이었습니다. 그러나 1.2절에서 다룬 것처럼 실제 시스템에서는 여러 에이전트가 협업합니다. 이 단원에서는 다중 에이전트 시스템(MAS)에서 메모리를 어떻게 공유하고 분배하는지를 다룹니다.

MAS에서 메모리 관리가 단일 에이전트와 근본적으로 다른 이유는 **규모 문제**에 있습니다. 여러 에이전트가 협업하면 상호작용 궤적의 양이 단일 에이전트 대비 10배 이상으로 폭증합니다. 에이전트 A가 에이전트 B에게 요청하고, B가 C에게 위임하고, C의 결과를 B가 종합하여 A에게 반환하는 과정에서 발생하는 발화의 총량은 단일 에이전트의 단독 작업과 비교할 수 없을 정도로 많습니다. 이 방대한 궤적을 단일 에이전트용 메모리 시스템에 그대로 넣으면 어떻게 될까요.

실험 결과는 직관에 반합니다. **단일 에이전트 메모리를 MAS에 직접 이식했을 때 성능이 오히려 하락**합니다. 역할과 무관한 정보가 각 에이전트에게 노출되어 판단을 혼란시키기 때문입니다. 코딩 에이전트에게 디자인 토론의 세부 내용이, 리뷰 에이전트에게 코드 디버깅의 시행착오가 그대로 전달되면, 각 에이전트가 자신의 역할에 집중하지 못합니다. 한 연구에서 단일 에이전트 메모리를 MAS에 이식했을 때 2.6%에서 10.4%까지 성능이 하락했습니다. 이것은 MAS에는 MAS 전용의 메모리 설계가 필요하다는 것을 보여줍니다.

MAS 메모리의 핵심 원칙은 **역할 인식 메모리 개인화(role-aware memory personalization)**입니다. 1.2.2에서 역할 기반 전략을 다루면서, 각 에이전트가 자신의 역할 범위 내에서만 행동하도록 설계한다고 설명했습니다. 메모리도 같은 원칙을 따라야 합니다. 팀 전체의 상호작용 기록은 공유 메모리에 저장하되, 각 에이전트가 검색할 때는 자신의 역할에 관련된 정보만 선택적으로 수신합니다. 2.4.2에서 다룬 역할 인식 필터링이 이 원칙의 검색 측면 구현입니다.

공유와 개인화를 동시에 달성하는 구체적 메커니즘을 살펴보겠습니다. 2.2.2에서 다계층 그래프 구조의 세 계층(Interaction, Query, Insight)을 소개했습니다. MAS 메모리에서 이 구조는 다음과 같이 활용됩니다. **Insight(추상화된 교훈) 공유**에서는 여러 쿼리에서 귀납된 고수준 전략을 팀 전체가 공유합니다. "이 유형의 버그는 대부분 타입 불일치에서 발생한다"는 교훈은 코딩 에이전트뿐 아니라 리뷰 에이전트에게도 유용하므로 공유 계층에 저장됩니다. **Interaction 기록의 역할별 선택적 노출**에서는 에이전트 발화 수준의 상세 기록을 역할에 따라 필터링합니다. 코딩 에이전트가 검색하면 코드 관련 Interaction만, 리뷰 에이전트가 검색하면 품질 관련 Interaction만 반환됩니다. 이 분리를 통해 각 에이전트가 자신의 역할에 필요한 깊이의 정보를 얻으면서도, 팀 전체의 전략적 교훈은 공유합니다.

메모리를 효율적으로 관리하기 위해 **다중 에이전트 병렬 메모리 업데이트**가 사용됩니다. 각 메모리 유형(에피소드, 의미, 절차 등)을 전담하는 메모리 매니저 에이전트가 병렬로 업데이트를 수행합니다. 에피소드 메모리 매니저는 대화 이력을, 의미 메모리 매니저는 사실 정보를, 절차 메모리 매니저는 워크플로우를 각각 독립적으로 관리합니다. 이 병렬화로 메모리 업데이트 지연을 줄입니다.

이 병렬 매니저들을 조율하는 것이 **Meta Memory Manager 패턴**입니다. 사용자의 입력이 들어오면 Meta Memory Manager가 입력을 분석하여 관련 메모리 유형을 판단하고, 해당 메모리 매니저에게 라우팅합니다. "어제 논의한 API 설계에서 인증 부분을 다시 확인해 줘"라는 요청이 들어오면, Meta Manager는 이것이 에피소드 메모리(어제의 논의)와 의미 메모리(API 인증 스펙) 모두에 관련된다고 판단하여, 두 매니저에게 동시에 검색을 요청합니다.

이 설계의 이론적 기반이 **조직 기억 이론(Organizational Memory Theory)**입니다. 경영학에서 개인이 아닌 조직 수준에서 지식을 관리하는 이론으로, 조직의 기억은 개인 기억의 단순 합이 아니라 조직 구조와 프로세스에 의해 형성되는 독자적 실체라는 관점입니다. MAS에서도 마찬가지입니다. 팀의 메모리는 개별 에이전트 메모리의 단순 합이 아니라, 역할 구조, 공유 메커니즘, 필터링 규칙에 의해 형성되는 조직 수준의 인지 자산입니다. 1.2.3에서 다룬 SOP 기반 협업에서 구조화된 산출물이 역할 간 지식 전달의 매개체 역할을 하는 것도 조직 기억의 한 형태입니다.

정리하면, MAS 메모리는 단일 에이전트 메모리의 단순 확장이 아니라 별도의 설계가 필요합니다. 역할 인식 메모리 개인화가 핵심 원칙이며, Insight 공유와 Interaction의 역할별 선택적 노출로 공유와 개인화를 동시에 달성합니다. 다중 에이전트 병렬 메모리 업데이트와 Meta Memory Manager 패턴이 효율적인 관리를 제공하며, 조직 기억 이론이 이 설계의 이론적 기반입니다.

다음 단원인 2.6.2에서는 개별 사용자에 대한 장기 개인화를 다루며, 사용자 선호 벡터, 선호 변화 감지, 세션 간 개인화의 메커니즘을 살펴봅니다.

이 단원을 마치면 단일 에이전트 메모리를 MAS에 직접 이식할 때의 역효과를 설명하고, 역할 인식 개인화, Insight 공유, Meta Memory Manager 패턴 등 MAS 전용 메모리 설계의 원리를 기술할 수 있습니다.
