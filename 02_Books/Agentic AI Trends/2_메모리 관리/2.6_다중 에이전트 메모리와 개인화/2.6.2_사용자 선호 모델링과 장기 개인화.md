# 2.6.2 사용자 선호 모델링과 장기 개인화

2.6.1에서 다중 에이전트 환경의 메모리 공유와 분배를 살펴보았습니다. 이 단원에서는 개별 사용자에 대한 장기 개인화를 다룹니다. 에이전트가 사용자와 오랜 기간 상호작용하면서 사용자의 선호, 습관, 스타일을 점진적으로 파악하고, 이를 바탕으로 응답을 맞춤화하는 메커니즘입니다.

사용자 개인화의 출발점은 사용자의 선호를 정량적으로 표현하는 것입니다. **다차원 사용자 선호 벡터**는 사용자의 선호를 여러 차원으로 분해하여 각 차원에 수치 값을 부여합니다. 대표적인 차원으로는 어조 스타일(공식적/비공식적), 응답 길이(간결/상세), 감정 톤(중립적/친근한), 정보 밀도(개괄적/기술적), 격식 수준(캐주얼/포멀) 등이 있습니다. 예를 들어 어떤 사용자의 선호 벡터가 [어조=공식적, 길이=상세, 톤=중립, 밀도=기술적, 격식=포멀]이라면, 에이전트는 이 벡터를 참조하여 해당 사용자에게 공식적이고 상세하며 기술적인 응답을 생성합니다. 이 벡터는 고정되지 않고 상호작용에 따라 계속 갱신됩니다.

선호 벡터를 갱신하는 데는 두 가지 시간 척도의 추정이 필요합니다. **Sliding Window(SW) 선호 추정**은 최근 W턴의 이동 평균으로 단기 선호 변동을 포착합니다. 사용자가 최근 5번의 대화에서 간결한 응답을 선호했다면, SW 추정은 이 단기 변화를 빠르게 반영합니다. 반면 **Exponential Moving Average(EMA) 선호 추정**은 지수 이동 평균으로 장기 선호 추세를 포착합니다. 사용자가 전반적으로 상세한 응답을 선호하지만 최근에만 간결한 응답을 원했다면, EMA는 장기적 경향이 여전히 "상세"임을 반영합니다.

이 두 추정을 결합하는 것이 **SW-EMA 이중 시간 스케일 선호 융합**입니다. 수학적으로 칼만 필터(Kalman filter)에 대응하는 최소 분산 추정으로, 단기 변동(SW)과 장기 추세(EMA)를 최적 비율로 결합합니다. 이를 통해 사용자의 선호가 일시적으로 변한 것인지, 근본적으로 바뀐 것인지를 구분할 수 있습니다.

사용자의 선호가 급격하게 변하는 시점을 감지하는 것이 **선호 변화 감지(Preference Change Detection)** 메커니즘입니다. SW 추정값과 EMA 추정값 사이의 차이를 임계값과 비교합니다. 차이가 임계값을 초과하면 "사용자의 선호가 급격히 변했다"고 판단합니다. 예를 들어 평소 상세한 응답을 선호하던 사용자가 갑자기 간결한 응답을 연속으로 요청하면, SW(간결)와 EMA(상세) 사이의 차이가 커져 변화 감지가 트리거됩니다. 이 시점에서 에이전트는 프롬프트를 재작성하여 새로운 선호를 즉시 반영합니다.

감지된 선호를 에이전트 행동에 반영하는 방식이 **선호 가이드 프롬프팅(Preference-Guided Prompting)**입니다. 추정된 선호 벡터를 자연어 지시로 변환하여 시스템 프롬프트에 삽입합니다. [어조=공식적, 길이=상세]라는 벡터가 "이 사용자는 공식적이고 상세한 응답을 선호합니다. 기술적 세부사항을 포함하여 답변하세요."라는 자연어 지시로 변환됩니다. 이 방식은 모델의 파라미터를 수정하지 않으므로 모델 불가지론적(model-agnostic)입니다. 어떤 LLM이든 시스템 프롬프트를 지원하면 적용할 수 있습니다.

이 모든 메커니즘이 작동하는 시간 범위가 **세션 간 개인화(Cross-Session Personalization)**입니다. 하나의 대화 세션 내에서만 사용자를 파악하는 것이 아니라, 여러 세션에 걸쳐 사용자 특성을 일관되게 유지하고 다음 세션에 활용합니다. 2.2.1에서 다룬 LTM(Long-Term Memory)이 세션 간 개인화의 저장 기반입니다. 사용자의 선호 벡터, 개인 사실(이름, 직업, 제약 조건), 과거 대화의 핵심 내용이 LTM에 저장되어, 새 세션이 시작될 때 자동으로 로드됩니다.

사용자 프로필의 동적 갱신도 세션 간 개인화의 일부입니다. 대화가 진행될수록 사용자에 대한 이해가 심화되며 프로필이 지속적으로 업데이트됩니다. 초기 세션에서는 "사용자가 프로그래머이다" 수준이었던 이해가, 세션이 축적되면서 "사용자는 Python 백엔드 개발자로 FastAPI를 주로 사용하며, 코드 리뷰 시 성능보다 가독성을 우선시한다" 수준으로 심화됩니다.

민감한 정보의 관리를 위한 **Knowledge Vault**도 개인화 메모리의 한 부분입니다. 자격 증명, 주소, API 키 등 민감한 원문(verbatim) 정보를 보안 등급별로 관리하는 특수 메모리 유형입니다. 일반 메모리와 달리 암호화 저장, 접근 권한 제어, 감사 로그 등 보안 메커니즘이 추가됩니다. 2.7.2에서 다룰 메모리 프라이버시 문제와 직접 연결됩니다.

정리하면, 사용자 선호 모델링은 다차원 선호 벡터로 사용자의 선호를 정량화하고, Sliding Window와 EMA의 이중 시간 스케일 융합으로 단기 변동과 장기 추세를 동시에 추적합니다. 선호 변화 감지는 SW와 EMA의 차이를 임계값과 비교하여 급격한 변화 시점을 포착하고, 선호 가이드 프롬프팅이 감지된 선호를 모델 불가지론적 방식으로 에이전트 행동에 반영합니다. 세션 간 개인화는 LTM을 기반으로 사용자 프로필을 여러 세션에 걸쳐 유지하고 심화시킵니다.

다음 단원인 2.7.1에서는 지금까지 다룬 메모리 시스템의 능력을 어떻게 측정하는지, 메모리 평가의 핵심 역량과 주요 벤치마크를 살펴봅니다.

이 단원을 마치면 다차원 사용자 선호 벡터와 SW-EMA 이중 시간 스케일 융합의 동작 원리를 설명하고, 선호 변화 감지와 선호 가이드 프롬프팅이 장기 개인화를 구현하는 메커니즘을 기술할 수 있습니다.
