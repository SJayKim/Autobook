# 2.5.1 메모리 업데이트와 통합

2.4절에서 저장된 메모리를 검색하는 기법을 살펴보았습니다. 메모리가 한 번 저장된 뒤 영원히 고정된 상태로 남아 있다면, 시간이 지남에 따라 메모리의 내용이 현실과 괴리됩니다. 사용자의 선호가 변하고, 환경이 달라지며, 에이전트가 새로운 경험을 축적합니다. 메모리는 살아 있는 시스템으로, 지속적으로 갱신되어야 합니다. 이 단원에서는 기존 메모리에 새로운 정보가 통합되는 과정과, 이 과정에서 발생하는 문제를 다룹니다.

메모리 진화(evolution)의 기본 과정을 수식으로 표현하면 M_{t+1} = E(M_t, M^{form}_{t+1})입니다. 2.3.1에서 다룬 형성 함수가 새로운 메모리 후보를 생성하는 것이라면, 진화 함수 E는 그 새로운 메모리를 기존 메모리에 통합하는 것입니다. 단순히 새 항목을 추가(append)하는 것이 아니라, 기존 메모리의 키워드, 태그, 문맥 설명, 링크까지 함께 갱신하는 재구조화 작업입니다. **LLM 기반 자율적 메모리 업데이트**에서는 새 경험이 들어올 때 LLM이 기존 메모리와의 관계를 분석하여 인접 메모리의 메타데이터를 자동으로 갱신합니다.

메모리 업데이트에서 가장 간단한 접근은 전체 재작성(full rewrite)입니다. 기존 메모리의 내용을 새로운 정보를 반영하여 처음부터 다시 작성하는 것입니다. 그러나 이 방식에는 심각한 문제가 있습니다. **Context Collapse**라는 현상입니다. 반복적인 전체 재작성이 축적된 지식을 짧은 요약으로 압축하여, 갑작스러운 성능 저하를 유발하는 것입니다. 예를 들어 에이전트가 100회의 상호작용에서 축적한 세밀한 사용자 선호 정보가, 전체 재작성을 거치면서 "사용자는 기술에 관심이 많다"는 한 줄 요약으로 압축될 수 있습니다. 재작성할 때마다 정보가 손실되며, 이 손실이 누적되면 메모리의 정보량이 급격히 감소합니다.

Context Collapse를 방지하는 접근이 **점진적 델타 업데이트(incremental delta update)**입니다. 전체를 재작성하는 대신, 변경된 부분만 소규모로 수정합니다. 기존 지식의 대부분을 그대로 보존하면서 새로운 정보만 추가하거나 수정합니다. 이 방식은 축적된 지식의 손실을 최소화합니다.

점진적 업데이트의 체계적 구현이 **Grow-and-Refine 패턴**입니다. "성장(Grow)"은 새로운 bullet(구조화된 정보 항목)을 메모리에 추가하는 것이고, "정제(Refine)"는 중복되거나 구식이 된 bullet을 제거하는 것입니다. 이 두 연산을 균형 있게 수행함으로써 메모리가 성장하되 비대해지지 않게 관리합니다. 2.2.3에서 다룬 구조화된 bullet 컬렉션의 helpful/harmful 카운터가 정제 결정을 안내합니다.

Grow-and-Refine을 더 정교하게 구현한 것이 **Generator-Reflector-Curator 3역할 분리**입니다. 세 가지 역할이 메모리 업데이트의 서로 다른 측면을 담당합니다. **Generator(생성자)**는 새로운 경험에서 메모리 후보를 생성합니다. 2.3.1에서 다룬 메모리 생성 과정이 여기에 해당합니다. **Reflector(반성자)**는 새로운 경험에서 교훈과 인사이트를 추출합니다. 단순히 경험을 기록하는 것이 아니라, "이 경험에서 배울 수 있는 추상적 원칙은 무엇인가"를 분석합니다. 1.1.2에서 다룬 Self-Reflection의 메모리 시스템 적용입니다. **Curator(큐레이터)**는 Generator가 생성하고 Reflector가 정제한 새 항목을 기존 컨텍스트에 병합합니다. 중복을 제거하고, 모순되는 정보를 해결하며, 전체 메모리의 일관성을 유지합니다.

이 3역할 분리가 필요한 이유는, 경험 생성, 교훈 추출, 컨텍스트 병합이 서로 다른 기술을 요구하기 때문입니다. 생성은 정보의 정확한 포착이 중요하고, 반성은 추상화 능력이 중요하며, 큐레이션은 기존 메모리와의 일관성 판단이 중요합니다. 이 세 기능을 단일 프롬프트로 한꺼번에 수행하면 각 기능의 품질이 저하됩니다. 역할을 분리하면 각 단계에서 최적화된 프롬프트를 사용할 수 있어 전체적인 메모리 업데이트 품질이 향상됩니다.

메모리 업데이트에서 주의해야 할 또 다른 현상이 **Brevity Bias**입니다. 프롬프트 최적화 과정에서 에이전트의 지시문이 점점 간결하고 범용적인 방향으로 수렴하여, 도메인별 세부 지식이 손실되는 경향입니다. "데이터 전처리 시 결측값 확인, 이상치 처리, 정규화 순서를 따르라"는 구체적 지시가 "데이터를 적절히 전처리하라"로 축약되는 것이 예입니다. Brevity Bias는 Context Collapse와 유사한 메커니즘이지만, 메모리 내용이 아니라 메모리의 활용 지시가 축약되는 차이가 있습니다. 점진적 델타 업데이트와 Curator 역할이 이 편향을 완화합니다.

정리하면, 메모리 업데이트와 통합은 새로운 경험이 기존 메모리에 반영되는 과정으로, 전체 재작성은 Context Collapse를 유발하므로 점진적 델타 업데이트가 권장됩니다. Grow-and-Refine 패턴은 새 정보 추가와 중복 제거를 균형 있게 수행하며, Generator-Reflector-Curator 3역할 분리는 경험 생성, 교훈 추출, 컨텍스트 병합의 세 기능을 각각 최적화합니다. Brevity Bias에 대한 경계도 필요합니다.

다음 단원인 2.5.2에서는 메모리가 성장하는 방향과 반대인, 더 이상 유효하지 않은 메모리를 삭제하고 감쇠시키는 망각과 삭제 전략을 살펴봅니다.

이 단원을 마치면 메모리 진화의 공식화와 점진적 델타 업데이트 전략을 설명하고, Context Collapse 문제와 이를 방지하는 Generator-Reflector-Curator 패턴의 동작 과정을 기술할 수 있습니다.
