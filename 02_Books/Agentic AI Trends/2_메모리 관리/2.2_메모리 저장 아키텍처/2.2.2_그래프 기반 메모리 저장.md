# 2.2.2 그래프 기반 메모리 저장

2.2.1에서 시간 범위에 따른 계층적 메모리 저장 구조를 살펴보았습니다. 계층적 구조는 정보의 시간적 중요도를 관리하는 데 효과적이지만, 정보 간의 의미적 관계를 명시적으로 표현하지는 못합니다. "사용자가 Python을 선호한다"는 사실과 "사용자가 데이터 분석 프로젝트를 진행 중이다"라는 사실이 각각 독립적으로 저장되면, 이 둘이 관련되어 있다는 것은 검색 시점에 LLM이 추론해야 합니다. 이 단원에서는 정보를 노드와 엣지로 연결하여 관계를 명시적으로 표현하는 그래프 기반 메모리 저장을 다룹니다.

그래프 기반 메모리 저장의 출발점은 **지식 그래프(Knowledge Graph)** 구조입니다. 지식 그래프에서 개념은 노드로, 개념 간의 관계는 엣지로 표현됩니다. "사용자 --선호언어--> Python", "프로젝트 --사용도구--> pandas"처럼 삼중항(triple) 형태로 정보가 저장됩니다. 2.1.2에서 Planar(2D) 메모리로 분류한 것이 바로 이 형태입니다. 지식 그래프의 강점은 관계 기반 검색이 가능하다는 것입니다. "사용자가 선호하는 언어와 관련된 도구는?"이라는 질문에 "Python" 노드에서 연결된 엣지를 따라가면 "pandas", "numpy" 같은 관련 개념에 도달할 수 있습니다. 평면적 벡터 검색으로는 이런 구조적 탐색이 어렵습니다.

에이전트 메모리에서 지식 그래프를 활용하는 가장 체계적인 방법론 중 하나가 **Zettelkasten 방법론**의 적용입니다. Zettelkasten은 20세기 독일의 사회학자 니클라스 루만이 고안한 지식 관리 체계로, 원자적 노트(atomic note)와 유연한 링크(flexible link)로 지식을 조직합니다. 각 노트는 하나의 아이디어만 담으며, 노트 간의 관계는 링크로 표현됩니다. 이 방법론이 에이전트 메모리에 적용되면, 각 메모리 항목이 하나의 노트가 되고, 메모리 항목 간의 관계가 링크가 됩니다.

구체적으로, 구조화된 노트 형식의 에이전트 메모리에서 각 메모리 항목은 여러 속성을 갖습니다. **내용(content)**은 메모리의 핵심 정보입니다. **타임스탬프(timestamp)**는 메모리가 생성된 시점입니다. **키워드(keywords)**는 메모리의 핵심 용어들입니다. **태그(tags)**는 메모리의 분류 레이블입니다. **문맥 설명(context description)**은 메모리가 형성된 상황에 대한 설명입니다. **임베딩(embedding)**은 의미 검색을 위한 벡터 표현입니다. **링크(links)**는 다른 메모리 항목과의 관계입니다. 이 다속성 구조 덕분에 검색 시 키워드 매칭, 의미 유사도, 관계 기반 탐색을 모두 활용할 수 있습니다.

링크의 역할이 특히 중요합니다. 새로운 메모리가 추가될 때 LLM이 기존 메모리들과의 관계를 분석하여 자동으로 링크를 생성합니다. "Python을 사용한 데이터 전처리 경험"이라는 새 메모리가 추가되면, 기존의 "Python 선호" 메모리와 "데이터 분석 프로젝트" 메모리에 각각 링크가 연결됩니다. 이렇게 메모리 간 링크가 축적되면서 네트워크가 성장하고, 여러 에피소드에 걸쳐 반복되는 패턴이 자연스럽게 드러납니다. 이것이 **고차 패턴 발견(higher-order pattern discovery)**입니다. 개별 메모리 항목에서는 보이지 않던 추상적 패턴이 링크 구조의 분석을 통해 발견됩니다.

계층적 메모리와 그래프 기반 메모리를 결합한 더 복잡한 구조도 있습니다. **다계층 그래프 구조**에서는 서로 다른 추상화 수준의 정보를 별도의 그래프 계층에 배치하고, 계층 간 연결을 통해 정보를 조직합니다. 대표적인 구현에서는 세 개의 그래프 계층이 사용됩니다. **Interaction Graph**는 가장 하위 계층으로, 에이전트 간 또는 에이전트-사용자 간의 개별 발화를 노드로 하는 그래프입니다. 구체적인 대화 내용이 시간순으로 기록됩니다. **Query Graph**는 중간 계층으로, 처리된 쿼리나 태스크를 노드로 하는 그래프입니다. 여러 Interaction을 하나의 쿼리로 묶어 중간 수준의 추상화를 제공합니다. **Insight Graph**는 최상위 계층으로, 여러 쿼리에서 귀납된 고수준 교훈이나 전략을 노드로 하는 그래프입니다. "이 유형의 문제에서는 A 접근이 B 접근보다 효과적이다"와 같은 추상화된 지식이 여기에 위치합니다.

이 세 계층 사이에는 **하이퍼엣지(hyperedge)**가 존재합니다. 하이퍼엣지는 일반 엣지가 두 노드를 연결하는 것과 달리, 여러 Insight 간의 관계를 쿼리 컨텍스트와 함께 인코딩하는 복합 엣지입니다. 하이퍼엣지를 통해 "이 인사이트들이 어떤 쿼리 맥락에서 함께 관련되었는가"를 기록할 수 있습니다.

다계층 그래프 구조가 특히 효과적인 것은 다중 에이전트 시스템에서의 메모리 관리입니다. 단일 에이전트 시스템에서는 상호작용 궤적의 양이 관리 가능한 수준이지만, 여러 에이전트가 협업하는 시스템에서는 궤적의 양이 단일 에이전트 대비 10배 이상으로 폭증합니다. 이 방대한 궤적을 평면적으로 저장하면 검색 효율이 급격히 저하됩니다. 다계층 그래프는 구체적인 상호작용(하위), 태스크 수준 요약(중간), 추상적 교훈(상위)으로 정보를 조직하여, 검색 시 필요한 추상화 수준에서 정보를 효율적으로 인출할 수 있게 합니다. 역할 인식 필터링을 통해 각 에이전트의 역할에 맞게 관련 정보만 선택적으로 제공하는 것도 그래프 구조에서 자연스럽게 구현됩니다. 이 주제는 2.6.1에서 다중 에이전트 메모리를 다룰 때 더 자세히 살펴봅니다.

그래프 기반 메모리의 장점을 정리하면, 첫째 정보 간 관계를 명시적으로 표현하여 관계 기반 검색이 가능합니다. 둘째 링크가 축적되면서 고차 패턴이 자연스럽게 발견됩니다. 셋째 다계층 구조를 통해 서로 다른 추상화 수준의 정보를 효율적으로 조직할 수 있습니다. 단점으로는, 구축과 유지에 더 많은 계산 비용이 듭니다. 새 메모리가 추가될 때마다 기존 메모리와의 관계를 분석하고 링크를 생성해야 하므로, LLM 호출이 추가됩니다. 또한 그래프가 커질수록 탐색 경로가 폭발적으로 증가할 수 있어, 효율적인 그래프 탐색 알고리즘이 필요합니다. 실제 시스템에서는 토큰 효율성과 관계 표현력 사이의 균형을 고려하여, 중요한 관계만 명시적으로 저장하고 나머지는 임베딩 기반 유사도 검색에 맡기는 하이브리드 접근이 흔합니다.

정리하면, 그래프 기반 메모리 저장은 지식 그래프의 노드-엣지 구조를 활용하여 정보 간 관계를 명시적으로 표현합니다. Zettelkasten 방법론에서 영감을 받은 구조화된 노트 형식은 내용, 타임스탬프, 키워드, 태그, 문맥 설명, 임베딩, 링크의 다속성 메모리를 제공하며, 링크의 축적이 고차 패턴 발견을 가능하게 합니다. Interaction-Query-Insight 3계층 그래프와 하이퍼엣지는 서로 다른 추상화 수준의 정보를 효율적으로 조직하며, 특히 다중 에이전트 환경에서 방대한 상호작용 궤적을 관리하는 데 효과적입니다.

다음 단원인 2.2.3에서는 메모리의 최소 단위를 어떻게 설정하는지, 턴, 세그먼트, 세션, 요약 등 메모리 세분성 스펙트럼의 각 단위와 최적 세분성 선택의 트레이드오프를 살펴봅니다.

이 단원을 마치면 지식 그래프 기반 메모리와 Zettelkasten 방법론이 에이전트 메모리에 적용되는 원리를 설명하고, 다계층 그래프 구조의 설계 이유와 동작 방식을 기술할 수 있습니다.
