# 5.3.2 멀티턴 Credit Assignment

강화학습에서 에이전트가 여러 단계를 거쳐 행동할 때, 최종 결과에 기여한 각 행동에 얼마만큼의 책임을 돌릴 것인지를 결정하는 문제를 크레딧 할당(credit assignment)이라고 합니다. 단일 턴 과제에서는 이 문제가 상대적으로 단순합니다. 하나의 답변에 하나의 보상을 부여하면 됩니다. 그러나 멀티턴 자기 교정 환경에서는 여러 개의 수정 턴이 순서대로 이어지며, 각 턴이 최종 결과에 미치는 영향이 다릅니다. 크레딧 할당을 잘못하면 모델은 실제로 결과를 개선한 행동이 아니라, 우연히 높은 보상과 시간적으로 가까운 행동을 강화하게 됩니다. 이 단원에서는 멀티턴 크레딧 할당의 핵심 문제와 해결 방향을 다룹니다.

희소 결과 보상(Sparse Outcome Reward)의 한계부터 이해해야 합니다. 멀티턴 대화에서 가장 단순한 보상 설계는 마지막 턴의 결과만 평가하여 보상을 한 번 부여하는 것입니다. 세 번의 수정 턴을 거쳤을 때, 최종 세 번째 턴의 답변이 정답이면 양의 보상을, 틀리면 보상 없음 또는 음의 보상을 받습니다. 이 설계의 문제는 첫 번째와 두 번째 턴이 어떠했는지에 관한 정보가 학습 신호에 반영되지 않는다는 것입니다. 첫 번째 턴에서 훌륭한 방향을 잡았더라도, 두 번째 턴에서 좋지 않은 수정을 했더라도, 마지막 결과가 같다면 모델은 두 경로를 동등하게 취급합니다. 이는 중간 행동의 질을 개선하는 방향으로 학습이 이루어지지 않음을 의미합니다.

밀집 중간 보상(Dense Intermediate Reward)은 각 턴마다 즉각적인 보상 신호를 제공하여 이 문제를 완화합니다. 예를 들어, 수정 이후 답변이 정답에 더 가까워졌다면 양의 중간 보상을, 오히려 멀어졌다면 음의 중간 보상을 부여합니다. 이를 통해 모델은 어떤 수정 행동이 실질적으로 품질을 높였는지를 턴 단위로 학습할 수 있습니다. 그러나 밀집 보상 설계는 중간 상태의 품질을 측정할 방법이 있어야 합니다. 수학 문제에서는 각 단계의 중간 결과가 정답에 얼마나 가까운지를 평가하기 어렵고, 자연어 과제에서는 더욱 어렵습니다. 잘못 설계된 밀집 보상은 오히려 모델이 표면적으로 보상을 높이는 방향으로 행동을 왜곡하는 보상 해킹을 유발할 수 있습니다.

턴 수준 보상 설계(turn-level reward design)는 멀티턴 대화의 각 턴에 어떤 기준으로 보상을 부여할지를 결정하는 작업입니다. 일반적으로 고려되는 기준에는 정확도 변화(답변이 정답에 더 가까워졌는가), 불필요한 수정 회피(이미 맞는 답을 틀리게 바꾸지 않았는가), 수정 효율성(같은 개선을 더 짧은 추론으로 달성했는가) 등이 있습니다. 이 기준들은 서로 충돌할 수 있습니다. 예를 들어, 수정을 길게 작성하면 정확도는 높아지지만 효율성 보상은 낮아집니다. 여러 기준을 하나의 스칼라 보상으로 결합하는 과정에서 가중치 선택이 결과에 크게 영향을 미칩니다.

MT-GRPO(Multi-Turn GRPO)와 MT-PPO(Multi-Turn PPO)는 5.2.2에서 소개한 GRPO와 PPO를 멀티턴 환경에 맞게 확장한 알고리즘입니다. 표준 GRPO는 단일 응답 쌍을 기준으로 상대적 보상을 계산합니다. MT-GRPO는 여러 턴에 걸친 궤적 전체를 하나의 그룹으로 묶어 비교합니다. 동일한 초기 답변에서 출발한 여러 수정 궤적을 비교하여, 더 좋은 최종 결과를 낸 궤적에 더 높은 가중치를 부여합니다. 이를 통해 멀티턴 맥락에서도 상대적 보상 정규화의 분산 감소 효과를 유지합니다. MT-PPO는 멀티턴 MDP(Markov Decision Process) 구조를 명시적으로 모델링하고, 각 턴을 상태 전이 단계로 취급합니다. 가치 함수가 각 턴의 상태 가치를 추정하도록 확장되어, 중간 상태에서의 예측 보상을 기반으로 보상을 할당합니다.

일반화된 이점 추정(Generalized Advantage Estimation, GAE)은 크레딧 할당의 분산과 편향 사이의 트레이드오프를 조절하는 기법입니다. 완전한 결과 보상만 사용하면 분산이 높고, 단일 단계의 즉각 보상만 사용하면 편향이 높습니다. GAE는 두 극단 사이를 람다(lambda) 매개변수로 조절합니다. 람다가 0에 가까울수록 즉각 보상에 가까워지고, 1에 가까울수록 전체 결과 보상에 가까워집니다. 실용적으로는 람다를 0.9에서 0.95 사이로 설정하는 경우가 많으며, 이는 즉각 보상과 장기 결과를 균형 있게 반영합니다. 멀티턴 자기 교정 환경에서 GAE를 올바르게 적용하려면, 각 턴 경계에서 에피소드가 계속되는지 종료되는지를 정확히 처리해야 합니다.

멀티턴 MDP(multi-turn MDP)로의 정식화는 이 모든 논의를 이론적으로 통합합니다. 각 상태는 현재까지의 대화 이력(문제, 이전 답변들, 이전 교정 시도들)이고, 행동은 현재 턴의 응답 생성이며, 전이는 응답이 이력에 추가되는 과정입니다. 보상은 최종 턴에서 한 번만 주어질 수도 있고, 각 턴마다 주어질 수도 있습니다. 할인 인자(discount factor)는 미래 보상의 현재 가치를 조절합니다. 수정 횟수가 많을수록 먼 미래의 보상 중요성이 낮아지므로, 적절한 할인 인자 설정이 수정 깊이에 영향을 줍니다. 이 정식화 아래에서 MT-GRPO와 MT-PPO, GAE는 같은 프레임워크의 서로 다른 구현 선택으로 이해됩니다.

정리하면, 희소 결과 보상은 중간 턴의 품질 신호를 제공하지 못하고, 밀집 중간 보상은 설계 어려움과 보상 해킹 위험을 수반합니다. 턴 수준 보상 설계는 여러 기준을 균형 있게 반영해야 합니다. MT-GRPO와 MT-PPO는 표준 알고리즘을 멀티턴 구조에 맞게 확장하며, GAE는 분산-편향 트레이드오프를 람다로 조절합니다. 멀티턴 MDP 정식화는 이 모든 요소를 하나의 이론 틀로 통합합니다.

다음 단원인 5.3.3에서는 학습 목표와 추론 시간 성능 사이의 정렬 문제를 다룹니다. 교차 엔트로피 손실로 훈련된 모델이 왜 pass@N 기준에서는 최적이 아닌지, 그리고 출력 다양성을 장려하는 방향으로 학습 목표를 조정하는 방법을 살펴봅니다.
