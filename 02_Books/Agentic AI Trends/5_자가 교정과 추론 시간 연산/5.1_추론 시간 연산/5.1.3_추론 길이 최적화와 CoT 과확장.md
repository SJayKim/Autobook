# 5.1.3 추론 길이 최적화와 CoT 과확장

체인 오브 생각(Chain-of-Thought, CoT)은 언어 모델이 최종 답을 내기 전에 중간 추론 단계를 명시적으로 생성하도록 유도하는 기법입니다. CoT는 TTC를 활용하는 가장 자연스러운 방법 중 하나로, 어려운 문제일수록 긴 추론 사슬이 정답률을 높인다는 것이 반복적으로 확인되었습니다. 그러나 추론이 무조건 길수록 좋은 것은 아닙니다. 이 단원에서는 CoT가 TTC와 맺는 관계, 추론이 지나치게 길어질 때 발생하는 역효과, 그리고 추론 노력을 적절히 제어하는 방법을 다룹니다.

CoT가 TTC를 활용하는 방식은 토큰 생성 수를 늘리는 것입니다. 각 토큰 생성은 모델이 수행하는 계산 한 단계에 해당합니다. 따라서 추론 단계가 많을수록 모델은 더 많은 계산을 수행하며, 이는 곧 TTC가 증가함을 의미합니다. 단순 암산으로 답할 수 없는 다단계 수학 문제나, 논리적 연쇄가 필요한 추론 과제에서 CoT가 효과적인 이유가 여기에 있습니다. 모델이 중간 결과를 토큰으로 기록함으로써 '외부 작업 기억'처럼 활용하고, 이를 바탕으로 다음 단계를 이어나가는 것입니다.

그러나 CoT 과확장(overthinking)이라고 불리는 현상이 관찰됩니다. 이는 모델이 이미 정답에 도달했음에도 불구하고 추론을 계속 이어가다가 결론을 번복하거나, 불필요하게 복잡한 방향으로 논리를 재구성하는 현상입니다. 과확장의 역효과는 여러 방면에서 나타납니다. 첫째, 추론 길이가 길어질수록 오류가 누적될 기회도 늘어납니다. 추론 초반에 올바른 방향을 잡았더라도 이후 단계에서 불필요한 분기를 탐색하다 틀린 경로로 빠질 수 있습니다. 둘째, 연산 자원이 낭비됩니다. 같은 연산량으로 여러 문제를 풀 수 있었을 예산을 한 문제의 과도한 추론에 소진하면, 전체 처리량이 줄어듭니다. 셋째, 생성된 텍스트가 길어질수록 사용자 입장에서 읽기 어렵고, 응답 지연이 증가합니다.

과확장을 방지하기 위한 접근 중 하나가 추론 노력 제어(reasoning effort control)입니다. 이는 모델에게 얼마나 깊이 생각할지를 사전에 지정하는 방법입니다. 'Low', 'Medium', 'High' 수준으로 추론 노력을 지정하면, 모델은 해당 수준에 맞는 추론 토큰 예산 안에서 답을 생성합니다. Low 설정에서는 직접 답을 내거나 최소한의 추론만 수행하고, High 설정에서는 충분히 긴 탐색과 검증을 수행합니다. 이 접근의 실용적 가치는 사용자가 응답 품질과 응답 속도 사이의 트레이드오프를 명시적으로 제어할 수 있다는 데 있습니다. 시간이 중요한 상황에서는 Low를 선택하고, 정확도가 중요한 상황에서는 High를 선택하는 방식입니다.

최적 추론 경로 선택(Shortest Correct Path)은 과확장 문제를 다른 각도에서 접근합니다. 동일한 결론에 도달하는 여러 추론 경로 중, 불필요한 우회 없이 가장 짧게 정답에 이르는 경로가 최적이라는 관점입니다. 이를 구현하는 방법으로 Test-time Optimal Path Selection(TOPS)이 제안되었습니다. TOPS는 여러 추론 경로를 샘플링한 뒤, 정답에 도달한 경로들 중 가장 짧은 것을 선택합니다. 이는 단순히 TTC를 줄이는 것이 아니라, TTC의 질을 높이는 접근입니다. 긴 추론이 반드시 더 나은 추론을 의미하지 않으며, 경제적이고 직선적인 추론이 더 신뢰할 만하다는 직관에 기반합니다.

자기 일관성(Self-Consistency, SC-CoT)은 CoT를 여러 번 실행하고 다수결 방식으로 최종 답을 결정하는 기법입니다. 단일 CoT 경로는 확률적 노이즈에 취약하지만, 여러 독립적인 경로를 거쳐 동일한 결론에 도달할 경우 그 결론의 신뢰성이 높아집니다. SC-CoT는 Best-of-N과 유사하게 병렬 샘플링을 활용하지만, 검증자 없이 다수결이라는 단순한 집계 방식을 사용한다는 점이 다릅니다. 다수결의 장점은 별도의 검증 모델이 필요 없다는 것이고, 단점은 다수가 같은 오류를 공유하는 체계적 오류(systematic bias) 상황에서 신뢰성이 무너진다는 것입니다.

오류 단계를 포함한 추론 경로(error-laden reasoning paths)는 역설적으로 학습 데이터로서 가치를 지닙니다. 모델이 틀린 방향으로 추론하다가 스스로 수정하는 경로는 오류 감지와 교정 능력을 학습하는 데 활용될 수 있습니다. 이는 5.2.3에서 다루는 오류-수정 궤적 학습과 연결됩니다. 그러나 학습 데이터 관점에서 이러한 경로를 그대로 포함하면, 모델이 불필요하게 긴 추론이나 오류를 반복하는 나쁜 습관을 학습할 위험도 있습니다. 따라서 오류 단계를 포함한 경로를 활용할 때는 어떤 부분을 학습 신호로 삼고 어떤 부분을 억제할지를 신중하게 설계해야 합니다.

추론 길이 최적화는 단순히 짧게 만드는 것이 목표가 아닙니다. 문제의 실질적 난이도에 맞는 길이, 즉 과부족 없이 충분한 추론을 수행하는 것이 목표입니다. 쉬운 문제를 빠르게 처리하고 어려운 문제에 집중적으로 연산을 쓰는 동적 배분이 TTC 효율성의 핵심입니다. 모델이 문제의 난이도를 판단하고 스스로 추론 깊이를 조절할 수 있다면, 이는 고정된 설정에 의존하는 것보다 더 강인한 시스템이 됩니다.

정리하면, CoT는 TTC를 토큰 생성으로 구현하는 자연스러운 기제이지만, 과확장이라는 역효과를 수반합니다. 추론 노력 제어는 사용자가 품질-속도 트레이드오프를 명시적으로 설정하는 방법이고, TOPS는 올바른 경로 중 가장 짧은 것을 선택하는 효율화 전략입니다. SC-CoT는 검증자 없이 다수결로 신뢰성을 높이는 방법이며, 오류를 포함한 경로는 학습 신호로 활용될 수 있으나 신중한 설계가 필요합니다.

다음 단원인 5.2.1에서는 자기 교정의 정의와 근본적 한계를 다룹니다. 모델이 스스로 오류를 발견하고 수정하는 행위가 어디까지 가능한지, 그리고 왜 내재적인 역설을 내포하는지를 살펴봅니다. 추론 길이 최적화와 자기 교정의 한계를 함께 이해하면, TTC 스케일링의 진짜 병목이 어디에 있는지가 명확해집니다.
