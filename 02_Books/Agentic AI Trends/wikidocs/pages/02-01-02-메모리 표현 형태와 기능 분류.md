# 2.1.2 메모리 표현 형태와 기능 분류

2.1.1에서 에이전트 메모리가 LLM 메모리, RAG, Context Engineering과 어떻게 다른지, 그리고 왜 필요한지를 살펴보았습니다. 에이전트 메모리가 "호출을 가로질러 축적되는 진화적 인지 상태"라는 것을 이해했다면, 다음 질문은 이 메모리가 구체적으로 어떤 형태로 저장되고 어떤 기능을 수행하는지입니다. 이 단원에서는 에이전트 메모리를 두 가지 축, 즉 표현 형태(Forms)와 기능(Functions)으로 분류합니다.

먼저 **표현 형태(Forms)**를 살펴보겠습니다. 메모리가 물리적으로 어떤 구조로 저장되는지를 나타내는 분류입니다. 크게 토큰 수준 메모리(Token-level Memory), 파라메트릭 메모리(Parametric Memory), 잠재 메모리(Latent Memory)의 세 범주가 있습니다.

**토큰 수준 메모리**는 정보를 사람이 읽을 수 있는 텍스트 형태로 저장하는 방식입니다. 가장 직관적이고 널리 사용되는 형태입니다. 토큰 수준 메모리는 데이터의 조직 방식에 따라 세 단계로 나뉩니다.

첫 번째는 **Flat(1D) 메모리**입니다. 정보를 단순한 목록이나 평면적 구조로 저장합니다. 벡터 데이터베이스에 텍스트 조각을 임베딩하여 저장하는 방식, 키-값 저장소에 사실 정보를 기록하는 방식, JSON 형태의 구조화된 텍스트로 경험을 저장하는 방식이 여기에 해당합니다. 구현이 단순하고 검색이 빠르지만, 정보 간의 관계를 명시적으로 표현하지 못합니다. "서울의 인구는 약 950만 명이다"와 "한국의 수도는 서울이다"라는 두 사실이 각각 독립적으로 저장되어, 이 둘이 관련되어 있다는 것은 검색 시점에 추론해야 합니다.

두 번째는 **Planar(2D) 메모리**입니다. 정보 간의 관계를 명시적으로 표현하는 구조입니다. 테이블 형태로 속성과 값의 관계를 저장하거나, 그래프와 지식 그래프 형태로 엔티티 간의 관계를 노드와 엣지로 표현합니다. Flat 메모리에서 독립적이었던 두 사실이 Planar 메모리에서는 "서울 --수도관계--> 한국", "서울 --인구--> 950만"처럼 관계로 연결됩니다. 관계 기반 검색이 가능해지지만, 구축과 유지에 더 많은 비용이 듭니다.

세 번째는 **Hierarchical(3D) 메모리**입니다. 정보를 여러 층위로 조직하는 구조입니다. 2.2.1에서 자세히 다룰 STM/MTM/LTM 계층이 대표적입니다. 짧은 시간 범위의 상세한 정보는 하위 계층에, 오랜 시간에 걸쳐 축적된 추상적 지식은 상위 계층에 배치합니다. 다계층 그래프도 이 범주에 속합니다. 에이전트 간 대화 기록(Interaction Graph), 처리된 쿼리(Query Graph), 추상화된 교훈(Insight Graph)을 각각 다른 계층에 두고, 계층 간 연결을 통해 다양한 추상화 수준에서 정보를 검색하는 구조가 예입니다.

**파라메트릭 메모리(Parametric Memory)**는 정보를 모델 파라미터(가중치)에 인코딩하는 방식입니다. 토큰 수준 메모리가 명시적으로 저장된 텍스트라면, 파라메트릭 메모리는 학습을 통해 모델 내부에 암묵적으로 저장된 지식입니다. 두 종류가 있습니다. 내부 파라메트릭 메모리(Internal)는 LLM의 사전학습으로 가중치에 인코딩된 지식으로, 파인튜닝을 통해서만 갱신할 수 있습니다. 외부 파라메트릭 메모리(External)는 별도로 학습된 외부 모듈의 가중치에 저장된 지식입니다. 검색기(retriever) 모듈이나 어댑터(adapter) 모듈이 이에 해당합니다. 파라메트릭 메모리의 장점은 검색 없이 즉시 활용할 수 있다는 것이고, 단점은 갱신에 재학습이 필요하여 실시간 업데이트가 어렵다는 것입니다.

**잠재 메모리(Latent Memory)**는 토큰도 아니고 학습된 가중치도 아닌, LLM이 추론 과정에서 생성하고 재활용하는 잠재 표현 기반의 메모리입니다. LLM이 중간 표현을 생성(Generate)하고, 이를 후속 추론에서 재활용(Reuse)하며, 필요에 따라 변환(Transform)합니다. 가장 새롭고 실험적인 형태의 메모리로, 아직 에이전트 시스템에서의 활용이 초기 단계입니다.

이제 **기능(Functions)** 축을 살펴보겠습니다. 메모리가 무엇을 저장하고 어떤 목적으로 사용되는지를 나타내는 분류입니다. 크게 사실 메모리(Factual Memory), 경험 메모리(Experiential Memory), 작업 메모리(Working Memory)의 세 범주가 있습니다.

**사실 메모리(Factual Memory)**는 변하지 않는(또는 느리게 변하는) 사실적 정보를 저장합니다. 두 종류로 나뉩니다. 사용자 사실 메모리(User Factual)는 사용자의 이름, 선호도, 프로필, 개인적 사실 등을 저장하여 개인화를 가능하게 합니다. "이 사용자는 채식주의자이다", "선호하는 프로그래밍 언어는 Python이다" 같은 정보입니다. 환경 사실 메모리(Environment Factual)는 에이전트가 활동하는 환경에 대한 지식을 저장합니다. 게임 에이전트가 맵의 구조를 기억하거나, 가정 환경 에이전트가 물건의 위치를 기록하는 것이 해당합니다.

**경험 메모리(Experiential Memory)**는 에이전트가 태스크를 수행하면서 축적한 경험을 저장합니다. 에이전트 메모리의 가장 독특한 기능이며, 2.1.1에서 RAG와 구분되는 핵심 지점입니다. 세 종류로 나뉩니다. 사례 기반 경험 메모리(Case-based)는 과거의 태스크 수행 궤적(trajectory) 전체를 저장하고, 유사한 새 태스크를 만났을 때 이를 few-shot 예시로 검색하여 활용합니다. "이전에 비슷한 문제를 이렇게 풀었다"를 그대로 보여주는 방식입니다. 전략 기반 경험 메모리(Strategy-based)는 궤적 전체가 아니라, 경험에서 추출한 전략, 인사이트, 교훈을 저장합니다. "이 유형의 문제에서는 A 접근보다 B 접근이 효과적이었다"처럼 추상화된 지식입니다. 1.1.2에서 다룬 Self-Reflection의 반성 요약이 전략 기반 경험 메모리의 대표적 사례입니다. 기술 기반 경험 메모리(Skill-based)는 경험을 재사용 가능한 도구, API 호출 패턴, 코드 스니펫으로 증류하여 저장합니다. 추상적 전략이 아니라 바로 실행 가능한 형태로 경험을 결정화하는 것입니다.

**작업 메모리(Working Memory)**는 현재 진행 중인 태스크의 맥락을 유지하는 임시 저장 공간입니다. 인지과학의 작업 기억(working memory) 개념에서 차용한 것으로, 현재 주의를 기울이고 있는 정보를 일시적으로 보유합니다. 두 시간 범위로 나뉩니다. 단일 턴 작업 메모리(Single-turn)는 하나의 추론 단계 내에서 중간 결과를 임시로 유지하는 스크래치패드입니다. 다중 턴 작업 메모리(Multi-turn)는 에이전트 루프 전체에 걸쳐 중간 상태를 유지합니다. 1.1.2에서 다룬 ReAct의 Thought-Action-Observation 이력이 다중 턴 작업 메모리의 대표적 예입니다.

표현 형태와 기능은 서로 독립적인 축입니다. 사실 메모리는 Flat 형태의 키-값 저장소에 저장될 수도 있고, Planar 형태의 지식 그래프에 저장될 수도 있습니다. 경험 메모리는 Flat 형태의 벡터 데이터베이스에 텍스트 궤적으로 저장될 수도 있고, Hierarchical 형태의 다계층 그래프에 추상화 수준별로 조직될 수도 있습니다. 이 두 축의 조합이 에이전트 메모리 시스템의 설계 공간을 형성합니다. 어떤 조합을 선택하느냐는 태스크의 성격, 요구되는 검색 방식, 메모리 갱신 빈도, 저장 효율성 등을 고려하여 결정합니다.

이 분류 체계를 이해하면, 앞으로 다룰 구체적인 메모리 아키텍처들이 어디에 위치하는지 파악하기 쉬워집니다. 2.2.1에서 다룰 계층적 메모리 저장 구조는 Hierarchical(3D) 표현 형태에 해당하고, 2.2.2의 그래프 기반 메모리는 Planar(2D) 표현 형태에 해당합니다. 2.5.3의 자기 진화 메모리에서 다루는 Think-Act-Refine 루프는 경험 메모리의 전략 기반 변형에 해당합니다. 2.6.2의 사용자 선호 모델링은 사실 메모리(User Factual)의 고도화된 형태입니다.

정리하면, 에이전트 메모리는 표현 형태(Forms)와 기능(Functions)의 두 축으로 분류됩니다. 표현 형태는 Flat(1D), Planar(2D), Hierarchical(3D)의 토큰 수준 메모리, Internal과 External의 파라메트릭 메모리, 그리고 잠재 메모리로 나뉩니다. 기능은 사실 메모리(User와 Environment), 경험 메모리(Case-based, Strategy-based, Skill-based), 작업 메모리(Single-turn과 Multi-turn)로 나뉩니다. 이 두 축은 독립적이며, 그 조합이 메모리 시스템의 설계 공간을 형성합니다.

다음 단원인 2.1.3에서는 인지과학의 에피소드 기억, 의미 기억, 절차 기억 분류가 에이전트 메모리 설계에 어떻게 적용되는지 살펴봅니다.

이 단원을 마치면 에이전트 메모리의 표현 형태(Flat, Planar, Hierarchical, Parametric, Latent)와 기능(Factual, Experiential, Working)을 체계적으로 분류하고, 각 범주의 특성과 사용 맥락을 설명할 수 있습니다.
