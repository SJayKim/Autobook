# 6.3.1 에이전틱 검색 오케스트레이션

에이전틱 검색 오케스트레이션은 단순한 검색 파이프라인을 넘어, 에이전트가 복잡한 정보 수집 목표를 달성하기 위해 여러 검색 도구를 자율적으로 계획하고 실행하며 조정하는 방식입니다. 이 접근법에서 검색은 일회성 조회가 아니라, 에이전트의 지식 상태를 점진적으로 발전시키는 반복적인 활동입니다. 이 단원에서는 에이전틱 검색의 구성 요소와 작동 원리를 구체적으로 살펴봅니다.

멀티툴 오케스트레이션은 에이전트가 여러 검색 도구를 목적에 맞게 선택하고 순서를 결정하는 능력입니다. 에이전트가 사용할 수 있는 도구 목록에는 전통적 벡터 검색, KG 엔티티 조회, KG 관계 탐색, 그래프 쿼리 언어 실행, 커뮤니티 요약 검색 등 다양한 종류가 포함됩니다. 각 도구는 특정 종류의 정보를 가져오는 데 최적화되어 있습니다. 에이전트는 현재 질문과 지금까지 수집된 정보를 바탕으로 다음에 어떤 도구를 사용할지 결정합니다. 이 결정 과정 자체가 추론의 일부입니다.

Cypher 쿼리 동적 생성은 에이전트가 KG에 특정 질문을 던질 때, 그 질문을 그래프 쿼리 언어인 Cypher로 변환하여 실행하는 기법입니다. Cypher는 Neo4j 그래프 데이터베이스에서 사용되는 선언적 쿼리 언어로, 노드와 엣지 패턴을 직관적인 문법으로 표현합니다. 에이전트는 자연어로 된 서브 질문을 LLM을 통해 Cypher 쿼리로 변환합니다. 예컨대 "A의 직속 상하 관계에 있는 모든 인물을 찾아라"는 Cypher의 방향 있는 관계 패턴 쿼리로 변환됩니다. 동적 생성이기 때문에 사전에 정의되지 않은 쿼리 유형도 처리할 수 있습니다.

쿼리 분해와 서브 질문 재작성은 복잡한 원래 질문을 작은 단위로 나누고, 각 단위를 현재 사용 가능한 정보와 도구에 맞게 재작성하는 과정입니다. 원래 질문이 "A 분야에서 최근 5년간 가장 많이 인용된 연구자의 소속 기관과 연구 방향은?"이라면, 에이전트는 먼저 (1) A 분야 정의, (2) 최근 5년간 상위 인용 연구자 목록 검색, (3) 각 연구자의 소속 기관 조회, (4) 각 연구자의 최근 연구 방향 요약으로 분해합니다. 각 서브 질문은 독립적으로 실행 가능한 형태로 재작성됩니다.

증거 수집-평가-후속 계획 사이클은 에이전틱 검색의 반복 구조입니다. 매 사이클에서 에이전트는 계획된 검색 행동을 실행하여 증거를 수집하고, 수집된 증거가 목표 달성에 얼마나 기여하는지 평가하며, 평가 결과를 바탕으로 후속 계획을 세웁니다. 이 사이클은 충분한 증거가 축적되었다고 판단될 때까지 반복됩니다. 증거 평가 단계에서는 새로 수집된 정보가 기존 지식을 확인하는지, 보완하는지, 아니면 모순되는지도 판단합니다.

ReAct RAG는 ReAct(Reasoning + Acting) 프레임워크를 RAG 시스템에 적용한 방식입니다. ReAct에서 에이전트는 생각(Thought), 행동(Action), 관찰(Observation)을 번갈아 수행합니다. RAG 환경에서 행동은 특정 검색 도구를 호출하는 것이고, 관찰은 그 검색 결과입니다. 에이전트는 매 관찰 후 다음 생각 단계에서 결과를 해석하고 다음 행동을 결정합니다. 이 구조는 검색 과정의 각 단계가 명시적으로 기록되고 추적 가능하다는 장점을 제공합니다.

Working Memory는 에이전틱 검색 과정에서 누적된 모든 중간 결과를 저장하는 단기 메모리 구조입니다. 이미 확인된 사실들, 방문한 KG 노드들, 검색한 문서 패시지들, 중간 추론 결과들이 모두 Working Memory에 저장됩니다. 에이전트는 매 단계에서 Working Memory를 참조하여 이미 알고 있는 정보는 다시 검색하지 않으며, 이전 검색에서 발견한 연관 노드들을 다음 검색의 시드로 활용합니다. Working Memory의 구조는 단순한 텍스트 목록일 수도 있고, 수집된 정보를 노드-엣지 구조로 저장하는 Graph Memory일 수도 있습니다.

서브골 기반 계획(subgoal-based planning)은 최종 목표를 달성하기 위한 중간 단계 목표들을 명시적으로 설정하고 순서를 정하는 방식입니다. 에이전트는 처음 쿼리를 분석하여 최종 답변을 생성하려면 어떤 중간 정보들이 필요한지를 나열합니다. 이 중간 정보들을 서브골로 설정하고, 각 서브골 사이의 의존 관계를 파악하여 최적의 실행 순서를 결정합니다. 의존 관계가 없는 서브골들은 병렬로 실행하여 전체 처리 시간을 단축합니다.

매 관찰마다 그래프 업데이트는 실시간 그래프 적응의 핵심입니다. 에이전트가 검색을 통해 새로운 엔티티나 관계를 발견하면, 이를 즉시 임시 작업 그래프(working graph)에 추가합니다. 이 임시 그래프는 원래 KG와 별도로 유지되며, 현재 추론 세션에서만 유효합니다. 후속 검색은 원래 KG뿐 아니라 이 임시 그래프도 참조합니다. 이 방식으로 에이전트는 탐색이 진행될수록 더 풍부한 컨텍스트 안에서 다음 검색을 결정합니다.

정리하면, 에이전틱 검색 오케스트레이션은 멀티툴 오케스트레이션, Cypher 동적 쿼리 생성, 쿼리 분해와 서브 질문 재작성, 증거 수집-평가-후속 계획 사이클, ReAct 구조의 RAG 적용, Working Memory 관리, 서브골 기반 계획, 그리고 매 관찰 후 그래프 업데이트가 결합된 자율적 검색 시스템입니다. 단순한 단일 검색 호출과 달리, 이 시스템은 목표 달성에 필요한 지식을 능동적으로 구성해 나갑니다.

다음 단원에서는 인간 뇌의 기억 구조에서 영감을 받은 RAG 아키텍처를 다룹니다. 해마와 신피질의 역할을 모방하여 연상 기억과 장기 기억을 구현하는 방법, 그리고 이 아키텍처가 단일 단계로 멀티홉 검색을 가능하게 하는 원리를 살펴볼 것입니다.
