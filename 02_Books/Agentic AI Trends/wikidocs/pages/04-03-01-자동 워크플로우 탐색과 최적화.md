# 4.3.1 자동 워크플로우 탐색과 최적화

에이전트 시스템의 워크플로우를 수동으로 설계하는 것은 많은 전문 지식과 시간을 요구합니다. 어떤 에이전트 조합이 가장 좋은 결과를 내는지, 에이전트들을 어떤 순서로 연결해야 하는지, 각 단계에서 어떤 프롬프트를 사용해야 하는지 같은 결정들은 경험적으로만 알 수 있는 경우가 많습니다. 이 설계 작업 자체를 자동화하려는 시도가 자동 워크플로우 탐색입니다. 이 접근법에서는 에이전트가 워크플로우 공간을 탐색하여 스스로 효과적인 구조를 발견합니다.

몬테카를로 트리 탐색(Monte Carlo Tree Search, MCTS)은 게임 플레이 AI에서 성공을 거둔 탐색 알고리즘으로, 워크플로우 탐색에도 적용될 수 있습니다. MCTS는 트리 구조에서 유망한 경로를 선택하는 선택(selection) 단계, 새로운 노드를 탐색하는 확장(expansion) 단계, 랜덤 또는 휴리스틱 방식으로 결과를 시뮬레이션하는 시뮬레이션(simulation) 단계, 결과를 상위 노드로 전파하는 역전파(backpropagation) 단계를 반복합니다. 워크플로우 탐색에 적용할 때, 각 노드는 워크플로우의 한 단계 또는 에이전트 조합을 나타내고, 결과는 해당 워크플로우의 성능 지표가 됩니다. MCTS는 완전한 탐색이 불가능한 큰 공간에서 탐색과 활용의 균형을 맞추며 좋은 솔루션을 효율적으로 찾는 데 효과적입니다.

오퍼레이터 패턴(operator pattern)은 워크플로우를 구성하는 기본 연산 단위를 정의하는 개념입니다. Ensemble 오퍼레이터는 동일한 태스크를 여러 에이전트가 독립적으로 수행한 후 결과를 통합합니다. 여러 관점에서 생성된 결과를 통합하면 단일 에이전트의 결과보다 더 안정적이고 균형 잡힌 출력을 얻을 수 있습니다. Review & Revise 오퍼레이터는 한 에이전트가 결과를 생성하면 다른 에이전트가 그것을 검토하고 개선하는 방식입니다. 이 패턴은 작성-검토-수정의 인간 작업 방식과 유사하며, 결과의 품질을 단계적으로 높이는 데 효과적입니다. 이 오퍼레이터들은 워크플로우 탐색의 기본 단위가 되어, 탐색 알고리즘이 이 오퍼레이터들을 조합하여 효과적인 워크플로우를 찾습니다.

코드 수정을 통한 워크플로우 변이(workflow mutation via code modification)는 워크플로우가 코드로 표현될 때 적용할 수 있는 방법입니다. 워크플로우 코드를 직접 수정하여 새로운 변형을 생성하고, 그 성능을 평가하여 좋은 변형을 선택합니다. 이 과정은 생물학적 진화에서의 돌연변이와 유사합니다. LLM이 워크플로우 코드를 이해하고 의미 있는 방식으로 수정할 수 있기 때문에, 단순한 랜덤 변이가 아닌 지능적인 변이가 가능합니다. 예를 들어 두 에이전트 사이에 검증 단계를 추가하거나, 순차 처리를 병렬 처리로 변경하거나, 특정 오퍼레이터를 다른 오퍼레이터로 교체하는 등의 변이를 LLM이 직접 제안하고 구현합니다.

프롬프트-토폴로지 동시 최적화(joint prompt-topology optimization)는 워크플로우의 구조(토폴로지)와 각 에이전트의 프롬프트를 함께 최적화하는 방식입니다. 워크플로우 구조만 최적화하면 각 에이전트가 최적의 프롬프트를 사용하지 않을 수 있고, 반대로 프롬프트만 최적화하면 전체 구조가 비효율적일 수 있습니다. 두 가지를 함께 최적화하면 더 높은 성능을 달성할 수 있지만, 탐색 공간이 훨씬 커지는 도전이 있습니다. 이를 효율적으로 처리하기 위해 먼저 토폴로지를 탐색하고 그 후 프롬프트를 최적화하는 단계적 접근법이나, 교대로 두 가지를 최적화하는 반복적 접근법이 사용됩니다.

토폴로지 자동 탐색(automatic topology search)은 에이전트들이 연결되는 방식의 구조 자체를 자동으로 탐색합니다. 순차 체인, 병렬 처리, 계층 구조, 피드백 루프 등 다양한 연결 패턴이 가능하며, 각 패턴이 특정 유형의 태스크에 더 적합할 수 있습니다. 토폴로지 탐색은 사전에 설계자가 예상하지 못했던 효과적인 구조를 발견할 수 있습니다. 특히 복잡한 멀티스텝 태스크에서, 단순한 구조로는 해결이 어렵지만 특정 비선형 구조에서는 높은 성능이 나오는 경우가 있습니다.

보상 모델 기반 사전 평가(reward model-based pre-evaluation)는 워크플로우를 실제로 실행하기 전에 그 성능을 예측하는 방법입니다. 많은 워크플로우 변형을 모두 실제 실행하여 평가하면 비용이 매우 높습니다. 보상 모델은 워크플로우의 구조와 구성 요소를 보고 예상 성능을 추정합니다. 이 사전 평가를 통해 탐색 공간을 효율적으로 줄여, 유망한 워크플로우에만 실제 실행 비용을 투자할 수 있습니다. 보상 모델 자체도 더 많은 실행 결과가 쌓일수록 정확해지므로, 탐색이 진행될수록 사전 평가의 신뢰도가 높아지는 선순환 구조가 형성됩니다.

Representative Works 메커니즘은 지금까지 탐색에서 발견된 가장 효과적인 워크플로우 패턴들을 저장하고, 새로운 탐색의 출발점으로 활용하는 방식입니다. 탐색 이력에서 반복적으로 좋은 성능을 보이는 패턴이 있다면, 그것을 대표 사례로 기록하고 이후 탐색에서 그 패턴을 변형하여 더 나은 구조를 찾습니다. 이 메커니즘은 탐색이 처음부터 다시 시작하는 것이 아니라 이전 경험을 축적하며 진행되도록 합니다. 마치 진화 과정에서 적응적 특성이 보존되고 축적되는 것과 같습니다.

재계획(replanning)과 계획 상세화(plan refinement)는 실행 중에 예상과 다른 결과가 나왔을 때의 대응 방식입니다. 재계획은 현재 계획을 버리고 새로운 계획을 수립하는 것이고, 계획 상세화는 기존 계획의 틀은 유지하면서 특정 부분을 더 구체화하거나 수정하는 것입니다. 탐색 기반 접근법에서는 이 두 가지가 모두 자연스럽게 이루어질 수 있습니다. 중간 결과를 평가하고, 남은 부분에 대해 새로운 탐색을 수행하거나, 기존 계획의 일부를 더 세밀한 서브워크플로우로 교체합니다.

자동 워크플로우 탐색과 최적화는 에이전트 시스템 설계의 경험적 지식을 자동화하는 강력한 방향입니다. 단, 이 접근법은 탐색 비용이 높고, 좋은 보상 신호를 정의하는 것이 어려우며, 탐색 결과가 태스크 유형에 특화될 수 있다는 한계도 있습니다. 효과적인 적용을 위해서는 탐색 비용을 관리하는 효율적인 알고리즘, 태스크의 진정한 목표를 반영하는 보상 함수, 탐색 결과를 다른 유형의 태스크에도 일반화하는 전략이 필요합니다.

정리하면, 자동 워크플로우 탐색은 MCTS를 비롯한 탐색 알고리즘을 통해 에이전트 워크플로우 공간을 효율적으로 탐색하며, 오퍼레이터 패턴과 코드 기반 변이를 통해 다양한 워크플로우 변형을 생성합니다. 프롬프트-토폴로지 동시 최적화와 보상 모델 기반 사전 평가는 탐색의 효율성과 품질을 높이고, Representative Works 메커니즘은 탐색 경험을 축적하여 이후 탐색을 더 효율적으로 만듭니다.

다음 단원(4.3.2)에서는 워크플로우를 탐색이 아닌 강화학습으로 최적화하는 방법, 특히 GRPO 기반 Planner 학습과 자기 진화 커리큘럼을 활용한 에이전트 학습을 살펴봅니다.

이 단원을 학습한 후에는 MCTS 기반 워크플로우 탐색의 원리를 설명하고, 오퍼레이터 패턴을 이용한 워크플로우 구성을 이해하며, 프롬프트-토폴로지 동시 최적화와 보상 모델 기반 사전 평가를 적용한 탐색 전략을 설계할 수 있습니다.
