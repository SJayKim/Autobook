# 2.7.1 메모리 평가 벤치마크

2.6절에서 다중 에이전트 메모리와 사용자 개인화를 살펴보았습니다. 메모리 시스템이 다양한 메커니즘으로 구성되어 있다면, 그 시스템이 실제로 얼마나 잘 동작하는지를 측정하는 방법도 필요합니다. 1.4.1에서 에이전트 벤치마크를 다루면서 "에이전트 능력은 텍스트 생성 능력과 구별되는 독자적 역량"이라고 했듯, 메모리 능력도 별도의 평가 체계가 필요합니다.

메모리 시스템을 평가할 때 측정해야 하는 핵심 역량은 크게 네 가지입니다. **Accurate Retrieval(AR, 정확한 검색)**은 쿼리에 대해 정확한 정보를 검색하는 능력입니다. 단일 홉 회상(한 번의 검색으로 답을 찾는 것)과 다중 홉 회상(여러 메모리를 연결하여 답을 도출하는 것)을 모두 포함합니다. **Test-Time Learning(TTL)**은 추가 학습 없이 배포 중에 새로운 경험을 메모리에 통합하여 행동을 개선하는 능력입니다. 2.5.3에서 다룬 자기 진화 메모리의 직접적 평가 항목입니다. **Long-Range Understanding(LRU, 장거리 이해)**은 10만 토큰 이상에 걸쳐 분산된 정보를 통합하고 이해하는 능력입니다. **Selective Forgetting(SF, 선택적 망각)**은 모순되는 새 정보가 주어졌을 때 기존 메모리를 적절히 수정하거나 폐기하는 능력입니다. 2.5.2에서 다룬 선택적 망각의 평가입니다.

이 외에 **Multi-Hop Reasoning(다중 홉 추론)**은 여러 메모리 항목을 연결하여 복합적 추론을 수행하는 능력이고, **Temporal Reasoning(시간적 추론)**은 시간적 순서, 간격, 변화를 정확하게 추론하는 능력입니다. **Adversarial Robustness(적대적 강건성)**는 의도적으로 혼동을 유발하는 입력에도 메모리 시스템이 안정적으로 동작하는 내성입니다.

이 역량들을 측정하는 주요 벤치마크를 살펴보겠습니다. **LoCoMo**는 평균 9,000 토큰, 최대 35세션에 걸친 장기 대화 질의응답 벤치마크입니다. 5개 카테고리(Multi-Hop, Temporal, Single-Hop, Open-Domain, Adversarial)의 질문으로 구성되어, 메모리 시스템의 다양한 측면을 종합적으로 평가합니다. 여러 메모리 시스템 연구에서 공통 평가 벤치마크로 사용됩니다.

**MemoryAgentBench**는 인지과학 기반 4대 역량(AR, TTL, LRU, SF)을 점진적 멀티턴 형식으로 평가하는 종합 벤치마크입니다. 이 벤치마크의 특징은 정적 질의응답이 아니라, 에이전트가 순차적으로 태스크를 수행하면서 메모리를 축적하고 활용하는 동적 과정을 평가한다는 것입니다. 특히 선택적 망각 평가에서 대부분의 메모리 시스템이 10~15% 수준의 낮은 성공률을 보여, 이 역량이 현재 가장 어려운 과제임을 확인했습니다.

**Evo-Memory Benchmark**는 10개 데이터셋에서 10가지 이상의 메모리 모듈을 동일 프로토콜로 스트리밍 방식 비교하는 자기 진화 메모리 특화 벤치마크입니다. **태스크 스트리밍(Task Streaming) 평가**를 사용하는데, 이것은 정적 질의응답이 아니라 순차적 태스크 스트림을 에이전트에게 제공하여 메모리의 경험 축적 효과를 시간에 따라 측정하는 동적 평가 방식입니다. 태스크가 진행될수록 메모리가 축적되면서 성능이 개선되는 곡선을 그리므로, 단순 정확도뿐 아니라 학습 속도와 장기 안정성도 평가할 수 있습니다.

평가 지표로는 **F1 Score**와 **BLEU** 계열 지표가 생성된 응답의 정보 커버리지와 표면적 유사도를 측정하는 기본 지표로 사용됩니다. **GPT-4 기반 평가(GPT4Score)**는 LLM을 자동 평가기로 사용하여 응답 품질을 정성적, 정량적으로 측정합니다. **Pairwise 비교 평가**는 두 시스템의 출력을 직접 비교하여 승률로 우열을 판단합니다. 메모리 시스템의 효율성을 종합적으로 평가하기 위해, 성능 향상 대비 토큰 소모량, 처리 지연시간, 스토리지 사용량을 동시에 측정하는 복합 효율성 지표도 사용됩니다.

흥미로운 평가 지표로 **Experience-Following 상관관계**가 있습니다. 입력 유사도와 출력 유사도 사이의 Pearson 상관(r 약 1)으로 에이전트의 메모리 의존 행동 패턴을 정량화합니다. 이 상관이 높으면 에이전트가 메모리에서 검색된 경험을 강하게 모방한다는 의미입니다. 이것은 유용할 수도 있지만, 2.7.2에서 다룰 오류 전파의 원인이 되기도 합니다. 또한 데이터셋 내 태스크 유사도와 메모리 시스템 성능 개선의 상관(Pearson r=0.717)도 보고되어, 유사한 태스크가 반복되는 환경에서 메모리의 효과가 더 크다는 것이 확인되었습니다.

정리하면, 메모리 평가의 핵심 역량은 정확한 검색(AR), Test-Time Learning(TTL), 장거리 이해(LRU), 선택적 망각(SF)의 네 가지입니다. LoCoMo는 장기 대화 QA, MemoryAgentBench는 인지과학 기반 4대 역량, Evo-Memory Benchmark는 자기 진화 메모리를 각각 특화 평가합니다. 태스크 스트리밍 방식의 동적 평가가 정적 QA보다 메모리의 경험 축적 효과를 더 정확히 측정합니다.

다음 단원인 2.7.2에서는 메모리 시스템의 프라이버시 위험과 오류 전파 메커니즘을 살펴봅니다.

이 단원을 마치면 AR, TTL, LRU, SF 등 메모리 평가 핵심 역량을 구별하고, LoCoMo, MemoryAgentBench, Evo-Memory 등 주요 벤치마크의 평가 방식을 비교할 수 있습니다.
