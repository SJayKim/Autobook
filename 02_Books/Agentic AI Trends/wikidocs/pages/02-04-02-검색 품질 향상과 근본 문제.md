# 2.4.2 검색 품질 향상과 근본 문제

2.4.1에서 의미 검색, 키워드 검색, 하이브리드 검색 등 기본 검색 기법을 살펴보았습니다. 이 단원에서는 기본 검색의 한계를 넘어서는 품질 향상 기법들과, 메모리 검색의 근본적인 트레이드오프를 다룹니다.

기본 검색 기법들은 쿼리와 개별 메모리 항목 사이의 유사도만 고려합니다. 가장 유사한 상위 k개 항목을 반환하는 top-k 검색이 전형적입니다. 그러나 이 방식은 메모리 항목들 사이의 관계를 활용하지 못합니다. 2.2.2에서 그래프 기반 메모리를 설명하면서, 메모리 항목 간에 링크가 존재한다는 것을 살펴보았습니다. **링크 기반 연관 메모리 확장**은 top-k로 검색된 메모리 항목에서 링크로 연결된 인접 메모리까지 함께 반환하는 기법입니다. "Python 데이터 전처리"라는 메모리가 검색되면, 이 메모리에 링크된 "pandas merge 오류 해결 경험"과 "데이터 분석 프로젝트 개요"도 함께 반환됩니다. 단순 유사도 상위 k개만 반환하는 것보다 훨씬 풍부한 맥락을 제공하며, 토큰 사용량은 기존 방식 대비 대폭 절감됩니다. 한 연구에서는 링크 기반 확장을 사용했을 때 기존 방식 대비 85~93%의 토큰 절감을 달성하면서도 동등하거나 우수한 검색 품질을 보였습니다.

검색 쿼리 자체를 풍부하게 만드는 **토픽 추출을 통한 쿼리 확장**도 효과적입니다. 사용자의 원래 입력에서 핵심 토픽을 자동 추출하여 검색 쿼리를 확장합니다. "이 코드가 왜 안 돼?"라는 짧은 질문에서 현재 대화 맥락을 분석하여 "Python", "pandas", "타입 오류"라는 토픽을 추출하고, 이 확장된 쿼리로 검색하면 원래 질문만으로는 찾기 어려웠던 관련 메모리를 발견할 수 있습니다.

다중 에이전트 환경에서는 **역할 인식 메모리 필터링(role-aware retrieval)**이 중요합니다. 2.2.2에서 다계층 그래프 구조를 설명할 때 언급한 것처럼, 팀 공유 메모리에서 각 에이전트의 역할에 맞는 정보만 선택적으로 반환하는 메커니즘입니다. 코딩 에이전트에게는 코드 관련 과거 경험을, 리뷰 에이전트에게는 품질 관련 피드백 이력을 우선적으로 제공합니다. 역할 필터링 없이 모든 에이전트에게 동일한 검색 결과를 제공하면, 역할과 무관한 정보가 에이전트의 판단을 오히려 혼란시킵니다. 단일 에이전트용 메모리를 다중 에이전트 시스템에 그대로 이식했을 때 성능이 오히려 하락하는 현상이 이 때문입니다.

2.2.2에서 다룬 다계층 그래프 구조에서는 **양방향 그래프 탐색(bi-directional traversal)**이 사용됩니다. 먼저 쿼리와 가장 관련 높은 노드를 거친(coarse) 수준에서 검색한 뒤, 그 노드로부터 상향(Insight 계층)과 하향(Interaction 계층) 방향으로 동시에 그래프를 탐색합니다. 상향 탐색은 추상적 교훈과 전략을, 하향 탐색은 구체적인 대화 내용을 반환합니다. 이렇게 서로 다른 추상화 수준의 정보를 함께 제공하면 에이전트가 더 풍부한 맥락에서 판단할 수 있습니다.

임베딩 기반 초기 검색의 결과를 LLM이 다시 평가하는 **LLM 기반 관련도 재순위화(reranking)**도 있습니다. 임베딩 유사도는 대략적인 관련성을 빠르게 파악하지만, 세밀한 맥락적 관련성 판단은 부족합니다. LLM에게 검색 결과 목록과 원래 쿼리를 함께 제공하면, LLM이 각 결과의 실제 관련성을 맥락적으로 평가하여 더 정밀한 순위를 매길 수 있습니다. 이 2단계 검색(초기 임베딩 검색 + LLM 재순위화)은 정밀도를 높이지만, LLM 호출이 추가되므로 지연 시간과 비용이 증가합니다.

검색된 결과가 너무 길어 컨텍스트를 과도하게 차지하는 문제를 해결하는 **스파시파이어(Sparsifier)**도 있습니다. 검색된 메모리 항목에서 LLM이 현재 쿼리와 가장 관련 높은 핵심 부분만 선별하여 컨텍스트에 포함합니다. 메모리 항목 전체를 넣는 대신 필요한 문장만 추출하는 것입니다. 이를 통해 컨텍스트 윈도우를 효율적으로 사용하면서도 관련 정보의 밀도를 높입니다.

이제 메모리 검색의 근본적인 문제들을 살펴보겠습니다. 첫 번째는 **검색 vs 생성(retrieval vs generation) 트레이드오프**입니다. 에이전트가 질문에 답할 때, 저장된 메모리를 검색하여 활용할 것인지, 아니면 LLM이 내부 지식으로 즉석에서 답변을 생성할 것인지의 전략적 선택입니다. 검색은 정확한 과거 정보를 활용할 수 있지만 검색 오류의 위험이 있고, 생성은 유연하지만 환각의 위험이 있습니다. 이 트레이드오프는 메모리 시스템의 품질에 따라 달라집니다. 메모리가 고품질이면 검색이 유리하고, 메모리가 노이즈로 가득 차 있으면 생성이 오히려 나을 수 있습니다.

두 번째는 2.4.1에서 예고한 **전체 이력 삽입의 역설**입니다. 모든 대화 이력을 그대로 컨텍스트에 주입하면 정보가 더 많으니 성능이 좋아질 것 같지만, 실제로는 오히려 성능이 하락합니다. 무관한 정보가 주의를 분산시키고, 2.1.1에서 다룬 Lost in the Middle 현상으로 중간에 위치한 관련 정보가 무시되기 때문입니다. 선택적 검색이 전체 삽입보다 우수하다는 것은 메모리 시스템의 존재 이유를 정당화하는 핵심 근거입니다.

세 번째는 **메모리 크기와 검색 품질의 비선형 관계**입니다. 2.3.2에서 메모리 입장 제어를 다루면서 "모든 경험을 무분별하게 저장하면 성능이 하락한다"는 현상을 언급했습니다. 이것은 검색의 관점에서도 같은 현상입니다. 메모리 크기가 증가하면 처음에는 검색 대상이 풍부해져 성능이 향상되지만, 일정 크기를 넘어서면 무관한 메모리가 관련 메모리와 경쟁하여 검색 정밀도가 하락합니다. 메모리 크기와 검색 품질의 관계가 단조 증가가 아니라 역U자 형태를 보인다는 것입니다. 이것이 메모리 입장 제어(2.3.2)와 메모리 망각(2.5.2)이 검색 품질에도 직접적으로 기여하는 이유입니다.

정리하면, 메모리 검색 품질은 링크 기반 연관 확장, 쿼리 확장, 역할 인식 필터링, 양방향 그래프 탐색, LLM 재순위화, 스파시파이어 등의 기법으로 향상됩니다. 동시에 검색 vs 생성 트레이드오프, 전체 이력 삽입의 역설, 메모리 크기와 검색 품질의 비선형 관계라는 근본 문제를 인식해야 합니다. 선택적이고 정밀한 검색이 무분별한 전체 삽입보다 우수하며, 메모리의 양보다 질이 검색 성능을 결정합니다.

다음 단원인 2.5.1에서는 저장된 메모리가 시간이 지남에 따라 어떻게 변화하는지, 메모리 업데이트와 통합의 메커니즘을 살펴봅니다.

이 단원을 마치면 링크 기반 연관 확장, 역할 인식 필터링, 양방향 그래프 탐색, LLM 재순위화 등 검색 품질 향상 기법을 설명하고, 메모리 크기와 검색 품질의 비선형 관계를 기술할 수 있습니다.
