# 2.2.3 메모리 세분성과 최적 단위 선택

2.2.1에서 계층적 저장 구조를, 2.2.2에서 그래프 기반 저장을 살펴보았습니다. 이 두 단원에서 다루지 않은 중요한 설계 결정이 있습니다. 메모리의 최소 단위를 어떻게 설정할 것인가, 즉 대화의 어떤 조각을 하나의 메모리 항목으로 저장할 것인가의 문제입니다. 이것을 **메모리 세분성(memory granularity)**이라고 합니다. 세분성 선택에 따라 메모리의 정확도, 검색 효율, 저장 비용이 크게 달라집니다.

메모리 세분성은 가장 세밀한 단위에서 가장 거친 단위까지 스펙트럼을 이룹니다. 가장 세밀한 단위는 **턴(turn)**입니다. 대화에서 하나의 발화(사용자 한 번 말하기 또는 에이전트 한 번 응답하기)를 하나의 메모리 단위로 저장합니다. 장점은 정보가 원자적이어서 정확한 검색이 가능하다는 것입니다. "사용자가 정확히 무슨 말을 했는지"를 그대로 인출할 수 있습니다. 단점은 개별 턴이 맥락에서 분리되면 의미가 불완전해질 수 있다는 것입니다. "그건 좋은 생각이야"라는 턴은 앞뒤 맥락 없이는 무엇을 가리키는지 알 수 없습니다. 또한 턴의 수가 매우 많으므로 저장 공간과 검색 대상이 폭발적으로 늘어납니다.

중간 수준의 단위가 **세그먼트(segment)**입니다. 여러 턴을 하나의 주제 단위로 묶은 것입니다. 대화에서 "여행 계획"에 관한 5개 턴, "저녁 식사"에 관한 3개 턴이 각각 하나의 세그먼트가 됩니다. 단일 턴보다 맥락이 풍부하면서도 전체 세션보다 초점이 분명합니다.

더 거친 단위는 **세션(session)**입니다. 하나의 대화 세션 전체를 하나의 메모리 단위로 저장합니다. 세션 수준에서는 대화의 전체 맥락이 보존되지만, 긴 세션의 경우 하나의 메모리 항목이 수천 토큰에 달하여 검색 시 노이즈가 증가합니다.

가장 거친 단위는 **요약(summary)**입니다. 하나 이상의 세션을 LLM이 압축한 요약문을 메모리로 저장합니다. 저장 공간이 매우 효율적이지만, 압축 과정에서 세부 정보가 손실됩니다.

이 스펙트럼에서 최적의 세분성을 선택하는 것에는 트레이드오프가 존재합니다. 세밀한 단위(턴)는 정보가 정확하지만 단편적이고 저장 비용이 높습니다. 거친 단위(세션, 요약)는 맥락이 풍부하지만 무관한 정보(노이즈)를 포함하여 검색 정밀도가 떨어집니다. 이 트레이드오프를 해결하기 위해 제안된 접근이 **토픽 일관 세그먼트(topic-coherent segment)** 분할입니다.

토픽 일관 세그먼트 분할은 대화를 주제의 흐름에 따라 자연스러운 경계에서 나누는 방식입니다. 하나의 세그먼트 안에서는 동일한 주제가 유지되므로, 턴 단위의 단편성과 세션 단위의 노이즈를 동시에 피합니다. 구체적으로 LLM이 분할 가이드라인을 사용하여 대화를 주제 일관 세그먼트로 분리합니다. 분할의 기준은 "주제가 전환되는 지점"입니다. 사용자가 여행 계획 이야기를 하다가 갑자기 내일 회의 일정을 묻는 지점이 세그먼트 경계가 됩니다.

이 분할의 품질을 높이기 위해 **Self-Reflection 기반 분할 가이드라인 학습**이 사용됩니다. 초기 가이드라인으로 분할한 결과 중 어려운 사례(hard examples)에 대해 LLM이 반성하여 가이드라인을 반복 갱신합니다. "이 지점에서 분할이 잘못된 이유는 무엇인가"를 분석하고, 그 교훈을 가이드라인에 반영합니다. 이 과정은 경사 하강법(SGD)과 유사한 반복 최적화입니다. 1.1.2에서 다룬 Self-Reflection의 원리가 메모리 세분성 최적화에도 적용되는 것입니다.

분할 후에는 **압축 기반 메모리 디노이징(denoising)**이 적용됩니다. 자연어에는 반복, 필러 표현, 불필요한 예의 표현 등 정보 전달에 기여하지 않는 중복성이 많습니다. 이 중복성을 제거하면 검색 시 신호 대 노이즈 비율이 개선됩니다. LLM 기반 텍스트 압축 기법을 사용하여 세그먼트의 핵심 정보만 남기고 불필요한 부분을 제거합니다. 중요한 것은 이 압축이 요약과 다르다는 점입니다. 요약은 정보를 재구성하여 새로운 텍스트를 생성하지만, 디노이징은 원문에서 불필요한 부분만 제거하고 핵심 표현은 그대로 보존합니다. 원문의 표현이 보존되므로 검색 시 키워드 매칭의 정확도가 유지됩니다.

세분성 설계의 또 다른 접근은 **구조화된 bullet 컬렉션**입니다. 대화에서 추출한 정보를 자유 형식 텍스트가 아니라, 메타데이터(고유 ID, helpful/harmful 카운터)와 콘텐츠(전략, 실패 모드)로 구성된 구조화된 항목으로 저장합니다. 각 bullet은 독립적인 메모리 단위로, 태스크 수행 중 유용했던 전략에는 helpful 카운터가 증가하고, 해로웠던 정보에는 harmful 카운터가 증가합니다. 이 카운터를 기반으로 시간이 지남에 따라 유용한 정보는 보존하고 해로운 정보는 제거하는 자율적 큐레이션이 가능해집니다. 2.5.1에서 다룰 메모리 업데이트와 통합에서 이 메커니즘을 더 자세히 살펴봅니다.

전체 이력을 그대로 삽입하는 것이 항상 좋을 것 같지만, 실제로는 그렇지 않다는 점도 주목해야 합니다. 모든 대화 이력을 그대로 컨텍스트에 주입하면 오히려 성능이 하락하는 현상이 관찰됩니다. 무관한 정보가 주의를 분산시키기 때문입니다. 이것은 2.1.1에서 다룬 Lost in the Middle 현상과도 연결됩니다. 적절한 세분성으로 분할하고, 현재 태스크에 관련된 세그먼트만 선택적으로 검색하는 것이 전체 이력 삽입보다 우수한 성능을 보입니다. 이것이 메모리 세분성 설계가 단순한 저장 효율의 문제가 아니라, 에이전트의 추론 품질에 직접 영향을 미치는 설계 결정인 이유입니다.

정리하면, 메모리 세분성은 턴, 세그먼트, 세션, 요약의 스펙트럼을 이루며, 세밀한 단위는 정확하지만 단편적이고, 거친 단위는 맥락이 풍부하지만 노이즈가 증가합니다. 토픽 일관 세그먼트 분할은 주제 흐름에 따른 자연스러운 경계에서 대화를 나누어 이 트레이드오프를 완화하며, Self-Reflection 기반 가이드라인 학습으로 분할 품질을 높입니다. 압축 기반 디노이징은 원문의 핵심 표현을 보존하면서 노이즈를 제거하고, 구조화된 bullet 컬렉션은 helpful/harmful 카운터로 자율적 큐레이션을 가능하게 합니다.

다음 단원인 2.3.1에서는 에이전트가 환경과 상호작용하면서 메모리를 형성하는 과정, 즉 메모리 후보를 식별하고 구조화된 메타데이터를 자동 생성하는 메커니즘을 살펴봅니다.

이 단원을 마치면 턴, 세그먼트, 세션, 요약 등 메모리 세분성 스펙트럼의 각 단위를 비교하고, 토픽 일관 세그먼트 분할 전략과 압축 기반 디노이징 기법의 작동 원리를 설명할 수 있습니다.
