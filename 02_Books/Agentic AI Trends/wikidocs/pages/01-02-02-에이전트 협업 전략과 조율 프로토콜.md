# 1.2.2 에이전트 협업 전략과 조율 프로토콜

1.2.1에서 다중 에이전트 시스템의 네 가지 구조 유형을 살펴보았습니다. 구조가 에이전트들이 어떻게 연결되는지를 결정한다면, 이 단원에서 다룰 협업 전략과 조율 프로토콜은 연결된 에이전트들이 실제로 어떤 방식으로 함께 일하는지를 결정합니다. 같은 중앙집중형 구조라 하더라도 에이전트들이 협력하는지, 경쟁하는지, 아니면 둘 다 하는지에 따라 시스템의 행동이 완전히 달라집니다.

에이전트들이 상호작용하는 유형은 크게 세 가지로 나뉩니다. 첫 번째는 **협력(Cooperation)**입니다. 모든 에이전트가 공동 목표를 공유하며, 개별 에이전트의 성공이 시스템 전체의 성공과 일치합니다. 1.1.3에서 다룬 소프트웨어 팀 구조가 전형적인 협력 유형입니다. Product Manager, Architect, Engineer, QA 에이전트들이 소프트웨어 완성이라는 단일 목표를 향해 분업합니다. 각 에이전트의 산출물이 다음 단계의 입력이 되는 파이프라인 구조에서, 태스크 분해가 명확하고 에이전트 간 역할 충돌이 없으며 설계가 단순하다는 장점이 있습니다. 반면 한 에이전트가 오류를 내면 전체 파이프라인이 영향을 받고, 에이전트 수가 늘어날수록 통신 비용이 급격히 증가합니다.

두 번째는 **경쟁(Competition)**이며, LLM 기반 다중 에이전트 시스템에서는 주로 디베이트(debate) 형식으로 구현됩니다. 동일한 질문에 대해 여러 LLM 인스턴스가 독립적으로 답변을 생성한 뒤, 서로의 답변을 비판하고 반론하는 라운드를 반복합니다. 최종 답변은 수렴된 공통 응답으로 결정됩니다. 이 접근은 수학 추론과 사실 확인 태스크에서 단일 에이전트 대비 일관된 성능 향상을 보였으며, 환각을 감소시켰습니다. 상호 비판을 통해 논리적 오류와 편향이 드러나는 것이 핵심 장점입니다. 그러나 에이전트들이 오류를 고집하며 거짓 합의에 도달하는 위험이 있고, 라운드마다 토큰 비용이 선형으로 증가합니다.

세 번째는 **협경(Coopetition)**으로, 협력과 경쟁이 동시에 존재하는 혼합 유형입니다. 일부 에이전트는 공동 목표를 향해 협력하면서 동시에 다른 에이전트들과는 경쟁합니다. 여러 전문가 모델이 최종 출력에 기여하려고 경쟁하는 동시에, 게이팅 메커니즘이 가장 적합한 전문가를 선택하면 선택된 전문가들이 협력하여 하나의 응답을 완성하는 구조가 대표적 사례입니다. 통신 네트워크 자원 관리에서도 각 기지국 에이전트가 전체 네트워크 성능 향상을 위해 협력하면서 제한된 주파수 자원 배분을 놓고 경쟁하는 협경 패턴이 사용됩니다. 다양성과 효율성을 동시에 추구할 수 있다는 장점이 있지만, 언제 협력하고 언제 경쟁할지를 결정하는 메타 전략 설계가 복잡합니다.

에이전트들이 이 세 가지 유형 중 어떤 방식으로 상호작용할지를 결정하는 것이 전략(strategy)입니다. **역할 기반 전략**은 에이전트마다 특정 역할과 페르소나를 부여하여 분업 구조를 만드는 방식으로, 1.1.3에서 다룬 내용의 직접적인 적용입니다. 각 에이전트는 시스템 프롬프트에 명시된 자신의 역할 범위 내에서만 행동합니다. 실제 구현에서는 대화의 시작마다 에이전트 자신의 역할, 목표, 허용된 행동을 명시적으로 재확인하는 "Inception Prompting" 기법이 사용되어 역할 혼동을 방지합니다. 복잡한 태스크를 전문화된 서브태스크로 분해할 수 있고 모듈성과 재사용성이 높지만, 역할이 잘못 명세되면 역할 이탈이나 책임 공백이 발생하고, 미리 정의되지 않은 태스크 유형에 대응하기 어렵습니다.

**규칙 기반 전략**은 에이전트 행동을 사전에 정의된 규칙으로 제어합니다. 규칙은 턴 순서, 발언 기회, 의사결정 절차 등을 명시합니다. 대표적인 규칙으로는 디베이트의 라운드 수 제한(예: 3개 에이전트, 2라운드), 최대 턴 수 도달 시 대화 강제 종료, 다수결 투표로 최종 답변 선택, 특정 에이전트가 특정 에이전트에게만 발언권을 넘길 수 있는 전환 규칙 등이 있습니다. 예측 가능성과 일관성이 높고 무한 루프나 예산 초과를 방지할 수 있지만, 예상치 못한 상황에 적응하지 못하며 규칙이 많아지면 확장성이 떨어집니다.

**모델 기반 전략**은 에이전트가 환경의 불확실성과 다른 에이전트의 심리적 상태를 내적으로 모델링하여 동적으로 전략을 결정하는 접근입니다. Theory of Mind(ToM) 추론이 대표적으로, 에이전트가 협력 파트너의 믿음, 의도, 지식 수준을 추론하고, 그 추론을 바탕으로 자신의 발화 방식과 행동을 조정합니다. "상대가 무엇을 아는가"(1차 ToM)를 넘어 "상대가 내가 무엇을 아는지 아는가"(2차 ToM)까지 추론 깊이를 확장하면 협력 성능이 향상된다는 연구 결과가 있습니다. 태스크 진행 상황에 따라 에이전트가 자신의 역할을 자율적으로 갱신하는 동적 역할 재할당, 대화 내용에 따라 활성화할 에이전트를 동적으로 선택하는 메커니즘 등도 모델 기반 전략에 해당합니다. 동적 환경에 적응할 수 있지만 구현 복잡도와 추론 비용이 높습니다.

협업 전략이 "무엇을 할 것인가"를 결정한다면, **조율 프로토콜**은 "어떤 형식으로, 어떤 순서로, 언제까지, 어떻게 합의할 것인가"를 결정합니다. 조율 프로토콜은 메시지 형식, 턴 순서, 종료 조건, 합의 메커니즘의 네 가지 요소로 구성됩니다.

**메시지 형식**은 에이전트 간 통신의 기본 단위를 정의합니다. 세 가지 형식이 실제로 사용됩니다. 자연어 메시지는 역할 정보와 내용을 포함한 사전(dictionary) 형태로 교환하며, 인간이 읽을 수 있어 디버깅이 쉽지만 구조화된 파싱에는 취약합니다. 구조화 메시지는 JSON-RPC 같은 형식의 요청과 응답을 사용하여 표준화된 도구 호출 패턴을 구현합니다. 산출물 기반 메시지는 1.2.1에서 언급한 어셈블리 라인 구조에서처럼 에이전트의 결과물 자체, 즉 요구사항 문서, 설계 다이어그램, 코드 파일 등이 메시지가 되는 방식입니다.

**턴 순서**는 에이전트들이 언제 발언하는지를 결정합니다. 다섯 가지 방식이 사용됩니다. 순차적 고정 순서는 역할 순서가 미리 정해져 있어 예측 가능하지만 유연하지 않습니다. 라운드로빈은 에이전트들이 돌아가며 발언하여 동등한 기회를 보장합니다. LLM 자동 선택은 현재 대화 맥락을 바탕으로 LLM이 다음 발언자를 결정하는 것으로 가장 유연하지만 선택 자체에 추가 비용이 듭니다. 제약된 전환은 허용된 발언자 전환 경로만 허용하여 통제된 비선형 흐름을 구현합니다. 전파(relay)는 에이전트 A의 결과를 에이전트 B에게 넘기는 선형 체인입니다.

**종료 조건**은 에이전트 루프가 언제 멈추는지를 결정합니다. 가장 단순한 것은 최대 턴 수 도달로, 지정된 라운드 수에 이르면 강제 종료됩니다. 텍스트 기반 종료 신호는 에이전트가 특정 문자열(예: "TERMINATE", "TASK_COMPLETE")을 출력하면 종료하는 방식입니다. 합의 임계값 도달은 지정된 결정자 에이전트가 충분한 정보가 수집되었다고 판단할 때 종료합니다. 품질 지표 충족은 외부 평가 메커니즘이 출력의 품질 기준 충족 여부를 판단하여 종료합니다. 오류 상태 감지는 예외 발생이나 응답 불능 상태에서 안전 종료하는 것입니다. 실제 시스템에서는 이러한 조건들이 병행 사용됩니다. 블랙보드 시스템 실험에서 합의율은 문제 난이도에 따라 크게 달랐는데, 상대적으로 쉬운 태스크에서는 약 90%, 어려운 추론 태스크에서는 약 30%에 그쳤습니다.

**합의 메커니즘**은 여러 에이전트의 의견이 갈릴 때 최종 결정을 내리는 방법입니다. 다수결 투표는 가장 많이 등장한 답변을 선택하는 단순한 방식이지만, 다수가 동일한 오류를 공유하면 오답이 선택될 수 있습니다. 유사도 기반 투표는 각 에이전트의 제안과 다른 모든 에이전트의 제안 사이의 의미적 유사도 합을 계산하여, 가장 높은 유사도 합을 가진 제안을 선택합니다. 토론 기반 수렴은 디베이트 방식으로 에이전트들이 서로의 주장을 읽고 자신의 입장을 갱신하면서 점진적으로 합의에 도달하는 것으로, 대화 자체가 합의 과정이 됩니다.

에이전트 간 통신의 시간적 구조도 중요한 설계 결정입니다. **동기식 통신**에서는 에이전트 A가 발언한 뒤 에이전트 B의 응답이 올 때까지 기다린 후 다음 단계로 진행합니다. 모든 에이전트가 항상 동일한 최신 맥락을 공유하므로 인과 관계와 맥락 일관성이 명확하고, 디버깅과 추적이 쉽습니다. 그러나 느린 에이전트가 전체 파이프라인의 병목이 되며 병렬화의 이점을 활용하지 못합니다. **비동기식 통신**에서는 에이전트들이 독립적으로 동작하며 응답을 기다리지 않습니다. 느린 에이전트가 빠른 에이전트를 차단하지 않아 처리량이 높고 확장성이 우수하지만, 에이전트들이 서로 다른 버전의 맥락 위에서 동작하여 일관성 문제가 생길 수 있습니다. 실제 시스템에서는 이벤트 트리거 방식이 시간 트리거 방식보다 효율적이지만 조율 복잡도가 높아지는 트레이드오프가 관찰됩니다.

마지막으로, 에이전트 간 통신의 매개체가 되는 **공유 메모리와 블랙보드 패턴**을 살펴보겠습니다. 공유 메모리는 모든 에이전트가 읽고 쓸 수 있는 공유 저장 공간으로, 개인 단기 메모리(현재 대화 컨텍스트), 개인 장기 메모리(벡터 데이터베이스의 이전 경험), 공유 외부 메모리(모든 에이전트가 접근하는 공유 저장소)로 구분됩니다. 블랙보드 패턴은 이 공유 메모리 개념을 체계화한 것으로, "에이전트들이 서로 직접 통신하지 않고 공유 공간(블랙보드)을 통해서만 상호작용한다"는 원칙입니다. 메인 에이전트가 블랙보드에 요청을 게시하면, 각 보조 에이전트가 자율적으로 자신이 해당 요청에 응답할 수 있는지 판단하여 응답 보드에 답변을 게시합니다. 메인 에이전트는 모든 답변을 취합하여 최종 해결책을 생성합니다. 블랙보드 패턴의 핵심 장점은 느슨한 결합(loose coupling)입니다. 에이전트들이 서로를 알 필요가 없고, 새 에이전트를 추가할 때 기존 코드를 수정하지 않아도 됩니다. 직접 메시지 전달 대비 에이전트 수 증가 시 연결 수가 선형으로만 증가하여 확장성이 우수합니다. 제어 유닛이 다음 행동 에이전트를 결정하면 토큰 소비를 효과적으로 관리할 수 있으며, 제어 유닛이 없는 경우 동일 태스크에서 토큰 소비가 약 4배로 폭증합니다.

정리하면, 다중 에이전트 시스템의 협업 유형은 협력, 경쟁(디베이트), 협경으로 나뉘며 각 유형은 목표 구조와 상호작용 동학이 다릅니다. 이 유형 위에서 역할 기반, 규칙 기반, 모델 기반 전략이 에이전트들의 구체적 행동 방식을 결정합니다. 조율 프로토콜은 메시지 형식, 턴 순서, 종료 조건, 합의 메커니즘으로 구성되어 에이전트 간 상호작용의 절차적 틀을 제공하며, 동기식과 비동기식 통신, 공유 메모리와 블랙보드 패턴이 통신의 시간적, 공간적 구조를 결정합니다.

다음 단원인 1.2.3에서는 이러한 전략과 프로토콜이 실제로 적용된 대표적 사례인 SOP 기반 협업 구조를 자세히 살펴보고, 구조화된 산출물이 연쇄 환각을 어떻게 억제하는지 다룹니다.

이 단원을 마치면 협력, 경쟁, 협경 세 가지 협업 유형의 차이를 설명하고, 역할 기반, 규칙 기반, 모델 기반 전략과 조율 프로토콜의 설계 원리를 기술할 수 있습니다.
