# 2.4.1 메모리 검색 기법

2.3절에서 메모리를 생성하고 저장하는 과정을 살펴보았습니다. 메모리가 아무리 잘 구축되어 있어도, 필요한 시점에 관련 정보를 정확하고 빠르게 찾아낼 수 없다면 쓸모가 없습니다. 이 단원에서는 저장된 메모리에서 현재 태스크에 관련된 정보를 인출하는 검색 기법들을 살펴봅니다.

가장 기본적이고 널리 사용되는 검색 기법은 **벡터 임베딩 기반 의미 검색(semantic search)**입니다. 이 기법은 쿼리(현재 사용자의 질문이나 에이전트의 현재 맥락)와 메모리 항목 각각을 동일한 임베딩 공간의 벡터로 변환한 뒤, 코사인 유사도(cosine similarity)를 계산하여 가장 유사한 메모리를 반환합니다. "Python 데이터 분석"이라는 쿼리는 "pandas를 사용한 CSV 처리"라는 메모리와 높은 유사도를 가지게 됩니다. 두 텍스트가 동일한 단어를 공유하지 않더라도 의미적으로 관련된 메모리를 찾을 수 있다는 것이 의미 검색의 핵심 장점입니다. 반면 정확한 키워드 매칭이 필요한 경우, 예를 들어 특정 함수명이나 오류 코드를 포함한 메모리를 찾을 때는 의미 검색이 부정확할 수 있습니다.

이 한계를 보완하는 것이 **BM25 키워드 기반 검색**입니다. BM25는 전통적인 정보 검색에서 사용되는 희소(sparse) 표현 기반 검색으로, 쿼리에 포함된 키워드가 메모리 항목에 정확히 등장하는 빈도를 기반으로 점수를 매깁니다. "pandas merge error"라는 쿼리가 주어지면, "pandas", "merge", "error"라는 단어가 정확히 포함된 메모리 항목이 높은 점수를 받습니다. 키워드 검색은 정확한 용어 매칭에 강하지만, 동의어나 의미적으로 관련된 표현을 놓칠 수 있습니다.

두 기법의 강점을 결합한 것이 **하이브리드 검색**입니다. 의미 검색과 키워드 검색을 동시에 수행하고, 두 결과를 결합하여 최종 검색 결과를 생성합니다. 의미 검색이 의미적 유사성을, 키워드 검색이 정확한 용어 매칭을 각각 담당하여, 단독 사용보다 검색 정밀도와 재현율 모두에서 우수한 성능을 보입니다. 결합 방식으로는 두 검색의 점수를 가중합하는 방법, Reciprocal Rank Fusion(RRF)으로 순위를 통합하는 방법 등이 사용됩니다.

에이전트 메모리 시스템에서는 하나의 검색만으로 충분하지 않은 경우가 많습니다. **다중 메모리 컴포넌트 병렬 검색**은 여러 메모리 유형을 동시에 조회하여 결과를 통합하는 방식입니다. 2.1.3에서 다룬 것처럼 에이전트 메모리는 에피소드 기억, 의미 기억, 절차 기억 등 여러 유형으로 나뉘며, 이들이 서로 다른 저장소에 보관됩니다. 사용자의 질문이 들어오면, 에피소드 기억 저장소에서 관련 과거 경험을, 의미 기억 저장소에서 관련 사실 지식을, 절차 기억 저장소에서 관련 워크플로우를 동시에 검색하고, 각 결과에 소스 태그를 붙여 통합합니다. 에이전트는 통합된 결과에서 소스 태그를 확인하여 어떤 유형의 메모리에서 온 정보인지 구별할 수 있습니다.

단일 검색으로 충분한 정보를 얻지 못하는 경우에는 **반복적 검색(iterative retrieval)**이 사용됩니다. 첫 번째 검색 결과를 바탕으로 쿼리를 정제하거나 새로운 검색 방향을 설정하여 추가 검색을 수행합니다. 2.2.1에서 다룬 OS 가상 메모리 비유의 에이전트 시스템에서, 에이전트가 함수 호출 체이닝(function chaining)을 통해 Recall Storage를 여러 번 검색하며 필요한 정보를 단계적으로 수집하는 것이 반복적 검색의 대표적 사례입니다. 첫 검색에서 "사용자가 Python 프로젝트를 진행 중"이라는 정보를 찾고, 이를 바탕으로 "Python 프로젝트에서 발생한 이전 오류들"을 두 번째로 검색하는 식입니다.

마지막으로, 검색의 시점을 결정하는 **Active Retrieval** 설계 원칙이 있습니다. Active Retrieval은 에이전트가 매 사용자 입력마다 능동적으로 메모리를 조회하는 방식입니다. 사용자가 질문을 하면 에이전트가 응답을 생성하기 전에 먼저 메모리를 검색하여, 관련 과거 경험이나 사용자 정보를 컨텍스트에 포함시킵니다. 이것은 사용자나 시스템이 명시적으로 검색을 요청할 때만 작동하는 수동적 저장소(passive storage)와 대조됩니다. Active Retrieval의 장점은 에이전트가 항상 가용한 메모리의 맥락 안에서 응답하므로 일관성이 높아진다는 것이고, 단점은 매 입력마다 검색 비용이 추가된다는 것입니다.

정리하면, 메모리 검색 기법은 벡터 임베딩 기반 의미 검색(의미적 유사성), BM25 키워드 검색(정확한 용어 매칭), 하이브리드 검색(두 기법의 결합)의 세 가지 기본 방식으로 나뉩니다. 다중 메모리 컴포넌트 병렬 검색은 여러 유형의 메모리를 동시에 조회하여 통합하고, 반복적 검색은 다중 홉 탐색으로 복잡한 정보를 단계적으로 수집합니다. Active Retrieval은 매 입력마다 능동적으로 메모리를 조회하는 설계 원칙입니다.

다음 단원인 2.4.2에서는 기본 검색의 품질을 높이는 고급 기법들과, 메모리 크기와 검색 품질의 비선형 관계 등 검색의 근본 문제를 살펴봅니다.

이 단원을 마치면 벡터 임베딩 기반 의미 검색, BM25 키워드 검색, 하이브리드 검색의 원리를 비교하고, 다중 메모리 컴포넌트 병렬 검색과 반복적 검색의 동작 방식을 설명할 수 있습니다.
