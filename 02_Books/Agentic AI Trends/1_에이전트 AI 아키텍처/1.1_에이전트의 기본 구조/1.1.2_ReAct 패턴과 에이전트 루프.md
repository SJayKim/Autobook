# 1.1.2 ReAct 패턴과 에이전트 루프

1.1.1에서 에이전트가 Thought-Action-Observation 사이클을 반복하며 환경과 상호작용한다는 것을 살펴보았습니다. 이 단원에서는 이 루프의 가장 대표적인 구현인 ReAct 패턴을 깊이 있게 다루고, 에이전트 루프를 확장하는 두 가지 핵심 메커니즘인 Self-Reflection과 Self-Refine을 살펴봅니다.

LLM 기반 에이전트가 등장하기 전까지, 언어 모델을 활용한 추론과 행동은 서로 분리된 문제로 취급되었습니다. 한쪽에는 Chain-of-Thought(CoT)로 대표되는 순수 추론 방식이 있었습니다. CoT는 최종 답변에 도달하기 전에 중간 추론 단계를 명시적으로 전개하는 기법으로, 수학 문제나 논리 퍼즐에서 현저한 성능 향상을 보여주었습니다. 그러나 CoT는 모델 내부에 이미 저장된 지식만을 사용합니다. 학습 시점 이후의 정보에는 접근할 수 없고, 초기 추론 단계에 잘못된 전제가 포함되면 이후 모든 추론이 그 위에 쌓여 오류가 증폭됩니다. 다른 한쪽에는 모방 학습이나 강화학습으로 행동 시퀀스를 학습하는 방식이 있었습니다. 이 방식은 외부 환경과 상호작용할 수 있지만, 왜 그 행동을 선택했는지에 대한 추론 과정이 없어 새로운 상황에 적응하기 어렵습니다.

**ReAct(Reasoning + Acting)**는 이 두 가지 접근을 하나로 합친 프레임워크입니다. 핵심 아이디어는 추론 단계와 행동 단계를 번갈아 수행하는 것입니다. CoT처럼 중간 추론을 언어로 전개하되, 추론 사이사이에 외부 도구를 호출하고 그 결과를 다음 추론의 입력으로 사용합니다. 이렇게 하면 추론이 외부 사실에 의해 지속적으로 교정되고, 동시에 행동이 추론에 의해 논리적으로 정당화됩니다.

ReAct의 세 요소를 구체적으로 살펴보겠습니다. **Thought(사고)**는 모델이 현재 상황을 분석하고 다음에 할 일을 언어로 계획하는 단계입니다. 이 단계는 외부에 어떤 부작용도 남기지 않는 순수한 내부 추론입니다. 예를 들어 "콜로라도 조로제니의 동쪽 섹터가 어디로 뻗어 있는지 검색해야 한다"처럼 구체적인 탐색 목표를 명시합니다. **Action(행동)**은 외부 도구나 환경과 실제로 인터페이스하는 단계입니다. 검색 API 호출, 코드 실행, 데이터베이스 쿼리 등 부작용을 발생시키는 모든 행위가 여기에 해당합니다. **Observation(관찰)**은 Action의 결과로 환경이 에이전트에게 돌려주는 피드백입니다. 검색 결과 텍스트, API 응답, 오류 메시지 등 모든 형태를 포함합니다.

이 세 요소가 실제로 어떻게 전개되는지 질의응답 예시로 설명하겠습니다. "콜로라도 조로제니의 동쪽 섹터가 뻗어 들어가는 고원의 해발고도는?" 이라는 질문이 주어졌다고 합시다. 에이전트의 첫 번째 Thought는 "콜로라도 조로제니를 먼저 검색해야 한다"입니다. 이에 따라 Action으로 검색 도구를 호출합니다. Observation으로 위키피디아 단락이 반환됩니다. 두 번째 Thought는 반환된 텍스트에서 동쪽 섹터 정보를 찾아야 한다고 판단합니다. Action으로 텍스트 내 조회를 실행하고, Observation에서 "동쪽 섹터는 High Plains로 뻗어 들어간다"는 정보를 얻습니다. 세 번째 Thought는 이제 High Plains의 해발고도를 찾아야 한다고 전환합니다. 검색 후 해발고도 정보를 확인한 에이전트는 최종적으로 답변을 출력하고 루프를 종료합니다.

여기서 주목할 점은, Observation에서 "High Plains"라는 새로운 정보가 나왔을 때 다음 Thought가 즉시 방향을 전환했다는 것입니다. 이것이 순수 CoT와의 근본적인 차이입니다. CoT는 모든 추론이 모델 내부 지식에만 의존하므로, 처음에 잘못된 가정을 하면 이후 추론 전체가 틀어집니다. ReAct는 매 Observation마다 외부 사실과 대조하여 추론을 교정합니다. 이 차이를 비교하면 다음과 같습니다. CoT는 Thought만 이어지는 선형 구조, 즉 T1에서 T2, T3를 거쳐 최종 답변에 이르는 흐름입니다. ReAct는 T1 다음에 Action과 Observation이 삽입되고, 그 결과를 반영한 T2가 이어지는 교차 구조입니다.

ReAct 루프는 에이전트가 명시적으로 종료 행동을 출력할 때 끝납니다. 충분한 정보를 수집했다고 Thought에서 판단하면 최종 답변을 담은 종료 행동을 생성하고 루프를 빠져나옵니다. 암묵적으로 멈추는 것이 아니라 명시적 종료 행동을 두면, 추론-행동 사이클과 최종 응답 생성 사이의 경계가 명확해집니다. 사실 검증 벤치마크와 질의응답 벤치마크에서 ReAct는 CoT 대비 환각과 오류 전파를 대폭 줄였고, 가상 가정 환경과 온라인 쇼핑 시뮬레이터에서는 모방학습 및 강화학습 기준선 대비 절대 성공률을 크게 향상시켰습니다. 이 결과는 문맥 내 예시 1~2개만 사용한 조건에서 달성되었습니다.

ReAct가 매 행동 직후에 추론을 교정하는 **미시 루프(micro loop)**라면, **Self-Reflection**은 한 번의 시도 전체가 실패한 뒤에 교훈을 추출하는 **중간 루프(meso loop)**입니다. Self-Reflection의 출발점은 이런 문제입니다. 전통적인 강화학습은 파라미터를 직접 수정하여 정책을 개선하지만, LLM의 파라미터를 매 실패마다 업데이트하는 것은 비용이 너무 높고 추론 시점에는 불가능합니다. 그렇다면 파라미터를 고치지 않고도 에이전트의 행동을 개선할 방법은 없을까요.

Self-Reflection의 해법은 언어 자체를 경사(gradient) 대신 사용하는 것입니다. 실패 후 모델이 자신의 잘못된 행동을 자연어로 분석하고, 그 분석 내용을 다음 시도의 프롬프트에 포함시킵니다. 이렇게 하면 같은 실수를 반복할 가능성이 낮아집니다. 이 메커니즘은 세 가지 구성 요소로 이루어집니다. **Actor(행동자)**는 ReAct 형식으로 환경과 상호작용하며 궤적을 생성하는 LLM입니다. **Evaluator(평가자)**는 생성된 궤적을 평가하여 성공 또는 실패를 판정합니다. 추론 태스크에서는 정답 일치 여부, 프로그래밍 태스크에서는 테스트 코드 실행 결과를 사용합니다. **Self-Reflection 모델**은 핵심 구성 요소로, 실패한 궤적과 평가 결과를 함께 입력받아 "어떤 행동이 잘못되었고, 대신 무엇을 했어야 하는지"를 자연어로 서술한 반성 요약을 생성합니다.

Self-Reflection의 동작 과정을 단계별로 설명하겠습니다. 첫째, Actor가 초기 궤적을 생성하고 Evaluator가 이를 평가합니다. 둘째, 실패한 경우 Self-Reflection 모델이 궤적을 분석하여 반성 요약을 생성합니다. 예를 들어 "3번째 행동에서 이미 방문한 위치를 다시 방문했다. 다음에는 방문 기록을 추적하여 중복을 피해야 한다"와 같은 구체적인 피드백입니다. 셋째, 이 반성 요약을 에피소딕 메모리 버퍼에 저장합니다. 이 버퍼는 이전 시도들에서 추출한 교훈의 누적 목록입니다. 넷째, 다음 시도에서 Actor는 메모리 버퍼의 반성들을 프롬프트에 포함하여 새로운 궤적을 생성합니다. "이전에 이런 실수를 했다"는 맥락이 이미 컨텍스트에 있으므로 같은 실수를 피하게 됩니다. 이 과정을 태스크 성공 또는 최대 시도 횟수에 도달할 때까지 반복합니다.

가상 가정 환경처럼 에이전트가 무한 루프에 빠질 수 있는 태스크에서는, 동일한 행동이 3회 이상 반복되거나 총 행동 수가 30을 초과하면 에피소드를 강제 종료하고 Self-Reflection을 실행하여 "왜 루프가 발생했는지"를 분석합니다. 프로그래밍 태스크에서는 더욱 구체적으로 동작합니다. 에이전트가 먼저 단위 테스트를 자동 생성하고, 코드를 실행하여 통과 여부를 확인한 뒤, 실패한 경우 어떤 테스트 케이스에서 어떤 이유로 실패했는지를 반성 텍스트로 작성합니다. 이 접근은 코딩 벤치마크에서 91%의 1회 통과율을 달성하여, 당시 가장 강력한 모델의 80%를 초과하는 결과를 보였습니다.

Self-Reflection이 시도 간(inter-trial) 개선이라면, **Self-Refine**은 단일 출력을 반복적으로 세밀하게 다듬는 시도 내(intra-trial) 개선입니다. Self-Reflection이 "다음번에는 더 잘하자"라면, Self-Refine은 "이 답을 지금 당장 더 좋게 만들자"입니다. Self-Refine의 핵심 아이디어는 단일 LLM이 생성자, 비평가, 개선자 역할을 동시에 수행한다는 것입니다. 별도의 학습 데이터나 추가 학습 없이 작동합니다.

Self-Refine의 루프는 다음과 같이 전개됩니다. 첫 번째 단계에서 LLM이 입력을 받아 초기 출력을 생성합니다. 이것은 일반적인 단일 생성과 동일합니다. 두 번째 단계에서 동일한 LLM이 자신의 출력을 평가하여 피드백을 생성합니다. 이 피드백은 두 부분으로 구성됩니다. 하나는 위치 지정으로, 어디가 문제인지 특정합니다. 예를 들어 "3번째 문장에서 감정 표현이 중립적이다"와 같이 구체적인 위치를 짚습니다. 다른 하나는 개선 지시로, 무엇을 어떻게 바꿔야 하는지 명시합니다. 코드 최적화라면 "이 루프는 이차 복잡도를 가지며, 해시맵을 사용하면 선형 복잡도로 개선 가능하다"처럼 구체적인 방향을 제시합니다. 세 번째 단계에서 LLM이 원본 출력, 피드백, 이전 이력을 모두 참조하여 개선된 출력을 생성합니다. 네 번째 단계에서 개선된 출력에 대해 다시 피드백을 생성하고 정제합니다. 이 사이클을 피드백이 "더 이상 개선할 점이 없다"를 반환하거나, 최대 반복 횟수에 도달하거나, 품질 지표가 임계값을 초과할 때까지 반복합니다. 7개의 서로 다른 태스크에서 Self-Refine은 동일 모델의 단일 생성 대비 평균 약 20퍼센트 포인트의 절대 성능 향상을 달성했습니다.

에이전트 루프에서 빠질 수 없는 요소가 **오류 복구 메커니즘**입니다. 에이전트가 실제 환경에서 동작하면 도구 호출 실패, 잘못된 매개변수, 환경 상태와 에이전트 내부 모델의 불일치, 일시적 네트워크 장애 등 다양한 오류가 발생합니다. ReAct의 Thought-Action-Observation 구조 자체가 기본적인 오류 복구를 내포하고 있습니다. Action이 예상과 다른 Observation을 반환하면, 예를 들어 검색 결과가 비어 있거나 오류 메시지가 돌아오면, 다음 Thought는 그 정보를 흡수하여 전략을 수정합니다. 이것은 관찰 기반 복구(observation-driven recovery)입니다. 오류를 특별한 예외로 처리하는 것이 아니라, 정상적인 관찰의 한 종류로 취급합니다.

그러나 이 내재적 복구가 항상 작동하는 것은 아닙니다. 올바른 궤적만 학습한 에이전트는 오류 상황을 경험한 적이 없기 때문에, 환경이 명시적 부정 피드백을 제공해도 동일한 잘못된 행동을 반복하는 현상이 관찰됩니다. 이 문제를 해결하기 위해, 의도적으로 오류를 포함한 궤적과 그 수정 과정을 학습 데이터로 구성하는 접근이 제안되었습니다. 에이전트 역할과 환경 심판 역할을 동시에 수행하는 LLM이 매개변수 오류, 논리 오류, 위치 오류 등을 발생시키고, 오류 피드백에 기반한 수정 행동을 생성합니다. 이때 핵심적인 기법은 오류 발생 시점의 사고와 행동에 대한 학습 손실을 차단(마스킹)하는 것입니다. 모델이 잘못된 추론 과정을 학습하지 않도록 방지하고, 오직 수정된 올바른 행동만 학습합니다. 이 마스킹을 제거하면 성능이 평균 75퍼센트 하락한다는 실험 결과는 이 기법의 중요성을 보여줍니다.

시스템 수준에서는 추가적인 오류 복구 패턴들이 적용됩니다. 일시적 오류에 대해서는 지수 백오프(exponential backoff)를 적용하여, 첫 번째 재시도는 1초 후, 두 번째는 2초 후, 세 번째는 4초 후처럼 대기 시간을 지수적으로 늘립니다. 장기 실행 에이전트에서는 각 단계 완료 시점에 체크포인트를 저장하여, 오류 발생 시 처음부터 재시작하지 않고 가장 최근 체크포인트부터 재개합니다. 행동의 부작용이 실제로 발생했는지 확인하는 환경 검증(post-action verification)도 중요합니다. 파일을 작성한 후 파일이 존재하는지, 데이터베이스에 레코드를 삽입한 후 조회가 되는지 확인하는 것입니다. 오류 없이 실행되었지만 실제로는 아무 변화도 없는 침묵 실패(silent failure)를 이 방법으로 탐지할 수 있습니다.

지금까지 살펴본 ReAct, Self-Reflection, Self-Refine, 오류 복구는 모두 **관찰-사고-행동의 반복 사이클**이라는 동일한 구조를 공유합니다. 이 사이클은 제어 이론의 피드백 제어 루프, 심리학의 OODA 루프(관찰-방향설정-결정-행동), 강화학습의 에이전트-환경 상호작용과 구조적으로 동형입니다. 다만 이 루프가 작동하는 시간 척도가 다릅니다. ReAct는 단일 행동-관찰 주기인 미시 루프에서 작동합니다. Self-Reflection은 하나의 태스크 시도 전체를 단위로 하는 중간 루프에서 작동합니다. 자율 집필이나 에이전트 설계 탐색처럼 복수의 태스크에 걸치는 경우는 거시 루프에 해당합니다. 이 중첩된 루프 구조가 현대 에이전트 시스템의 기본 뼈대입니다.

정리하면, ReAct 패턴은 추론과 행동을 교차 삽입하여 LLM의 내부 추론을 외부 사실로 지속적으로 교정하는 프레임워크입니다. Self-Reflection은 시도 실패 후 자연어 반성을 메모리에 축적하여 파라미터 업데이트 없이 행동 정책을 개선합니다. Self-Refine은 단일 LLM이 생성-비평-개선 역할을 순환하며 출력 품질을 단계적으로 끌어올립니다. 오류 복구는 ReAct 루프에 내재된 관찰 기반 복구를 기본으로 하되, 오류-수정 궤적 학습과 시스템 수준 패턴으로 보강됩니다. 이 모든 메커니즘은 관찰-사고-행동의 반복 사이클이라는 동일한 구조 위에서 서로 다른 시간 척도로 작동합니다.

다음 단원인 1.1.3에서는 에이전트에 특정 역할과 전문성을 부여하는 페르소나 설계를 다루고, 정적 페르소나와 동적 페르소나의 차이, 그리고 에이전트 주체성 스펙트럼의 개념을 살펴봅니다.

이 단원을 마치면 ReAct 패턴의 Reasoning-Acting 교대 구조를 단계별로 설명하고, Self-Reflection 및 Self-Refine 메커니즘이 에이전트 루프를 어떻게 확장하는지 기술할 수 있습니다.
