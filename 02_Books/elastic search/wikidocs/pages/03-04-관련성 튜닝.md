# 3.4 관련성 튜닝

이 섹션에서는 검색 결과의 순위를 결정하는 관련성 스코어링의 원리와 튜닝 방법을 다룹니다. BM25 알고리즘의 수식을 이해하고, 비즈니스 로직을 스코어에 반영하는 방법, 그리고 스코어 이상을 진단하는 방법을 익힙니다.

## 토픽 목록

**3.4.1 BM25 알고리즘**
- BM25 스코어링 공식의 구성 요소를 설명할 수 있다
- TF-IDF와 BM25의 차이를 비교할 수 있다

TF-IDF 기반 스코어링 역사, BM25 공식 구조(TF 포화, 문서 길이 정규화), IDF(역문서 빈도) 계산, TF 포화(saturation) 개념, 문서 길이 정규화, Elasticsearch 기본 유사도로 BM25 사용, similarity 설정으로 변경 방법, DFR/DFI/LMDirichlet 대안 유사도를 다룹니다.

**3.4.2 k1/b 파라미터 튜닝**
- BM25의 k1, b 파라미터가 스코어에 미치는 영향을 설명할 수 있다
- 도메인 특성에 맞는 k1, b 값을 조정할 수 있다

k1 파라미터(TF 포화 속도 조절, 기본값 1.2), b 파라미터(문서 길이 정규화 강도, 기본값 0.75), b=0(문서 길이 무시), b=1(완전 길이 정규화), 짧은 필드(제목)에서의 b 조정 전략, 매핑 similarity 설정으로 인덱스별 파라미터 적용, Explain API로 스코어 검증을 다룹니다.

**3.4.3 function_score 쿼리**
- function_score 쿼리로 비즈니스 로직을 스코어에 반영할 수 있다
- gauss, linear, exp decay 함수를 적용할 수 있다

function_score 쿼리 구조, weight 함수, field_value_factor 함수, gauss/linear/exp decay 함수, random_score 함수, script_score 함수, score_mode: sum/avg/max/min/multiply, boost_mode: replace/multiply/sum/avg/max/min을 다룹니다.

**3.4.4 Painless 스크립팅**
- Painless 스크립트로 커스텀 스코어 계산과 필드 변환을 구현할 수 있다
- 스크립트 캐싱과 파라미터화로 성능을 최적화할 수 있다

Painless 언어 특징(Java 유사, 샌드박스), script_score 쿼리에서 Painless 사용, doc['field'].value 접근 방식, params 파라미터화로 컴파일 캐시 활용, 스크립트 필드(script_fields), update 스크립트에서 ctx._source 사용, Painless 디버깅 방법을 다룹니다.

**3.4.5 Explain API와 스코어 디버깅**
- Explain API로 특정 문서의 스코어 계산 과정을 추적할 수 있다
- 스코어 이상 원인을 진단하고 수정 방향을 제시할 수 있다

GET /{index}/_explain/{id} API, explain: true 파라미터, explanation 객체 구조 해석, IDF 값이 낮게 나오는 원인(샤드별 통계), search_type: dfs_query_then_fetch, 스코어 정규화 문제 진단, 관련성 튜닝 반복 프로세스를 다룹니다.
