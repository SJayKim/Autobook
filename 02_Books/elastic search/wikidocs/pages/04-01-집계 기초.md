# 4.1 집계 기초

이 섹션에서는 Elasticsearch 집계(Aggregations)의 기본 구조와 가장 자주 사용하는 Bucket 집계, Metric 집계를 다룹니다. 집계의 문법 구조를 익히면 이후 중첩 집계와 Pipeline 집계를 자연스럽게 이해할 수 있습니다.

## 토픽 목록

**4.1.1 집계 구문 구조**
- aggs 파라미터의 기본 구조를 작성할 수 있다
- 쿼리와 집계를 결합한 필터링 집계를 수행할 수 있다

aggs/aggregations 키워드, 집계 이름(사용자 정의), 집계 타입 지정, size: 0으로 검색 결과 제외, 쿼리와 집계 조합, 집계 응답 구조(aggregations 객체), 집계 정확도와 doc_count_error_upper_bound를 다룹니다.

**4.1.2 Bucket 집계**
- terms, date_histogram, range, filter 집계로 문서를 그룹화할 수 있다

terms 집계(상위 N개 버킷), date_histogram 집계(시계열 그룹화), calendar_interval vs fixed_interval, range 집계(숫자/날짜 범위 버킷), filter 집계(단일 필터 버킷), filters 집계(복수 명명 필터 버킷), histogram 집계, missing 집계를 다룹니다.

**4.1.3 Metric 집계**
- avg, sum, min, max, stats, cardinality 집계로 수치를 계산할 수 있다

avg/sum/min/max 집계, stats 집계(한번에 통계량 반환), extended_stats 집계(분산, 표준편차), cardinality 집계(HyperLogLog++ 근사), value_count 집계, percentiles 집계, top_hits 집계, geo_centroid 집계를 다룹니다.
