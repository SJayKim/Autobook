# 2.3 텍스트 분석

이 섹션에서는 Elasticsearch가 텍스트를 어떻게 처리하는지, 분석기(analyzer)의 구조와 활용 방법을 다룹니다. 분석기 설계는 검색 품질에 직접적인 영향을 미치며, 한국어 검색에는 별도의 형태소 분석기가 필요합니다.

## 토픽 목록

**2.3.1 분석기 파이프라인**
- 분석기의 3단계(character filter → tokenizer → token filter)를 설명할 수 있다
- _analyze API로 분석 결과를 확인할 수 있다

analyzer 구성요소(character filter, tokenizer, token filter), character filter 역할(html_strip, mapping), tokenizer 역할(standard, whitespace, ngram), token filter 역할(lowercase, stop, synonym), _analyze API 사용법, 인덱스 시 분석기 vs 검색 시 분석기, search_analyzer 별도 지정을 다룹니다.

**2.3.2 내장 분석기**
- standard, simple, whitespace, language 분석기의 차이를 설명할 수 있다
- 각 분석기의 적합한 사용 사례를 선택할 수 있다

standard 분석기(기본값), simple 분석기, whitespace 분석기, stop 분석기, keyword 분석기(분석 없음), fingerprint 분석기, english/french 등 언어별 분석기, snowball 어간 추출기를 다룹니다.

**2.3.3 커스텀 분석기 구성**
- 인덱스 settings에서 커스텀 분석기를 정의하고 매핑에 연결할 수 있다
- 동의어(synonym) 필터를 적용한 분석기를 구성할 수 있다

index.analysis.analyzer 설정 블록, 커스텀 tokenizer 정의, 커스텀 token filter 정의, synonym token filter와 동의어 파일, synonym_graph와 인덱스 시 사용 주의, ngram tokenizer와 edge_ngram tokenizer, 분석기 업데이트와 reindex 필요성을 다룹니다.

**2.3.4 한국어 분석: Nori**
- Nori 플러그인을 설치하고 한국어 형태소 분석을 적용할 수 있다
- nori_tokenizer의 decompound_mode 옵션을 상황에 맞게 선택할 수 있다

Nori 플러그인 설치(elasticsearch-plugin install), nori_tokenizer 설정, decompound_mode: none/discard/mixed, nori_part_of_speech 토큰 필터, nori_readingform 토큰 필터(한자→한글), 사용자 정의 사전(user_dictionary), Nori 분석 결과 검증 예시를 다룹니다.
