# 8.2 ELK 로그 분석

이 섹션에서는 ELK 스택(Elasticsearch + Logstash + Kibana)을 활용하여 서버 로그를 수집, 파싱, 저장, 시각화하는 전체 파이프라인을 구축합니다. 실제 서비스 운영에서 에러를 빠르게 탐지하고 분석하는 방법을 익힙니다.

## 토픽 목록

**8.2.1 로그 수집 파이프라인**
- Filebeat → Logstash → Elasticsearch 로그 수집 파이프라인을 구성할 수 있다

Filebeat 설정과 로그 파일 모니터링, Filebeat modules(nginx, apache, system), Logstash grok 파싱 파이프라인, ECS(Elastic Common Schema) 필드 정규화, 타임스탬프 파싱과 시간대 설정, 인덱스 네이밍 규칙(logs-{service}-{date}), Data Stream 기반 로그 적재, ILM과 로그 보존 정책을 다룹니다.

**8.2.2 Kibana 로그 분석 대시보드**
- Kibana에서 에러율, 응답시간, 트래픽 분석 대시보드를 구성할 수 있다

Kibana Logs UI 활용, 에러 로그 필터링(KQL), 시간별 요청 수 date_histogram, 상위 에러 코드 terms 집계, 응답 시간 percentiles 집계, 지역별 트래픽 geo 시각화, Kibana Alerting으로 에러율 임계치 알림을 다룹니다.
