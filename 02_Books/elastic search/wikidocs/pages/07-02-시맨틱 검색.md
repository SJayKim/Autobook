# 7.2 시맨틱 검색

이 섹션에서는 Elastic의 자체 AI 모델인 ELSER(Elastic Learned Sparse EncodeR)를 활용한 시맨틱 검색을 다룹니다. ELSER의 희소 벡터 방식을 이해하고, dense 벡터 방식과 비교하며, 실제 배포와 검색 구현 방법을 익힙니다.

## 토픽 목록

**7.2.1 ELSER 작동 원리**
- ELSER의 희소 벡터 표현 방식과 BM25와의 차이를 설명할 수 있다

ELSER(Elastic Learned Sparse EncodeR) 개념, 희소 벡터(sparse vector) 표현, learned token expansion 메커니즘, ELSER가 생성하는 확장 토큰 예시, rank_features 타입과 ELSER 연동, ELSER 모델 버전(ELSER v1 vs v2), 쿼리 시 text_expansion 쿼리를 다룹니다.

**7.2.2 ELSER vs dense 벡터 비교**
- ELSER와 dense 벡터 검색의 성능, 정확도, 리소스 트레이드오프를 비교할 수 있다

ELSER 장점(영어 out-of-box), dense 벡터 장점(다국어, 도메인 파인튜닝), 인덱싱 속도 비교, 검색 레이턴시 비교, 메모리/스토리지 사용량 비교, 재현율(recall) 벤치마크, 사용 사례별 선택 가이드를 다룹니다.

**7.2.3 ELSER 배포와 시맨틱 검색 구현**
- ELSER 모델을 배포하고 semantic_text 필드로 시맨틱 검색을 구현할 수 있다

ELSER 모델 다운로드(Kibana ML UI), semantic_text 필드 타입(8.x), semantic 쿼리 타입, text_expansion 쿼리(저수준), 인제스트 파이프라인의 sparse_encoding 프로세서, ELSER 추론 지연과 배포 수 조정, ELSER 인덱싱 파이프라인 최적화를 다룹니다.
