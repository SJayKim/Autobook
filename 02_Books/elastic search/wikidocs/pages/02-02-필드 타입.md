# 2.2 필드 타입

이 섹션에서는 Elasticsearch가 제공하는 다양한 필드 타입을 다룹니다. 올바른 타입을 선택하는 것은 검색 품질, 집계 성능, 저장 효율성 모두에 영향을 미칩니다.

## 토픽 목록

**2.2.1 text vs keyword**
- text와 keyword 타입의 분석 여부 차이를 설명할 수 있다
- 각 타입에 적합한 사용 사례를 구분하여 적용할 수 있다

text 타입(분석기 적용, 전문 검색용), keyword 타입(분석 없음, 정렬·집계·exact match용), text 필드에서 집계 불가 이유, fielddata 활성화(레거시, 비권장), doc_values 구조와 역할, index: false 옵션을 다룹니다.

**2.2.2 숫자/날짜/불리언 타입**
- integer, long, float, double 타입의 선택 기준을 설명할 수 있다
- date 타입의 포맷 설정과 ISO 8601 형식을 적용할 수 있다

integer/long/short/byte 정수 타입, float/double/half_float/scaled_float 실수 타입, date 타입과 포맷 문자열, epoch_millis/strict_date_optional_time, boolean 타입, 숫자 타입 선택 기준(범위·정밀도), 범위 타입(integer_range, date_range 등)을 다룹니다.

**2.2.3 특수 타입: dense_vector, geo_point, nested, join**
- dense_vector 타입으로 벡터 임베딩을 저장할 수 있다
- nested와 join 타입의 사용 목적을 비교할 수 있다

dense_vector 타입과 dims 설정, geo_point 타입과 위도/경도 표현, nested 타입(객체 배열 독립 색인), join 타입(parent-child 관계), IP 타입, binary 타입, completion 타입(자동완성용)을 다룹니다.

**2.2.4 멀티필드 패턴**
- fields 파라미터로 text+keyword 멀티필드를 설정할 수 있다
- 멀티필드를 활용한 전문 검색과 정렬 동시 지원 패턴을 구현할 수 있다

fields 파라미터 문법, title.keyword 패턴, 멀티필드에 다른 분석기 지정, 멀티필드 쿼리 대상 지정 방법, 동적 매핑의 기본 멀티필드 생성 동작, 멀티필드 추가 시 reindex 필요 여부를 다룹니다.
