# 4.3.2 Kibana Dashboard

4.3.1에서 Kibana Lens를 사용하여 개별 시각화를 만드는 방법을 다뤘습니다. 시간에 따른 요청 수 추이를 선 차트로 그리거나, 카테고리별 매출을 막대 차트로 비교하는 등 각각의 시각화는 데이터의 한 단면을 보여 줍니다. 그런데 실무에서는 하나의 차트만으로 시스템 상태를 파악하기 어렵습니다. 요청 수, 오류율, 응답 시간, CPU 사용률 등 여러 지표를 한 화면에 모아 놓고 한눈에 살펴볼 수 있어야 합니다. 이때 사용하는 것이 **Kibana Dashboard**입니다. Dashboard는 여러 시각화 패널을 하나의 화면에 배치하고, 공통 필터를 적용하며, 이를 저장하여 팀과 공유할 수 있는 기능입니다.

Dashboard를 만들려면 Kibana 좌측 메뉴에서 Dashboard를 선택하고, Create dashboard 버튼을 누릅니다. 빈 대시보드 화면이 열리면 상단의 "Add panel" 버튼을 클릭하여 패널을 추가합니다. 패널을 추가하는 방법은 두 가지입니다.

첫째, 4.3.1에서 Visualize Library에 저장해 둔 기존 시각화를 선택하여 추가하는 방법입니다. "Add from library" 탭에서 저장된 시각화 목록이 나타나며, 원하는 항목을 클릭하면 대시보드에 패널로 삽입됩니다. 같은 시각화를 여러 대시보드에서 재사용할 수 있어서, 팀 내에서 표준화된 차트를 공유할 때 유용합니다.

둘째, 대시보드 화면에서 곧바로 새 시각화를 만드는 방법입니다. "Create visualization" 탭을 선택하면 Lens 편집기가 대시보드 안에서 열립니다. 여기서 4.3.1에서 다룬 것과 동일한 방식으로 필드를 축에 매핑하고 차트를 구성한 뒤 저장하면, 해당 시각화가 대시보드 패널로 바로 추가됩니다.

패널을 추가한 뒤에는 크기와 위치를 조정합니다. 각 패널의 오른쪽 하단 모서리를 드래그하면 크기가 변경됩니다. 패널 상단의 제목 영역을 드래그하면 패널 위치를 이동할 수 있습니다. 패널끼리 겹치지 않도록 자동 정렬이 적용되므로, 한 패널을 이동하면 주변 패널이 자동으로 밀려납니다. 중요도가 높은 지표는 크게, 보조 지표는 작게 배치하면 시선이 자연스럽게 핵심 정보로 향합니다.

대시보드의 패널 배치를 설계할 때 실무에서 자주 사용하는 구성이 있습니다. 화면 최상단에 메트릭 패널을 가로로 나열하여 핵심 수치(총 요청 수, 평균 응답 시간, 오류율 등)를 한 줄로 보여 줍니다. 그 아래에 시계열 선 차트나 막대 차트를 넓게 배치하여 시간에 따른 추이를 표시합니다. 하단에는 상세 테이블이나 파이 차트를 두어 세부 내역을 확인할 수 있게 합니다.

```
+----------+----------+----------+----------+
| 총 요청수 | 평균 응답 | 오류율    | 활성 사용자|
| (Metric) | (Metric) | (Metric) | (Metric) |
+----------+----------+----------+----------+
|                                            |
|      시간대별 요청 수 추이 (Line)            |
|                                            |
+--------------------------------------------+
|                    |                        |
| 상태 코드별 분포    |  상위 엔드포인트 목록   |
| (Pie)              |  (Table)              |
+--------------------+------------------------+
```

대시보드에는 **전역 시간 필터**가 적용됩니다. 4.3.1에서 설명한 Kibana 상단의 시간 필터가 대시보드의 모든 패널에 동일하게 적용됩니다. 시간 필터를 "Last 1 hour"로 변경하면 대시보드의 모든 패널이 최근 1시간 데이터만 표시합니다. 이를 통해 여러 지표를 동일한 시간 범위에서 비교할 수 있습니다.

그런데 특정 패널에서는 전역 시간 필터와 다른 시간 범위를 보여 주고 싶을 때가 있습니다. 예를 들어 대시보드 전체는 최근 1시간을 보고 있지만, "지난 24시간 오류 추이" 패널만 24시간 범위를 유지하고 싶은 경우입니다. 이때 **패널 시간 오버라이드**를 사용합니다. 패널 우측 상단의 톱니바퀴 아이콘을 클릭하고 "Customize time range"를 선택하면, 해당 패널에만 별도의 시간 범위를 지정할 수 있습니다. 오버라이드가 설정된 패널은 전역 시간 필터가 바뀌어도 자신이 가진 시간 범위를 유지합니다.

대시보드에서 데이터를 세밀하게 필터링하고 싶을 때 **Controls 패널**을 사용합니다. Controls 패널은 드롭다운 형태의 필터를 대시보드 상단에 배치하는 기능입니다. 예를 들어 "국가", "서비스 이름", "환경(production/staging)" 같은 필드를 Controls로 추가하면, 사용자가 드롭다운에서 값을 선택할 때마다 대시보드의 모든 패널이 해당 조건으로 필터링됩니다.

Controls 패널을 추가하는 절차는 다음과 같습니다. 대시보드 편집 모드에서 "Add panel" 버튼을 누르고 "Controls"를 선택합니다. Controls 설정 화면에서 "Add control" 버튼을 눌러 필터링에 사용할 필드를 지정합니다. 필드 타입에 따라 드롭다운(Options list)과 슬라이더(Range slider) 중 하나를 선택합니다. keyword 필드에는 드롭다운이, 숫자 필드에는 슬라이더가 적합합니다. 여러 필드를 Controls에 추가하면, 사용자가 여러 조건을 조합하여 대시보드를 필터링할 수 있습니다.

대시보드를 완성했으면 저장하고 팀원과 공유합니다. 가장 간편한 공유 방법은 **URL 공유**입니다. 대시보드 상단의 "Share" 버튼을 클릭하면 공유 옵션이 나타납니다. "Get links" 또는 "Copy link"를 선택하면 현재 대시보드의 URL이 복사됩니다. 이 URL에는 현재 적용된 시간 필터와 Controls 필터 상태가 포함되므로, 링크를 받은 사람은 동일한 시점과 조건의 대시보드를 바로 확인할 수 있습니다. 단, URL을 통해 대시보드에 접근하려면 해당 사용자에게 Kibana 접근 권한이 있어야 합니다.

Kibana 접근 권한이 없는 사람에게 대시보드를 전달해야 하거나, 정기 보고서를 만들어야 하는 경우에는 **PDF 또는 PNG 내보내기**를 사용합니다. "Share" 버튼에서 "PDF Reports" 또는 "PNG Reports"를 선택하면, Kibana가 현재 대시보드 화면을 이미지 또는 PDF 파일로 렌더링합니다. 생성이 완료되면 다운로드 링크가 제공됩니다. PDF 내보내기는 대시보드의 모든 패널을 페이지에 맞추어 배치하므로, 인쇄용 보고서로 사용하기에 적합합니다.

정기적으로 보고서를 생성해야 하는 경우, 매번 수동으로 내보내기를 실행하는 것은 번거롭습니다. 이때 **Reporting 스케줄링** 기능을 사용합니다. Kibana의 Stack Management에서 Reporting 설정으로 이동하면, 대시보드 보고서를 일정한 주기로 자동 생성하도록 예약할 수 있습니다. 매일 아침 특정 대시보드의 PDF를 생성하여 이메일로 발송하는 식의 자동화가 가능합니다. Reporting 스케줄링을 사용하려면 Kibana 인스턴스에 Reporting 기능이 활성화되어 있어야 하며, 구독 등급에 따라 사용 가능 여부가 달라질 수 있습니다.

대시보드를 구성하고 공유하는 전체 흐름을 정리합니다.

```
Dashboard 구성 흐름

1. Create dashboard
   - 빈 대시보드 생성

2. Add panels
   - 라이브러리에서 기존 시각화 추가
   - 또는 대시보드 안에서 새 시각화 생성

3. Arrange panels
   - 크기 조정 (드래그)
   - 위치 이동 (드래그)

4. Configure filters
   - 전역 시간 필터 설정
   - 필요 시 패널별 시간 오버라이드
   - Controls 패널로 드롭다운 필터 추가

5. Save & Share
   - URL 링크 공유
   - PDF/PNG 내보내기
   - Reporting 스케줄로 자동 생성
```

정리하면, Kibana Dashboard는 여러 시각화 패널을 한 화면에 모아 종합적인 데이터 모니터링 환경을 제공합니다. 패널을 추가하고 크기와 위치를 조정하여 레이아웃을 구성하고, 전역 시간 필터와 Controls 패널로 데이터 범위를 제어합니다. 완성된 대시보드는 URL 공유, PDF/PNG 내보내기, Reporting 스케줄링을 통해 팀과 공유할 수 있습니다.

다음 단원인 4.3.3에서는 KQL(Kibana Query Language)을 다룹니다. Kibana 검색 바에서 필터 조건을 직접 작성하는 쿼리 언어를 살펴봅니다.

이 단원을 마치면 여러 시각화 패널을 조합한 대시보드를 구성하고 공유할 수 있습니다.
