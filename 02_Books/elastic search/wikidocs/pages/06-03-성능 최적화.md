# 6.3 성능 최적화

이 섹션에서는 Elasticsearch 클러스터의 인덱싱 성능과 검색 성능을 향상시키는 방법, 그리고 JVM과 OS 수준의 시스템 최적화를 다룹니다. 성능 문제는 대부분 설정 한두 가지로 극적으로 개선될 수 있습니다.

## 토픽 목록

**6.3.1 인덱싱 성능 최적화**
- 대량 인덱싱 시 처리량을 높이는 설정과 기법을 적용할 수 있다

refresh_interval 증가(-1 비활성화), index.number_of_replicas 일시 0 설정, Bulk API 배치 크기 최적화, index.translog.durability: async, mapping 최적화(불필요한 _all 비활성화), 인덱싱 스레드 풀 설정, OS 페이지 캐시 활용, force merge 후 최적화를 다룹니다.

**6.3.2 검색 성능 최적화**
- 슬로우 로그를 분석하여 검색 성능 병목을 찾고 최적화할 수 있다

슬로우 로그 설정(index.search.slowlog.threshold), request cache 활성화와 무효화, fielddata cache 설정, routing으로 샤드 대상 축소, _source 필드 필터링, store: true vs doc_values 선택, 쿼리 캐시 활용(filter context), profile API로 쿼리 실행 계획 분석을 다룹니다.

**6.3.3 JVM/파일시스템 최적화**
- JVM heap 크기와 GC 설정을 올바르게 조정할 수 있다
- OS 레벨 파일시스템 설정 최적화를 수행할 수 있다

JVM heap 크기 설정(물리 메모리의 50%, 최대 31GB), Compressed OOP와 31GB 경계, G1GC vs ZGC 선택 기준, bootstrap.memory_lock 설정, 파일 디스크립터 제한(ulimit -n 65535), vm.max_map_count 설정, SSD vs HDD와 io 스케줄러, NUMA 설정을 다룹니다.
