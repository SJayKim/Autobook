# Phase 6. 운영 및 관리 (Operations)

이 Phase를 마치면 ILM 정책을 설계하고, 데이터 수집 파이프라인을 구성하며, 보안·스냅샷·모니터링을 포함한 프로덕션 클러스터를 운영할 수 있게 됩니다.

실제 프로덕션 환경에서 Elasticsearch를 운영하려면 단순한 쿼리 작성 이상의 역량이 필요합니다. 데이터가 시간이 지남에 따라 어떻게 관리되어야 하는지, 데이터를 어떻게 수집하는지, 클러스터의 성능을 어떻게 유지하고, 보안을 어떻게 설정하는지를 알아야 합니다. Phase 6에서는 ILM 정책, 데이터 수집 파이프라인(Logstash, Beats, 인제스트 파이프라인), 성능 최적화, 보안 및 운영 절차를 종합적으로 다룹니다.

## 이 Phase의 섹션 구성

**6.1 인덱스 수명주기**
ILM(Index Lifecycle Management) 정책 설계, Data Streams, Searchable Snapshots를 통한 Cold/Frozen 티어 구성을 다룹니다.
- 6.1.1 ILM 정책
- 6.1.2 Data Streams
- 6.1.3 Searchable Snapshots

**6.2 데이터 수집 파이프라인**
Logstash 파이프라인 구성, Beats 경량 에이전트, 인제스트 파이프라인을 이용한 데이터 전처리를 다룹니다.
- 6.2.1 Logstash
- 6.2.2 Beats
- 6.2.3 인제스트 파이프라인

**6.3 성능 최적화**
인덱싱 성능 최적화, 검색 성능 최적화(슬로우 로그, 캐시), JVM/파일시스템 수준 최적화를 다룹니다.
- 6.3.1 인덱싱 성능 최적화
- 6.3.2 검색 성능 최적화
- 6.3.3 JVM/파일시스템 최적화

**6.4 보안 및 운영**
TLS/RBAC/API Key 보안 설정, 스냅샷과 복구, 클러스터 모니터링, 롤링 업그레이드를 다룹니다.
- 6.4.1 TLS/RBAC/API Key 보안
- 6.4.2 스냅샷과 복구
- 6.4.3 클러스터 모니터링
- 6.4.4 롤링 업그레이드
