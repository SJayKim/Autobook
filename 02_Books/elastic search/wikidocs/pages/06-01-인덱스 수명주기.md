# 6.1 인덱스 수명주기

이 섹션에서는 시간이 지남에 따라 인덱스를 자동으로 관리하는 방법을 다룹니다. ILM 정책으로 데이터의 Hot→Warm→Cold→Delete 전환을 자동화하고, Data Streams와 Searchable Snapshots로 대용량 시계열 데이터를 효율적으로 관리합니다.

## 토픽 목록

**6.1.1 ILM 정책**
- ILM 정책을 생성하고 인덱스에 연결하여 단계별 전환을 자동화할 수 있다

ILM(Index Lifecycle Management) 개념, Hot → Warm → Cold → Frozen → Delete 단계, 단계별 전환 조건(min_age, rollover), ILM 정책 생성 API, 인덱스 템플릿에 ILM 연결, ILM 상태 확인(_ilm/explain), ILM 재시도 및 수동 이동을 다룹니다.

**6.1.2 Data Streams**
- Data Stream을 생성하고 시계열 데이터를 인덱싱할 수 있다
- Data Stream의 backing index 구조를 설명할 수 있다

Data Stream 개념과 시계열 데이터 적합성, @timestamp 필수 필드, backing index 자동 롤오버, Data Stream 생성(인덱스 템플릿 기반), Data Stream 검색과 인덱싱, Data Stream과 ILM 연동, Data Stream 삭제와 backing index 관리를 다룹니다.

**6.1.3 Searchable Snapshots**
- Searchable Snapshot으로 Cold/Frozen 티어를 구성하여 스토리지 비용을 절감할 수 있다

Searchable Snapshot 개념, Cold 티어 vs Frozen 티어, 스냅샷 저장소 등록(S3, GCS, Azure), mount API로 스냅샷 마운트, 부분 마운트 vs 전체 마운트, Frozen 티어 캐시 설정, ILM cold/frozen 단계 설정을 다룹니다.
