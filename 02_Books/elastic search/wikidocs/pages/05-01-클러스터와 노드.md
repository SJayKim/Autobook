# 5.1 클러스터와 노드

이 섹션에서는 Elasticsearch 클러스터를 구성하는 노드의 역할과 내부 동작을 다룹니다. 각 노드가 어떤 책임을 맡는지, 마스터 선출은 어떻게 이루어지는지, 그리고 인덱싱과 검색 요청이 클러스터 내에서 어떻게 처리되는지를 이해합니다.

## 토픽 목록

**5.1.1 노드 역할**
- Elasticsearch 노드 역할 유형을 열거하고 각 역할의 책임을 설명할 수 있다

master-eligible 노드, data 노드(data/data_hot/data_warm/data_cold/data_frozen), ingest 노드, coordinating 노드, machine_learning 노드, transform 노드, 노드 역할 혼용 vs 전용 설정, node.roles 설정을 다룹니다.

**5.1.2 클러스터 아키텍처**
- 마스터 선출 과정과 split-brain 방지 메커니즘을 설명할 수 있다
- 프로덕션 권장 클러스터 토폴로지를 설계할 수 있다

마스터 선출(master election) 과정, Quorum 기반 합의, discovery.seed_hosts 설정, cluster.initial_master_nodes 부트스트랩, split-brain 문제와 minimum_master_nodes 역사, 투표 전용 노드(voting_only), dedicated master 노드 3개 권장 구성, _cluster/state API를 다룹니다.

**5.1.3 분산 읽기/쓰기 흐름**
- 인덱싱 요청이 coordinating → primary → replica 경로로 처리되는 과정을 설명할 수 있다
- 검색 요청의 scatter/gather 패턴을 설명할 수 있다

인덱싱 라우팅(coordinating → primary shard), 레플리카 복제 흐름, wait_for_active_shards 설정, 검색 scatter 단계(coordinating → 각 샤드), 검색 gather 단계(결과 병합, 전역 정렬), preference 파라미터(_local, _prefer_nodes), adaptive replica selection을 다룹니다.
