# Phase 1. 기초 (Foundation)

이 Phase를 마치면 Elasticsearch를 로컬에 설치하고, 문서를 인덱싱·검색하며, 역색인과 세그먼트 구조를 설명할 수 있게 됩니다.

Elasticsearch를 제대로 다루려면 단순히 명령어를 외우는 것만으로는 부족합니다. 검색 엔진이 어떻게 텍스트를 저장하고 빠르게 찾아내는지, 그 근본 원리를 이해해야 합니다. Phase 1은 그 출발점입니다.

첫 번째 섹션에서는 Elasticsearch가 무엇인지, ELK Stack에서 어떤 역할을 하는지 살펴봅니다. 이어서 로컬 환경에 설치하고 첫 번째 문서를 인덱싱해보는 실습으로 이어집니다.

두 번째 섹션에서는 Elasticsearch의 검색 성능을 가능하게 하는 핵심 원리인 Lucene과 역색인을 다룹니다. 역색인이 어떻게 구성되고, 세그먼트가 어떻게 생성·병합되며, NRT(Near Real-Time) 검색과 Translog가 어떻게 함께 동작하는지 이해합니다.

세 번째 섹션에서는 Elasticsearch의 데이터 모델을 다룹니다. JSON 문서, 인덱스, 샤드와 레플리카의 개념을 익히고, 이것이 RDBMS와 어떻게 다른지 비교합니다.

Elasticsearch의 정의, ELK Stack 구성요소, 설치 방법, 첫 번째 문서 인덱싱과 검색을 다룹니다.
- 1.1.1 Elasticsearch란 무엇인가
- 1.1.2 설치 및 환경 구성
- 1.1.3 첫 번째 문서 인덱싱과 검색

**1.2 핵심 원리: Lucene과 역색인**
Apache Lucene, 역색인 동작 원리, 세그먼트 구조, NRT 검색과 Translog를 다룹니다.
- 1.2.1 Lucene과 Elasticsearch의 관계
- 1.2.2 역색인 동작 원리
- 1.2.3 세그먼트 계층 구조
- 1.2.4 NRT 검색과 Translog

**1.3 데이터 모델: 문서/인덱스/샤드**
JSON 문서 구조, 인덱스의 개념, 샤드와 레플리카의 역할과 설계 기준을 다룹니다.
- 1.3.1 JSON 문서 구조
- 1.3.2 인덱스 개념과 RDBMS 비교
