# 7.4 RAG 통합

이 섹션에서는 Elasticsearch를 벡터 저장소로 활용하여 LLM(Large Language Model)과 통합하는 RAG(Retrieval-Augmented Generation) 파이프라인을 다룹니다. 문서 청킹부터 임베딩 인덱싱, 컨텍스트 검색, LLM 프롬프트 구성까지 전체 파이프라인을 익힙니다.

## 토픽 목록

**7.4.1 RAG 파이프라인 구성**
- Elasticsearch를 벡터 저장소로 사용하는 RAG 파이프라인을 설계할 수 있다

RAG(Retrieval-Augmented Generation) 아키텍처, 청킹(chunking) 전략(고정 크기, 문장, 단락), 임베딩 생성과 인덱싱, 컨텍스트 검색(kNN 또는 하이브리드), LLM 프롬프트에 검색 결과 삽입, LangChain/LlamaIndex와 ES 통합, 답변 품질 평가 방법을 다룹니다.

**7.4.2 ML Inference 파이프라인**
- ML Inference 프로세서로 인덱싱 시 자동 임베딩 생성 파이프라인을 구성할 수 있다

inference 인제스트 프로세서, model_id 지정, input_output 필드 매핑, 파이프라인 체이닝으로 임베딩 + 메타데이터 처리, batch inference 성능, ML Inference 모니터링, Elastic AI Assistant 연동 개요를 다룹니다.
