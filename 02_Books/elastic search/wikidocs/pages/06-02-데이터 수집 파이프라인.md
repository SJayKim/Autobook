# 6.2 데이터 수집 파이프라인

이 섹션에서는 외부 데이터를 Elasticsearch로 가져오는 다양한 방법을 다룹니다. Logstash를 이용한 ETL 파이프라인, Beats를 이용한 경량 에이전트 수집, Elasticsearch 내부의 인제스트 파이프라인 전처리를 익힙니다.

## 토픽 목록

**6.2.1 Logstash**
- Logstash 파이프라인의 input → filter → output 구조를 설명할 수 있다
- Grok 필터로 비정형 로그를 파싱할 수 있다

Logstash 파이프라인 구조, input 플러그인(file, beats, kafka), filter 플러그인(grok, mutate, date, geoip), Grok 패턴 문법, output 플러그인(elasticsearch), Dead Letter Queue, Logstash 파이프라인 다중화, Persistent Queue를 다룹니다.

**6.2.2 Beats**
- Filebeat, Metricbeat, Packetbeat의 역할을 구분하고 기본 설정을 적용할 수 있다

Beats 경량 에이전트 개념, Filebeat(로그 파일 수집), Metricbeat(시스템/서비스 메트릭), Packetbeat(네트워크 패킷), Heartbeat(가용성 모니터링), Auditbeat(보안 감사), Beats → Logstash → ES vs Beats → ES 직접 비교, filebeat.yml 기본 설정을 다룹니다.

**6.2.3 인제스트 파이프라인**
- 인제스트 파이프라인으로 문서 인덱싱 전 전처리를 구성할 수 있다
- grok, date, rename, set 프로세서를 활용할 수 있다

인제스트 파이프라인 개념, PUT _ingest/pipeline API, grok 프로세서, date 프로세서, rename/set/remove 프로세서, foreach 프로세서, pipeline 프로세서(파이프라인 체이닝), on_failure 오류 처리, simulate API로 파이프라인 테스트를 다룹니다.
