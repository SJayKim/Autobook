# 7.3 하이브리드 검색

이 섹션에서는 키워드 기반 BM25 검색과 벡터 기반 kNN 검색을 결합하는 하이브리드 검색을 다룹니다. 두 방식의 결과를 공정하게 병합하는 RRF 알고리즘과 retriever API, 그리고 검색 품질을 평가하고 튜닝하는 방법을 익힙니다.

## 토픽 목록

**7.3.1 RRF 원리**
- Reciprocal Rank Fusion의 수식과 rank_constant 파라미터를 설명할 수 있다

RRF(Reciprocal Rank Fusion) 공식, rank_constant 파라미터(기본값 60), 다중 순위 목록 병합 방식, 스코어 정규화 불필요 이점, RRF vs 선형 결합(linear combination) 비교, ES 8.8+에서 rrf retriever 지원, Kibana에서 RRF 시각화를 다룹니다.

**7.3.2 BM25+kNN 결합 구현**
- retriever API로 BM25와 kNN을 결합한 하이브리드 검색을 구현할 수 있다

retriever API 구조(ES 8.9+), standard retriever(BM25), knn retriever, rrf retriever로 두 결과 병합, sub_searches 기반 구현(레거시), 하이브리드 검색 응답 구조, explain으로 하이브리드 스코어 분석을 다룹니다.

**7.3.3 하이브리드 검색 튜닝**
- 하이브리드 검색의 rank_constant와 num_candidates를 조정하여 검색 품질을 최적화할 수 있다

rank_constant 값에 따른 순위 변화, num_candidates 증가와 recall 향상, 하이브리드 검색 평가 지표(NDCG, MAP), 오프라인 평가와 온라인 A/B 테스트, 검색 품질 평가 데이터셋 구성, Rank Evaluation API(_rank_eval)를 다룹니다.
