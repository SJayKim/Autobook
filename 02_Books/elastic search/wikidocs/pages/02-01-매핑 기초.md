# 2.1 매핑 기초

이 섹션에서는 Elasticsearch가 문서의 필드를 어떻게 인식하고 저장하는지를 결정하는 매핑(mapping)의 기초를 다룹니다. 매핑을 올바르게 설계하지 않으면 검색 결과가 의도와 다르게 나오거나, 나중에 변경하기 어려운 상황이 발생합니다.

## 토픽 목록

**2.1.1 동적 매핑 vs 명시적 매핑**
- 동적 매핑과 명시적 매핑의 차이를 설명할 수 있다
- 동적 매핑이 추론하는 타입 규칙을 열거할 수 있다

동적 매핑(dynamic mapping) 개념, JSON 타입에서 ES 타입으로의 자동 추론 규칙, dynamic: true/false/strict 설정, 명시적 매핑(explicit mapping) PUT 방법, GET /{index}/_mapping API, 동적 매핑의 장단점을 다룹니다.

**2.1.2 매핑 설정과 업데이트 제약**
- 기존 필드 타입 변경이 불가능한 이유를 역색인 관점에서 설명할 수 있다
- reindex API를 사용해 매핑을 변경하는 절차를 수행할 수 있다

필드 타입 변경 불가 제약, 새 필드 추가(additive 변경)는 가능함, PUT mapping API로 새 필드 추가, reindex API 워크플로, 인덱스 별칭을 활용한 무중단 재매핑, _source 비활성화 주의사항, copy_to 활용 패턴을 다룹니다.

**2.1.3 매핑 폭발 방지**
- 매핑 폭발(mapping explosion)이 발생하는 원인과 위험을 설명할 수 있다
- flattened 타입과 dynamic: strict로 매핑 폭발을 방지할 수 있다

매핑 폭발 개념과 클러스터 메모리 영향, index.mapping.total_fields.limit 설정, flattened 필드 타입, dynamic: strict 설정, 와일드카드 동적 매핑 템플릿, 구조화되지 않은 로그 데이터 처리 전략을 다룹니다.
