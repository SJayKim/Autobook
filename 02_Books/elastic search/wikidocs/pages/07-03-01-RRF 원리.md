# 7.3.1 RRF 원리

7.1.2에서 kNN 검색을, 3.2.2에서 match 쿼리 기반의 BM25 검색을 다뤘습니다. kNN 검색은 의미가 비슷한 문서를 잘 찾지만, 정확한 키워드 매칭에서는 BM25보다 약할 수 있습니다. 반대로 BM25는 키워드가 정확히 일치하는 문서를 잘 찾지만, 표현이 다른 유사 문서는 놓칠 수 있습니다. 두 방식을 함께 사용하면 각각의 약점을 보완할 수 있습니다. 이것이 하이브리드 검색의 핵심 아이디어입니다.

그런데 두 검색 방식의 결과를 합치려면 문제가 하나 있습니다. BM25가 돌려주는 점수와 kNN이 돌려주는 점수는 척도가 다릅니다. BM25 점수는 TF-IDF 기반의 관련성 점수로, 값의 범위에 이론적 상한이 없습니다. kNN 점수는 벡터 유사도로, cosine의 경우 0에서 1 사이입니다. 이 서로 다른 점수를 단순히 더하거나 비교하면 한쪽 점수가 다른 쪽을 압도할 수 있습니다. 점수를 정규화하는 방법이 있지만, 적절한 정규화 기준을 정하는 것 자체가 어려운 문제입니다.

**RRF**(Reciprocal Rank Fusion)는 이 문제를 우아하게 해결합니다. RRF는 점수를 아예 무시하고, 각 결과 목록에서 문서가 몇 번째에 있는지, 즉 **순위**(rank)만 사용합니다.

RRF의 공식은 다음과 같습니다.

```
RRF_score(d) = 0.0
for q in queries:
    if d in result(q):
        RRF_score(d) += 1.0 / (k + rank(result(q), d))
```

d는 문서, q는 각 검색 쿼리(예: BM25 쿼리와 kNN 쿼리), k는 **rank_constant**라는 상수입니다. rank 함수는 해당 검색 결과에서 문서 d가 몇 번째인지를 반환합니다. 1등이면 1, 2등이면 2입니다.

구체적인 예를 들겠습니다. BM25 검색과 kNN 검색을 수행했고, rank_constant(k)가 60이라고 가정합니다.

```
BM25 검색 결과:           kNN 검색 결과:
1등: 문서A                1등: 문서C
2등: 문서B                2등: 문서A
3등: 문서C                3등: 문서D
4등: 문서D                4등: 문서E
```

문서A의 RRF 점수를 계산합니다. BM25에서 1등이므로 1/(60+1) = 0.01639입니다. kNN에서 2등이므로 1/(60+2) = 0.01613입니다. 합산하면 0.03252입니다.

문서C의 RRF 점수를 계산합니다. BM25에서 3등이므로 1/(60+3) = 0.01587입니다. kNN에서 1등이므로 1/(60+1) = 0.01639입니다. 합산하면 0.03226입니다.

문서B의 RRF 점수는 BM25에서만 2등이므로 1/(60+2) = 0.01613입니다. kNN 결과에 없으므로 기여분이 없습니다.

최종 RRF 순위는 문서A(0.03252) > 문서C(0.03226) > 문서B(0.01613) > 문서D > 문서E 순이 됩니다. 문서A가 양쪽 결과에 모두 높은 순위로 등장했기 때문에 최종 1등을 차지했습니다. 이것이 RRF의 핵심 원리입니다. 여러 결과 목록에서 고루 높은 순위를 받는 문서가 최종 결과에서도 높은 순위를 받습니다.

**rank_constant** 파라미터는 순위가 낮은 문서에 얼마나 기회를 줄지를 조절합니다. 기본값은 60입니다. k가 작으면 상위 순위 문서와 하위 순위 문서 사이의 점수 차이가 커집니다. 1등은 1/(0+1)=1.0이고 10등은 1/(0+10)=0.1로, 10배 차이가 납니다. k가 60이면 1등은 1/(60+1)=0.01639이고 10등은 1/(60+10)=0.01429로, 차이가 1.15배 정도로 줄어듭니다. k가 클수록 하위 순위 문서도 최종 결과에 영향을 미칠 여지가 커집니다.

RRF가 점수 정규화 없이 동작한다는 것은 큰 장점입니다. **선형 결합**(linear combination) 방식과 비교하면 차이가 명확합니다. 선형 결합은 각 검색 방식의 점수에 가중치를 곱한 뒤 더합니다. 예를 들어 "최종 점수 = 0.7 x BM25 점수 + 0.3 x kNN 점수"처럼 설정합니다. 이 방식은 두 가지 문제가 있습니다.

첫째, BM25 점수와 kNN 점수의 스케일이 다르므로 정규화가 필요합니다. 정규화 방법에 따라 결과가 크게 달라질 수 있습니다. 둘째, 가중치(0.7과 0.3)를 적절히 정하기 어렵습니다. 데이터셋과 쿼리 패턴에 따라 최적값이 달라지므로 실험이 필요합니다.

RRF는 이 두 문제를 모두 피합니다. 점수를 사용하지 않으므로 정규화가 불필요하고, rank_constant 하나만 조정하면 됩니다. 실무에서 기본값 60이 대부분의 경우 좋은 결과를 보여 별도 튜닝이 필요 없는 경우가 많습니다.

Elasticsearch는 **8.8 버전**부터 RRF를 retriever API의 일부로 공식 지원합니다. retriever API에서 rrf retriever를 지정하면 RRF 병합이 자동으로 수행됩니다. 구체적인 구현 방법은 7.3.2에서 다룹니다.

Kibana에서도 RRF의 효과를 시각적으로 확인할 수 있습니다. Discover 화면에서 하이브리드 검색 결과를 조회하면, 각 문서가 어떤 검색 방식에서 몇 번째로 나왔는지 확인할 수 있습니다. Dev Tools에서 explain 옵션을 함께 사용하면 RRF 점수 계산 과정을 상세히 볼 수 있습니다.

정리하면, RRF는 서로 다른 검색 방식의 결과를 순위 기반으로 병합하는 알고리즘입니다. 원시 점수를 무시하고 순위만 사용하므로 점수 정규화가 필요 없습니다. rank_constant로 상위 순위 집중도를 조절하며, 기본값 60이 대부분의 상황에서 잘 동작합니다. 선형 결합 방식 대비 설정이 간단하고 안정적인 결과를 제공합니다.

다음 단원인 7.3.2에서는 retriever API를 사용하여 BM25와 kNN을 실제로 결합하는 하이브리드 검색을 구현합니다.

이 단원을 마치면 Reciprocal Rank Fusion의 수식과 rank_constant 파라미터를 설명할 수 있습니다.
