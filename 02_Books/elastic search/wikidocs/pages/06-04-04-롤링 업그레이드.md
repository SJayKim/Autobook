# 6.4.4 롤링 업그레이드

Elasticsearch 새 버전이 출시되면 보안 패치, 성능 개선, 새 기능을 적용하기 위해 업그레이드를 수행해야 합니다. 하지만 프로덕션 클러스터를 업그레이드하는 것은 단순한 소프트웨어 교체가 아닙니다. 분산 시스템에서 여러 노드의 바이너리를 동시에 바꾸면, 버전 불일치로 인한 통신 오류나 데이터 손상이 발생할 수 있습니다. Elasticsearch는 두 가지 업그레이드 전략을 제공하며, 상황에 따라 적합한 방법을 선택합니다.

**풀 클러스터 재시작(full cluster restart)** 방식은 모든 노드를 동시에 중지하고, 전부 업그레이드한 뒤 다시 시작하는 방법입니다. 절차가 단순하지만, 모든 노드가 내려가는 동안 클러스터를 사용할 수 없습니다. 서비스 중단이 허용되는 유지보수 시간대가 확보된 경우에 적합합니다.

**롤링 업그레이드(rolling upgrade)**는 노드를 한 대씩 순서대로 업그레이드하여, 클러스터가 계속 동작하는 상태에서 버전을 올리는 방법입니다. 업그레이드 도중에도 인덱싱과 검색이 가능하므로, 서비스 중단 없이 업그레이드를 완료할 수 있습니다. 프로덕션 환경에서 일반적으로 선택하는 방법입니다.

롤링 업그레이드에는 제약이 있습니다. 모든 버전 간에 롤링 업그레이드가 가능한 것은 아닙니다. 마이너 버전 간 업그레이드(예: 8.10에서 8.11)는 롤링 업그레이드를 지원합니다. 메이저 버전 간에는 특정 징검다리 버전을 거쳐야 합니다. 예를 들어 7.x에서 8.x로 올리려면 먼저 7.17 이상으로 업그레이드한 뒤 8.x로 전환해야 합니다. 6.x에서 8.x로 직접 올릴 수는 없고, 6.x에서 7.17로 먼저 올린 뒤 8.x로 올려야 합니다.

업그레이드 전에 반드시 수행해야 하는 준비 작업이 있습니다.

첫째, 6.4.2에서 다룬 스냅샷으로 클러스터 전체를 백업합니다. 업그레이드 중 예상치 못한 문제가 발생했을 때 원래 상태로 되돌릴 수 있는 유일한 안전장치입니다.

둘째, 대상 버전의 **브레이킹 체인지(breaking changes)**를 확인합니다. Elasticsearch는 메이저 버전마다 이전 버전과 호환되지 않는 변경 사항을 공개합니다. 제거된 설정, 변경된 API 동작, 폐기된 기능 등이 포함됩니다. 이를 확인하지 않으면 업그레이드 후 애플리케이션이 오동작할 수 있습니다.

셋째, Elasticsearch의 **디프리케이션 로그(deprecation log)**를 확인합니다. 현재 버전에서 사용 중이지만 다음 버전에서 제거될 기능을 사용하고 있다면 경고가 기록됩니다. 업그레이드 전에 이 경고를 해결해야 합니다.

넷째, 사용 중인 **플러그인의 호환성**을 확인합니다. 플러그인 버전은 Elasticsearch 버전과 정확히 일치해야 합니다. 대상 버전에 맞는 플러그인이 없으면 업그레이드를 진행할 수 없습니다.

다섯째, 클러스터 상태가 **green**인지 확인합니다. 6.4.3에서 다룬 _cluster/health로 확인합니다. yellow나 red 상태에서 업그레이드를 시작하면, 노드를 내리는 과정에서 데이터 유실이 발생할 수 있습니다.

롤링 업그레이드의 구체적인 절차를 단계별로 설명합니다.

```
┌──────────────────────────────────────────────────┐
│ 1. 클러스터 상태 확인 (green 확인)                  │
│ 2. 샤드 할당 비활성화                               │
│ 3. synced flush 수행 (7.x 이하)                    │
│ 4. 노드 하나 중지                                   │
│ 5. 소프트웨어 및 플러그인 업그레이드                   │
│ 6. 노드 재시작                                      │
│ 7. 샤드 할당 재활성화                               │
│ 8. 클러스터 green 상태 대기                         │
│ 9. 다음 노드로 반복 (4~8)                           │
│ 10. 모든 노드 완료 후 최종 확인                      │
└──────────────────────────────────────────────────┘
```

먼저 **샤드 할당을 비활성화**합니다. 노드가 내려가면 Elasticsearch는 해당 노드의 샤드를 다른 노드로 재배치하려고 합니다. 업그레이드를 위해 잠시 내리는 것이므로, 불필요한 샤드 재배치를 방지해야 합니다.

```json
PUT _cluster/settings
{
  "persistent": {
    "cluster.routing.allocation.enable": "primaries"
  }
}
```

cluster.routing.allocation.enable을 primaries로 설정하면, 프라이머리 샤드의 할당만 허용하고 레플리카 샤드의 재배치를 중단합니다. 노드가 다시 올라왔을 때 기존 샤드가 그대로 복원되므로, 대량의 데이터 복사를 피할 수 있습니다.

다음으로 대상 노드를 **정상 종료(graceful shutdown)**합니다. 노드의 Elasticsearch 프로세스를 중지한 뒤, 새 버전의 바이너리를 설치합니다. 플러그인도 대상 버전에 맞는 것으로 업데이트합니다. 플러그인 버전이 Elasticsearch 버전과 일치하지 않으면 노드가 시작되지 않습니다.

노드를 **재시작**합니다. 시작 후 노드가 클러스터에 합류하고, 이전에 보유하던 샤드가 다시 할당됩니다.

샤드 할당을 **재활성화**합니다.

```json
PUT _cluster/settings
{
  "persistent": {
    "cluster.routing.allocation.enable": null
  }
}
```

값을 null로 설정하면 기본 동작(모든 샤드 할당 허용)으로 복원됩니다. Elasticsearch가 샤드를 재배치하고 복구하기 시작합니다.

_cluster/health API로 **클러스터 상태가 green으로 돌아올 때까지 기다립니다**. 이 과정이 중요합니다. green이 되기 전에 다음 노드를 내리면, 레플리카가 부족한 상태에서 또 다른 노드가 사라지는 셈이므로 데이터 유실 위험이 있습니다.

```json
GET _cluster/health?wait_for_status=green&timeout=5m
```

green 상태를 확인한 뒤, 다음 노드에 대해 같은 절차를 반복합니다.

노드를 업그레이드하는 순서도 중요합니다. 새 버전의 노드는 이전 버전의 마스터를 따를 수 있지만, 이전 버전의 노드는 새 버전의 마스터를 따르지 못할 수 있습니다. 따라서 **마스터 적격 노드가 아닌 노드를 먼저** 업그레이드합니다. 데이터 티어가 있는 경우에는 Frozen, Cold, Warm, Hot 순으로 업그레이드합니다. 마스터 적격 노드는 가장 마지막에 업그레이드합니다.

모든 노드의 업그레이드가 완료되면, Kibana도 업그레이드합니다. Kibana 버전은 Elasticsearch 버전과 일치해야 합니다. Elasticsearch를 먼저 업그레이드하고, 그 다음에 Kibana를 업그레이드하는 순서를 따릅니다.

업그레이드가 완료된 뒤에는 새 버전에서 추가된 기능이 모든 노드에서 동작하는지 확인합니다. 주의할 점은, 클러스터 내에 구버전 노드가 남아 있는 동안에는 새 버전 전용 기능을 사용할 수 없다는 것입니다. 새 기능은 모든 노드가 새 버전으로 업그레이드된 후에야 활성화됩니다.

정리하면, 롤링 업그레이드는 노드를 한 대씩 중지하고 업그레이드하여 서비스 중단 없이 버전을 올리는 방법입니다. 샤드 할당을 비활성화한 뒤 노드를 업그레이드하고, 재시작 후 할당을 복원하고, green 상태를 확인한 뒤 다음 노드로 진행합니다. 업그레이드 전에 반드시 스냅샷을 생성하고, 브레이킹 체인지와 플러그인 호환성을 확인하며, 마스터 적격 노드가 아닌 노드부터 업그레이드합니다.

Phase 6 "운영 및 관리"를 모두 다루었습니다. 다음 Phase 7 "AI 검색"의 첫 단원인 7.1.1에서는 dense_vector 매핑과 HNSW 알고리즘을 다룹니다. 텍스트를 벡터로 변환하여 의미 기반 검색을 수행하는 현대적인 검색 기법으로 넘어갑니다.

이 단원을 마치면 무중단 롤링 업그레이드 절차를 단계별로 수행할 수 있습니다.
