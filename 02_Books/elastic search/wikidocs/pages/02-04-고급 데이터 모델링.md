# 2.4 고급 데이터 모델링

이 섹션에서는 기초 매핑을 넘어 더 복잡한 데이터 구조와 운영 패턴을 다룹니다. 런타임 필드, 관계 데이터 모델링, 인덱스 템플릿, 인덱스 별칭은 실무에서 자주 사용되는 핵심 패턴입니다.

## 토픽 목록

**2.4.1 런타임 필드**
- 런타임 필드를 정의하고 검색 시 동적으로 계산된 값을 조회할 수 있다
- 런타임 필드의 성능 트레이드오프를 설명할 수 있다

런타임 필드(runtime field) 개념, 매핑에 runtime 섹션 추가, Painless 스크립트로 필드 값 계산, 검색 요청 시 runtime_mappings 임시 정의, 런타임 필드 vs 인덱스 시 계산 필드 성능 비교, 런타임 필드를 인덱스 필드로 전환하는 전략을 다룹니다.

**2.4.2 중첩 문서 vs Parent-Child 조인**
- nested 타입과 join 타입의 저장 방식 차이를 설명할 수 있다
- 각 방식의 쿼리 성능 특성을 비교하여 적합한 방식을 선택할 수 있다

nested 타입 저장 방식(숨겨진 문서), nested 쿼리 구조, join 타입 설정(relations), has_child 쿼리와 has_parent 쿼리, parent_id 쿼리, nested vs join 성능 비교, join 타입의 동일 샤드 라우팅 필요성, 사용 권장 시나리오를 다룹니다.

**2.4.3 인덱스 템플릿과 Component Template**
- 인덱스 템플릿으로 인덱스 자동 설정을 구성할 수 있다
- Component Template을 조합하여 재사용 가능한 매핑 블록을 관리할 수 있다

레거시 _template vs 현행 _index_template, 인덱스 패턴(index_patterns) 매칭, 우선순위(priority) 설정, Component Template(_component_template API), composed_of 필드로 컴포넌트 조합, data stream과 인덱스 템플릿 연계, 시뮬레이션 API(_index_template/_simulate)를 다룹니다.

**2.4.4 인덱스 별칭**
- 인덱스 별칭을 생성하고 원자적 교체(atomic swap)를 수행할 수 있다
- 필터 별칭을 활용한 가상 인덱스 뷰를 구성할 수 있다

인덱스 별칭(alias) 개념, 별칭 생성 및 삭제(_aliases API), 원자적 add/remove swap 패턴, 필터 별칭(alias filter), 라우팅 별칭(routing alias), write 별칭과 is_write_index, 무중단 인덱스 교체 시나리오를 다룹니다.
