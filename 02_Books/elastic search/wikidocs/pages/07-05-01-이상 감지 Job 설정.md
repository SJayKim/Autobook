# 7.5.1 이상 감지 Job 설정

지금까지 Phase 7에서는 벡터 검색, 시맨틱 검색, 하이브리드 검색, RAG 파이프라인을 다뤘습니다. 이 기능들은 모두 "검색"에 초점이 맞춰져 있었습니다. Elasticsearch의 ML 기능에는 검색 외에 또 하나의 중요한 영역이 있습니다. 시계열 데이터에서 비정상적인 패턴을 자동으로 찾아내는 **이상 감지**(Anomaly Detection)입니다.

이상 감지가 왜 필요한지부터 살펴봅니다. 서버 모니터링을 예로 들겠습니다. 웹 서버의 응답 시간을 매 분마다 기록한다고 가정합니다. 평소에는 50~100밀리초 사이인데, 어느 순간 갑자기 500밀리초로 치솟습니다. 이런 변화는 서버 장애, 네트워크 문제, 또는 비정상적인 트래픽 증가를 의미할 수 있습니다. 서버가 한두 대라면 사람이 그래프를 보면서 발견할 수 있지만, 수백 대의 서버에서 수십 가지 지표를 동시에 감시하는 것은 사람의 눈으로 불가능합니다. ML 이상 감지는 이 작업을 자동화합니다.

Elasticsearch의 **ML Anomaly Detection**은 시계열 데이터를 분석하여 정상 패턴을 학습하고, 학습된 패턴에서 크게 벗어나는 데이터 포인트를 자동으로 식별합니다. 이 과정의 중심에 있는 것이 **이상 감지 Job**입니다.

이상 감지 Job은 두 가지 유형이 있습니다. **단일 메트릭 Job**(single metric job)은 하나의 지표만 분석합니다. 예를 들어 전체 서버의 평균 응답 시간만 추적합니다. 구성이 간단하고 결과 해석이 직관적이어서, 이상 감지를 처음 시작할 때 적합합니다.

**다중 메트릭 Job**(multi-metric job)은 여러 지표를 동시에 분석합니다. 응답 시간, CPU 사용률, 메모리 사용량을 하나의 Job에서 함께 감시할 수 있습니다. 또한 호스트별, 서비스별로 분리하여 분석할 수도 있습니다. 여러 지표 사이의 상관관계까지 고려하므로, 단일 메트릭보다 정밀한 이상 탐지가 가능합니다.

이상 감지 Job을 구성하는 핵심 요소는 **detector 함수**입니다. detector는 "무엇을 어떻게 분석할 것인가"를 정의합니다. 대표적인 detector 함수를 살펴봅니다.

**mean** 함수는 특정 필드의 평균값을 추적합니다. 평균 응답 시간이 평소보다 높거나 낮으면 이상으로 판단합니다. **sum** 함수는 합계를 추적합니다. 시간당 총 거래 금액이 급변하면 이상을 감지합니다. **count** 함수는 문서 수를 추적합니다. 특정 시간대에 로그인 시도 횟수가 급증하면 이상으로 식별합니다. **rare** 함수는 드물게 나타나는 값을 찾습니다. 평소에 보이지 않던 사용자 에이전트(user agent)나 IP 주소가 갑자기 등장하면 이상으로 표시합니다.

**bucket_span**은 시계열을 나누는 시간 간격입니다. 이상 감지 모델은 데이터를 이 간격 단위로 묶어 분석합니다. bucket_span을 15분으로 설정하면, 매 15분마다 해당 구간의 데이터를 집계하여 정상 범위와 비교합니다.

bucket_span이 짧으면 세밀한 변화를 감지할 수 있지만, 노이즈에 민감해져 거짓 양성(false positive)이 늘어납니다. bucket_span이 길면 안정적이지만, 짧은 시간 동안의 이상을 놓칠 수 있습니다. 데이터의 특성에 따라 적절한 값을 선택합니다. 서버 모니터링에서는 5분에서 15분 사이가 일반적입니다.

**influencer** 필드는 이상의 원인을 좁히는 데 사용됩니다. 이상이 감지되었을 때, "어떤 호스트에서", "어떤 사용자가", "어떤 서비스에서" 이상이 발생했는지를 식별합니다. influencer로 hostname, user, service_name 같은 필드를 지정하면, 이상 감지 결과에서 어떤 값이 이상에 가장 크게 기여했는지 확인할 수 있습니다.

Kibana에서 이상 감지 Job을 생성하는 과정을 단계별로 정리합니다.

```
1. Kibana > Machine Learning > Anomaly Detection > Create Job
      │
2. 데이터 소스 선택 (인덱스 패턴)
      │
3. Job 유형 선택 (단일 메트릭 / 다중 메트릭)
      │
4. detector 함수와 대상 필드 설정
   예: mean(response_time)
      │
5. bucket_span 설정 (예: 15m)
      │
6. influencer 필드 지정 (예: hostname, service)
      │
7. Job 저장 및 Datafeed 시작
```

REST API로도 동일한 Job을 생성할 수 있습니다.

```json
PUT _ml/anomaly_detectors/response-time-job
{
  "analysis_config": {
    "bucket_span": "15m",
    "detectors": [
      {
        "function": "mean",
        "field_name": "response_time",
        "by_field_name": "hostname"
      }
    ],
    "influencers": ["hostname", "service"]
  },
  "data_description": {
    "time_field": "timestamp"
  }
}
```

analysis_config 안에 bucket_span, detectors, influencers를 설정합니다. detectors 배열에서 function을 mean으로, field_name을 response_time으로 지정하면 응답 시간의 평균을 추적합니다. by_field_name을 hostname으로 두면 호스트별로 별도의 모델을 학습합니다. data_description의 time_field는 시계열의 시간 기준 필드입니다.

Job을 만든 후에는 **Datafeed**를 시작해야 데이터 분석이 시작됩니다. Datafeed는 Elasticsearch 인덱스에서 시계열 데이터를 읽어 Job에 공급하는 역할을 합니다. 일정한 주기로 쿼리를 실행하여 새로 들어온 데이터를 가져옵니다.

```
POST _ml/datafeeds/datafeed-response-time-job/_start
```

Datafeed가 시작되면 모델이 데이터를 받아 정상 패턴을 학습합니다. 충분한 데이터가 쌓이면(보통 bucket_span의 수십~수백 배), 정상 범위를 벗어나는 데이터를 이상으로 표시하기 시작합니다. 이 과정은 실시간으로 진행됩니다. 새 데이터가 들어올 때마다 모델이 업데이트되고, 즉시 이상 여부를 판단합니다.

이상 감지 결과에서 가장 중요한 값은 **anomaly score**입니다. 0에서 100 사이의 숫자로, 값이 클수록 이상의 심각도가 높습니다. 0~25는 낮은 수준의 이상, 25~50은 경고 수준, 50~75는 주요 이상, 75~100은 심각한 이상으로 해석합니다. 이 점수는 해당 데이터가 정상 패턴에서 얼마나 벗어났는지와, 과거에 비슷한 편차가 얼마나 드물었는지를 종합하여 산출됩니다.

이상 감지 결과는 **.ml-anomalies-*** 패턴의 내부 인덱스에 저장됩니다. Kibana의 Anomaly Explorer에서 시각적으로 확인할 수 있습니다. Anomaly Explorer는 시간축을 따라 이상 점수를 히트맵으로 표시하여, 언제 어떤 심각도의 이상이 발생했는지 한눈에 파악할 수 있습니다. Single Metric Viewer는 단일 지표의 실제값과 예측 범위를 겹쳐 보여주어, 데이터가 정상 범위를 어떻게 벗어났는지 직관적으로 확인할 수 있습니다.

**Job 스케줄러**는 Datafeed의 실행 주기를 관리합니다. 기본적으로 Datafeed는 실시간으로 동작하여 새 데이터를 지속적으로 분석합니다. 과거 데이터를 분석하고 싶다면 Datafeed의 시작 시간을 과거로 지정하여 historical analysis를 수행할 수도 있습니다.

정리하면, Elasticsearch의 이상 감지는 시계열 데이터에서 비정상 패턴을 자동으로 찾아내는 ML 기능입니다. Job에서 detector 함수로 분석 대상을 정의하고, bucket_span으로 시간 간격을, influencer로 원인 추적 필드를 설정합니다. Datafeed가 인덱스에서 데이터를 읽어 Job에 공급하면, 모델이 정상 패턴을 학습하고 이상을 실시간으로 감지합니다. anomaly score로 이상의 심각도를 판단하며, Kibana의 Anomaly Explorer와 Single Metric Viewer로 결과를 시각적으로 확인합니다.

다음 단원인 7.5.2에서는 이상 감지 결과를 Elastic Security(SIEM)와 연동하여 보안 모니터링에 활용하는 방법을 다룹니다.

이 단원을 마치면 Kibana에서 이상 감지 Job을 생성하고 결과를 해석할 수 있습니다.
