# 2.3.4 한국어 분석: Nori

2.3.3에서 캐릭터 필터, 토크나이저, 토큰 필터를 직접 조합하는 커스텀 분석기를 만들어 보았습니다. 영어처럼 단어 사이에 공백이 명확한 언어는 standard 토크나이저만으로도 기본적인 분리가 가능합니다. 그런데 한국어는 사정이 다릅니다. "선생님께서"라는 어절 하나 안에 "선생", "님", "께서"라는 세 개의 형태소가 붙어 있습니다. 공백으로 나누면 "선생님께서"가 통째로 하나의 토큰이 되어, "선생"이나 "선생님"으로 검색해도 일치하지 않습니다. 한국어 텍스트를 제대로 검색하려면 어절을 형태소 단위로 쪼개는 별도의 분석기가 필요합니다.

Elasticsearch는 한국어 형태소 분석을 위해 **Nori**라는 공식 플러그인을 제공합니다. Nori는 Elasticsearch 6.4 버전에서 처음 도입되었으며, 내부적으로 Apache Lucene의 한국어 분석 모듈을 사용합니다. 한국어에 특화된 사전과 알고리즘이 내장되어 있어, 설치만 하면 별도의 외부 사전 없이도 형태소 분석이 동작합니다.

Nori가 사용하는 사전은 **mecab-ko-dic**입니다. 이 사전은 대한민국 정부가 1998년부터 추진한 21세기 세종 프로젝트에서 구축한 한국어 데이터를 기반으로 만들어졌습니다. 원본 사전은 81만 개 이상의 단어를 포함하며 용량이 219MB에 달합니다. Nori는 이 사전을 24MB 크기의 바이너리 형식으로 압축하여 내장합니다. 압축에는 FST(Finite State Transducer)라는 자료구조가 사용됩니다. FST는 공통 접두어와 접미어를 공유하는 방식으로 수십만 단어를 수 MB 수준으로 줄입니다. 이 덕분에 Nori는 메모리를 적게 사용하면서도 빠른 사전 조회가 가능합니다.

사전이 준비되었으면 텍스트를 어떤 방식으로 분석하는지 살펴봅니다. Nori는 **비터비 알고리즘**(Viterbi algorithm)을 사용하여 최적의 형태소 분리 경로를 찾습니다. 비터비 알고리즘은 여러 가능한 분리 후보 중에서 비용이 가장 낮은 경로를 선택하는 동적 프로그래밍 기법입니다. 여기서 비용이란 사전에 미리 계산된 연결 비용(connection cost)으로, 형태소와 형태소가 이어질 때 자연스러운 정도를 수치로 나타낸 것입니다.

구체적인 분석 과정을 단계별로 살펴봅니다.

```
입력: "선생님께서"
  |
  v
[1단계] 사전 조회
  글자를 하나씩 읽으며 사전에서 가능한 형태소를 찾습니다.
  "선", "선생", "님", "께서" 등이 후보로 올라옵니다.
  |
  v
[2단계] 격자(lattice) 구성
  모든 후보 형태소를 격자 구조에 배치합니다.
  각 후보에는 사전에서 가져온 비용이 붙습니다.
  |
  v
[3단계] 비터비 경로 탐색
  후보 형태소를 연결하는 모든 경로의 비용을 계산합니다.
  비용이 가장 낮은 경로를 최종 분리 결과로 선택합니다.
  |
  v
출력: "선생" + "님" + "께서"
```

1단계에서 Nori는 텍스트를 한 글자씩 읽으며 사전에서 일치하는 단어를 찾습니다. 2단계에서 찾아낸 후보 형태소를 격자(lattice) 구조에 배치합니다. 격자란 텍스트의 각 위치에서 가능한 형태소를 노드로 나열한 그래프입니다. 3단계에서 비터비 알고리즘이 격자 위의 모든 경로를 탐색하여 총 비용이 가장 낮은 경로를 선택합니다. 이 과정에서 비용이 높은 경로는 중간에 가지치기(pruning)되므로 전체 경우의 수를 모두 계산하지 않아도 됩니다. Nori는 문장 경계나 1,024자마다 처리를 나누어 메모리가 과도하게 사용되지 않도록 합니다.

이제 Nori 플러그인을 설치하는 방법을 알아봅니다. Nori는 Elasticsearch에 기본 내장되어 있지 않고 별도로 설치해야 합니다. 설치 명령은 다음과 같습니다.

```
sudo bin/elasticsearch-plugin install analysis-nori
```

`bin/elasticsearch-plugin`은 Elasticsearch가 제공하는 플러그인 관리 명령입니다. `install` 뒤에 플러그인 이름인 `analysis-nori`를 지정하면 자동으로 다운로드와 설치가 진행됩니다. 클러스터 환경에서는 모든 노드에 각각 설치해야 합니다. 한 노드에만 설치하면 해당 노드에서는 Nori 분석기가 동작하지만, 설치하지 않은 노드에서는 인덱스를 열 수 없어 오류가 발생합니다. 설치 후에는 각 노드를 재시작해야 플러그인이 활성화됩니다.

Nori 플러그인을 설치하면 세 가지 구성요소가 추가됩니다. 토크나이저 하나와 토큰 필터 두 개입니다. 2.3.3에서 배운 커스텀 분석기의 구조를 떠올려 봅니다. 커스텀 분석기는 캐릭터 필터, 토크나이저, 토큰 필터를 조합하여 만들었습니다. Nori도 같은 방식으로 이 구성요소들을 `settings.analysis` 블록에서 조합합니다.

먼저 **nori_tokenizer**를 살펴봅니다. nori_tokenizer는 한국어 형태소 분석을 수행하는 토크나이저입니다. 앞에서 설명한 mecab-ko-dic 사전과 비터비 알고리즘을 내장하고 있어, 한국어 어절을 형태소 단위로 분리합니다.

nori_tokenizer에는 **decompound_mode**라는 중요한 옵션이 있습니다. 이 옵션은 복합 명사를 어떻게 처리할지 결정합니다. 복합 명사란 "삼성전자"처럼 두 개 이상의 명사가 결합한 단어를 말합니다. "삼성전자"는 "삼성"과 "전자"라는 두 명사가 합쳐진 것입니다. 검색 시스템에서 복합 명사를 통째로만 인덱싱하면 "삼성"이나 "전자"로는 검색할 수 없습니다. 반대로 분리된 부분만 인덱싱하면 "삼성전자"라는 정확한 표현으로 검색할 때 정밀도가 떨어질 수 있습니다. decompound_mode는 이런 상황에 맞게 세 가지 선택지를 제공합니다.

첫 번째 모드는 **none**입니다. 복합 명사를 분리하지 않고 원래 형태 그대로 유지합니다. "삼성전자"가 하나의 토큰으로 나옵니다.

두 번째 모드는 **discard**입니다. 복합 명사를 구성 요소로 분리하고 원래의 복합 형태는 버립니다. "삼성전자"가 "삼성"과 "전자" 두 개의 토큰으로 분리되고, "삼성전자"라는 토큰은 생성되지 않습니다. decompound_mode의 기본값이 discard입니다.

세 번째 모드는 **mixed**입니다. 복합 명사를 구성 요소로 분리하면서 원래의 복합 형태도 함께 출력합니다. "삼성전자", "삼성", "전자" 세 개의 토큰이 모두 생성됩니다.

세 모드의 차이를 정리합니다.

```
입력: "삼성전자"

+----------+-------------------------------+
| 모드     | 출력 토큰                     |
+----------+-------------------------------+
| none     | 삼성전자                      |
| discard  | 삼성, 전자                    |
| mixed    | 삼성전자, 삼성, 전자          |
+----------+-------------------------------+
```

none은 정확히 일치하는 검색에 적합합니다. discard는 구성 요소별 검색 범위를 넓히고 싶을 때 사용합니다. mixed는 정확 일치와 부분 일치를 모두 지원하고 싶을 때 유용합니다. 다만 mixed는 토큰 수가 늘어나므로 인덱스 크기가 다소 커집니다. 일반적인 한국어 검색 서비스에서는 discard나 mixed를 많이 사용합니다.

다음은 nori_tokenizer를 사용하는 커스텀 분석기를 설정하는 예입니다.

```
PUT my_korean_index
{
  "settings": {
    "analysis": {
      "tokenizer": {
        "my_nori_tokenizer": {
          "type": "nori_tokenizer",
          "decompound_mode": "mixed"
        }
      },
      "analyzer": {
        "my_nori_analyzer": {
          "type": "custom",
          "tokenizer": "my_nori_tokenizer"
        }
      }
    }
  }
}
```

`tokenizer` 블록에서 "my_nori_tokenizer"를 정의했습니다. `type`을 "nori_tokenizer"로 지정하면 Nori 플러그인이 제공하는 한국어 형태소 분석 토크나이저를 사용합니다. `decompound_mode`를 "mixed"로 설정하여 복합 명사의 원형과 구성 요소를 모두 토큰으로 출력하도록 했습니다. `analyzer` 블록에서는 이 토크나이저를 조합한 커스텀 분석기를 선언합니다. 2.3.3에서 배운 커스텀 분석기 구조와 동일합니다.

이제 Nori가 제공하는 첫 번째 토큰 필터인 **nori_part_of_speech**를 살펴봅니다. 한국어 형태소 분석을 하면 명사, 동사, 조사, 어미 등 다양한 품사의 토큰이 생성됩니다. 검색에서는 의미를 담고 있는 명사나 동사가 중요하고, "은", "는", "이", "가" 같은 조사나 "습니다", "ㅂ니다" 같은 어미는 검색 정밀도에 기여하지 않는 경우가 많습니다. nori_part_of_speech 필터는 지정한 품사 태그에 해당하는 토큰을 제거합니다. 영어에서 불용어(stopword) 필터가 "the", "a" 등을 제거하는 것과 비슷한 역할입니다.

nori_part_of_speech 필터의 설정 방식을 봅니다.

```
PUT my_korean_index
{
  "settings": {
    "analysis": {
      "tokenizer": {
        "my_nori_tokenizer": {
          "type": "nori_tokenizer",
          "decompound_mode": "mixed"
        }
      },
      "filter": {
        "my_pos_filter": {
          "type": "nori_part_of_speech",
          "stoptags": [
            "E",
            "J",
            "IC"
          ]
        }
      },
      "analyzer": {
        "my_nori_analyzer": {
          "type": "custom",
          "tokenizer": "my_nori_tokenizer",
          "filter": ["my_pos_filter"]
        }
      }
    }
  }
}
```

`filter` 블록에서 "my_pos_filter"를 정의했습니다. `type`이 "nori_part_of_speech"이고, `stoptags` 배열에 제거할 품사 태그를 나열합니다. "E"는 어미(Ending), "J"는 조사(Josa), "IC"는 감탄사(Interjection)를 뜻합니다. 이 필터를 적용하면 형태소 분석 결과에서 어미, 조사, 감탄사에 해당하는 토큰이 제거되고 명사, 동사 같은 의미 있는 토큰만 남습니다. stoptags에 어떤 품사를 넣을지는 검색 요구사항에 따라 조정합니다. 조사와 어미를 제거하는 것이 가장 일반적인 설정입니다.

두 번째 토큰 필터는 **nori_readingform**입니다. 이 필터는 한자(Hanja)를 한글 발음으로 변환합니다. 한국어 텍스트에는 "大韓民國"처럼 한자가 포함되는 경우가 있습니다. 사용자는 보통 한자가 아니라 한글로 검색합니다. "대한민국"이라고 검색하면 "大韓民國"이 포함된 문서도 찾아야 합니다. nori_readingform 필터를 적용하면 "大韓民國"이라는 토큰이 "대한민국"으로 변환되어 인덱싱됩니다. 이 필터는 별도의 설정 옵션 없이 `type`만 지정하면 됩니다.

```
"filter": {
  "my_reading": {
    "type": "nori_readingform"
  }
}
```

이 필터를 분석기에 추가하면 한자가 포함된 문서를 한글로 검색할 수 있습니다. 법률 문서, 인명, 지명 등 한자가 자주 등장하는 분야에서 유용합니다.

지금까지 살펴본 nori_tokenizer, nori_part_of_speech, nori_readingform을 모두 조합하면 한국어 검색에 적합한 분석기가 완성됩니다. 분석 흐름을 정리합니다.

```
입력: "大韓民國의 선생님께서"
  |
  v
[nori_tokenizer (mixed)]
  "大韓民國", "대한", "민국", "의", "선생", "님", "께서"
  |
  v
[nori_part_of_speech (stoptags: E, J)]
  조사 "의", "께서" 제거, 접미사 "님" 제거
  --> "大韓民國", "대한", "민국", "선생"
  |
  v
[nori_readingform]
  한자를 한글로 변환
  --> "대한민국", "대한", "민국", "선생"
```

nori_tokenizer가 형태소를 분리하고, nori_part_of_speech가 불필요한 품사를 걸러내고, nori_readingform이 한자를 한글로 바꿉니다. 이 파이프라인을 거치면 "大韓民國의 선생님께서"라는 텍스트에서 "대한민국", "대한", "민국", "선생"이라는 의미 있는 한글 토큰이 추출됩니다.

Nori는 내장 사전만으로도 일반적인 한국어 텍스트를 잘 처리하지만, 도메인에 특화된 단어는 사전에 없을 수 있습니다. 예를 들어 특정 회사의 제품명, 신조어, 전문 용어 같은 단어는 내장 사전에 등록되어 있지 않아 의도하지 않은 방식으로 분리될 수 있습니다. 이런 경우 **사용자 정의 사전**(user dictionary)을 등록하여 해결합니다.

사용자 정의 사전은 텍스트 파일로 작성합니다. 파일을 Elasticsearch의 config 디렉터리 아래에 저장하고, nori_tokenizer 설정에서 해당 파일의 경로를 지정합니다.

```
PUT my_korean_index
{
  "settings": {
    "analysis": {
      "tokenizer": {
        "my_nori_tokenizer": {
          "type": "nori_tokenizer",
          "decompound_mode": "mixed",
          "user_dictionary": "analysis/my_dict.txt"
        }
      },
      "analyzer": {
        "my_nori_analyzer": {
          "type": "custom",
          "tokenizer": "my_nori_tokenizer",
          "filter": ["nori_part_of_speech", "nori_readingform"]
        }
      }
    }
  }
}
```

`user_dictionary` 필드에 사전 파일의 경로를 지정합니다. 이 경로는 Elasticsearch config 디렉터리를 기준으로 한 상대 경로입니다. 2.3.3에서 동의어 파일의 경로를 `synonyms_path`로 지정했던 것과 같은 방식입니다.

사용자 정의 사전 파일의 형식은 한 줄에 하나의 단어를 적습니다. 복합 명사의 경우 분리할 방법을 함께 지정할 수 있습니다. 첫 번째 토큰이 등록할 단어이고, 뒤에 이어지는 토큰이 분리 결과입니다.

```
삼성전자
엘라스틱서치 엘라스틱 서치
```

첫 번째 줄의 "삼성전자"는 분리 방법을 지정하지 않았으므로 통째로 하나의 명사로 인식됩니다. 두 번째 줄의 "엘라스틱서치"는 뒤에 "엘라스틱"과 "서치"를 적어 두었으므로, 이 단어가 나오면 "엘라스틱"과 "서치"로 분리합니다. 이처럼 사용자 정의 사전을 활용하면 도메인 고유의 단어를 원하는 방식으로 분석할 수 있습니다.

분석기를 설정한 뒤에는 결과를 검증해야 합니다. Elasticsearch의 _analyze API를 사용하면 특정 분석기가 텍스트를 어떤 토큰으로 분리하는지 확인할 수 있습니다.

```
POST my_korean_index/_analyze
{
  "analyzer": "my_nori_analyzer",
  "text": "대한민국의 수도는 서울입니다"
}
```

이 요청은 "my_nori_analyzer"를 사용하여 주어진 텍스트를 분석한 결과를 반환합니다. 응답에는 각 토큰의 텍스트, 품사 태그, 시작 위치, 끝 위치가 포함됩니다. 이 정보를 통해 형태소가 의도한 대로 분리되는지, 불필요한 품사가 제거되는지, 한자가 한글로 변환되는지를 확인합니다. 새로운 사용자 정의 사전 단어를 등록한 뒤에도 _analyze API로 분리 결과를 검증하는 것이 좋습니다.

Nori의 성능에 대해서도 짚어 봅니다. 약 41만 건의 한국어 위키피디아 문서로 테스트했을 때, Nori는 초당 3,000건 이상의 문서를 인덱싱했습니다. 같은 테스트에서 다른 한국어 분석기는 메모리 부족으로 장애가 발생하거나 처리 속도가 초당 수백 건에 그쳤습니다. Nori는 사전 데이터를 Java 힙 바깥의 직접 바이트 버퍼(direct byte buffer)에 올려 메모리를 효율적으로 사용합니다. 이 덕분에 메모리가 제한된 환경에서도 안정적으로 동작합니다.

정리하면, Nori는 Elasticsearch 공식 한국어 형태소 분석 플러그인으로, mecab-ko-dic 사전과 비터비 알고리즘을 사용하여 한국어 어절을 형태소 단위로 분리합니다. nori_tokenizer의 decompound_mode 옵션으로 복합 명사 처리 방식을 none, discard, mixed 중 선택할 수 있고, nori_part_of_speech 필터로 불필요한 품사를 제거하며, nori_readingform 필터로 한자를 한글로 변환합니다. 도메인 고유 단어는 사용자 정의 사전을 등록하여 분석 품질을 높일 수 있습니다.

다음 단원인 2.4.1에서는 런타임 필드를 다룹니다.

이 단원을 마치면 Nori 플러그인을 설치하고 한국어 형태소 분석을 적용할 수 있으며, nori_tokenizer의 decompound_mode 옵션을 상황에 맞게 선택할 수 있습니다.
