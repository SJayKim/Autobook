# 6.4 보안 및 운영

이 섹션에서는 프로덕션 Elasticsearch 클러스터의 보안 설정, 데이터 백업 및 복구, 모니터링, 그리고 무중단 업그레이드 절차를 다룹니다.

## 토픽 목록

**6.4.1 TLS/RBAC/API Key 보안**
- Transport/HTTP TLS를 활성화하고 RBAC으로 사용자 권한을 관리할 수 있다
- API Key를 생성하고 애플리케이션 인증에 적용할 수 있다

xpack.security.enabled 활성화, Transport TLS 설정(transport.ssl), HTTP TLS 설정(http.ssl), elasticsearch-certutil 도구, 내장 사용자(elastic, kibana_system) 비밀번호 설정, RBAC 역할(role) 정의, 역할 매핑(role mapping), API Key 생성 및 사용, 필드 수준 보안(FLS)과 도큐먼트 수준 보안(DLS)을 다룹니다.

**6.4.2 스냅샷과 복구**
- 스냅샷 저장소를 등록하고 스냅샷을 생성·복구하는 절차를 수행할 수 있다
- SLM(Snapshot Lifecycle Management)으로 자동 스냅샷을 스케줄링할 수 있다

스냅샷 저장소 등록(S3, shared filesystem), 스냅샷 생성(_snapshot API), 증분 스냅샷 메커니즘, 스냅샷 복구(_restore API), 부분 복구(indices 필터), SLM 정책 생성, 스냅샷 상태 모니터링, 재해 복구(DR) 시나리오를 다룹니다.

**6.4.3 클러스터 모니터링**
- Kibana Stack Monitoring으로 클러스터 상태와 성능을 모니터링할 수 있다
- 주요 지표를 해석하여 이상을 감지할 수 있다

Stack Monitoring UI, 클러스터 상태(green/yellow/red) 의미, 노드 JVM heap 사용률 지표, 인덱싱/검색 처리량 지표, 스레드 풀 거부(rejected) 지표, 디스크 워터마크 경보, Metricbeat 기반 모니터링 설정, Watcher 알림 설정을 다룹니다.

**6.4.4 롤링 업그레이드**
- 무중단 롤링 업그레이드 절차를 단계별로 수행할 수 있다

롤링 업그레이드 vs 풀 클러스터 재시작, 업그레이드 전 체크리스트, 샤드 할당 비활성화(cluster.routing.allocation.enable), 노드 순서별 업그레이드 절차, 클러스터 상태 확인 후 다음 노드 진행, 업그레이드 후 샤드 재활성화, Kibana 업그레이드 순서, 브레이킹 체인지 마이그레이션 가이드 확인을 다룹니다.
