# 3.3 고급 검색 기법

이 섹션에서는 실무 검색 서비스에서 자주 필요한 고급 검색 기능을 다룹니다. 여러 필드에 걸친 검색, 자동완성, 하이라이팅, 대용량 페이지네이션, 위치 기반 검색을 구현하는 방법을 익힙니다.

## 토픽 목록

**3.3.1 multi_match와 cross-field 검색**
- multi_match 쿼리의 type 옵션을 비교하여 적합한 방식을 선택할 수 있다
- cross_fields 타입으로 여러 필드에 분산된 키워드를 효과적으로 검색할 수 있다

multi_match 쿼리 기본 구조, best_fields 타입(기본값), most_fields 타입, cross_fields 타입, phrase 타입, phrase_prefix 타입, tie_breaker 파라미터, 필드별 boost(^N 표기법)를 다룹니다.

**3.3.2 자동완성 구현**
- completion suggester로 실시간 자동완성을 구현할 수 있다
- search_as_you_type 필드 타입을 적용할 수 있다

completion 필드 타입과 FST 구조, suggest API 사용법, completion suggester weight 설정, search_as_you_type 필드 타입, edge_ngram 기반 자동완성 패턴, match_phrase_prefix 기반 자동완성, 자동완성 방식 성능 비교를 다룹니다.

**3.3.3 하이라이팅**
- highlight 파라미터로 검색 결과에 하이라이팅을 적용할 수 있다
- unified, plain, fvh 하이라이터의 차이를 설명할 수 있다

highlight 파라미터 구조, pre_tags/post_tags 커스터마이징, unified 하이라이터(기본값), plain 하이라이터, fvh(fast vector highlighter)와 term_vector 설정, fragment_size/number_of_fragments 설정, require_field_match 옵션을 다룹니다.

**3.3.4 페이지네이션**
- from/size, search_after, scroll API의 차이와 적합한 사용 사례를 설명할 수 있다
- search_after로 deep pagination 문제를 해결할 수 있다

from/size 페이지네이션과 deep pagination 한계, index.max_result_window 설정(기본 10000), search_after 파라미터, PIT(Point In Time) API, search_after + PIT 조합, scroll API(배치 처리용), scroll 컨텍스트 메모리 비용을 다룹니다.

**3.3.5 지리 검색**
- geo_distance 쿼리로 반경 내 문서를 검색할 수 있다
- geo_bounding_box 쿼리로 직사각형 범위 검색을 수행할 수 있다

geo_point 타입 매핑, geo_distance 쿼리, geo_bounding_box 쿼리, geo_polygon 쿼리, geo_shape 타입과 쿼리, 거리 단위(km, mi, m), sort by geo_distance, geohash와 geotile grid 집계를 다룹니다.
