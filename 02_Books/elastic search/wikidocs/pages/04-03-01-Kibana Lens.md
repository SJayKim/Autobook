# 4.3.1 Kibana Lens

Elasticsearch에 데이터를 인덱싱하고, 4.1.2에서 다룬 Bucket 집계와 Metric 집계로 통계를 뽑아 봤다면, 다음 단계로 자연스럽게 떠오르는 질문이 있습니다. "이 집계 결과를 차트로 보려면 어떻게 해야 하는가?" JSON 응답에 담긴 숫자를 눈으로 읽어 가며 추세를 파악하는 일은 문서가 몇 건일 때는 가능하지만, 수천 건 이상의 데이터를 다루기 시작하면 시각적 표현이 필수입니다. Kibana는 Elasticsearch 데이터를 시각화하는 웹 인터페이스이며, 그중 **Kibana Lens**는 드래그 앤 드롭 방식으로 차트를 만드는 시각화 편집기입니다.

Lens가 등장하기 전에는 Kibana에서 시각화를 만들 때 차트 종류를 미리 결정하고, 인덱스 패턴을 선택한 뒤, 집계 설정을 하나하나 입력해야 했습니다. 이 과정은 Elasticsearch의 집계 구조를 이미 알고 있는 사용자에게는 괜찮지만, 처음 접하는 사용자에게는 진입 장벽이 높았습니다. Lens는 이 문제를 해결합니다. 사용자가 필드를 화면으로 끌어다 놓기만 하면 Lens가 데이터의 특성에 따라 적절한 차트 타입을 자동으로 추천합니다. 차트를 먼저 그린 뒤에 타입을 바꿀 수도 있어서, 데이터를 탐색하면서 가장 적합한 시각화를 찾아갈 수 있습니다.

Lens를 사용하려면 먼저 **데이터 뷰**(Data View)가 필요합니다. 데이터 뷰는 Kibana가 Elasticsearch의 어떤 인덱스를 바라볼지 정의하는 설정입니다. Elasticsearch에는 여러 인덱스가 존재할 수 있고, 하나의 데이터 뷰가 하나의 인덱스를 가리킬 수도 있고, 와일드카드 패턴으로 여러 인덱스를 한꺼번에 가리킬 수도 있습니다. 예를 들어 "logs-*"라는 패턴을 사용하면 logs-2025-01, logs-2025-02 등 이름이 logs-로 시작하는 모든 인덱스를 하나의 데이터 뷰로 묶습니다.

데이터 뷰를 생성하는 방법은 다음과 같습니다. Kibana 좌측 메뉴에서 Stack Management를 열고, Data Views 항목으로 이동합니다. Create data view 버튼을 누르면 인덱스 패턴을 입력하는 화면이 나타납니다. 패턴을 입력하면 해당 패턴에 매칭되는 인덱스 목록이 아래에 표시됩니다. 타임스탬프 필드가 있는 인덱스라면, 시간 기준으로 사용할 필드를 선택하는 옵션이 추가로 나타납니다. 이 타임스탬프 필드 설정은 뒤에서 설명할 시간 필터 연동에 직접 관여하므로, 시계열 데이터를 다룬다면 올바른 필드를 지정해야 합니다.

데이터 뷰가 준비되었으면 Lens 화면으로 이동합니다. Kibana 좌측 메뉴에서 Visualize Library를 선택하고, Create new visualization 버튼을 누른 뒤 Lens를 선택합니다. 화면이 열리면 크게 세 영역이 보입니다. 왼쪽에는 현재 데이터 뷰에 속한 필드 목록이 나열됩니다. 가운데에는 차트가 표시되는 작업 영역이 있습니다. 오른쪽에는 현재 차트의 축과 레이어를 설정하는 패널이 있습니다.

왼쪽 필드 목록에서 필드 이름을 클릭하면 해당 필드의 값 분포를 미리 볼 수 있습니다. keyword 필드라면 상위 값과 비율이 표시되고, 숫자 필드라면 히스토그램 형태의 분포가 나타납니다. 이 미리보기를 통해 어떤 필드를 차트에 사용할지 판단할 수 있습니다. 필드 수가 많을 때는 상단의 검색창에 필드 이름을 입력하여 빠르게 찾을 수 있고, 필드 타입별로 필터링하는 기능도 제공됩니다.

시각화를 만드는 핵심 동작은 **필드를 축에 매핑**하는 것입니다. 가장 기본적인 패턴은 X축과 Y축에 각각 필드를 지정하는 것입니다. 예를 들어 시간에 따른 요청 수를 보고 싶다면, 타임스탬프 필드를 X축에 놓고 Y축에는 Count(문서 수)를 지정합니다. 가격 필드의 평균값을 카테고리별로 비교하고 싶다면, 카테고리 필드를 X축에, 가격 필드의 Average를 Y축에 놓습니다.

축 매핑을 수행하는 방법은 두 가지입니다. 첫째, 왼쪽 필드 목록에서 필드를 오른쪽 설정 패널의 축 영역으로 직접 드래그합니다. 둘째, 오른쪽 설정 패널에서 축 영역을 클릭한 뒤 필드를 선택하고 집계 함수를 고릅니다. Y축에 사용할 수 있는 집계 함수로는 Count, Sum, Average, Minimum, Maximum, Unique Count, Percentile 등이 있습니다. 이 함수들은 4.1.2에서 다룬 Metric 집계와 동일한 연산입니다. Lens가 내부적으로 Elasticsearch에 집계 요청을 보내고, 그 결과를 차트로 렌더링하는 것입니다.

필드를 축에 놓으면 Lens가 데이터 특성에 맞는 차트를 자동으로 선택합니다. 그러나 사용자가 원하는 차트 타입으로 언제든 변경할 수 있습니다. 오른쪽 설정 패널 상단에 현재 차트 타입이 표시되며, 이를 클릭하면 선택 가능한 타입 목록이 나타납니다. 주요 차트 타입과 적합한 상황을 정리합니다.

```
차트 타입          적합한 상황
--------------    -------------------------------------------
막대(Bar)          카테고리별 수량 비교, 시간대별 건수 추이
선(Line)           시간에 따른 연속적 추세 표현
파이(Pie)          전체 대비 각 항목의 비율 파악
히트맵(Heatmap)    두 축의 교차점마다 밀도 표현
영역(Area)         누적 추세를 면적으로 강조
테이블(Table)      정확한 수치를 표 형태로 확인
트리맵(Treemap)    계층적 비율을 면적 크기로 비교
메트릭(Metric)     단일 수치(총 주문 수, 평균 응답 시간 등) 강조
```

예를 들어 CPU 사용률 상위 5개 프로세스를 시간대별로 비교하려면 다음과 같이 설정합니다. X축에 타임스탬프 필드를 놓습니다. Y축에 system.process.cpu.total.pct 필드의 Maximum을 지정합니다. 그런 다음 차트를 process.executable 필드로 분할하여 상위 5개 값만 표시하도록 설정합니다. 차트 타입을 막대(Bar)로 선택하면, 시간대마다 어떤 프로세스가 CPU를 많이 사용했는지 한눈에 비교할 수 있습니다.

Kibana 화면 상단에는 **시간 필터**가 있습니다. 이 필터는 데이터 뷰에서 지정한 타임스탬프 필드를 기준으로 작동합니다. "Last 15 minutes", "Last 1 hour", "Last 7 days"와 같이 상대적인 시간 범위를 선택하거나, 달력에서 시작일과 종료일을 직접 지정하는 절대 범위를 설정할 수 있습니다. 시간 필터를 변경하면 Lens 차트가 즉시 갱신됩니다. 시계열 데이터를 다루는 시각화에서 이 시간 필터는 데이터의 범위를 결정하는 가장 기본적인 도구입니다.

하나의 차트에 서로 다른 관점을 겹쳐 보고 싶을 때 **Breakdown 레이어**를 사용합니다. Breakdown은 하나의 축 위에 추가 필드를 기준으로 데이터를 세분화하는 기능입니다. 앞에서 CPU 사용률 예시에서 process.executable 필드로 차트를 분할한 것이 Breakdown의 한 형태입니다. 오른쪽 설정 패널에서 "Break down by" 영역에 필드를 지정하면 됩니다.

Lens는 하나의 시각화 안에 여러 레이어를 겹칠 수도 있습니다. 예를 들어 첫 번째 레이어에서는 선 차트로 전체 소켓 수의 추이를 그리고, 두 번째 레이어에서는 막대 차트로 TCP와 UDP의 소켓 상태 분포를 표시할 수 있습니다. 레이어를 추가하려면 설정 패널 하단의 "Add layer" 버튼을 사용합니다. 각 레이어는 독립적으로 차트 타입과 필드, 집계 함수를 설정할 수 있어서, 서로 다른 지표를 한 화면에서 비교하는 복합 시각화를 만들 수 있습니다.

기본 집계 함수만으로 표현하기 어려운 계산이 필요할 때 Lens의 **수식(Formula)** 기능을 사용합니다. Y축 설정에서 집계 함수 대신 "Formula"를 선택하면 수식 입력창이 열립니다. 이 입력창에 Lens가 제공하는 함수를 조합하여 계산식을 작성합니다.

예를 들어 오류 비율을 백분율로 표시하려면 다음과 같은 수식을 입력합니다.

```
count(kql='status: 500') / count() * 100
```

이 수식의 각 부분을 설명합니다. count(kql='status: 500')은 status 필드 값이 500인 문서의 수를 셉니다. kql= 부분은 필터 조건을 수식 안에 직접 포함하는 문법입니다. count()은 전체 문서 수를 셉니다. 두 값을 나누고 100을 곱하면 오류 문서의 백분율이 됩니다. Formula에서 사용할 수 있는 함수로는 count(), sum(), average(), min(), max(), unique_count(), percentile() 등이 있으며, 사칙연산과 괄호를 조합하여 복합 지표를 정의할 수 있습니다.

시각화를 완성했으면 우측 상단의 Save 버튼을 눌러 저장합니다. 저장된 시각화는 Visualize Library에 보관되며, 이후 대시보드에 패널로 추가할 수 있습니다.

정리하면, Kibana Lens는 드래그 앤 드롭으로 Elasticsearch 데이터를 시각화하는 편집기입니다. 데이터 뷰를 생성하여 인덱스를 지정하고, 필드를 X축과 Y축에 매핑하면 차트가 만들어집니다. 차트 타입은 막대, 선, 파이, 히트맵 등으로 자유롭게 전환할 수 있고, Breakdown 레이어로 데이터를 세분화하거나 여러 레이어를 겹쳐 복합 시각화를 구성할 수 있습니다. 기본 집계 함수로 부족할 때는 Formula 수식으로 계산식을 직접 작성합니다.

다음 단원인 4.3.2에서는 Kibana Dashboard를 다룹니다. Lens로 만든 개별 시각화를 한 화면에 배치하여 종합적인 모니터링 대시보드를 구성하는 방법을 살펴봅니다.

이 단원을 마치면 Kibana Lens로 드래그 앤 드롭 방식으로 시각화를 생성할 수 있습니다.
