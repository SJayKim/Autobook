# 7.1 벡터 검색

이 섹션에서는 텍스트나 이미지를 수치 벡터로 표현하고 유사도 기반으로 검색하는 벡터 검색을 다룹니다. dense_vector 타입과 HNSW 알고리즘, kNN 검색 실행 방법, 그리고 임베딩 모델을 Elasticsearch에 배포하는 방법을 익힙니다.

## 토픽 목록

**7.1.1 dense_vector와 HNSW**
- dense_vector 매핑을 설정하고 HNSW 인덱스 파라미터를 조정할 수 있다

dense_vector 타입 dims 설정, index: true로 HNSW 인덱스 활성화, similarity 메트릭(cosine, dot_product, l2_norm), HNSW(Hierarchical Navigable Small World) 알고리즘, m 파라미터(연결 수), ef_construction 파라미터(빌드 품질), 벡터 차원 수와 메모리 사용량, byte 양자화 dense_vector(8비트)를 다룹니다.

**7.1.2 kNN 검색**
- knn 파라미터로 근사 최근접 이웃 검색을 수행할 수 있다
- exact kNN과 approximate kNN의 차이를 설명할 수 있다

knn 쿼리 파라미터(k, num_candidates), query_vector vs query_vector_builder, exact kNN(script_score 기반), approximate kNN(HNSW 기반), num_candidates와 recall 트레이드오프, pre-filter와 post-filter 차이, kNN 검색 응답 구조를 다룹니다.

**7.1.3 임베딩 모델 배포**
- Elasticsearch ML 노드에 외부 임베딩 모델을 배포하고 인제스트 파이프라인에 연결할 수 있다

Elasticsearch ML 모델 저장소, eland 라이브러리로 모델 업로드, PUT _ml/trained_models API, 모델 배포(_deploy API), inference 프로세서로 인제스트 파이프라인 통합, text_embedding 태스크 타입, 모델 성능 모니터링을 다룹니다.
