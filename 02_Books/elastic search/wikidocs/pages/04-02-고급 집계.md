# 4.2 고급 집계

이 섹션에서는 기본 집계를 조합하여 더 복잡한 분석을 수행하는 방법을 다룹니다. 집계 중첩, 집계 결과를 재가공하는 Pipeline 집계, 대용량 다차원 집계를 위한 composite 집계, 그리고 집계 성능 최적화를 다룹니다.

## 토픽 목록

**4.2.1 중첩 집계**
- Bucket 집계 내부에 Metric 집계를 중첩하여 그룹별 통계를 계산할 수 있다

집계 내부에 하위 집계 정의, terms + avg 중첩 패턴, date_histogram + sum 중첩 패턴, 3단계 중첩 집계, 중첩 집계의 메모리 사용, nested 타입 문서에서의 nested 집계, reverse_nested 집계를 다룹니다.

**4.2.2 Pipeline 집계**
- Pipeline 집계로 집계 결과를 입력으로 재계산하는 이중 집계를 구현할 수 있다

Pipeline 집계 개념(집계의 집계), avg_bucket/sum_bucket/min_bucket/max_bucket, derivative 집계(증분 변화), cumulative_sum 집계(누적 합), moving_avg/moving_fn 집계, bucket_script 집계, buckets_path 문법을 다룹니다.

**4.2.3 composite 집계**
- composite 집계로 대용량 다차원 집계를 페이지네이션하며 처리할 수 있다

composite 집계 개념, sources 배열 정의, after_key를 이용한 페이지네이션, terms/date_histogram/histogram/geotile_grid 소스, composite 집계와 terms 집계 비교, Kibana 사용 사례를 다룹니다.

**4.2.4 집계 성능 최적화**
- 집계 성능 저하 원인을 진단하고 최적화 방법을 적용할 수 있다

doc_values 기반 집계 성능, eager global ordinals(terms 집계 최적화), shard_size 파라미터 조정, execution_hint: global_ordinals vs map, 집계 캐시(request cache), index.number_of_replicas와 집계 병렬성, 샤드 크기와 집계 정확도 트레이드오프를 다룹니다.
