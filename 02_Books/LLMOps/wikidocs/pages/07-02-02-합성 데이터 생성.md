# 7.2.2 합성 데이터 생성

파인튜닝을 진행하려면 학습 데이터가 필요합니다. 7.2.1에서 다룬 것처럼 기존 데이터를 큐레이션하고 품질을 관리하는 것이 첫 단계입니다. 그런데 현실에서는 양질의 학습 데이터를 충분히 확보하기 어려운 상황이 자주 발생합니다. 도메인이 특수하여 공개 데이터셋이 없거나, 사람이 직접 데이터를 작성하는 비용이 지나치게 크거나, 기존 데이터만으로는 모델이 다양한 상황에 대응하기에 부족한 경우가 이에 해당합니다. 이런 문제를 해결하기 위해 등장한 방법이 **합성 데이터**(synthetic data) 생성입니다.

합성 데이터란 사람이 직접 작성하거나 실제 환경에서 수집한 것이 아니라, 알고리즘이나 모델을 활용하여 인공적으로 만들어 낸 데이터를 뜻합니다. 파인튜닝 맥락에서 합성 데이터는 주로 "지시문(instruction)과 그에 대한 응답(output)" 쌍의 형태를 띱니다. 예를 들어, "다음 문장을 요약해 주세요"라는 지시와 그에 맞는 요약 결과를 LLM이 자동으로 생성하는 것입니다.

합성 데이터가 필요한 이유를 좀 더 구체적으로 살펴보겠습니다. 첫째, 사람이 작성하는 데이터는 비용이 높습니다. 전문 어노테이터가 하나의 고품질 지시-응답 쌍을 만드는 데 수 분에서 수십 분이 걸릴 수 있습니다. 수만 건의 학습 데이터를 이렇게 확보하려면 비용과 시간이 막대합니다. 둘째, 데이터의 다양성이 부족해지기 쉽습니다. 소수의 어노테이터가 데이터를 만들면 표현 방식, 질문 유형, 난이도 분포가 편향됩니다. 셋째, 새로운 도메인이나 언어에 빠르게 적응해야 할 때 기존 데이터셋에 의존하면 대응 속도가 느려집니다. 합성 데이터는 이 세 가지 문제를 동시에 완화할 수 있습니다. LLM을 활용하면 적은 비용으로 다양한 유형의 데이터를 빠르게 대량 생성할 수 있기 때문입니다.

합성 데이터 생성의 대표적인 기법으로 **Self-Instruct**가 있습니다. Self-Instruct는 LLM 스스로 지시문과 그에 대한 응답을 생성하도록 하는 방법입니다. 이 기법이 등장한 배경을 먼저 짚겠습니다. 기존의 인스트럭션 튜닝(instruction tuning)은 사람이 작성한 지시-응답 쌍을 대규모로 수집하여 모델을 미세조정하는 방식이었습니다. 그런데 이 과정에서 가장 큰 병목은 고품질 지시문을 충분히 확보하는 것이었습니다. Self-Instruct는 이 병목을 해소하기 위해, 모델 자체에게 지시문 생성을 맡기는 아이디어에서 출발했습니다.

Self-Instruct의 작동 과정은 다음과 같습니다. 먼저 소수의 시드(seed) 지시문을 준비합니다. 시드 지시문이란 사람이 수작업으로 작성한 초기 예시로, 보통 수십 개에서 백여 개 정도입니다. 이 시드 지시문을 LLM에 입력하면서 "이와 유사하되 새로운 지시문을 만들어 달라"고 요청합니다. LLM은 시드를 참고하여 새로운 지시문을 생성합니다. 생성된 지시문에 대해 다시 LLM이 응답을 작성합니다. 마지막으로 품질 필터링을 거쳐, 기존 시드와 너무 유사하거나 형식이 맞지 않는 것을 제거합니다. 이 과정을 반복하면 소수의 시드에서 출발하여 수만 건의 지시-응답 쌍을 만들어 낼 수 있습니다.

이 과정을 단계별로 정리하면 다음과 같습니다.

```
Self-Instruct 생성 과정

1. 시드 지시문 준비 (사람이 수십~백여 개 작성)
         |
         v
2. LLM에 시드 지시문 제공 + "새로운 지시문 생성" 요청
         |
         v
3. LLM이 새로운 지시문 생성
         |
         v
4. 생성된 지시문에 대해 LLM이 응답 작성
         |
         v
5. 품질 필터링 (중복 제거, 형식 검증)
         |
         v
6. 통과한 데이터를 시드 풀에 추가
         |
         v
   2~6 반복 --> 대규모 지시-응답 데이터셋 완성
```

Self-Instruct의 핵심은 모델이 생성한 데이터로 다시 모델을 학습시킨다는 점입니다. 이 방식을 통해 사람의 개입을 최소화하면서도 방대한 학습 데이터를 확보할 수 있습니다. 다만, 시드 지시문의 품질과 다양성이 최종 데이터셋의 품질에 큰 영향을 미칩니다. 시드가 특정 유형에 편중되어 있으면 생성되는 데이터도 그 방향으로 쏠립니다.

Self-Instruct에서 한 단계 더 발전한 기법이 **Evol-Instruct**입니다. Evol-Instruct는 "진화(evolution)"라는 개념을 지시문 생성에 도입했습니다. Self-Instruct가 시드와 유사한 새로운 지시문을 "수평적으로" 확장하는 데 초점을 맞춘다면, Evol-Instruct는 기존 지시문의 난이도와 복잡도를 "수직적으로" 높이는 방향으로 진화시킵니다.

Evol-Instruct가 필요해진 이유는 Self-Instruct의 한계에 있습니다. Self-Instruct로 생성한 데이터는 시드 지시문과 비슷한 난이도에 머무르는 경향이 있습니다. 단순한 시드에서 출발하면 단순한 지시문이 대량으로 만들어지고, 복잡한 추론이나 다단계 작업을 요구하는 고난이도 지시문은 부족해집니다. 모델이 어려운 문제에도 대응하려면, 학습 데이터에 다양한 난이도의 지시문이 골고루 포함되어야 합니다. Evol-Instruct는 이 문제를 해결합니다.

Evol-Instruct의 진화는 두 가지 방향으로 진행됩니다. 첫 번째는 **깊이 진화**(depth evolution)입니다. 기존 지시문에 제약 조건을 추가하거나, 추론 단계를 늘리거나, 더 구체적인 요구사항을 붙여서 난이도를 높입니다. 예를 들어, "Python으로 리스트를 정렬하는 코드를 작성하세요"라는 지시문이 있다면, 깊이 진화를 거치면 "Python으로 중첩 딕셔너리의 특정 키 값을 기준으로 리스트를 정렬하되, 값이 없는 항목은 맨 뒤로 보내는 코드를 작성하세요"처럼 변합니다. 두 번째는 **넓이 진화**(breadth evolution)입니다. 기존 지시문의 주제나 맥락을 바꾸어 새로운 변형을 만듭니다. 정렬에 관한 지시문이 검색, 필터링, 변환 등 다른 주제의 지시문으로 확장되는 것입니다.

이 두 방향의 진화를 반복 적용하면, 단순한 초기 지시문 하나에서 점차 복잡하고 다양한 지시문 트리가 만들어집니다. 생물학에서 하나의 종이 환경에 적응하면서 다양한 변이를 만들어 내는 것과 유사한 구조입니다.

```
Evol-Instruct 진화 방향

원본 지시문: "리스트를 정렬하는 코드를 작성하세요"
     |
     +-- [깊이 진화] 제약 추가, 단계 증가, 난이도 상승
     |     |
     |     +--> "중첩 딕셔너리를 특정 키로 정렬하되,
     |     |     값이 없는 항목은 뒤로 보내세요"
     |     |
     |     +--> "대용량 CSV에서 메모리 제한 내에서
     |           외부 정렬을 구현하세요"
     |
     +-- [넓이 진화] 주제 변경, 맥락 전환
           |
           +--> "리스트에서 조건에 맞는 항목만
           |     필터링하는 코드를 작성하세요"
           |
           +--> "두 리스트의 교집합을 구하는
                 코드를 작성하세요"
```

합성 데이터 생성은 Self-Instruct와 Evol-Instruct 외에도 다양한 방식으로 확장됩니다. 그중 하나가 **LLM을 활용한 데이터 증강**(data augmentation)입니다. 데이터 증강이란 이미 존재하는 데이터를 변형하여 새로운 데이터를 만들어 내는 기법을 뜻합니다. 이미지 처리 분야에서 사진을 회전하거나 색상을 바꾸어 학습 데이터를 늘리는 것과 같은 원리입니다. 텍스트에서는 LLM을 활용하여 기존 지시문을 다른 표현으로 바꾸거나(paraphrase), 같은 의미의 응답을 다른 어조로 재작성하거나, 기존 데이터의 맥락을 변형하여 새로운 시나리오를 만드는 방식으로 증강합니다.

Self-Instruct나 Evol-Instruct가 아예 새로운 지시문을 생성하는 것과 비교하면, 데이터 증강은 기존 데이터를 기반으로 변형한다는 점에서 차이가 있습니다. 기존 데이터의 품질이 이미 검증되어 있다면, 증강된 데이터도 일정 수준의 품질을 유지하기 쉽습니다. 반면, 원본 데이터의 오류나 편향이 증강 과정에서 그대로 복제되거나 증폭될 수 있다는 점은 주의해야 합니다.

구체적인 증강 방식을 예로 들어 보겠습니다. "서울의 오늘 날씨를 알려주세요"라는 지시문이 있을 때, LLM에게 "이 지시문과 같은 의도이지만 다른 표현을 5개 만들어 달라"고 요청하면, "오늘 서울 날씨가 어떤가요?", "서울 현재 기온을 확인해 주세요", "지금 서울에 비가 오나요?" 같은 변형이 생성됩니다. 응답 측면에서도 같은 사실 정보를 격식체, 구어체, 간결체 등 다양한 어조로 재작성할 수 있습니다. 이렇게 하면 모델이 같은 의도의 다양한 표현에 대응하는 능력을 갖추게 됩니다.

합성 데이터를 생성했다면, 그 데이터가 실제로 학습에 도움이 되는지 **품질 검증**을 거쳐야 합니다. 합성 데이터는 사람이 직접 확인하지 않은 상태에서 대량으로 만들어지기 때문에, 오류, 불일관성, 반복, 의미 없는 내용이 섞일 수 있습니다. 품질이 낮은 데이터로 모델을 학습시키면 오히려 성능이 떨어지므로, 생성과 검증은 분리할 수 없는 한 쌍입니다.

품질 검증은 여러 단계로 이루어집니다. 먼저 **형식 검증**이 있습니다. 생성된 데이터가 지시문-응답 쌍의 구조를 제대로 갖추었는지, 빈 필드가 없는지, 응답이 지시문과 관련 있는 내용인지를 확인합니다. 다음으로 **중복 검증**이 있습니다. 같은 지시문이 반복 생성되었거나, 표현만 미세하게 다른 사실상 동일한 데이터가 대량으로 포함되어 있으면 학습 효율이 떨어집니다. 텍스트 유사도를 측정하여 일정 임계값 이상 유사한 항목을 제거합니다. 그다음으로 **내용 정확성 검증**이 있습니다. 응답에 사실과 다른 정보가 포함되어 있지 않은지를 검토합니다. 이 단계는 자동화가 가장 어려운 부분입니다. LLM이 생성한 응답을 또 다른 LLM이 평가하는 방식(LLM-as-Judge)을 사용하거나, 사람이 샘플링하여 직접 확인하는 방식을 결합합니다.

검증 과정을 표로 정리하면 다음과 같습니다.

```
합성 데이터 품질 검증 단계

+----------+---------------------+---------------------------+
| 단계     | 검증 대상           | 방법                      |
+----------+---------------------+---------------------------+
| 형식     | 구조, 필드 완전성   | 규칙 기반 자동 검사       |
+----------+---------------------+---------------------------+
| 중복     | 유사 데이터 제거    | 텍스트 유사도 측정        |
+----------+---------------------+---------------------------+
| 정확성   | 응답 내용의 사실성  | LLM-as-Judge + 사람 샘플링|
+----------+---------------------+---------------------------+
| 다양성   | 지시문 유형 분포    | 클러스터링 후 분포 확인   |
+----------+---------------------+---------------------------+
| 난이도   | 쉬운/어려운 비율    | 난이도 태깅 후 분포 확인  |
+----------+---------------------+---------------------------+
```

마지막으로 **합성 데이터의 윤리적 고려사항**을 짚겠습니다. 합성 데이터는 기술적으로 편리하지만, 그 사용에는 주의가 필요한 윤리적 쟁점이 있습니다.

첫째, **모델 붕괴**(model collapse) 위험입니다. LLM이 생성한 데이터로 다시 LLM을 학습시키는 과정을 여러 세대 반복하면, 데이터의 다양성이 점차 줄어들고 모델이 특정 패턴에 갇히는 현상이 발생할 수 있습니다. 원본 데이터에 있던 분포의 꼬리(tail) 부분, 즉 드물지만 중요한 사례들이 세대를 거듭할수록 사라지게 됩니다. 이는 실제 사용 환경에서 소수 의견이나 비주류 표현이 체계적으로 배제되는 결과를 낳을 수 있습니다.

둘째, **편향 증폭** 문제입니다. 생성에 사용하는 LLM 자체가 학습 데이터에서 물려받은 편향을 가지고 있습니다. 이 모델이 합성 데이터를 생성하면, 원래 모델의 편향이 데이터에 반영됩니다. 그 데이터로 학습한 새 모델은 편향이 더 강화될 수 있습니다. 특정 성별, 인종, 문화에 대한 고정관념이 합성 데이터를 통해 증폭되지 않도록 의도적인 점검이 필요합니다.

셋째, **지적 재산권과 데이터 출처** 문제입니다. LLM이 생성한 텍스트가 학습 데이터에 포함된 원본 텍스트를 거의 그대로 재현하는 경우가 있습니다. 합성 데이터에 이런 재현이 포함되면, 원본 저작자의 권리를 침해할 소지가 생깁니다. 합성 데이터를 대규모로 활용할 때는 이 점을 인식하고, 원본 텍스트와의 유사도 검사를 병행하는 것이 바람직합니다.

넷째, **투명성** 문제입니다. 합성 데이터로 학습된 모델이 서비스에 배포될 때, 학습 데이터의 출처와 생성 방식을 문서화하지 않으면, 모델의 한계와 잠재적 문제를 파악하기 어려워집니다. 합성 데이터 사용 여부와 비율, 생성에 사용한 모델, 품질 검증 절차 등을 기록으로 남기는 것이 운영 투명성 확보에 중요합니다.

정리하면, 합성 데이터 생성은 파인튜닝에 필요한 학습 데이터를 LLM을 활용하여 인공적으로 만들어 내는 기법입니다. Self-Instruct는 소수의 시드에서 출발하여 모델이 스스로 지시-응답 쌍을 대량 생성하는 방식이고, Evol-Instruct는 기존 지시문의 난이도와 범위를 점진적으로 진화시켜 데이터의 복잡도와 다양성을 높이는 방식입니다. LLM 기반 데이터 증강은 기존 데이터를 변형하여 양을 늘립니다. 생성된 합성 데이터는 형식, 중복, 정확성, 다양성, 난이도 측면에서 품질 검증을 반드시 거쳐야 하며, 모델 붕괴, 편향 증폭, 지적 재산권, 투명성 등 윤리적 고려사항도 함께 관리해야 합니다.

다음 단원인 7.2.3에서는 사용자 피드백을 학습 데이터로 순환시키는 데이터 플라이휠 구축을 다룹니다.

이 단원을 마치면 LLM을 활용한 합성 데이터 생성 기법을 설명하고 적용할 수 있습니다.
