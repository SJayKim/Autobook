# 2.2 스트림 처리

이 섹션에서는 실시간 이벤트를 처리하는 다양한 엔진의 특성과 적합한 사용 사례를 다룹니다. 다음 다섯 개의 토픽으로 구성되어 있습니다.

**2.2.1 이벤트 기반 아키텍처와 ML**에서는 이벤트 기반 아키텍처가 ML 시스템에서 어떤 역할을 하는지 살펴봅니다. 이벤트 소싱, CQRS, 그리고 배치와 스트리밍의 수렴 추세를 이해할 수 있습니다.

**2.2.2 Kafka Streams**에서는 Kafka Streams의 임베디드 라이브러리 특성과 상태 저장소 메커니즘을 살펴봅니다. KStream과 KTable, RocksDB 기반 상태 저장소, 윈도우 연산과 조인 연산의 구조를 이해할 수 있습니다.

**2.2.3 ksqlDB**에서는 ksqlDB로 Kafka 토픽에 SQL 기반 실시간 질의와 집계를 수행하는 방법을 살펴봅니다. STREAM과 TABLE, 지속 쿼리, 윈도우 집계 등 SQL 인터페이스로 스트림을 처리하는 방법을 이해할 수 있습니다.

**2.2.4 Apache Flink**에서는 Apache Flink의 클러스터 기반 아키텍처와 체크포인트 메커니즘을 살펴봅니다. JobManager와 TaskManager, DataStream API, 워터마크와 이벤트 시간 처리의 원리를 이해할 수 있습니다.

**2.2.5 스트림 처리 엔진 선택 기준**에서는 Kafka Streams, ksqlDB, Flink의 사용 사례별 적합성과 선택 기준을 비교합니다. 임베디드와 클러스터 방식의 차이, 처리량과 지연시간, 운영 복잡도를 종합적으로 판단하는 기준을 이해할 수 있습니다.
