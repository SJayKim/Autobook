# 1.3 Apache Kafka 기초

이 섹션에서는 실시간 데이터 파이프라인의 중추가 되는 Apache Kafka의 구조와 동작 원리를 다룹니다. 다음 여섯 개의 토픽으로 구성되어 있습니다.

**1.3.1 Kafka 아키텍처 개요**에서는 Kafka의 프로듀서-브로커-컨슈머 3계층 구조와 발행-구독 모델을 살펴봅니다. 분산 이벤트 스트리밍 플랫폼으로서 Kafka가 어떤 구조로 동작하는지 이해할 수 있습니다.

**1.3.2 토픽과 파티션**에서는 Kafka 토픽과 파티션의 관계, 파티션이 병렬 처리와 수평 확장에 기여하는 방식을 살펴봅니다. 메시지의 논리적 분류와 물리적 분할이 어떻게 연결되는지 이해할 수 있습니다.

**1.3.3 오프셋과 메시지 소비**에서는 Kafka 오프셋의 역할과 컨슈머가 메시지 읽기 위치를 추적하는 메커니즘을 살펴봅니다. 자동 커밋과 수동 커밋의 차이, earliest와 latest 설정의 의미를 이해할 수 있습니다.

**1.3.4 컨슈머 그룹**에서는 컨슈머 그룹의 파티션 할당 메커니즘과 리밸런싱 과정을 살펴봅니다. 여러 컨슈머가 협력하여 메시지를 처리하는 구조를 이해할 수 있습니다.

**1.3.5 복제와 내결함성**에서는 Kafka의 리더-팔로워 복제 메커니즘과 ISR이 내결함성을 보장하는 방식을 살펴봅니다. Replication Factor, min.insync.replicas, acks 설정 등 데이터 안전성을 좌우하는 핵심 설정을 이해할 수 있습니다.

**1.3.6 KRaft 모드와 컨트롤러 쿼럼**에서는 KRaft가 ZooKeeper를 대체하는 이유와 컨트롤러 쿼럼의 동작 원리를 살펴봅니다. Kafka 4.0 이후의 아키텍처 변화를 이해할 수 있습니다.
