# 4.2 LLM 추론 서빙

이 섹션에서는 대규모 언어 모델의 추론 비용과 지연시간을 줄이는 핵심 기술과 아키텍처 패턴을 다룹니다. 다음 네 개의 토픽으로 구성되어 있습니다.

**4.2.1 LLM 추론 최적화 기법**에서는 Continuous Batching, PagedAttention, 투기적 디코딩 등 핵심 LLM 추론 최적화 기법의 원리를 살펴봅니다. KV 캐시 관리와 Prefix Caching을 포함한 최적화 기법의 동작 방식을 이해할 수 있습니다.

**4.2.2 vLLM**에서는 vLLM의 PagedAttention 구현과 HuggingFace 대비 처리량 개선 원리를 살펴봅니다. OpenAI 호환 API, TensorRT-LLM과 SGLang과의 비교를 통해 서빙 엔진의 선택 기준을 이해할 수 있습니다.

**4.2.3 배치 추론 패턴**에서는 대량 데이터에 대한 배치 추론의 설계 패턴과 Ray Data + vLLM 활용 방법을 살펴봅니다. 오프라인 추론에서 GPU 활용을 최적화하는 방법을 이해할 수 있습니다.

**4.2.4 실시간 모델 추론 패턴**에서는 원격 추론과 내장 추론 패턴의 차이점과 Kafka/Flink 기반 실시간 추론 아키텍처를 살펴봅니다. HTTP/gRPC 모델 서버와 스트림 처리 엔진 내장 추론의 지연시간 차이를 이해할 수 있습니다.
