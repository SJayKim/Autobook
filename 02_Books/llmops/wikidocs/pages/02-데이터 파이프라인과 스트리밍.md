# Phase 2. 데이터 파이프라인과 스트리밍

이 Phase에서는 Kafka 기반 실시간 데이터 파이프라인을 설계하고 피처 엔지니어링 체계를 구축하는 방법을 다룹니다. 이 단계를 마치면 실시간 데이터 흐름을 설계하고 ML 모델에 필요한 피처를 체계적으로 관리할 수 있습니다.

**2.1 Kafka 심화**에서는 정확히 한 번 의미론, 스키마 레지스트리, Kafka Connect, CDC와 Debezium, 파티션 전략과 백프레셔를 다룹니다. Phase 1에서 익힌 Kafka 기초를 프로덕션 수준으로 확장합니다.

**2.2 스트림 처리**에서는 이벤트 기반 아키텍처와 ML의 관계, Kafka Streams, ksqlDB, Apache Flink, 스트림 처리 엔진 선택 기준을 다룹니다. 실시간 이벤트를 처리하는 다양한 엔진의 특성과 적합한 사용 사례를 비교합니다.

**2.3 피처 엔지니어링**에서는 피처 스토어 아키텍처, 데이터 검증과 품질 관리, 스트리밍 피처 엔지니어링, 온라인 피처 스토어, 데이터 파이프라인 패턴을 다룹니다. ML 모델이 필요로 하는 피처를 안정적으로 생산하고 서빙하는 체계를 설계합니다.
