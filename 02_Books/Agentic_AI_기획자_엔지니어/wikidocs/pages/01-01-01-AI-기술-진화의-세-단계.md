# 1.1.1 AI 기술 진화의 세 단계 — GenAI에서 Agentic AI까지

AI를 활용한 서비스나 제품을 기획하려면, 지금 다루는 AI가 정확히 어떤 수준의 능력을 갖고 있는지 알아야 합니다. "AI가 글을 써 준다"는 말과 "AI가 알아서 업무를 처리한다"는 말은 기술적으로 전혀 다른 이야기입니다. 이 단원에서는 AI 기술이 거쳐 온 세 단계를 구분하고, 각 단계가 기획자에게 어떤 의미를 갖는지 살펴봅니다.

첫 번째 단계는 **생성형 AI(GenAI)**입니다. 생성형 AI는 대규모 언어 모델(LLM)을 기반으로 텍스트, 이미지, 코드 같은 콘텐츠를 만들어 내는 기술입니다. 사용자가 프롬프트를 입력하면 모델이 결과를 돌려주는 단방향 흐름으로 동작합니다. 입력이 들어오고, 모델이 처리하고, 출력이 나옵니다. ChatGPT에 질문을 던지고 답변을 받는 상황이 전형적인 예입니다.

생성형 AI의 구조적 특징은 **무상태(Stateless)**라는 점입니다. 무상태란 각 요청이 독립적으로 처리된다는 뜻입니다. 이전 대화에서 무엇을 주고받았는지 모델이 장기적으로 기억하지 않습니다. 새 대화를 시작하면 사용자가 맥락을 처음부터 다시 제공해야 합니다.

이 구조에서 세 가지 한계가 드러납니다. 첫째, 대화 세션 사이에 상태를 유지하지 못합니다. 둘째, 외부 도구나 시스템과 직접 상호작용할 수 없습니다. 생성형 AI는 글을 '생성'할 뿐, 그 결과를 가지고 이메일을 보내거나 데이터베이스를 조회하는 '실행'은 하지 못합니다. 셋째, 여러 단계로 이루어진 작업을 스스로 계획하고 수행하지 못합니다. 사용자가 매 단계마다 직접 개입해야 합니다.

이런 한계를 극복하기 위해 등장한 것이 두 번째 단계, **AI 에이전트(AI Agent)**입니다. AI 에이전트는 LLM의 언어 능력에 세 가지를 더한 형태입니다. 계획 수립 능력, 도구 사용 능력, 그리고 단기 메모리입니다. 정보를 알려주기만 하던 '조언자(oracle)' 역할에서, 실제로 일을 처리하는 '실행자(worker)' 역할로 바뀐 것이 핵심 변화입니다.

AI 에이전트가 가능해진 데는 기술적 이정표들이 있었습니다. 2022년 10월, 구글 연구팀이 ReAct 프레임워크를 발표했습니다. ReAct는 '추론(Reason) 후 행동(Act)하고, 결과를 관찰(Observe)한다'는 순환 구조를 정립한 것입니다. 같은 시기 LangChain이 출시되어 LLM과 외부 도구를 연결하는 기반이 마련되었습니다. 2023년 3월에는 GPT-4가 출시되면서 복잡한 멀티스텝 계획이 실현 가능해졌고, AutoGPT가 깃허브에서 10만 스타를 받으며 자율 에이전트에 대한 관심이 폭발했습니다. 2023년 6월 OpenAI가 함수 호출(Function Calling) 기능을 네이티브로 지원하면서 모든 API를 에이전트의 '도구'로 연결할 수 있게 되었습니다. 2024년 11월에는 Anthropic이 MCP(Model Context Protocol)를 출시하여 도구 통합 비용을 획기적으로 줄였습니다.

AI 에이전트의 아키텍처는 생성형 AI와 다릅니다. 생성형 AI가 '입력 -> 모델 -> 출력'이라면, AI 에이전트는 '목표 -> 에이전트 -> 도구 -> 출력'입니다. 사용자가 목표를 주면, 에이전트가 어떤 도구를 어떤 순서로 쓸지 판단하여 결과를 만들어 냅니다. 메모리 측면에서는 **에피소딕(Episodic)** 메모리를 갖습니다. 에피소딕 메모리란 하나의 작업 세션 안에서 일어난 일을 기억하는 단기 기억입니다. 생성형 AI의 무상태보다는 발전했지만, 세션이 끝나면 기억이 사라진다는 제한이 있습니다. 자율성도 '제한적(Limited)' 수준입니다. 정해진 목표 안에서는 스스로 단계를 밟지만, 목표 자체를 설정하거나 장기적으로 학습하지는 못합니다.

세 번째 단계가 **에이전틱 AI(Agentic AI)**입니다. 에이전틱 AI는 인간의 지속적인 지시 없이 스스로 목표를 설정하고, 전략을 수립하며, 실행한 뒤, 그 결과를 다음 행동에 반영하는 학습 루프를 내장한 자율 실행형 AI입니다.

예를 들어 마케팅 팀에서 에이전틱 AI에게 "다음 분기 신규 고객 유입을 20% 늘려라"라는 목표만 부여한다고 가정합니다. 에이전틱 AI는 먼저 과거 캠페인 데이터를 분석해 성과가 좋았던 채널을 파악합니다. 그 결과를 바탕으로 이메일, 소셜 미디어, 검색 광고에 예산을 배분하는 계획을 스스로 세웁니다. 캠페인을 실행한 뒤 일주일 단위로 유입 수치를 확인하고, 성과가 낮은 채널의 예산을 성과가 높은 채널로 재배분합니다. 사람이 매 단계를 지시하지 않아도, 목표 달성을 향해 판단-실행-학습 루프를 반복하는 것입니다. AI 에이전트라면 "이메일 캠페인을 작성해 줘"처럼 구체적인 작업 단위로 지시해야 하지만, 에이전틱 AI는 상위 목표만으로 움직입니다.

에이전틱 AI를 정의하는 데 학계와 산업계가 합의하는 핵심 요소가 있습니다. MIT 슬론에서는 "인간 주체를 대신하여 디지털 환경에서 인식하고, 추론하고, 행동하는 자율 소프트웨어 시스템"이라 정의합니다. 도구 사용, 경제적 거래, 전략적 상호작용 능력을 포함합니다. 삼성SDS에서는 "인간의 지시나 개입 없이도 스스로 목표를 설정하고, 문제를 인식하며, 적절한 도구를 활용해 작업을 수행하고, 그 결과를 학습에 반영하는 자율 실행형 AI"라 정의합니다. 두 정의가 공통으로 짚는 핵심은 자율성(Autonomy), 인식(Perception), 추론(Reasoning), 행동(Action), 도구 사용(Tool Use), 그리고 지속적 메모리(Persistent Memory)입니다.

AI 에이전트와 에이전틱 AI의 차이를 명확히 하면 이렇습니다. AI 에이전트는 주어진 목표를 도구로 실행하는 데 초점이 있습니다. 에이전틱 AI는 여기에 더해 목표 자체를 스스로 정의하고, 여러 에이전트를 하위 모듈로 오케스트레이션하며, 실행 결과를 장기 메모리에 저장해 다음 판단에 활용합니다. '자발성(Proactiveness)'이 에이전틱 AI의 본질적 특성입니다.

세 단계를 자율성 기준으로 비교하면 하나의 스펙트럼 위에 놓입니다.

```
자율성 스펙트럼

  낮음                                                높음
   |                    |                    |
   GenAI               AI Agent          Agentic AI
   (없음)              (제한적)            (완전)
   Stateless           Episodic           Persistent
   Input->Model->Out   Goal->Agent->      Objective->Sub-Agents->
                        Tools->Out         Tools+Memory->Out
```

| 구분 | 자율성 | 메모리 | 아키텍처 |
|------|--------|--------|----------|
| GenAI | 없음 | Stateless (무상태) | 입력 -> 모델 -> 출력 |
| AI Agent | 제한적 | Episodic (단기) | 목표 -> 에이전트 -> 도구 -> 출력 |
| Agentic AI | 완전 | Persistent (지속) | 목표 -> 하위 에이전트 -> 도구+메모리 -> 출력 |

이 스펙트럼에서 어디에 위치할지는 순전히 기술 수준의 문제만이 아닙니다. 설계 결정이기도 합니다. 실무에서는 자율성 수준을 위험도와 반비례로 설정하는 것이 원칙입니다. 잘 정의된 반복 업무부터 자동화하고, 점진적으로 자율 범위를 넓혀 가는 접근이 권장됩니다. 제품 설계 시 자율성을 세 수준으로 나누어 생각할 수 있습니다. 첫째, 에이전트가 행동을 제안만 하고 사람이 모든 실행을 결정하는 '제안' 단계입니다. 둘째, 에이전트가 부분적으로 실행하되 핵심 단계에서 사람의 승인을 요구하는 '부분 승인' 단계입니다. 경비 보고서를 자동 작성한 뒤 관리자 승인을 요청하는 경우가 이에 해당합니다. 셋째, 정의된 가드레일 안에서 에이전트가 독립적으로 실행하는 '자율 실행' 단계입니다.

이제 이 세 단계가 기획자에게 무엇을 의미하는지 살펴봅니다. 생성형 AI 시대의 기획은 프롬프트 품질, 데이터 기반, 출력 정확도에 집중했습니다. 사용자가 프롬프트를 입력하고, 모델이 처리하고, 출력을 사용자가 확인하는 선형 흐름이었습니다.

에이전틱 AI 시대의 기획은 근본적으로 달라집니다. 사용자가 목표를 설정하면, 에이전트가 계획을 세우고, 실행하고, 피드백을 받아 다시 조정하는 비선형 루프가 됩니다. 기획자가 설계해야 하는 대상이 바뀝니다. 프롬프트 한 줄이 아니라, 목표 정의, 워크플로, 결정 경계(어디까지 AI가 판단하고 어디서 사람이 개입할지), 실세계 시스템과의 통합을 설계해야 합니다.

```
GenAI 기획 흐름 (선형)
  사용자 --> 프롬프트 --> 모델 --> 출력 --> 사용자

Agentic AI 기획 흐름 (비선형 루프)
  사용자 --> 목표 설정
                |
                v
            에이전트 --> 계획 수립
                |            |
                v            v
            도구 실행    피드백 반영
                |            |
                +-----<------+
                |
                v
              결과
```

구체적으로, 에이전틱 AI 시대에 새롭게 등장하는 기획 과제는 네 가지입니다. 첫째, 목표와 가드레일을 정의해야 합니다. AI가 자율적으로 움직이므로 "어디까지 허용할 것인가"를 사전에 설계해야 합니다. 둘째, 메모리 관리를 설계해야 합니다. AI가 무엇을 얼마나 오래 기억할지 결정하는 것이 제품 경험에 직결됩니다. 셋째, 실패 시 복구 경로를 마련해야 합니다. 자율 실행 중 오류가 발생했을 때 어떻게 되돌리고 사람에게 알릴지 미리 설계해야 합니다. 넷째, KPI를 재정의해야 합니다. 단순 출력 품질이 아니라, 태스크 성공률, 인간 개입률, 신뢰도 같은 새로운 지표가 필요합니다.

기획자의 역할 자체도 바뀝니다. 실제로 많은 프로덕트 매니저가 직접 만드는 '제작자(maker)' 역할에서, AI 시스템을 지휘하는 '디렉터(director)' 역할로 이동하고 있습니다. 다만, 아무리 AI가 자율화되어도 결과에 대한 책임은 인간 기획자에게 있다는 점은 변하지 않습니다. 기획자는 이제 여러 AI 에이전트로 구성된 시스템을 조율하는 '오케스트레이터'로서, AI 행동의 윤리적 감독과 조직 목표와의 전략적 정렬이라는 새로운 책임을 갖게 됩니다.

정리하면, AI 기술은 콘텐츠를 생성하는 GenAI에서, 도구를 활용해 목표를 실행하는 AI 에이전트를 거쳐, 스스로 목표를 설정하고 학습하며 실행하는 에이전틱 AI로 진화해 왔습니다. 이 세 단계는 자율성, 메모리, 아키텍처에서 뚜렷한 차이를 보이며, 기획자의 역할도 프롬프트 설계자에서 AI 시스템 오케스트레이터로 전환되고 있습니다.

다음 단원인 1.1.2에서는 AI 에이전트를 구성하는 세 가지 핵심 요소인 LLM, 도구, 메모리를 다룹니다.

이 단원을 마치면 GenAI, AI 에이전트, Agentic AI의 차이를 설명하고, 각 단계가 기획 관점에서 어떤 의미를 갖는지 구별할 수 있습니다.
