# 3.2.1 자가 지도 도구 학습

언어 모델이 도구를 사용할 수 있으려면, 어떤 상황에서 어떤 도구를 어떻게 호출해야 하는지를 학습해야 합니다. 이 학습에는 두 가지 경로가 있습니다. 하나는 사람이 직접 도구 호출 예시를 레이블링하여 지도 학습(supervised learning)을 수행하는 방법이고, 다른 하나는 모델 자신이 도구 호출의 유용성을 평가하여 스스로 학습 신호를 생성하는 자가 지도(self-supervised) 방법입니다. 전자는 레이블 비용이 크고 확장이 어려우며, 후자는 인간의 개입 없이도 광범위한 도구 사용 능력을 익힐 수 있다는 점에서 최근 연구의 주된 관심을 받고 있습니다.

자가 지도 도구 학습의 핵심 아이디어는 간단합니다. 모델이 어떤 텍스트를 생성할 때, 특정 위치에서 도구를 호출했을 경우와 호출하지 않았을 경우의 결과를 비교하여, 도구 호출이 실질적으로 도움이 되었는지를 손실(loss) 차이로 측정합니다. 도구를 호출한 쪽의 손실이 현저히 낮다면, 그 호출은 유용했다는 신호로 간주됩니다. 이 방식은 인간이 "이 호출은 좋다 / 나쁘다"를 판단할 필요 없이, 모델 자체의 예측 품질을 기준으로 도구 사용을 평가한다는 점에서 자율적입니다.

**손실 기반 유용성 필터링의 원리**

구체적으로, 손실 기반 유용성 필터링(loss-based utility filtering)은 다음과 같이 작동합니다. 먼저 모델이 텍스트를 생성하는 도중, 특정 위치에서 도구 호출 후보를 삽입합니다. 도구를 호출한 결과를 맥락에 포함한 채로 이후 텍스트를 생성했을 때의 손실 L_tool과, 도구를 호출하지 않고 이후 텍스트를 생성했을 때의 손실 L_nontool을 계산합니다. 만약 L_tool이 L_nontool보다 충분히 낮다면, 이 도구 호출은 "유용했다"고 판단하여 학습 데이터로 채택합니다. 반대로 L_tool이 더 높거나 거의 동일하다면, 해당 호출은 불필요하거나 오히려 방해가 됐다는 의미이므로 버립니다.

이 필터링 과정은 모델이 인터넷 텍스트나 지식 베이스와 같은 대규모 코퍼스를 처리하면서 자동으로 수행될 수 있습니다. 예를 들어, 수학 계산이 필요한 문장을 생성하는 도중 계산기 API를 호출한다면, 그 결과가 이후 문장 생성에 도움이 되었는지를 손실로 측정합니다. 날짜 조회, 단위 변환, 검색 쿼리 등 다양한 유형의 도구에 동일한 원리를 적용할 수 있습니다.

**Few-shot 데모 기반 API 호출 후보 샘플링**

자가 지도 학습이 작동하려면, 먼저 "이 위치에서 이런 도구를 이렇게 호출할 수 있다"는 후보 호출 자체를 생성해야 합니다. 무작위로 도구 호출을 삽입하면 대부분 무의미한 결과를 낳기 때문에, 적절한 후보를 생성하는 것이 중요합니다.

Few-shot 데모 기반 샘플링은 이 문제를 해결하는 실용적인 방법입니다. 소수의 도구 호출 예시(few-shot demonstrations)를 프롬프트에 제공하면, 언어 모델은 해당 패턴을 따라 새로운 맥락에서도 적절한 호출 형식을 생성할 수 있습니다. 예를 들어, "숫자 계산이 필요한 문장이 나오면 'CALC(수식)' 형태로 호출한다"는 예시 세 개를 보여주면, 모델은 유사한 맥락에서 같은 패턴의 후보 호출을 생성합니다. 이렇게 생성된 후보들 중에서 손실 기반 필터링을 통해 실제로 유용한 것만 학습 데이터로 선택합니다.

이 접근법의 장점은 도메인 지식이나 사람의 개입 없이도 다양한 도구에 대한 학습 데이터를 대규모로 생성할 수 있다는 점입니다. API 명세(사용법, 입력 형식, 출력 형식)를 few-shot 예시와 함께 제공하면, 모델은 해당 API를 언제 어떻게 사용해야 하는지를 스스로 학습하게 됩니다.

**범용 언어 모델 능력 보존의 문제**

도구 사용 능력을 학습시키는 과정에서 발생하는 중요한 문제 중 하나는 파국적 망각(catastrophic forgetting)입니다. 특정 도구 사용에 편향된 데이터로 파인튜닝을 수행하면, 모델이 기존에 보유하던 범용 언어 이해, 추론, 생성 능력이 저하될 수 있습니다. 특히 도구 호출 형식에만 최적화되면, 도구가 없는 상황에서의 자연스러운 텍스트 생성 능력이 감소할 수 있습니다.

이를 방지하기 위한 주된 전략은 혼합 학습(mixed training)입니다. 도구 관련 데이터와 일반 언어 모델링 데이터를 적절한 비율로 섞어 학습시키면, 도구 사용 능력을 강화하는 동시에 기존 능력도 유지할 수 있습니다. 또한 학습률(learning rate)을 낮게 유지하거나, 점진적 파인튜닝(progressive fine-tuning) 전략을 채택하여 기존 파라미터가 급격히 변하지 않도록 하는 방법도 사용됩니다.

자가 지도 도구 학습에서는 이 문제가 특히 중요합니다. 자가 지도 방식으로 수집된 도구 사용 데이터는 기존 학습 데이터와 도메인이 겹칠 수 있지만, 호출 형식이라는 새로운 문법 구조를 포함하기 때문에 모델이 이를 학습하는 과정에서 기존의 자유로운 텍스트 생성 능력과 충돌이 생길 수 있습니다. 이 균형을 잡는 것이 실제 구현에서 가장 어려운 부분 중 하나입니다.

**인스트럭션 튜닝과 도구 사용의 결합**

사전 학습된 언어 모델에 도구 사용 능력을 부여하는 또 다른 접근법은 인스트럭션 튜닝(instruction tuning)을 도구 사용에 확장하는 것입니다. 기존의 인스트럭션 튜닝은 "지시문 → 응답" 형태의 데이터로 모델이 다양한 지시를 따르도록 훈련합니다. 이를 도구 사용에 적용하면, "지시문 → 도구 호출 결정 → 도구 호출 실행 → 결과 해석 → 최종 응답"이라는 다단계 구조를 학습시키게 됩니다.

인스트럭션 튜닝 기반 도구 학습에서는 각 학습 예시가 단순한 질문-답변 쌍이 아니라, 도구 사용이 필요한 복합적인 태스크와 그에 대한 단계별 해결 과정을 포함합니다. 예를 들어, "오늘 서울의 날씨는?"이라는 질문에 대해, 날씨 API를 호출하고 그 결과를 자연어로 변환하는 전체 과정이 하나의 학습 예시가 됩니다.

자가 지도 방식과 인스트럭션 튜닝은 상호 보완적입니다. 자가 지도 방식은 레이블이 없는 대규모 데이터에서 기본적인 도구 호출 패턴을 학습하는 데 유리하고, 인스트럭션 튜닝은 특정 태스크와 도구 조합에 대한 정밀한 지시 따르기 능력을 강화하는 데 유리합니다. 실제 고성능 도구 사용 모델은 두 방법을 단계적으로 결합하여 훈련하는 경우가 많습니다.

**자가 지도 도구 학습의 한계**

자가 지도 도구 학습은 강력한 방법이지만 몇 가지 한계가 있습니다. 첫째, 손실 기반 필터링은 도구 호출이 텍스트 생성에 도움이 되는지만 측정하므로, 도구 호출이 "올바른" 방식으로 이루어졌는지는 별도로 검증해야 합니다. 잘못된 API 호출이 우연히 낮은 손실을 보이는 경우도 있을 수 있습니다. 둘째, few-shot 데모에 의존하는 후보 생성은 데모의 품질에 크게 좌우됩니다. 나쁜 데모는 모델이 잘못된 호출 패턴을 학습하게 할 수 있습니다. 셋째, 모든 도구가 텍스트 손실로 측정 가능한 유용성을 제공하는 것은 아닙니다. 부수 효과(side effect)만 있는 도구, 예를 들어 파일 저장이나 이메일 전송 같은 액션형 도구는 손실 비교만으로 유용성을 평가하기 어렵습니다.

이러한 한계를 극복하기 위해 후속 연구들은 더 정교한 데이터 생성 전략을 개발했습니다. 그 대표적인 예가 다음 단원에서 다룰 API 검색 기반 학습과 합성 데이터 기반 학습입니다.

정리하면, 자가 지도 도구 학습은 인간의 레이블 없이도 도구 사용 능력을 대규모로 학습할 수 있는 패러다임입니다. 손실 기반 유용성 필터링으로 유익한 도구 호출을 자동 선별하고, few-shot 데모로 호출 후보를 생성하며, 혼합 학습으로 범용 언어 모델 능력을 보존합니다. 인스트럭션 튜닝과 결합하면 특정 태스크에 대한 정밀한 도구 사용 능력도 갖출 수 있습니다.

다음 단원인 3.2.2에서는 수천 개 이상의 API가 존재하는 대규모 생태계에서 모델이 필요한 API를 어떻게 검색하고 활용하는지, 그리고 검색기를 학습에 통합하는 Retriever-Aware Fine-tuning을 다룹니다.

이 단원을 학습한 후에는 자가 지도 도구 학습의 작동 원리, 손실 기반 유용성 필터링의 메커니즘, few-shot 기반 후보 샘플링 방식, 범용 능력 보존 전략, 그리고 인스트럭션 튜닝과의 결합 방법을 설명할 수 있습니다.
