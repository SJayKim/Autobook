# 2.5.2 메모리 망각과 삭제 전략

2.5.1에서 새로운 경험이 기존 메모리에 통합되는 과정을 살펴보았습니다. 메모리가 성장하기만 하고 줄어들지 않는다면, 2.3.2에서 다룬 것처럼 노이즈가 축적되어 검색 품질이 하락합니다. 인간의 뇌도 모든 것을 기억하지 않으며, 중요하지 않은 정보를 자연스럽게 잊어버리는 것이 효율적인 인지 활동의 전제 조건입니다. 이 단원에서는 에이전트 메모리에서 더 이상 유효하지 않거나 중요하지 않은 정보를 감쇠하고 삭제하는 전략을 다룹니다.

가장 기본적인 메모리 감쇠 메커니즘은 **망각 곡선(Forgetting Curve) 기반 감쇠**입니다. 19세기 독일의 심리학자 헤르만 에빙하우스가 발견한 망각 곡선은, 새로 학습한 정보의 기억 보존율이 시간이 지남에 따라 지수적으로 감소한다는 것을 보여줍니다. 처음에는 빠르게 잊어지고, 시간이 지날수록 감쇠 속도가 완화됩니다. 에이전트 메모리에 이 원리를 적용하면, 각 메모리 항목에 시간에 따라 감소하는 가중치를 부여합니다. 최근에 저장되거나 접근된 메모리는 높은 가중치를 가지고, 오래 사용되지 않은 메모리는 낮은 가중치를 갖습니다. 검색 시 가중치가 낮은 메모리는 우선순위가 떨어지며, 가중치가 임계값 아래로 내려가면 메모리에서 삭제됩니다.

2.2.1에서 간략히 소개한 **열도(Heat) 기반 메모리 승격**은 망각 곡선의 확장입니다. 메모리의 "열도"는 접근 빈도와 중요도에 따라 변화합니다. 자주 참조되거나 에이전트의 성능에 기여한 메모리의 열도는 올라가고, 오래 참조되지 않은 메모리의 열도는 내려갑니다. 열도가 높은 메모리는 STM에서 MTM으로, MTM에서 LTM으로 승격되어 더 영속적인 저장소에 보관됩니다. 열도가 낮은 메모리는 반대 방향으로 이동하거나 삭제됩니다. 이 메커니즘은 인간의 기억에서 반복 노출된 정보가 장기 기억으로 공고화되는 과정의 공학적 구현입니다.

2.2.1에서 다룬 FIFO Queue의 동작도 메모리 삭제의 한 형태입니다. **FIFO 큐 플러시**에서는 컨텍스트가 가득 차면 가장 오래된 메시지부터 밀려납니다. 밀려난 메시지는 단순히 사라지는 것이 아니라, LLM에 의해 요약되어 Working Context의 요약 영역에 반영되고, 원본 메시지는 Recall Storage에 보존됩니다. 이것은 **재귀적 요약 생성**으로 이어질 수 있습니다. 요약이 다시 축적되면 요약의 요약을 생성하여, 점점 더 추상적인 수준의 정보를 유지합니다. 세부 사항은 점진적으로 사라지지만 핵심 맥락은 보존됩니다.

2.2.3에서 소개한 **세그먼트 페이징(Segment Paging)**도 삭제 전략의 한 형태입니다. 대화 블록(세그먼트) 단위로 메모리를 동적으로 교체합니다. 현재 주제와 관련 없는 세그먼트는 외부 저장소로 이동시키고, 관련 있는 세그먼트만 활성 메모리에 유지합니다. 이것은 FIFO의 시간 순서 기반 삭제와 달리, 주제적 관련성을 기준으로 메모리를 관리합니다.

더 능동적인 삭제 전략이 **Selective Forgetting(선택적 망각)**입니다. 모순되는 새 정보가 들어왔을 때, 이전에 저장된 정보를 수정하거나 덮어쓰거나 제거하는 능력입니다. "사용자가 채식주의자이다"라는 메모리가 저장되어 있는데, 사용자가 "요즘은 고기도 먹습니다"라고 말했을 때, 에이전트는 이전 메모리를 갱신하거나 삭제해야 합니다. 선택적 망각은 현재 에이전트 메모리 연구에서 가장 어려운 과제 중 하나입니다. 벤치마크 평가에서 대부분의 메모리 시스템이 선택적 망각 태스크에서 10~15% 수준의 낮은 성공률을 보였습니다. 이전 정보를 지우는 것이 새 정보를 추가하는 것보다 훨씬 어렵다는 것입니다.

삭제 전략의 시간적 기준에 따라 **주기적 삭제(Periodic Deletion)**와 **이력 기반 삭제(History-based Deletion)**로 나눌 수 있습니다. 주기적 삭제는 FIFO처럼 일정 기간이 지난 메모리를 자동으로 제거하는 방식입니다. 구현이 단순하고 예측 가능하지만, 오래되었지만 여전히 유효한 중요 정보를 잃을 수 있습니다. 이력 기반 삭제는 메모리의 실제 활용 이력을 기반으로 삭제를 결정합니다. 자주 검색되고 에이전트 성능에 기여한 메모리는 오래되어도 유지하고, 검색된 적이 없거나 성능에 기여하지 못한 메모리는 최근 것이라도 삭제합니다. 2.2.3에서 다룬 helpful/harmful 카운터가 이력 기반 삭제의 판단 근거입니다. 이력 기반 삭제가 메모리 품질 유지에 더 효과적이지만, 활용 이력을 추적하는 추가 비용이 듭니다.

정리하면, 메모리 망각과 삭제는 노이즈 축적을 방지하고 검색 품질을 유지하기 위한 필수적 메커니즘입니다. 망각 곡선 기반 감쇠는 시간에 따른 자연스러운 가중치 하락을, 열도 기반 승격은 접근 빈도에 따른 계층 이동을, FIFO 플러시는 컨텍스트 용량 관리를 제공합니다. 선택적 망각은 모순되는 정보를 능동적으로 처리하는 가장 어려운 과제이며, 주기적 삭제와 이력 기반 삭제는 삭제 시점 결정의 두 가지 접근입니다.

다음 단원인 2.5.3에서는 메모리가 단순히 갱신되는 것을 넘어, 에이전트가 경험을 통해 메모리 시스템 자체를 진화시키는 자기 진화 메모리를 살펴봅니다.

이 단원을 마치면 Ebbinghaus 망각 곡선, 열도 기반 승격, FIFO 플러시, 선택적 망각 등 메모리 감쇠와 삭제 전략을 비교하고, 주기적 삭제와 이력 기반 삭제의 트레이드오프를 설명할 수 있습니다.
