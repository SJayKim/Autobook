# 3.2.3 합성 데이터 기반 도구 학습

도구 학습을 위한 고품질 데이터를 수집하는 것은 근본적으로 어렵습니다. 실제 API 호출 로그는 대부분 내부 시스템에 갇혀 있고, 사람이 도구 사용 궤적을 직접 작성하는 것은 비용이 크며, 자가 지도 방식으로 자연어 코퍼스에서 추출한 데이터는 범위가 제한됩니다. 이 문제를 극복하기 위해 등장한 것이 합성 데이터(synthetic data) 기반 접근법입니다. 실제 API를 실행하거나 사람이 직접 작성하는 대신, 언어 모델 자체를 활용하여 도구 사용 학습 데이터를 생성하는 방식입니다. 합성 데이터 기반 도구 학습은 현재 가장 활발하게 연구되는 분야 중 하나로, 여러 가지 정교한 방법론이 개발되어 있습니다.

**Bottom-Up 지시 구성**

합성 데이터 생성의 첫 번째 전략은 Bottom-Up 방식입니다. 이 방식에서는 먼저 단순한 원자 태스크(atomic task)를 정의하고, 이를 조합하여 복합적인 태스크를 구성합니다.

원자 태스크란 단일 API 호출 하나로 해결 가능한 최소 단위의 태스크입니다. 예를 들어, "특정 도시의 현재 기온을 조회한다", "두 수를 더한다", "파일에서 특정 줄을 읽는다" 같은 것들입니다. 각 API에 대해 이런 원자 태스크를 정의하면, 각 API의 기능과 사용법을 명확히 표현하는 기초 데이터가 만들어집니다.

조합 태스크(compositional task)는 여러 원자 태스크를 연결하여 만들어집니다. "서울과 부산의 기온 차이를 구한다"는 태스크는 두 번의 기온 조회와 한 번의 빼기 연산을 순서대로 수행해야 합니다. "지금 비가 오면 우산을 챙기도록 알림을 보낸다"는 태스크는 날씨 조회, 조건 판단, 알림 전송을 결합합니다. Bottom-Up 방식에서는 이러한 조합 패턴을 체계적으로 생성하여, 단순한 API 사용법을 넘어 복수의 API를 협력시키는 능력을 학습시킵니다.

이 방식의 장점은 학습 데이터의 범위와 복잡도를 정밀하게 제어할 수 있다는 점입니다. 원자 태스크의 조합 방식을 조절함으로써, 2단계 조합 태스크, 3단계 조합 태스크, n단계 조합 태스크를 원하는 비율로 생성할 수 있습니다.

**Top-Down 궤적 생성**

Top-Down 방식은 Bottom-Up과 반대 방향으로 접근합니다. 먼저 복잡한 사용자 요청을 설정하고, 그 요청을 해결하는 전체 궤적(trajectory)을 역방향으로 생성합니다.

구체적으로, 강력한 언어 모델(교사 모델)에게 "다음 도구들을 사용하여 해결할 수 있는 복잡한 사용자 요청을 만들고, 그 해결 과정을 단계별로 기술하라"는 지시를 줍니다. 교사 모델은 창의적인 태스크 시나리오와 함께 도구 호출 순서, 각 호출의 입력과 출력, 최종 답변을 생성합니다. 이렇게 생성된 전체 궤적이 학생 모델의 학습 데이터가 됩니다.

Top-Down 방식은 Bottom-Up보다 훨씬 다양하고 자연스러운 태스크 시나리오를 생성할 수 있습니다. 사람이 실제로 할 법한 복잡한 요청(예: "이번 주 서울 날씨를 확인하고, 비가 예보된 날에는 실내 활동 일정으로 자동 변경해줘")을 자연스럽게 만들어낼 수 있습니다. 다만 교사 모델의 품질에 크게 의존하며, 생성된 궤적에 오류가 포함될 수 있다는 단점이 있습니다.

**3-에이전트 시뮬레이션**

합성 데이터의 현실성과 다양성을 높이기 위한 더 정교한 방법으로, 3-에이전트 시뮬레이션이 있습니다. 이 방식에서는 세 개의 역할이 분리됩니다.

첫 번째는 사용자 에이전트(User Agent)입니다. 이 에이전트는 실제 사용자의 역할을 합니다. 다양한 페르소나(전문 개발자, 일반 사용자, 특정 도메인 전문가 등)를 설정하고, 그에 맞는 자연어 요청을 생성합니다. 사용자 에이전트는 단순히 초기 요청만 생성하는 것이 아니라, 어시스턴트의 답변에 대해 후속 질문을 하거나, 불만족스러울 때 재요청을 하는 대화적 행동도 시뮬레이션합니다.

두 번째는 어시스턴트 에이전트(Assistant Agent)입니다. 이 에이전트는 도구를 사용하여 사용자 요청을 해결하는 역할을 합니다. 어시스턴트 에이전트는 주어진 도구 목록에서 적절한 도구를 선택하고, 호출 형식을 결정하며, 도구 실행 결과를 해석하여 다음 행동을 결정합니다.

세 번째는 도구 에이전트(Tool Agent)입니다. 이 에이전트는 실제 API 호출을 시뮬레이션합니다. 실제 서버에 연결하지 않고도 현실적인 API 응답을 생성합니다. 예를 들어, 날씨 API 호출에 대해 형식에 맞는 가상의 응답을 반환하고, 오류 상황(타임아웃, 파라미터 오류, 서버 오류 등)도 확률적으로 발생시킵니다.

이 세 에이전트의 상호작용을 시뮬레이션하면, 단순한 태스크 해결 궤적뿐만 아니라 도구 호출 실패 처리, 대화적 맥락에서의 도구 사용, 다중 턴 상호작용 등 현실적인 시나리오를 대량으로 생성할 수 있습니다.

**일관성 기반 데이터 필터링**

합성 데이터는 생성 과정에서 오류, 불일관성, 논리적 결함을 포함할 수 있습니다. 이를 그대로 학습에 사용하면 모델이 잘못된 패턴을 학습할 수 있습니다. 일관성 기반 데이터 필터링(consistency-based filtering)은 이 문제를 해결하기 위한 품질 관리 방법입니다.

일관성 검사의 기본 원리는 간단합니다. 같은 태스크를 여러 번 독립적으로 생성하고, 결과들이 서로 일관된 경우에만 학습 데이터로 채택합니다. 만약 동일한 입력에 대해 서로 다른 도구 호출 궤적이 생성되고, 그 결과도 다르다면 어떤 것이 맞는지 알 수 없으므로 해당 데이터를 버립니다. 반면 여러 번의 생성이 같은 결과를 냈다면, 그 궤적이 신뢰할 수 있다고 판단합니다.

더 정교한 방법은 역방향 일관성(backward consistency) 검사입니다. 생성된 궤적으로부터 원래의 질문이나 태스크를 역추론할 수 있는지 확인합니다. 도구 호출 궤적을 보고 "이 궤적은 어떤 질문에 대한 답인가?"를 다른 모델로 추론했을 때, 원래 질문과 일치하지 않는다면 그 궤적에 문제가 있을 가능성이 높습니다.

**자기 진화 API 합성**

도구 사용 능력이 향상된 모델을 다시 활용하여 더 좋은 합성 데이터를 생성하는 자기 진화(self-evolution) 방식도 있습니다. 초기 데이터로 학습된 모델 v1이 새로운 합성 데이터를 생성하고, 이 데이터로 v2를 학습시키며, v2가 다시 더 나은 데이터를 생성하는 반복 과정입니다. 각 세대의 모델은 이전 세대보다 더 복잡하고 정확한 도구 사용 궤적을 생성할 수 있으므로, 학습 데이터의 품질이 점진적으로 향상됩니다.

이 방식의 핵심 과제는 각 세대에서 데이터 품질이 실제로 향상되고 있는지를 검증하는 것입니다. 모델이 자신의 편향을 데이터에 반영하면, 자기 강화(self-reinforcement)를 통해 오류가 증폭될 수 있습니다. 이를 방지하기 위해 외부 기준(실제 API 실행 결과, 독립적인 검증 모델)을 통한 검증이 필요합니다.

**모델 인식 복잡도 조절**

합성 데이터의 복잡도를 현재 모델의 능력에 맞게 조절하는 것도 중요합니다. 너무 쉬운 데이터만으로 학습하면 성장이 없고, 너무 어려운 데이터는 모델이 학습하지 못합니다. 모델 인식 복잡도 조절(model-aware difficulty adaptation)은 현재 모델의 성능을 측정하고, 그에 맞는 난이도의 합성 데이터를 우선적으로 생성하는 방식입니다.

구체적으로, 모델이 쉽게 맞추는 태스크는 학습 데이터에서 비중을 줄이고, 모델이 어려워하는 패턴의 태스크를 더 많이 생성합니다. 커리큘럼 학습(curriculum learning) 관점에서, 초반에는 단순한 태스크부터 학습하고 점차 복잡도를 높여가는 전략도 효과적입니다.

**이중 검증**

합성 데이터의 품질을 보장하기 위해, 두 가지 검증 방법을 함께 사용하는 이중 검증(dual verification) 방식이 실용적입니다.

첫 번째는 규칙 기반 검증(rule-based verification)입니다. API 호출의 형식이 올바른지(파라미터 타입, 필수 파라미터 존재 등), 도구 실행 순서에 논리적 오류가 없는지, 결과값의 범위가 타당한지 등을 정해진 규칙으로 자동 검사합니다. 이 검증은 빠르고 저렴하며, 명백한 오류를 걸러냅니다.

두 번째는 모델 기반 검증(model-based verification)입니다. 별도의 검증 모델이 합성 데이터를 평가하여 내용적 타당성을 확인합니다. "이 도구 호출 결과로 이 답변이 나오는 것이 합리적인가?", "이 단계에서 다른 도구를 사용하는 것이 더 나았을 것인가?" 같은 판단을 합니다. 모델 기반 검증은 계산 비용이 높지만, 규칙으로 포착하기 어려운 논리적 오류나 비효율을 찾아냅니다.

**대조적 추론 기반 프롬프트 최적화**

합성 데이터 생성 과정에서 프롬프트의 품질은 데이터 품질에 직접적인 영향을 미칩니다. 대조적 추론 기반 프롬프트 최적화(contrastive reasoning-based prompt optimization)는 좋은 데이터를 생성한 프롬프트와 나쁜 데이터를 생성한 프롬프트를 대조 분석하여, 더 나은 프롬프트를 자동으로 학습하는 방법입니다.

어떤 프롬프트가 고품질 도구 사용 궤적을 생성하고, 어떤 프롬프트가 오류 많은 궤적을 생성하는지 비교하면, 효과적인 프롬프트의 특성(명확한 도구 사용 지침, 구체적인 예시 포함 여부, 오류 처리 방법 명시 등)을 파악할 수 있습니다. 이 분석 결과를 반영하여 프롬프트를 개선하면, 이후 생성되는 합성 데이터의 품질이 향상됩니다. 이 과정을 반복하면 프롬프트가 점차 최적화됩니다.

정리하면, 합성 데이터 기반 도구 학습은 실제 API 실행이나 인간 레이블 없이 도구 사용 학습 데이터를 대규모로 생성하는 접근법입니다. Bottom-Up 방식은 원자 태스크 조합으로 복잡도를 제어하고, Top-Down 방식은 자연스러운 복합 시나리오를 생성합니다. 3-에이전트 시뮬레이션은 현실적인 다중 에이전트 상호작용을 모사하고, 일관성 기반 필터링과 이중 검증으로 데이터 품질을 보장합니다. 자기 진화를 통해 데이터 품질은 세대를 거쳐 향상되며, 대조적 추론으로 생성 프롬프트도 최적화됩니다.

다음 단원인 3.3.1에서는 도구 학습을 통해 능력을 갖춘 모델이 실제 문제를 해결할 때 어떤 추론 전략을 사용하는지 다룹니다. ReAct 패턴, Chain-of-Thought와 도구 사용의 결합, 그리고 선형 추론을 넘어선 트리 탐색 기반 DFSDT를 살펴봅니다.

이 단원을 학습한 후에는 합성 데이터 기반 도구 학습의 주요 방법론(Bottom-Up, Top-Down, 3-에이전트 시뮬레이션), 데이터 품질 보장을 위한 일관성 기반 필터링과 이중 검증의 역할, 자기 진화의 원리와 위험, 그리고 모델 인식 복잡도 조절의 의미를 설명할 수 있습니다.
