# 3.1.2 Function Calling의 기본 구조

3.1.1에서 LLM의 세 가지 구조적 한계와 외부 도구의 필요성을 살펴보았습니다. 이 단원에서는 에이전트가 실제로 도구를 호출하는 기본 메커니즘인 Function Calling의 구조를 다룹니다.

Function Calling은 LLM이 텍스트 응답을 생성하는 대신, 외부 함수를 호출하기 위한 구조화된 명령을 출력하는 메커니즘입니다. 일반적인 LLM 대화에서는 사용자가 질문하면 모델이 텍스트로 답합니다. Function Calling이 활성화되면, 모델은 "이 질문에 답하려면 외부 도구를 호출해야 한다"고 판단하고, 텍스트 대신 함수 호출 명령을 출력합니다. 이 명령에는 호출할 **함수명(function name)**, 전달할 **인자(arguments)**, 그리고 기대하는 **반환값(return value)**의 구조가 포함됩니다.

구체적인 예를 들어보겠습니다. 사용자가 "서울의 현재 기온은?"이라고 질문하면, 일반 LLM은 "서울의 평균 기온은 약 12도입니다"같은 부정확한 텍스트를 생성할 수 있습니다. Function Calling이 가능한 에이전트는 다음과 같은 구조화된 명령을 출력합니다. 함수명은 "get_weather"이고, 인자는 {"location": "Seoul", "unit": "celsius"}입니다. 시스템은 이 명령을 실제 날씨 API에 전달하고, 반환된 결과(예: {"temperature": 15, "condition": "cloudy"})를 에이전트에게 돌려줍니다. 에이전트는 이 결과를 바탕으로 "서울의 현재 기온은 15도이며, 흐린 날씨입니다"라는 정확한 응답을 생성합니다.

Function Calling의 작동 과정에서 가장 중요한 판단이 **도구 사용 시점 결정**입니다. 모든 질문에 도구가 필요한 것은 아닙니다. "머신러닝이 무엇인지 설명해 줘"라는 질문은 LLM의 내부 지식만으로 충분히 답할 수 있으므로 도구 호출이 불필요합니다. 반면 "지금 비트코인 가격은?"이라는 질문은 실시간 데이터가 필요하므로 도구 호출이 필수적입니다. 에이전트는 각 입력에 대해 "도구를 호출할 것인가(tool-use), 직접 답변할 것인가(non-tool-use)"를 판단해야 합니다. 이 판단이 잘못되면, 도구가 필요한 질문에 환각으로 답하거나, 도구가 불필요한 질문에 불필요한 API 호출을 발생시킵니다.

도구 사용 시점 결정과 밀접한 것이 **입력 위치별 API 호출 삽입**입니다. 초기 도구 학습 연구에서는 텍스트 스트림 중 어느 위치에 API 호출을 삽입할지를 결정하는 것이 핵심 문제였습니다. 모델이 텍스트를 생성하다가 "여기서 계산이 필요하다"고 판단하는 지점에 API 호출을 삽입하고, 반환값을 받아 텍스트 생성을 이어갑니다. 현대의 Function Calling 구현에서는 이 과정이 메시지 레이어(system/user/assistant/tool) 수준에서 구조화되어, 호출 위치 결정이 더 명시적입니다.

**단일 턴(single-turn) Function Calling**은 한 번의 사용자 질문에 대해 한 번의 함수 호출로 완결되는 가장 기본적인 형태입니다. 사용자가 질문하면 에이전트가 함수를 호출하고, 결과를 받아 응답을 생성하고, 상호작용이 끝납니다. "서울 날씨"를 묻고 답하는 위의 예가 단일 턴 Function Calling입니다.

단일 턴 Function Calling만으로는 해결할 수 없는 태스크가 많습니다. **도구 체이닝(tool chaining)**은 여러 도구를 순서대로 호출하여 복합적 태스크를 해결하는 것입니다. 첫 번째 도구의 출력이 두 번째 도구의 입력이 됩니다. 예를 들어 "서울에서 내일 비가 오면 우산을 가져가야 하는데, 근처 편의점에서 우산을 살 수 있어?"라는 질문은 먼저 날씨 API로 내일 서울의 날씨를 확인하고, 비가 오는 경우 지도 API로 근처 편의점을 검색하는 두 단계의 체이닝이 필요합니다. 도구 체이닝은 1.1.2에서 다룬 ReAct의 Thought-Action-Observation 루프가 여러 도구에 걸쳐 반복되는 것으로, 3.1.3에서 더 복잡한 형태를 자세히 다룹니다.

정리하면, Function Calling은 LLM이 텍스트 대신 구조화된 함수 호출 명령(함수명, 인자, 반환값)을 출력하는 메커니즘입니다. 도구 사용 시점 결정은 tool-use와 non-tool-use를 구분하는 핵심 판단이며, 단일 턴 Function Calling은 한 번의 호출로 완결되고, 도구 체이닝은 여러 호출을 연결하여 복합 태스크를 해결합니다.

다음 단원인 3.1.3에서는 멀티턴 Function Calling, 순차적-병렬-의존적 호출 패턴, 중첩 파라미터 등 더 복잡한 Function Calling 형태를 살펴봅니다.

이 단원을 마치면 API 호출의 기본 구조를 설명하고, 단일 턴 Function Calling에서 도구 사용 시점 결정과 입력 위치별 호출 삽입의 동작 방식을 기술할 수 있습니다.
