# 5.2.1 자기 교정의 정의와 근본적 한계

자기 교정(self-correction)은 언어 모델이 자신이 생성한 출력을 스스로 검토하고 오류를 발견하여 더 나은 출력으로 수정하는 과정을 가리킵니다. 개념 자체는 직관적이고 매력적입니다. 사람도 글을 쓴 뒤 다시 읽으며 고치고, 계산을 검산하며 오류를 잡아냅니다. 모델도 같은 방식으로 품질을 높일 수 있다면, 단일 생성 패스의 한계를 넘어설 수 있습니다. 그러나 자기 교정에는 근본적인 역설과 여러 실용적 한계가 존재합니다. 이 단원에서는 자기 교정의 개념적 범위, 분류 체계, 그리고 그것이 왜 생각처럼 잘 작동하지 않는지를 살펴봅니다.

자기 교정의 방식은 교정 신호의 출처에 따라 세 가지로 분류됩니다. 첫 번째는 내재적 교정(intrinsic correction)으로, 모델이 외부 정보 없이 오로지 자신의 내부 능력만으로 출력을 검토하고 수정하는 방식입니다. 두 번째는 외부 교정(extrinsic correction)으로, 외부 도구, 코드 실행 결과, 검색 엔진 조회 결과, 또는 다른 모델이 제공하는 피드백을 활용하여 수정하는 방식입니다. 세 번째는 파인튜닝 기반 교정(fine-tuning-based correction)으로, 교정 행동을 학습 목표로 삼아 모델을 직접 훈련함으로써 교정 능력을 내재화하는 방식입니다.

계층적으로는 S1, S2, S3의 세 단계로 구분하기도 합니다. S1은 단순 재시도로, 다른 프롬프트나 샘플링 설정으로 동일 문제를 다시 시도하는 것입니다. S2는 명시적 검토로, 이전 출력을 입력으로 받아 비판을 생성한 뒤 이를 바탕으로 수정본을 작성합니다. S3는 학습된 교정으로, 교정 행동이 모델 가중치에 반영되어 있어서 특별한 프롬프트 없이도 자연스럽게 교정 패턴이 발현됩니다.

내재적 교정의 근본적 역설은 다음과 같습니다. 모델이 오류를 식별할 능력이 있다면, 그 능력은 애초에 오류 없는 출력을 생성하는 데도 쓰였을 것입니다. 처음 생성할 때 오류를 피하지 못한 모델이, 검토 단계에서 갑자기 그 오류를 발견할 수 있다는 보장이 없습니다. 이는 생성 능력과 검증 능력이 완전히 분리되어 있지 않기 때문입니다. 동일한 매개변수 집합이 두 역할을 모두 수행하며, 생성 시 놓친 오류는 검토 시에도 놓칠 가능성이 높습니다.

교정 방향성 비대칭(correction asymmetry)은 이 역설의 구체적 발현입니다. 실험적으로 관찰되는 현상은 모델이 자기 교정을 수행할 때, 틀린 답을 맞는 답으로 교정하는 경우보다 맞는 답을 틀린 답으로 교정하는 경우가 비슷하거나 더 많다는 것입니다. 이는 자기 교정이 품질을 높이는 것이 아니라 임의적인 변동을 도입하는 역할을 한다는 의미입니다. 이미 정확한 답변을 불필요하게 수정하는 경향은 특히 추론 LLM(reasoning LLM)에서 두드러지게 나타납니다. 이 모델들은 긴 추론 과정에서 스스로의 결론을 여러 번 재검토하다가 원래는 맞았던 답변을 뒤집어버리는 패턴을 보입니다. 이를 추론 LLM 교정 포화(reasoning LLM correction saturation)라고 부르기도 합니다.

오라클 의존(oracle dependency) 문제도 자기 교정의 현실적 한계를 이해하는 데 중요합니다. 많은 자기 교정 연구에서 "모델이 수정을 수행했을 때 성능이 향상된다"는 결과를 보고하지만, 실험 설계를 자세히 들여다보면 교정 대상을 선택하거나 교정의 방향을 안내하는 데 오라클 정보(실제 정답 또는 실제 오류 여부)가 사용된 경우가 많습니다. 즉, 모델이 스스로 언제 수정해야 할지를 판단한 것이 아니라, 오류가 있다는 정보를 외부에서 제공받은 상황에서 수정을 수행한 것입니다. 실제 배포 환경에서는 이런 오라클 정보가 없으므로, 이런 실험 결과는 내재적 자기 교정의 능력을 과대평가하게 만들 수 있습니다.

불공정 비교(unfair comparison) 문제도 있습니다. 자기 교정 시스템의 성능을 평가할 때, 교정 과정에서 추가로 소비된 연산량을 고려하지 않는 경우가 많습니다. 초기 답변을 생성하고 추가로 비판 및 수정을 생성하는 데 쓰인 FLOPs를, 단순히 더 긴 초기 생성이나 Best-of-N 샘플링에 투입했을 때와 비교하지 않으면 자기 교정이 실제로 연산 효율적인지 알 수 없습니다. 여러 연구에서 동일한 연산 예산을 기준으로 비교했을 때, 자기 교정이 단순한 Best-of-N보다 우위에 있지 않다는 결과가 보고되고 있습니다.

이러한 한계들은 내재적 자기 교정이 완전히 무용하다는 것을 의미하지 않습니다. 외부 피드백이 보강된 교정, 즉 코드 실행 오류 메시지나 검색 결과 같은 환경 신호를 활용하는 방식은 분명한 효과를 보입니다. 또한 교정 행동을 명시적으로 학습시킨 파인튜닝 기반 교정은 내재적 교정의 역설을 일부 우회할 수 있습니다. 이를 5.2.2와 5.2.3에서 자세히 다룹니다.

정리하면, 자기 교정은 내재적, 외부, 파인튜닝 기반 방식으로 분류되며 S1~S3 계층으로 나뉩니다. 내재적 교정의 근본적 역설은 생성과 검증이 같은 매개변수에 의존한다는 데 있으며, 교정 방향성 비대칭은 이미 맞는 답을 틀리게 바꾸는 역효과로 나타납니다. 오라클 의존과 불공정 비교 문제는 자기 교정 성능을 과대평가하는 실험 설계를 경계해야 함을 시사합니다. 추론 LLM에서는 교정 포화 현상이 특히 두드러집니다.

다음 단원인 5.2.2에서는 강화학습(RL)을 이용하여 자기 교정 능력을 직접 학습시키는 방법을 살펴봅니다. 내재적 교정의 근본적 한계를 학습 목표 설계와 온 폴리시 훈련으로 어떻게 극복하려는지, 그 구체적인 메커니즘을 살펴봅니다.
