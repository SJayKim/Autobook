# 5.3.1 검증 메커니즘과 보상 모델

모델이 답변을 생성하거나 교정하고 나면, 그 결과가 실제로 올바른지를 판단하는 과정이 필요합니다. 이 판단을 담당하는 것이 검증 메커니즘(verification mechanism)이며, 강화학습 맥락에서는 보상 모델(reward model)이 같은 역할을 합니다. 검증 메커니즘의 품질은 TTC 스케일링의 효과를 직접적으로 결정합니다. 아무리 많은 후보를 생성해도 잘못된 검증자가 틀린 것을 고른다면 TTC 투자는 의미를 잃습니다. 이 단원에서는 주요 검증 방식들의 원리와 각각의 한계를 살펴봅니다.

결과 보상 모델(Outcome Reward Model, ORM)은 가장 단순한 형태의 검증자입니다. ORM은 최종 답변만을 입력으로 받아 그것이 올바른지 여부를 이진 판단하거나 연속적인 점수로 평가합니다. 중간 추론 과정은 고려하지 않습니다. ORM의 장점은 훈련 데이터 구성이 상대적으로 쉽다는 것입니다. 문제와 정답 쌍만 있으면 되므로, 데이터 수집 비용이 낮습니다. 단점은 최종 답이 맞더라도 추론 과정이 잘못되었을 수 있다는 점입니다. 예를 들어, 틀린 논리로 우연히 맞는 답에 도달했을 때 ORM은 높은 점수를 부여합니다. 반대로 거의 맞았지만 답 표현의 사소한 차이 때문에 낮은 점수를 받는 경우도 있습니다.

과정 보상 모델(Process Reward Model, PRM)은 각 추론 단계에 대해 개별적인 보상을 부여합니다. 수학적 풀이에서 각 계산 단계가 올바른지, 논리 추론에서 각 추론 단계가 유효한지를 단계별로 평가합니다. PRM은 ORM보다 훨씬 세밀한 신호를 제공하므로, 오류가 발생한 정확한 단계를 식별하고 학습하는 데 유리합니다. 그러나 단계별 레이블을 얻으려면 각 추론 단계에 대한 사람의 주석이나 자동 검증이 필요하며, 이는 수집 비용이 상당합니다. 또한 추론 과정을 명확한 단계로 분할하기 어려운 과제에는 적용이 제한됩니다.

PRM 기반 트리 탐색(PRM-guided tree search)은 PRM을 5.1.2에서 다룬 트리 탐색과 결합한 방식입니다. 트리의 각 노드, 즉 각 추론 단계에서 PRM이 그 단계의 품질을 평가하고, 낮은 점수를 받은 가지는 조기에 가지치기합니다. 이를 통해 탐색 공간을 효율적으로 좁히고, 유망한 경로에 연산을 집중할 수 있습니다. PRM의 정확도가 높을수록 트리 탐색의 효율도 높아지므로, 두 방법은 서로의 효과를 증폭시키는 상보적 관계입니다.

LLM-as-a-Judge는 별도의 훈련된 보상 모델 대신, 언어 모델 자체를 판단자로 활용하는 방식입니다. 판단자 모델에게 평가 대상 답변과 함께 평가 기준을 프롬프트로 제공하면, 모델이 해당 답변의 품질을 설명하고 점수를 매깁니다. 이 방식의 강점은 별도의 보상 모델을 훈련하지 않아도 된다는 유연성입니다. 다양한 과제와 기준에 쉽게 적용할 수 있고, 판단 근거를 텍스트로 확인할 수 있습니다. 단점은 판단 일관성이 훈련된 보상 모델보다 낮을 수 있고, 판단자 모델도 편향과 오류를 가진다는 것입니다. 특히 판단자 모델이 응답 모델과 같은 모델이거나 같은 모델 패밀리에서 나온 경우, 동일한 오류를 공유하여 편향된 평가를 내릴 수 있습니다.

검증 가능 보상(Verifiable Reward)은 정답을 자동으로 확인할 수 있는 과제에서 사용하는 방식입니다. 수학 문제의 숫자 답, 코드 문제의 테스트 케이스 통과 여부처럼, 규칙 기반으로 즉시 검증이 가능한 경우에 해당합니다. 이 방식은 사람의 주석이나 별도 모델 없이도 정확하고 일관된 보상 신호를 제공합니다. 강화학습 기반 교정 훈련에서 가장 신뢰할 만한 보상 신호로 꼽힙니다. 단, 검증 가능한 과제로 적용 범위가 제한된다는 것이 근본적인 한계입니다. 개방형 질문, 창의적 작성, 다단계 추론의 중간 과정 등에서는 자동 검증이 어렵습니다.

자기 검증(self-verification)은 모델이 스스로 검증자가 되는 방식입니다. 5.2.1에서 논의한 내재적 자기 교정의 역설과 동일한 문제를 안고 있습니다. 그러나 자기 검증이 완전히 무용한 것은 아닙니다. 모델이 답변을 검증하는 역할을 맡았을 때와 생성하는 역할을 맡았을 때의 작동 방식이 미묘하게 다르기 때문입니다. 검증 프롬프트가 모델로 하여금 다른 각도에서 문제를 바라보도록 유도하면, 생성 시 놓쳤던 오류를 발견하는 경우도 있습니다. 다만 이것이 체계적으로 신뢰할 만한 수준인지는 과제와 모델에 따라 크게 달라집니다.

생성적 보상 모델(Generative Reward Model, GenRM)은 이진 점수 대신 언어 모델의 생성 능력을 활용하여 보상을 계산하는 방식입니다. GenRM은 평가 대상 답변에 대한 체인 오브 생각 스타일의 비판 텍스트를 생성한 뒤, 그 텍스트를 바탕으로 점수를 부여합니다. 텍스트 형태의 비판은 어디가 잘못되었는지를 설명하므로, 단순한 스칼라 점수보다 정보량이 풍부합니다. 이 비판은 교정 과정에서 수정 방향을 안내하는 데도 직접 활용될 수 있습니다. GenRM은 교정 효율성(correction efficiency)을 높이는 방향으로 작용합니다. 단순히 "틀렸다"고 알리는 것이 아니라, 무엇이 왜 틀렸는지를 설명함으로써 수정 방향을 좁히기 때문입니다.

정리하면, ORM은 구현이 단순하지만 추론 과정의 오류를 놓치고, PRM은 단계별 정밀 피드백을 제공하지만 레이블 비용이 높습니다. LLM-as-a-Judge는 유연하지만 일관성이 낮을 수 있고, 검증 가능 보상은 가장 신뢰할 만하지만 적용 범위가 제한됩니다. 자기 검증은 보조적 역할에 그치며, GenRM은 텍스트 비판으로 교정 효율성을 높입니다. PRM 기반 트리 탐색은 검증과 탐색을 결합한 강력한 방식입니다.

다음 단원인 5.3.2에서는 멀티턴 환경에서 보상을 각 턴에 어떻게 배분할 것인지를 다루는 크레딧 할당(credit assignment) 문제를 살펴봅니다. 최종 결과만으로 보상을 부여하는 방식의 한계와, 중간 턴에 밀집 보상을 설계하는 방법을 함께 다룹니다.
