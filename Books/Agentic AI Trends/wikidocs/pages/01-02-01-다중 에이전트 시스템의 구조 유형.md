# 1.2.1 다중 에이전트 시스템의 구조 유형

1.1.1에서 LLM 에이전트의 네 가지 핵심 모듈을 살펴보았고, 1.1.3에서 역할 기반 페르소나로 에이전트를 차별화하는 방법을 다루었습니다. 하나의 에이전트만으로도 많은 작업을 수행할 수 있지만, 현실의 복잡한 문제는 여러 전문 분야의 지식과 관점이 동시에 필요한 경우가 많습니다. 소프트웨어를 개발하려면 기획자, 설계자, 개발자, 테스터가 협력해야 하고, 연구 보고서를 작성하려면 자료 수집, 분석, 집필, 검토가 분업되어야 합니다. 이처럼 여러 에이전트가 각자의 역할을 맡아 함께 작업하는 시스템을 다중 에이전트 시스템(Multi-Agent System, MAS)이라고 합니다. 이 단원에서는 MAS의 에이전트 구성 방식과 네 가지 구조 유형, 그리고 에이전트 수 확장 시 발생하는 통신 비용 문제를 살펴봅니다.

MAS를 설계할 때 가장 먼저 결정해야 할 사항은 에이전트들의 구성이 **동질적(homogeneous)**인지 **이질적(heterogeneous)**인지입니다. 동질적 구성은 모든 에이전트가 동일한 LLM 모델과 역량을 공유하는 방식입니다. 구현과 조율이 단순하고 에이전트 간 출력 형식이 일관됩니다. 그러나 모든 에이전트가 같은 편향과 약점을 공유하기 때문에, 한 에이전트에서 나타나는 오류 패턴이 시스템 전체에 동일하게 나타날 수 있습니다. 이질적 구성은 서로 다른 LLM, 역할, 전문성을 가진 에이전트들이 협력하는 방식입니다. 1.1.3에서 살펴본 것처럼 Product Manager, Architect, Engineer, QA 에이전트가 각각 다른 역할과 프롬프트를 갖는 구조가 대표적입니다. 역할 분업으로 각 에이전트가 자신의 전문 도메인에만 집중할 수 있어 품질이 높아지지만, 에이전트 간 출력 형식 불일치와 상호운용성 문제가 발생할 수 있습니다. 100편 이상의 논문을 분석한 서베이에서 역할 기반 이질적 구성이 현존하는 MAS 시스템에서 압도적으로 많이 사용되는 것으로 나타났습니다.

에이전트들이 어떤 방식으로 서로 연결되고 통신하는지를 결정하는 것이 MAS의 구조입니다. 크게 네 가지 구조 유형이 있습니다.

첫 번째는 **중앙집중형(Centralized) 구조**입니다. 단일 오케스트레이터 에이전트가 전체 상태를 유지하고 모든 라우팅 결정을 내립니다. 작업자(Worker) 에이전트들은 오케스트레이터의 지시에 따라 하위 작업을 수행하고 결과를 반환합니다. 1.1.3에서 다룬 소프트웨어 팀 구조가 이 방식의 전형적인 예입니다. 사용자 요구사항이 입력되면 Product Manager가 요구사항 문서를 작성하고, Architect가 설계 문서를 생성하며, Engineer가 코드를 구현하고, QA가 검증합니다. 각 단계의 산출물이 구조화된 형식으로 다음 에이전트에게 전달되는 어셈블리 라인 구조입니다. 이 구조의 장점은 전역 상태 일관성이 보장되고, 의사결정 흐름의 추적과 디버깅이 용이하며, 중복 작업이 최소화되어 토큰 효율이 높다는 것입니다. 감사 가능한 결정 흐름이 필요한 엔터프라이즈 환경에 적합합니다. 반면 오케스트레이터가 단일 장애점이 되며, 순차적 조율로 인한 지연 병목이 발생합니다. 에이전트 수가 20개를 초과하면 오케스트레이터의 컨텍스트 윈도우와 처리 능력의 한계로 확장성이 급격히 저하됩니다.

두 번째는 **분산형(Distributed) 구조**입니다. 중앙 관리자 없이 에이전트들이 **공유 메시지 풀**(shared message pool)을 통해 통신합니다. 이 구조를 블랙보드 아키텍처라고도 부릅니다. 각 에이전트는 메시지 풀에 결과를 게시(publish)하고, 자신과 관련된 메시지를 구독(subscribe)합니다. 전체 그림을 보는 에이전트는 없으며, 시스템의 지능은 국소적 상호작용의 집합에서 창발(emerge)합니다. 가상 마을에서 25명의 에이전트가 독립적으로 활동하며 중앙 조율 없이 메모리 스트림과 공유 환경을 통해 사회적 행동을 만들어 내는 시뮬레이션이 이 구조의 대표적 사례입니다. 분산형의 장점은 부분 장애에 강한 내결함성, 에이전트 수에 따른 처리량의 선형 확장, 새 에이전트 추가 시 기존 에이전트의 코드 수정이 불필요하다는 점입니다. 수백 개의 에이전트 규모에서도 작동할 수 있습니다. 그러나 전역 일관성 유지가 어렵고, 중복 작업으로 인한 토큰 효율 저하, 시스템 전체 우선순위 강제 집행의 어려움, 디버깅의 복잡성이 단점입니다.

세 번째는 **계층형(Hierarchical) 구조**입니다. 여러 감독 레이어가 트리 구조를 형성합니다. 결정이 위에서 아래로 전달되고 정보는 아래에서 위로 집계됩니다. 각 레이어는 상위 레이어를 위해 복잡성을 추상화합니다. 중앙집중형의 단일 오케스트레이터를 다층 구조로 확장한 형태로 볼 수 있습니다. 예를 들어 뉴스 플랫폼에서 최상위 감독자가 콘텐츠 팀, 팩트체킹 팀, 퍼블리싱 팀을 조율하고, 각 팀 내의 중간 감독자가 도메인별(정치, 기술, 스포츠) 에이전트를 감독하며, 실행 에이전트가 개별 소스 스크래핑, 검증, 편집을 수행하는 구조입니다. 자연스러운 문제 분해, 병렬 팀 실행으로 인한 높은 처리량, 각 레이어에서의 의미 있는 추상화가 장점입니다. 중앙집중형보다 훨씬 큰 규모로 확장할 수 있습니다. 단점은 레이어 간 조율 오버헤드, 감독자 용량의 병목 가능성, 그리고 상하 계층 간 정보가 왜곡되는 이른바 "전언 게임" 효과입니다.

네 번째는 **P2P(Peer-to-Peer) 구조**입니다. 모든 에이전트가 동등한 지위를 갖고 필요에 따라 서로 직접 통신합니다. 중앙 관리자나 감독 계층 없이 합의를 통해 결정이 이루어집니다. 주요 에이전트 프레임워크들은 1대1 대화, 그룹 대화, 중첩 대화를 모두 지원하여 P2P 구조를 유연하게 구성할 수 있도록 합니다. P2P 구조의 가장 대표적인 활용 사례가 LLM 디베이트(debate) 패턴입니다. 여러 에이전트가 동등한 지위에서 서로의 주장을 논박하고 검증하는 과정을 거쳐 더 높은 품질의 추론 결과를 도출합니다. 특정 단일 구성요소에 대한 의존성이 없고 합의 형성으로 출력 품질이 향상될 수 있다는 점이 장점입니다. 그러나 에이전트 수 증가 시 통신 경로가 기하급수적으로 증가하고, 합의 도달까지의 지연이 커질 수 있으며, 전역 상태 관리가 가장 어려운 구조입니다.

각 구조가 적합한 상황을 정리하면 다음과 같습니다. 일관성, 감사 가능성, 소규모 배포가 필요하면 중앙집중형이 적합합니다. 디베이트나 검증 시나리오에서 소수의 에이전트가 긴밀하게 협력하는 경우에는 P2P가 적합합니다. 자연스러운 도메인 계층이 존재하는 대규모 시스템에는 계층형이, 수백 개 에이전트의 고복원성 시스템에는 분산형이 적합합니다. 실제 시스템에서는 하나의 구조만 순수하게 사용하기보다, 상위 레벨에서 계층형으로 조직하고 각 팀 내부에서는 P2P로 소통하는 하이브리드 구조가 흔합니다.

마지막으로, MAS 설계에서 가장 중요한 실용적 제약인 **에이전트 수 확장 시 통신 비용 폭발 문제**를 살펴보겠습니다. 에이전트 수가 늘어나면 통신 비용이 어떻게 증가하는지를 실증 분석한 연구에 따르면, 에이전트 수 n에 대한 턴 수 T는 대략 n의 1.7제곱에 비례합니다. 이것은 통신 비용이 에이전트 수에 대해 거의 제곱에 가깝게 증가한다는 의미입니다. n명의 에이전트가 모두 서로 통신할 수 있는 완전 연결 구조에서는 이론적으로 n(n-1)/2개의 통신 경로가 존재합니다. 실용적 대역폭 한계로 이보다는 완화되지만, 여전히 초선형(super-linear) 증가입니다.

이 초선형 증가의 실질적 결과는 심각합니다. 고정된 계산 예산 아래에서 에이전트당 추론 용량은 3~4개의 에이전트를 초과하는 시점부터 급격히 얇아집니다. 통신 오버헤드가 실제 추론에 사용 가능한 인지 예산을 잠식하기 때문입니다. 구체적인 수치로 보면, 16개 도구를 활용하는 태스크에서 중앙집중형 다중 에이전트는 단일 에이전트 대비 285%의 추가 토큰을 소비했고, 하이브리드 다중 에이전트는 515%의 추가 토큰을 소비했습니다. 성능과 메시지 밀도의 관계는 로그 포화를 따르며, 최적 메시지 밀도를 초과하면 오버헤드를 두 배로 늘려도 성능 개선은 2~3%에 불과합니다.

구조별 확장성을 비교하면, 중앙집중형은 약 20개, P2P는 5개 이하, 계층형은 수십에서 수백 개, 분산형은 수백 개 이상의 에이전트를 지원할 수 있습니다. 이러한 통신 비용 폭발을 완화하기 위해 실제 시스템에서는 여러 전략이 사용됩니다. 비동기 통신으로 동기 대기를 제거하고, 공유 메시지 풀로 점대점(point-to-point) 연결 수를 줄이며, 계층화로 각 에이전트가 접촉하는 에이전트 수를 제한하고, 구조화된 산출물로 메시지당 정보 밀도를 높이는 것입니다. 단순히 에이전트를 추가한다고 성능이 선형으로 개선되지 않는다는 점을 인식하는 것이 MAS 설계의 출발점입니다.

정리하면, 다중 에이전트 시스템은 동질적 또는 이질적 에이전트 구성을 기반으로 중앙집중형, 분산형, 계층형, P2P의 네 가지 구조 유형으로 설계됩니다. 중앙집중형은 일관성과 추적성이 강하지만 확장성에 한계가 있고, 분산형은 내결함성과 확장성이 뛰어나지만 전역 일관성이 어렵습니다. 계층형은 자연스러운 문제 분해를 지원하며, P2P는 합의 기반 품질 향상에 적합합니다. 에이전트 수 확장 시 통신 비용은 초선형으로 증가하므로, 비동기 통신, 공유 메시지 풀, 계층화, 구조화된 산출물 등의 완화 전략이 필수적입니다.

다음 단원인 1.2.2에서는 이러한 구조 위에서 에이전트들이 실제로 어떻게 협업하는지, 협력, 경쟁, 협경 세 가지 협업 유형과 조율 프로토콜의 설계 원리를 다룹니다.

이 단원을 마치면 다중 에이전트 시스템의 중앙집중형, 분산형, 계층형, P2P 구조를 비교하고, 동질적 에이전트와 이질적 에이전트 구성의 차이를 설명할 수 있습니다.
