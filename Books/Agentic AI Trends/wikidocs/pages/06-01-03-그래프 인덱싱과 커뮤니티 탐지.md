# 6.1.3 그래프 인덱싱과 커뮤니티 탐지

지식 그래프가 구축되면 이를 효과적으로 검색하고 활용하기 위한 인덱싱 전략이 필요합니다. 수백만 개의 노드와 엣지로 이루어진 그래프에서 원하는 정보를 찾으려면, 그래프의 구조적 특성을 활용한 별도의 조직화 방법이 있어야 합니다. 이 단원에서는 그래프를 의미 있는 단위로 분할하는 커뮤니티 탐지 알고리즘과, 그 결과를 인덱싱하여 검색에 활용하는 방법을 다룹니다.

커뮤니티 탐지(community detection)는 그래프에서 서로 밀접하게 연결된 노드 집합을 자동으로 식별하는 기술입니다. 현실 세계의 지식 그래프에는 자연스러운 군집 구조가 존재합니다. 특정 학문 분야의 개념들, 특정 산업의 기업들, 특정 시대의 역사적 인물들은 그래프 내에서 서로 더 빽빽하게 연결되어 있는 반면, 다른 군집과는 상대적으로 느슨하게 연결됩니다. 커뮤니티 탐지는 이런 자연적 클러스터를 알고리즘적으로 찾아내는 작업입니다.

Leiden 알고리즘은 현재 가장 널리 사용되는 커뮤니티 탐지 알고리즘 중 하나입니다. 기존의 Louvain 알고리즘이 모듈성(modularity) 최적화 과정에서 단절된 커뮤니티를 생성하는 버그를 가지고 있었던 반면, Leiden은 이를 보정하여 항상 연결된 커뮤니티만 생성함을 보장합니다. Leiden은 각 노드를 개별 커뮤니티로 시작하여, 주변 커뮤니티로 이동할 때 모듈성 이득이 발생하면 합치는 과정을 반복합니다. 이 과정에서 그래프 전체의 모듈성 점수가 최대화되는 방향으로 커뮤니티 구조가 형성됩니다.

계층적 커뮤니티 탐지는 Leiden을 단일 해상도가 아닌 여러 해상도에서 반복 적용하여 계층 구조를 만드는 접근법입니다. 낮은 해상도에서는 소수의 큰 커뮤니티가 형성되고, 높은 해상도에서는 다수의 작은 커뮤니티가 형성됩니다. 이 다중 해상도 결과를 트리 구조로 조직화하면, 최상위 레벨에는 넓은 주제 군집이, 하위 레벨에는 세부 주제 군집이 배치됩니다. 이 계층 구조는 글로벌 질문에는 상위 레벨 커뮤니티를, 세부 질문에는 하위 레벨 커뮤니티를 활용할 수 있게 해줍니다.

재귀적 분할은 커뮤니티 계층을 구성하는 구체적인 방법입니다. 먼저 전체 그래프에 Leiden을 적용하여 최상위 커뮤니티들을 얻습니다. 그런 다음 각 커뮤니티를 독립된 서브그래프로 보고 다시 Leiden을 적용합니다. 이 과정을 커뮤니티 크기가 일정 기준 이하로 내려갈 때까지 재귀적으로 반복합니다. 결과적으로 루트에서 리프까지 내려갈수록 더 구체적이고 좁은 주제 군집이 형성되는 트리가 만들어집니다.

각 커뮤니티에 대해서는 LLM이 해당 커뮤니티에 속한 노드와 엣지들을 읽고 요약문을 생성합니다. 이것이 커뮤니티별 LLM 요약입니다. 요약문은 해당 커뮤니티가 어떤 주제를 다루는지, 핵심 엔티티들은 무엇인지, 커뮤니티 내의 주요 관계 패턴은 어떤지를 자연어로 기술합니다. 이 요약문은 이후 검색 단계에서 쿼리와의 관련성을 판단하는 데 사용됩니다.

상위 요약 통합은 계층 구조를 따라 올라가며 하위 커뮤니티 요약들을 결합하여 상위 커뮤니티의 요약을 생성하는 과정입니다. 단순히 하위 요약들을 이어붙이는 것이 아니라, LLM이 하위 요약들을 읽고 상위 수준에서의 통합된 관점을 새로 작성합니다. 이 통합 요약은 하위 요약들에서 드러나지 않는 상위 수준의 패턴이나 주제적 일관성을 포착할 수 있습니다.

쿼리 독립적 인덱싱은 GraphRAG 방식의 핵심 특징 중 하나입니다. 전통적 RAG에서는 문서가 그대로 인덱스에 저장되며, 인덱스의 구조는 쿼리와 무관합니다. GraphRAG의 커뮤니티 요약 인덱스도 특정 쿼리를 염두에 두지 않고 미리 구축됩니다. 그러나 커뮤니티 구조와 요약이 이미 그래프의 의미적 조직을 반영하기 때문에, 어떤 종류의 쿼리에도 관련된 커뮤니티를 빠르게 찾을 수 있습니다. 이는 온라인 처리 부하를 미리 오프라인으로 흡수하는 설계입니다.

그래프 저장 방식으로는 키-값 구조가 실용적입니다. 노드 ID를 키로 하고 해당 노드의 속성(이름, 유형, 설명, 커뮤니티 소속 등)을 값으로 저장하며, 엣지도 유사하게 관리합니다. 커뮤니티 요약은 커뮤니티 ID를 키로 하는 별도의 테이블에 저장됩니다. 이 구조는 Neo4j 같은 그래프 데이터베이스나, 단순한 경우 파일 시스템 기반 키-값 스토어로도 구현할 수 있습니다.

엔티티 노드가 허브 역할을 한다는 점은 그래프 인덱스의 중요한 특성입니다. 실제 지식 그래프에서는 소수의 노드가 매우 많은 엣지를 가지는 멱법칙(power law) 분포가 자주 나타납니다. 이런 허브 노드는 여러 커뮤니티에 걸쳐 연결되어 있어 커뮤니티 간 정보 통로 역할을 합니다. 검색 시 허브 노드에서 출발하면 그래프의 넓은 영역에 빠르게 접근할 수 있습니다.

그래프+벡터 결합 인덱싱은 그래프 구조 인덱스와 전통적 벡터 인덱스를 함께 유지하는 하이브리드 방식입니다. 노드와 엣지의 설명 텍스트, 커뮤니티 요약문을 임베딩하여 벡터 인덱스에도 저장합니다. 이렇게 하면 자연어 쿼리로 가장 관련성 높은 노드나 커뮤니티를 벡터 유사도로 빠르게 찾은 뒤, 그래프 구조 탐색으로 깊이 있는 추론을 이어갈 수 있습니다. 두 인덱스를 결합함으로써 각각의 장점을 취하는 것입니다.

정리하면, 그래프 인덱싱과 커뮤니티 탐지는 Leiden 알고리즘의 계층적 재귀 분할, 커뮤니티별 LLM 요약 생성, 상위 통합 요약, 쿼리 독립적 사전 인덱싱, 키-값 저장 구조, 허브 노드 활용, 그리고 그래프+벡터 결합 인덱싱으로 구성됩니다. 이 단계를 완료한 그래프 인덱스는 전통적 벡터 인덱스가 포착하지 못했던 글로벌 구조와 관계 정보를 검색에 제공할 준비가 된 상태입니다.

다음 단원에서는 이렇게 구축된 그래프 인덱스를 실제 검색에 활용하는 전략들을 다룹니다. Map-Reduce 패턴, PPR 기반 연상 검색, 서브그래프 검색, 하이브리드 검색과 리랭커를 결합하여 복잡한 쿼리에도 정밀하게 답변하는 방법을 살펴볼 것입니다.
