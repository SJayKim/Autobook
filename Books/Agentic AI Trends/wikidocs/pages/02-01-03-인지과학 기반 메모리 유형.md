# 2.1.3 인지과학 기반 메모리 유형

2.1.2에서 에이전트 메모리를 표현 형태(Forms)와 기능(Functions)의 두 축으로 분류했습니다. 이 분류는 메모리의 기술적 구현에 초점을 맞춘 것입니다. 그런데 에이전트 메모리 연구에서는 인간의 기억 체계를 참조하여 메모리를 설계하는 접근도 널리 사용됩니다. 인간의 뇌가 수십 년에 걸쳐 정보를 효율적으로 저장하고 검색하는 메커니즘을 진화시켜 왔다면, 그 구조를 에이전트 메모리에 차용하는 것은 자연스러운 발상입니다. 이 단원에서는 인지과학의 에피소드 기억, 의미 기억, 절차 기억 분류가 에이전트 메모리 설계에 어떻게 적용되는지, 그리고 Core Memory와 Declarative/Procedural 이분법의 실용적 의미를 살펴봅니다.

인지과학에서 장기 기억(long-term memory)은 크게 세 유형으로 분류됩니다. 이 분류는 캐나다의 심리학자 엔델 툴빙(Endel Tulving)이 1972년에 제안한 것으로, 반세기가 지난 지금도 기억 연구의 기본 틀로 사용되고 있습니다.

첫 번째 유형은 **에피소드 기억(Episodic Memory)**입니다. 특정 시간과 장소에서 일어난 개인적 경험의 기억입니다. "어제 점심에 카페에서 친구를 만났다", "지난주 회의에서 A 방안이 거부되었다"처럼 시간 태그가 붙은 사건의 기억입니다. 에피소드 기억의 핵심 특성은 시간적 맥락(temporal context)입니다. 언제 일어났는지, 어떤 순서로 일어났는지가 기억의 필수적인 구성 요소입니다.

에이전트 메모리에서 에피소드 기억은 에이전트가 특정 태스크를 수행한 경험의 기록으로 구현됩니다. 에이전트가 웹 쇼핑 태스크를 수행하면서 거친 행동과 관찰의 시퀀스, 가정 환경에서 물건을 찾으며 시도한 탐색 경로, 코딩 태스크에서 겪은 오류와 수정의 이력이 모두 에피소드 기억에 해당합니다. 타임스탬프가 함께 저장되어 "언제 이 경험을 했는지"를 추적할 수 있습니다. 2.1.2에서 다룬 경험 메모리 중 사례 기반(Case-based) 경험 메모리가 에피소드 기억의 직접적 대응입니다. 과거 궤적을 그대로 저장하고, 유사한 상황에서 이를 검색하여 참조합니다.

에피소드 기억의 강점은 구체적이고 풍부한 맥락 정보를 보존한다는 것입니다. 단점은 저장 공간을 많이 차지하고, 개별 에피소드에서 일반적 패턴을 추출하는 것은 별도의 처리가 필요하다는 것입니다. 또한 에피소드 기억만으로는 "일반적으로 이 유형의 문제에서는 어떤 접근이 효과적인가"라는 질문에 직접 답하기 어렵습니다.

두 번째 유형은 **의미 기억(Semantic Memory)**입니다. 시간과 무관하게 축적된 일반적 지식과 사실, 개념 간의 관계입니다. "서울은 한국의 수도이다", "물은 100도에서 끓는다", "Python은 인터프리터 언어이다"처럼 특정 경험에 묶이지 않은 추상적 지식입니다. 의미 기억의 핵심 특성은 시간 독립성(time independence)입니다. 언제 그 사실을 알게 되었는지는 중요하지 않으며, 사실 자체의 진위만이 중요합니다. 의미 기억은 흔히 계층적 트리 구조나 네트워크 구조로 조직됩니다. "동물 > 포유류 > 개 > 리트리버"처럼 상위 개념에서 하위 개념으로 이어지는 계층이나, 개념 간의 연관 관계를 나타내는 네트워크입니다.

에이전트 메모리에서 의미 기억은 에이전트가 축적한 일반 지식 트리플(예: "함수 A는 모듈 B에 속한다", "에러 코드 404는 리소스 미발견을 의미한다")이나, 환경에 대한 영속적 사실(예: "이 API의 응답 형식은 JSON이다")로 구현됩니다. 2.1.2에서 다룬 사실 메모리(Factual Memory)가 의미 기억의 직접적 대응입니다. 지식 그래프 형태로 저장되어 관계 기반 검색이 가능한 경우, 2.1.2의 Planar(2D) 표현 형태와 결합됩니다.

세 번째 유형은 **절차 기억(Procedural Memory)**입니다. 어떤 일을 수행하는 방법에 대한 기억으로, "어떻게(how to)"에 해당합니다. 자전거를 타는 방법, 피아노를 치는 방법처럼 의식적으로 설명하기 어렵지만 몸이 기억하는 기술입니다. 인간에게서 절차 기억은 대부분 암묵적(implicit)이지만, 에이전트에서는 명시적인 형태로 구현됩니다. 목표 지향적 절차, 워크플로우, 단계별 지침이 텍스트나 코드로 저장됩니다.

에이전트 메모리에서 절차 기억은 반복적으로 수행하는 태스크의 최적 절차를 저장하는 데 사용됩니다. "데이터를 분석할 때는 먼저 결측값을 확인하고, 이상치를 처리한 뒤, 정규화하는 순서를 따른다"와 같은 절차적 지식입니다. 2.1.2에서 다룬 경험 메모리 중 기술 기반(Skill-based) 경험 메모리가 절차 기억의 직접적 대응입니다. 경험을 재사용 가능한 코드 스니펫이나 API 호출 패턴으로 증류하여 저장하는 것은 절차 기억의 공학적 구현입니다.

이 세 가지 외에 에이전트 메모리 연구에서 자주 등장하는 개념이 **Core Memory**입니다. Core Memory는 에이전트에 항상 노출되는 최우선 영속 정보로, 사용자의 이름, 핵심 선호, 에이전트의 역할 정의 등 매 추론 호출에서 컨텍스트에 반드시 포함되어야 하는 정보를 담습니다. OS 가상 메모리 비유를 사용한 에이전트 시스템에서 Working Context의 시스템 프롬프트 영역에 해당합니다. Core Memory는 인지과학의 분류에 직접 대응하지는 않지만, 시스템 설계 관점에서 "절대 페이지 아웃되지 않는 상주 메모리"라는 독자적 역할을 합니다.

또 하나의 자주 사용되는 분류가 **선언적 기억(Declarative Memory)과 절차적 기억(Procedural Memory)의 이분법**입니다. 선언적 기억은 "무엇을 아는가(knowing what)"에 해당하며, 에피소드 기억과 의미 기억을 포괄합니다. 절차적 기억은 "어떻게 하는가(knowing how)"에 해당합니다. 이 이분법은 에이전트 메모리 시스템을 설계할 때 실용적인 가이드를 제공합니다. 선언적 메모리는 검색과 추론에 적합한 구조(벡터 데이터베이스, 지식 그래프)로, 절차적 메모리는 실행에 적합한 구조(코드 라이브러리, 워크플로우 템플릿)로 각각 저장하는 것이 자연스럽습니다.

인간의 기억 모델을 에이전트에 적용하는 것이 유용한 이유는, 인간의 기억 체계가 수십만 년의 진화를 통해 최적화된 정보 관리 시스템이기 때문입니다. 에피소드 기억으로 구체적 경험을 보존하면서 의미 기억으로 일반 지식을 추상화하고, 절차 기억으로 반복적 행동을 자동화하는 분업 구조는, 제한된 인지 자원으로 최대의 적응력을 발휘하는 전략입니다. 에이전트도 제한된 컨텍스트 윈도우와 연산 예산 안에서 최대의 성능을 내야 하므로, 같은 전략이 유효합니다. 다만 인간의 기억이 암묵적이고 연속적인 반면, 에이전트의 기억은 명시적이고 이산적이라는 차이가 있습니다. 인간은 에피소드 기억에서 의미 기억으로의 전환이 수면 중에 무의식적으로 일어나지만, 에이전트에서는 이 전환을 명시적인 알고리즘으로 설계해야 합니다. 2.5절에서 다룰 메모리 진화가 바로 이 전환 과정을 다룹니다.

정리하면, 인지과학의 에피소드 기억(시간 태그가 붙은 경험), 의미 기억(시간 독립적 추상 지식), 절차 기억(목표 지향 절차)은 에이전트 메모리의 각각 사례 기반 경험 메모리, 사실 메모리, 기술 기반 경험 메모리에 대응합니다. Core Memory는 매 추론 호출에 상주하는 최우선 정보이며, 선언적/절차적 이분법은 메모리의 저장 구조 설계를 안내합니다. 인간 기억 모델의 분업 구조는 제한된 자원으로 최대의 적응력을 발휘하는 전략으로, 에이전트 메모리 설계에 유효한 참조 틀을 제공합니다.

다음 단원인 2.2.1에서는 이 인지과학적 분류가 구체적으로 어떤 저장 아키텍처로 구현되는지, STM/MTM/LTM 계층 구조와 OS 가상 메모리 비유를 살펴봅니다.

이 단원을 마치면 인지과학의 에피소드 기억, 의미 기억, 절차 기억 분류가 에이전트 메모리 설계에 어떻게 적용되는지 설명하고, Core Memory와 Declarative/Procedural 이분법의 실용적 의미를 기술할 수 있습니다.
