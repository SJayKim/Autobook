# 5.2.3 증류와 오류-수정 궤적 학습

RL 기반 교정 학습은 강력하지만 안정적인 훈련을 위해 정교한 설계가 필요하고 연산 비용도 상당합니다. 이 단원에서는 보완적인 접근, 즉 교사 모델의 행동을 학생 모델에게 전수하는 증류(distillation)와, 오류를 포함한 수정 과정 전체를 학습 데이터로 활용하는 궤적 학습(trajectory learning)을 다룹니다.

사고 템플릿 증류(thought template distillation)는 교사 모델이 문제를 푸는 방식, 특히 스스로 검토하고 수정하는 사고 패턴을 학생 모델에게 가르치는 방법입니다. 교사 모델이 "이 부분은 잘못 계산했다. 다시 살펴보면..." 같은 자기 검토 구절을 포함한 풀이를 생성하면, 이 풀이를 그대로 학습 데이터로 삼아 학생 모델을 파인튜닝합니다. 학생 모델은 교사의 사고 패턴을 모방하면서 검토-수정 루프를 내재화합니다. 이 방식의 강점은 RL처럼 보상 함수를 설계하거나 불안정한 온라인 탐색을 수행할 필요가 없다는 점입니다. 단점은 학생이 교사의 패턴을 표면적으로 복사할 수는 있지만, 패턴 뒤에 있는 추론 원칙을 깊이 이해하지 못할 수 있다는 것입니다.

교차 모델 DPO(cross-model DPO, Direct Preference Optimization)는 동일 모델 내 비교가 아니라 서로 다른 모델 사이의 비교를 활용하는 방법입니다. 표준 DPO에서는 선호 응답과 비선호 응답을 쌍으로 구성하여 모델이 선호 응답 방향으로 학습하게 합니다. 교차 모델 DPO에서는 더 강력한 교사 모델의 응답을 선호, 학생 모델(또는 더 약한 모델)의 응답을 비선호로 설정합니다. 자기 교정 맥락에서는 교사 모델이 교정한 답변을 선호로, 교정 전 초기 답변을 비선호로 구성할 수 있습니다. 이를 통해 학생 모델은 교정 이후의 품질을 목표로 학습하게 됩니다.

오류-수정 궤적(error-correction trajectory) 학습은 이름 그대로 오류가 발생하고 그것이 수정되는 전 과정을 학습 데이터로 삼는 접근입니다. 단순히 최종 정답만 학습 대상으로 삼는 방식과 달리, 오류를 저지르고 인식하고 수정하는 전체 흐름을 학습합니다. 이 방식의 중요한 세부 사항은 오류 턴 손실 마스킹(error turn loss masking)입니다. 오류가 발생한 턴의 토큰에 대해서는 손실을 계산하지 않고 마스킹합니다. 이렇게 하면 모델이 오류를 내는 행동을 강화하는 것을 방지하면서도, 오류를 인식하고 수정하는 과정에서 올바른 방향으로의 전환을 학습할 수 있습니다. 마스킹 없이 오류 턴까지 학습하면 모델이 오류 패턴 자체를 모방할 위험이 있습니다.

AgentRefine은 이 궤적 학습 아이디어를 에이전트 환경으로 확장한 프레임워크입니다. 에이전트가 다단계 작업을 수행하는 과정에서 도구 호출 실패, 잘못된 계획 실행 등 다양한 오류가 발생합니다. AgentRefine은 이러한 실패 궤적을 단순히 버리지 않고, 실패 후 수정을 시도하는 경로를 인위적으로 구성하거나 수집하여 학습 데이터에 포함합니다. 에이전트가 오류 상황에서 어떻게 복구하는지를 직접 학습함으로써, 실제 환경에서의 강인성을 높입니다.

올바른 궤적 암기의 한계(limitations of correct trajectory memorization)는 증류 기반 접근이 공통으로 직면하는 문제입니다. 교사 모델이 제공하는 정답 경로만 학습하면, 학생 모델은 익숙한 패턴은 잘 재현하지만 새로운 오류 유형이나 전에 본 적 없는 수정 시나리오에서는 취약합니다. 지도 학습의 일반화 한계와 동일한 문제입니다. 이 한계를 극복하려면 다양한 오류 유형과 수정 전략을 포함하는 폭넓은 궤적 데이터가 필요하며, 이를 수동으로 수집하는 것은 비용이 큽니다.

환경 피드백 기반 수정(environment feedback-based correction)은 이 한계를 우회하는 실용적 대안입니다. 코드 실행기, 수식 검증기, 외부 데이터베이스 등 환경에서 자동으로 얻을 수 있는 피드백을 수정 신호로 활용합니다. 예를 들어, 모델이 생성한 코드가 테스트를 통과하지 못하면 오류 메시지가 자동으로 반환됩니다. 이 메시지를 다음 수정 턴의 입력으로 제공하면 모델은 명확한 피드백을 바탕으로 수정을 시도할 수 있습니다. 이 방식은 내재적 자기 교정의 역설을 우회합니다. 수정 신호가 모델 자신의 내부 판단이 아니라 외부 환경에서 오기 때문입니다.

반복적 자기 개선(iterative self-improvement)은 모델이 생성한 데이터로 자신을 재훈련하는 과정을 여러 차례 반복하는 방법입니다. 초기 모델이 문제를 풀고, 그 결과로부터 고품질 샘플을 선별하여 파인튜닝하고, 개선된 모델이 다시 데이터를 생성합니다. 이 루프를 반복하면 이론적으로 성능이 점진적으로 향상됩니다. 실제로는 선별 기준이 너무 엄격하면 데이터 다양성이 줄어들고, 너무 느슨하면 저품질 데이터가 누적되는 문제가 생깁니다. 또한 루프를 오래 반복할수록 출력 분포가 좁아져 새로운 전략을 탐색하는 능력이 감소하는 분포 붕괴(distribution collapse) 현상이 나타날 수 있습니다.

정리하면, 사고 템플릿 증류는 교사 모델의 검토-수정 패턴을 모방 학습으로 전수하는 방법이고, 교차 모델 DPO는 교사-학생 응답 쌍으로 선호 방향을 설정합니다. 오류-수정 궤적 학습은 수정 과정 전체를 학습 데이터로 활용하되, 오류 턴을 마스킹하여 나쁜 행동의 강화를 방지합니다. AgentRefine은 이를 에이전트 환경으로 확장하고, 환경 피드백 기반 수정은 내재적 교정의 역설을 외부 신호로 우회합니다. 반복적 자기 개선은 강력하지만 분포 붕괴에 주의가 필요합니다.

다음 단원인 5.3.1에서는 검증 메커니즘과 보상 모델을 다룹니다. 교정을 수행하는 측면이 아니라, 교정이 잘 이루어졌는지를 평가하는 검증 측면을 살펴봅니다. ORM, PRM, LLM-as-a-Judge 등 다양한 검증 도구들의 원리와 장단점을 비교합니다.
