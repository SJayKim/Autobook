# 12. Phrase Bank & Drills — 표현사전 + 드릴

> 면접 직전·자투리 시간용 빠른 복습 시트. 소리 내어 읽으며 입에 붙이세요.

---

## A. 8개 프로젝트 한 줄 요약 (Headline drill)

> 각 프로젝트를 **한 문장**으로 못 박는 연습. "Tell me about a project"에 곧바로 나오게.

| # | Project | One-liner (외우기) |
|---|---------|--------------------|
| 1 | Reflexion Agent | "I built a self-improving agent that learns from its own failures, cutting repeated retries by about **55 percent**." |
| 2 | Smishing Detection | "I replaced a flaky external API with a fine-tuned model and took real-world F1 from **28 to over 95 percent**." |
| 3 | Agentic RAG | "I built a hybrid-search RAG with a safety layer that lifted Top-5 recall by about **35 percent**." |
| 4 | vLLM Serving | "I moved our text processing to in-house LLM serving, cutting external API cost by **100 percent** and latency by **65**." |
| 5 | LLM Eval Pipeline | "I built a pipeline that evaluated **600,000** production LLM outputs automatically, at a **fifth** of the cost." |
| 6 | Visit Busan Chatbot | "I led the AI on a travel chatbot that's **live in production** on the official Busan tourism site." |
| 7 | Skylife Recommender | "I built a two-stage vector recommender that improved recommendation **diversity** — the foundation for my later search work." |
| 8 | MarketScope AI | "I built and shipped a full-stack AI analytics SaaS **solo**, from planning to production." |

> **드릴법:** 위 8개를 가리고, 프로젝트 이름만 보고 한 문장을 말해보세요. 막힘없이 나올 때까지.

---

## B. 숫자 드릴 (Numbers — 또박또박 말하기)

> 숫자가 이 인터뷰의 무기입니다. 영어로 정확히 발음하는 게 핵심.

| 숫자 | 말하기 | 어느 프로젝트 |
|------|--------|---------------|
| 28.79% → 96.10% | "twenty-eight point seven nine percent up to ninety-six point one percent" | Smishing |
| 95%+ | "over ninety-five percent" | Smishing (실환경) |
| 600,000 | "six hundred thousand" | Eval Pipeline |
| 1/5 | "a fifth" / "one fifth" | Eval Pipeline |
| 100% | "one hundred percent" | vLLM (비용 제거) |
| 65% | "about sixty-five percent" | vLLM (응답 단축) |
| 92% | "around ninety-two percent" | vLLM (요약 수렴) |
| 35% | "about thirty-five percent" | RAG (Recall) |
| 55% | "about fifty-five percent" | Reflexion (재시도 감소) |
| 12% / 150,000 | "twelve percent out of a hundred fifty thousand" | RAG (노이즈 제거) |
| 1,650 | "one thousand six hundred fifty" | MarketScope (상권) |
| 66/66 | "sixty-six out of sixty-six" | MarketScope (E2E) |

> **흔한 실수:** "퍼센트"가 아니라 "per-**CENT**". "thousand"의 th. "fifth"의 -fth 끝소리.

## C. 만능 전환·연결 표현 (Transitions)

| English | 뜻 |
|---------|-----|
| "Let me give you some context first." | 배경부터 말할게요 |
| "The way it works is ..." | 작동 방식은 ~ |
| "Here's the key part." | 여기가 핵심입니다 |
| "To put it simply, ..." | 간단히 말하면 |
| "On top of that, ..." | 게다가 |
| "That said, ..." | 그렇긴 하지만 |
| "Which is why ..." | 그래서 ~인 것이다 |
| "Long story short, ..." | 길게 말할 것 없이 |

---

## D. 비교·결정 표현 (Comparing & deciding)

| English | 뜻 |
|---------|-----|
| "I compared A, B, and C, and went with C because ..." | A·B·C 비교 후 C 선택 |
| "The tradeoff was between X and Y." | X와 Y 사이 트레이드오프 |
| "A was cheaper, but B was more reliable." | A는 저렴, B는 안정적 |
| "It came down to ___." | 결국 ~로 귀결됐다 |
| "In hindsight, I'd ___." | 돌이켜보면 ~하겠다 |

---

## E. 기술 설명 표현 (Explaining tech clearly)

> 면접관이 비전문가일 수도 있습니다. **쉬운 비유 + 정확한 용어**를 함께.

| English | 뜻 / 쓰임 |
|---------|-----------|
| "Think of it as ___." | ~라고 생각하면 됩니다 (비유 진입) |
| "It's basically ___." | 기본적으로 ~입니다 |
| "fine-tuning means taking a pretrained model and ___" | 파인튜닝을 한 줄로 풀어 설명 |
| "RAG is just retrieval plus generation — find the right text, then answer from it." | RAG를 한 문장으로 |
| "an agent is a model that can decide which tools to call." | 에이전트를 한 문장으로 |
| "hybrid search combines keyword and meaning-based search." | 하이브리드 검색 한 문장 |
| "in-process means the model runs inside the same service, not behind another API." | in-process 한 문장 |
| "LLM-as-a-judge means using one LLM to score another LLM's output." | 평가 핵심 용어 |

---

## F. 압박 상황 표현 (Under pressure)

| English | 뜻 |
|---------|-----|
| "That's a great question — let me think for a second." | 좋은 질문이네요, 잠깐 생각할게요 |
| "Just to make sure I answer the right thing — do you mean ___?" | 질문 확인 (시간 확보) |
| "I haven't worked with that directly, but here's how I'd approach it." | 안 해봤지만 이렇게 접근하겠다 |
| "I don't want to guess — but I'd find out by ___." | 추측 대신 알아내는 법 |
| "Can I come back to that in a second?" | 잠시 후 다시 답해도 될까요 |

> **핵심:** 모르면 **지어내지 말 것.** "여기까진 안다 + 이렇게 접근한다"가 정답입니다.

---

## G. 마무리 표현 (Closing an answer)

| English | 뜻 |
|---------|-----|
| "So in the end, ___." | 결국 ~ |
| "The takeaway for me was ___." | 제 교훈은 ~ |
| "That's the project in a nutshell." | 요약하면 이게 다입니다 |
| "Happy to go deeper on any part." | 어느 부분이든 더 깊이 들어갈 수 있습니다 |
