# 01. Speaking Framework — 영어로 프로젝트를 말하는 법

> 이 파일은 모든 프로젝트 정답지의 "뼈대"입니다. 먼저 이 구조를 손에 익히면,
> 어떤 프로젝트를 묻든 같은 틀로 안정적으로 말할 수 있습니다.
> (정답 문장은 영어, 학습용 설명은 한국어로 표기합니다.)

---

## A. 왜 "구조"가 먼저인가

영어 면접에서 무너지는 이유는 단어가 부족해서가 아니라 **순서를 즉석에서 만들려다** 말이 꼬이기 때문입니다. 모국어가 아니면 즉흥 구성이 어렵습니다. 그래서 우리는 **고정된 5-비트 뼈대**를 외워둡니다. 뼈대가 고정되면, 머리는 "내용"에만 집중하면 되고, 말이 끊겨도 다음 비트로 자연스럽게 넘어갑니다.

핵심 원칙 3가지:
1. **구조 > 유창함.** 더듬어도 구조가 있으면 유능해 보입니다.
2. **숫자로 시작하고 숫자로 끝낸다.** 결과 지표(F1 96%, 60만 건, 비용 100% 제거)가 곧 신뢰입니다.
3. **결정과 이유를 말한다.** "무엇을 했다"보다 "왜 그렇게 정했다(트레이드오프)"가 시니어 신호입니다.

---

## B. The 5-Beat Project Story (핵심 뼈대)

모든 프로젝트 답변은 이 5개 비트 순서로 말합니다. 각 비트마다 **고정 시작 표현(trigger phrase)**을 외워두면 즉석에서도 막히지 않습니다.

| # | Beat | 한국어 | 이 비트의 시작 표현 (trigger) |
|---|------|--------|------------------------------|
| 1 | **Headline** | 한 줄 정의 + 최고의 결과 | "In short, I built **X** that **[result]**." |
| 2 | **Context & Problem** | 이전 상태와 진짜 문제 | "Before this, ... The real problem was ..." |
| 3 | **Approach & Key Decision** | 한 일 + 자랑할 결정 1개 + **이유** | "So I ... The key decision was to ... I chose A over B because ..." |
| 4 | **The Hard Part** | 가장 어려웠던 점 + 해결 | "The tricky part was ... I solved it by ..." |
| 5 | **Result & Insight** | 정량 결과 + 배운 점 | "As a result, ... What I took away was ..." |

```
[1 Headline] ─ "뭘 만들었고 결과가 뭐다"
      │
[2 Problem] ─ "왜 필요했나 (전 상태 + 고통)"
      │
[3 Decision] ─ "어떻게 했나 + 핵심 결정 + 왜 (트레이드오프)"
      │
[4 Hard Part] ─ "가장 어려웠던 1가지 + 어떻게 뚫었나"
      │
[5 Result] ─ "숫자 결과 + 한 줄 교훈"
```

> **암기 팁:** "**H-P-D-H-R**" (Headline, Problem, Decision, Hard part, Result).
> 면접 중에는 손가락으로 5개를 세며 속으로 체크하세요.

---

## C. 길이별 버전은 이 뼈대에서 나온다

세 가지 버전은 다른 내용이 아니라, **같은 5-비트를 얼마나 펼치느냐**의 차이입니다.

| 버전 | 길이 | 구성 | 언제 쓰나 |
|------|------|------|-----------|
| 🟢 **Version A — Elevator** | 25~35초 | Beat 1 + Beat 5 (헤드라인 + 결과) | "Tell me about a project." 첫 답, 또는 시간 없을 때 |
| 🟡 **Version B — Standard** | 60~90초 | 5비트 각 1~2문장 | 기본값. 대부분의 "이 프로젝트 설명해줘"에 |
| 🔴 **Version C — Deep Dive** | 2~3분 | 5비트 확장 + Beat 3·4에 기술 디테일 1개 | "더 자세히", "어떻게 구현했나" 후속 질문 |

전략: **항상 Version B로 시작** → 면접관이 더 파면 Version C의 디테일을 꺼냅니다. 먼저 다 쏟지 말고, 후속 질문용 카드를 남겨두세요.

---

## D. 즉석에서 생각하는 법 (Thinking on your feet)

영어가 막힐 때를 위한 안전장치입니다.

**1) 시간 버는 표현 (Buying time) — 침묵 대신 이걸 말하세요**
- "That's a good question. Let me think for a second."
- "So, the way I'd describe it is ..."
- "There were basically three parts to this ..." (← 숫자를 먼저 던지면 뇌가 구조를 잡습니다)

**2) 질문 되짚기 (Rephrase) — 이해 확인 + 생각할 시간 확보**
- "If I understand correctly, you're asking how I handled ___, right?"
- "Just to make sure I answer the right thing — do you mean ___ or ___?"

**3) 소리 내어 구조 잡기 (Signpost out loud)**
- "Let me start with the problem, then how I solved it, then the result."
- "There are three reasons. First, ... Second, ... And finally, ..."

**4) 모르는 걸 우아하게 (Graceful "I don't know")**
- "I haven't worked with that directly, but my understanding is ___. How I'd approach it is ___."
- "I don't want to guess — but here's how I'd find out: ___."
- 절대 지어내지 마세요. "여기까진 안다 + 이렇게 접근하겠다"가 정답입니다.

## E. Phrase Bank (만능 표현 — 외워두면 어디든 씀)

> 프로젝트가 바뀌어도 그대로 재사용하는 표현들입니다. 소리 내어 10번씩 읽어 입에 붙이세요.

**① Openers (시작)**
| English | 뜻 / 쓰임 |
|---------|-----------|
| "In short, I built / designed / led ..." | 요점부터: 헤드라인에 |
| "The project was about ___, basically." | 가볍게 주제 던지기 |
| "At a high level, the goal was to ___." | 큰 그림부터 |

**② Sequencing / Transitions (순서·전환)**
| English | 뜻 |
|---------|-----|
| "First ... then ... after that ... finally ..." | 1-2-3-4 단계 |
| "The way it works is ..." | 작동 방식 설명 진입 |
| "On top of that, ..." | 게다가 (추가) |
| "That said, ..." | 그렇긴 한데 (반전) |
| "Which brings me to ..." | 자연스럽게 다음 포인트로 |

**③ Cause & Effect (인과)**
| English | 뜻 |
|---------|-----|
| "The problem was that ___, which meant ___." | 문제 → 그래서 ~됨 |
| "Because of ___, we ended up with ___." | ~때문에 ~결과 |
| "This led to ___." | 이것이 ~로 이어짐 |
| "As a result, ___." | 그 결과 |

**④ Quantifying results (결과 정량화 — 가장 중요)**
| English | 뜻 |
|---------|-----|
| "We took the F1 score from 28% up to 96%." | 28%→96%로 끌어올림 |
| "That cut costs by 100% / by a fifth." | 비용을 100% / 1/5로 절감 |
| "It improved recall by about 35%." | 약 35% 향상 |
| "We processed around 600,000 items with zero manual review." | 60만 건을 사람 개입 0으로 |
| "Response time dropped by roughly 65%." | 응답 65% 단축 |

**⑤ Decisions & Tradeoffs (결정·트레이드오프 — 시니어 신호)**
| English | 뜻 |
|---------|-----|
| "The key decision was to ___." | 핵심 결정은 ~ |
| "I chose A over B because ___." | B 대신 A를 고른 이유 |
| "There was a tradeoff between ___ and ___." | ~와 ~ 사이 트레이드오프 |
| "I went with the simpler option, since ___." | 더 단순한 쪽을 택함 |
| "In hindsight, I'd ___." | 돌이켜보면 ~하겠다 |

**⑥ Hedging / Uncertainty (조심스럽게)**
| English | 뜻 |
|---------|-----|
| "Roughly / around / about ___." | 대략 |
| "If I remember correctly, ___." | 내 기억이 맞다면 |
| "It depends on ___." | ~에 따라 다름 |

**⑦ Emphasis (강조)**
| English | 뜻 |
|---------|-----|
| "The really important part here is ___." | 진짜 핵심은 |
| "What made this hard was ___." | 어렵게 만든 건 |
| "This is the part I'm most proud of." | 가장 자랑스러운 부분 |

**⑧ Wrapping up (마무리)**
| English | 뜻 |
|---------|-----|
| "So in the end, ___." | 결국 |
| "The takeaway for me was ___." | 내 교훈은 |
| "That's the project in a nutshell." | 요약하면 이게 다입니다 |

## F. 전달·발음 (Delivery for Korean speakers)

**1) 속도와 멈춤 (Pace & pause)**
- 빨리 말하면 발음이 뭉개집니다. **천천히 + 비트 사이 0.5초 멈춤**이 더 유능해 보입니다.
- 멈춤은 약점이 아닙니다. 생각하는 사람처럼 보입니다.

**2) 기술 용어 강세 (Word stress) — 강세 위치가 틀리면 안 들립니다**
| 단어 | 강세 (대문자가 강) |
|------|---------------------|
| architecture | AR-ki-tek-cher |
| asynchronous | ay-SIN-kruh-nus |
| deploy / deployment | di-PLOY / di-PLOY-ment |
| latency | LAY-ten-see |
| parameter | puh-RAM-uh-ter |
| inference | IN-fer-ence |
| schema | SKEE-muh |
| query | KWEER-ee |
| retrieval | ri-TREE-vul |
| embedding | em-BED-ding |
| orchestration | or-kuh-STRAY-shun |
| deterministic | di-ter-MIN-is-tic |
| threshold | THRESH-hold |
| PostgreSQL | "POST-gres" (또는 post-gres-Q-L) |

**3) 한국인이 자주 놓치는 소리**
- **F vs P:** "F1" = "**ef** one" (에프), "performance"의 f. (P로 새지 않기)
- **R vs L:** "retrieval", "parallel", "reranker".
- **TH:** "threshold", "throughput", "method", "month". 혀를 살짝 물기.
- **V vs B:** "**v**ector", "Mil**v**us", "**vL**LM", "ser**v**ing". (B로 새지 않기)
- **끝소리 자음:** "deployed", "cached", "fixed" 의 -d/-t 를 끝까지.

**4) 숫자 읽기 (자주 씀)**
- 28.79% → "twenty-eight point seven nine percent"
- 96.10% → "ninety-six point one percent"
- 600,000 → "six hundred thousand"
- 1/5 → "a fifth" / "one fifth"
- ~35% → "about thirty-five percent"
- v0.4.0 → "version zero point four point zero"

**5) 필러(filler) 관리**
- "uh, um"을 줄이고, 대신 **의미 있는 필러**로 대체: "so", "basically", "right", "let me see".
- 막히면 "uh..." 대신 **그냥 멈추세요.** 침묵이 "음..."보다 낫습니다.

## G. 연습법 & 면접 직전 루틴

**연습법 (집에서)**
1. **Shadowing:** Version B를 들으며(또는 읽으며) 0.5초 뒤따라 말하기.
2. **Record & compare:** 핸드폰으로 녹음 → 정답지와 비교 → 강세·끝소리 체크.
3. **무작위 드릴:** 8개 프로젝트 카드를 섞어 뽑고, 뽑힌 것의 Version A를 즉석에서.
4. **꼬리질문 스파링:** 각 정답지 3번 섹션(Follow-up)을 소리 내어.

**면접 직전 5분 루틴**
- Headline 8개(프로젝트별 한 줄)만 빠르게 훑기.
- 핵심 숫자 5개 입으로: "28 to 96", "600k", "100 percent", "65 percent", "35 percent".
- 심호흡. 천천히 말하기로 마음먹기.
