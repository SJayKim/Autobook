# 2.1.2 AGENTS.md 작성 패턴

같은 레포에서 Claude Code와 Codex CLI를 함께 쓰면 두 도구가 서로 다른 지시문 파일을 읽는다는 점에서 출발하는 문제가 생깁니다. 2.1.1에서 다룬 CLAUDE.md는 Claude Code 전용이며 Codex CLI는 이 파일을 자동으로 열어 보지 않습니다. Codex CLI가 같은 위상으로 자동 로딩하는 파일은 AGENTS.md이며, 이 단원에서는 어떤 위치에 두는지, 어떤 섹션 구성을 따르는지, CLAUDE.md와 톤이 어떻게 다른지, 두 파일을 어떻게 동기화하는지를 차례로 정리합니다.

AGENTS.md의 자동 로딩 규칙부터 풀어 두겠습니다. Codex CLI는 세션을 시작하면 세 위치를 순서대로 확인합니다. 첫째는 사용자 홈의 ~/.codex/AGENTS.md로, 사용자가 어느 프로젝트에서 작업하든 항상 따라붙는 공통 규칙을 담습니다. 둘째는 현재 작업 디렉토리에서 위로 올라가며 만나는 첫 AGENTS.md로, 대개 프로젝트 루트에 둡니다. 셋째는 현재 작업 디렉토리의 하위에 있는 AGENTS.md로, 모노레포의 패키지별 지시문을 격리할 때 씁니다. 세 파일이 동시에 존재하면 위에서 아래로 누적되어 합쳐지며, 더 구체적인 위치의 지시가 더 넓은 위치의 지시를 보강합니다.

AGENTS.md의 표준 섹션은 Build, Test, Run, Style 네 가지로 굳어져 있습니다. **Build**는 의존성 설치와 산출물을 만드는 명령을, **Test**는 단위·통합 테스트를 돌리는 명령을, **Run**은 실제 애플리케이션이나 스크립트를 실행하는 명령을, **Style**은 코드 컨벤션·포맷터·린터의 기준을 적습니다. Codex CLI는 작업 단위를 받아 한 번에 처리하는 모드가 기본이므로, 이 네 섹션이 곧 작업이 끝날 때마다 모델이 스스로 돌릴 명령의 카탈로그가 됩니다.

```markdown
# AGENTS.md

## Build
- 의존성 설치: `pnpm install`
- 산출물 빌드: `pnpm build`

## Test
- 단위 테스트: `pnpm test`
- 변경 파일만: `pnpm test -- --changed`

## Run
- 개발 서버: `pnpm dev`
- 마이그레이션: `pnpm db:migrate`

## Style
- 포맷터: Prettier (`pnpm format`)
- 린터: ESLint (`pnpm lint`)
- TypeScript strict 모드 유지, any 금지
```

이 예시에서 가장 중요한 점은 각 항목이 **실행 가능한 한 줄 명령** 형태로 적혀 있다는 것입니다. Codex CLI는 자연어 설명보다 실행 가능한 명령을 우선하여 인식하므로, '테스트를 돌리려면 의존성을 먼저 설치한 뒤 pnpm test를 실행한다' 같은 서술형 문장보다 명령 두 줄을 그대로 두는 편이 더 유리합니다. 명령에 인자가 필요하면 백틱 안에 그대로 노출하고, 주의 사항은 명령 다음 줄에 짧게 덧붙이는 식으로 작성합니다.

CLAUDE.md와 톤 차이를 함께 짚어 두는 것이 좋습니다. CLAUDE.md는 대화형 루프를 통제하는 지시문이 많아 '어떤 상황에서 무엇을 우선하라'는 행동 규범을 자연어 문단으로 담는 비중이 큽니다. AGENTS.md는 작업 단위 실행 모드에 맞춰 '어떤 명령을 어떤 순서로 부르라'는 작업 명세에 가깝습니다. 둘을 같은 톤으로 쓰면 한쪽에서 불필요한 문장이 생기거나 다른 쪽에서 명령이 빠집니다.

```
도구          파일         주요 톤           대표 섹션
Claude Code   CLAUDE.md    대화형 지시       Conventions, Gotchas, Self-reflection
Codex CLI     AGENTS.md    작업 명세         Build, Test, Run, Style
```

이제 두 파일을 한 레포에서 어떻게 동기화하는지를 정합니다. 같은 정보를 두 파일에 두 번 복사하면 한쪽만 갱신되어 어긋나는 사고가 반복됩니다. 가장 안전한 방법은 공통 정보를 한 곳에 한 번만 쓰고 다른 쪽에서 참조하도록 만드는 것입니다. Claude Code는 CLAUDE.md에서 `@경로` 형태의 import 문법을 지원하므로, 명령 카탈로그를 AGENTS.md에 두고 CLAUDE.md에서 `@AGENTS.md`로 끌어다 쓰는 구성이 깔끔합니다. 반대로 행동 규범처럼 Claude Code에만 필요한 내용은 CLAUDE.md에만 두고 AGENTS.md에 옮기지 않습니다.

동기화의 두 번째 축은 갱신 시점을 한 군데로 모으는 것입니다. 신규 명령이 생겼거나 테스트 러너가 바뀌었다면 AGENTS.md를 먼저 고치고, 그 변경이 Claude Code의 행동 규범과 충돌하지 않는지 CLAUDE.md를 점검합니다. PR 템플릿에 'AGENTS.md/CLAUDE.md 동기화 확인'을 한 줄 체크박스로 넣어 두면 빠지지 않습니다. 모노레포라면 패키지 하위의 AGENTS.md가 루트의 명령을 일부 덮어쓰는 형태를 활용해 중복을 줄입니다.

정리하면, AGENTS.md는 Codex CLI가 ~/.codex/, 프로젝트 루트, 하위 디렉토리 순서로 자동 로딩하는 작업 명세 파일이며 Build·Test·Run·Style 네 섹션에 실행 가능한 한 줄 명령을 모아 두는 것이 핵심입니다. CLAUDE.md는 대화형 행동 규범을 담고 AGENTS.md는 작업 명세를 담으므로, 같은 레포에서는 공통 정보를 한 곳에 쓰고 다른 쪽에서 import 또는 참조로 끌어다 쓰는 방식으로 동기화합니다.

다음 단원인 2.2.1에서는 Claude Code의 동작을 세밀하게 조정하는 settings.json의 계층과 핵심 필드를 살펴봅니다.

이 단원을 마치면 AGENTS.md를 작성해 Codex CLI 세션에 적용하고, CLAUDE.md와의 차이를 한 문단으로 설명할 수 있게 됩니다.
