# 1.2.1 Claude Code 설치와 첫 세션

도구를 처음 켤 때 가장 흔한 실수는 권한 모드를 모른 채 바로 명령을 시키는 것입니다. 첫 세션을 안전한 모드로 띄우면, 모델이 무엇을 시도하는지 보면서 정책을 다듬을 시간이 생깁니다. 이 단원에서는 Claude Code의 설치와 인증, 첫 세션 진입, 안전 모드 사용까지를 한 번에 정리합니다.

Claude Code는 npm 전역 패키지로 배포됩니다. 다음 명령으로 설치합니다.

```bash
npm install -g @anthropic-ai/claude-code
```

운영 체제별로 주의할 점이 조금 있습니다. macOS와 Linux에서는 보통 문제가 없지만, npm 전역 디렉터리에 쓰기 권한이 없으면 권한 오류가 납니다. 이때 `sudo`로 해결하기보다는 npm prefix를 사용자 디렉터리로 옮기는 쪽을 권장합니다. Windows에서는 `cmd`보다 PowerShell이나 Git Bash가 호환이 좋습니다. 한글 출력이 깨지면 터미널 코드 페이지를 UTF-8로 맞춥니다. 설치가 끝나면 `claude --version`이 응답하는지 확인합니다.

다음은 **최초 인증**입니다. 설치 직후 `claude /login`을 실행하면 브라우저가 열리고, 사용 중인 Anthropic 계정으로 로그인합니다. 계정 인증 대신 API 키를 쓰는 환경이라면 환경 변수 `ANTHROPIC_API_KEY`로 키를 주입할 수 있습니다. 인증이 끝나면 토큰이 사용자 홈 디렉터리 아래 `~/.claude/` 폴더에 저장됩니다. 이 폴더에는 토큰 외에 사용자 전역 설정(`~/.claude/settings.json`)과 자동 메모리도 함께 보관됩니다. 다른 컴퓨터에서 같은 계정을 쓰려면 `claude /login`을 다시 수행하면 되며, 토큰 파일을 직접 복사하지 않습니다.

인증을 마치면 작업할 프로젝트 폴더로 이동해 첫 세션을 띄웁니다.

```bash
cd ~/code/my-project
claude
```

`claude` 한 줄이 인터랙티브 세션을 시작합니다. 첫 화면에 모델 이름과 작업 디렉터리, 그리고 활성화된 MCP 서버·hook이 짧게 표시됩니다. 이 화면을 무심코 넘기지 말고, 5가지를 확인하는 습관을 들이면 도구를 통제하기 쉬워집니다. 첫째는 **모델**입니다. 표시된 모델이 작업 성격에 맞는지, 비용이 큰 모델이 기본값으로 잡혀 있지 않은지 봅니다. 둘째는 **작업 디렉터리**입니다. 의도한 폴더에서 세션이 시작됐는지, 상위 폴더가 자동 포함되지는 않았는지 확인합니다. 셋째는 **MCP 서버** 목록입니다. 의도하지 않은 외부 서버가 켜져 있으면 권한 범위가 넓어집니다. 넷째는 **hook** 목록입니다. 어떤 이벤트에 어떤 스크립트가 걸려 있는지, 새 프로젝트라면 비어 있는지를 확인합니다. 다섯째는 **권한** 설정입니다. 현재 적용된 permissions 정책을 짧게 출력해 두면 첫 작업 중 예기치 못한 자동 허용이 줄어듭니다.

기본 권한 모드는 **default**입니다. 이 모드는 settings.json의 permissions에 따라 매처가 일치하는 호출만 자동 허용하고, 매치되지 않은 호출은 사용자에게 물어 봅니다. 권한 모델을 처음 접하면 default 그대로 두는 편이 안전합니다.

작업 전 계획만 합의하고 싶다면 **plan 모드**로 진입합니다. plan 모드는 파일을 쓰거나 명령을 실행하는 도구를 막은 채, 모델이 무엇을 어떻게 바꿀지 계획만 출력하도록 제한합니다. 사용자가 계획을 검토하고 `ExitPlanMode`를 통해 명시적으로 동의하면 실제 변경 단계로 넘어갑니다. default가 도구를 쓰며 즉시 작업하는 모드라면, plan은 변경 직전에 한 번 멈추는 안전망입니다.

두 모드의 차이를 짧게 정리하면 다음과 같습니다.

| 항목 | default 모드 | plan 모드 |
| --- | --- | --- |
| 파일 변경 도구 | permissions에 따라 허용 | 차단 |
| 명령 실행 | permissions에 따라 허용 | 차단 |
| 계획 출력 | 선택적 | 기본 |
| 종료 절차 | 자동 | `ExitPlanMode` 후 동의 |
| 권장 시점 | 작은 작업, 신뢰된 정책 | 큰 변경, 새 레포 |

첫 세션을 끄려면 `/exit` 또는 Ctrl+D를 누릅니다. 다음 세션은 같은 폴더에서 `claude`를 다시 실행하면 됩니다. 이전 대화를 이어 가고 싶을 때는 `claude --resume`이 도움이 됩니다.

정리하면, npm 전역 설치 후 `claude /login`으로 토큰을 받고, 프로젝트 폴더에서 `claude`로 첫 세션을 띄우며, default 모드와 plan 모드 중 작업 위험도에 맞춰 한 쪽을 고릅니다. 진입 직후 모델·작업 디렉터리·MCP·hook·권한 5가지를 점검하는 습관이 이후 모든 자동화의 출발점이 됩니다.

다음 단원인 1.2.2에서는 Codex CLI를 설치하고 sandbox 모드를 켠 채 첫 작업을 띄우는 절차를 다룹니다.
