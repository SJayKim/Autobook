# 4.1.1 Hook 6종 개요와 동작 순서

3.2.3에서 위험 명령 차단을 다루며 PreToolUse hook을 잠깐 언급했습니다. hook은 Claude Code가 세션 중 특정 순간에 외부 스크립트를 호출해 주는 기능입니다. 사용자가 메시지를 보내는 순간, 모델이 도구를 부르려는 순간, 도구가 결과를 돌려준 순간, 세션이 끝나는 순간처럼 미리 정해진 시점마다 hook이 끼어들 수 있습니다. 이 자리에서 정책을 강제하거나, 자동 포맷·테스트를 돌리거나, 환경을 점검하는 일을 한 곳에 묶을 수 있습니다.

hook을 잘 쓰려면 어떤 순간에 어떤 hook이 동작하고, 그때 어떤 데이터가 전달되는지 그림을 먼저 잡아야 합니다. 한 세션은 사용자 입력에서 시작해 도구 호출과 결과 보고를 여러 번 반복하다가 종료에 이릅니다. 그 흐름을 따라가며 hook이 호출되는 지점을 짚어 봅니다.

```text
세션 시작 ─▶ SessionStart hook
              │
              ▼
사용자 입력 ─▶ UserPromptSubmit hook ─▶ 모델 추론
                                       │
                                       ▼
                              도구 호출 시도 ─▶ PreToolUse hook ─▶ 도구 실행 ─▶ PostToolUse hook
                                       │
                                       ▼
                              자동 압축 임박 ─▶ PreCompact hook
                                       │
                                       ▼
모델 응답 ─▶ Stop hook (또는 SubagentStop)
              │
              ▼
알림 발생 ─▶ Notification hook
              │
              ▼
세션 종료 ─▶ SessionEnd hook
```

이 흐름에는 자주 쓰는 핵심 hook 여섯 가지가 있습니다. **PreToolUse**, **PostToolUse**, **UserPromptSubmit**, **Stop**, **SessionStart**, **PreCompact**입니다. 이 외에 보조로 동작하는 **Notification**, **SubagentStop**, **SessionEnd**까지 합쳐 총 아홉 가지가 정의되어 있습니다. 표로 한 번 정리합니다.

| Hook 이벤트 | 트리거 시점 | 대표 용도 |
| --- | --- | --- |
| PreToolUse | 모델이 도구를 부르기 직전 | 위험 명령 차단, 인자 검증 |
| PostToolUse | 도구 실행이 끝난 직후 | 포맷·린트·테스트 자동 실행 |
| UserPromptSubmit | 사용자 메시지가 모델에 전달되기 직전 | 정책 문구·환경 정보 주입 |
| Notification | Claude Code가 알림을 띄울 때 | 외부 알림 시스템 연동 |
| Stop | 모델 응답이 끝나려는 순간 | 응답 후 정리, 추가 검사 |
| SubagentStop | subagent 응답이 끝나려는 순간 | 위임 결과 검토 |
| SessionStart | 세션이 시작될 때 | git 상태·환경 점검 |
| SessionEnd | 세션이 종료될 때 | 작업 로그 보존 |
| PreCompact | 자동 압축 직전 | 중요 컨텍스트 외부 보존 |

각 hook은 표준 입력으로 JSON을 받습니다. 공통 필드는 `session_id`, `transcript_path`, `hook_event_name`이고, 이벤트마다 추가 필드가 붙습니다. PreToolUse와 PostToolUse는 `tool_name`과 `tool_input`을 함께 받아 호출하려는 도구와 그 인자를 검사할 수 있게 합니다. UserPromptSubmit은 `prompt` 필드로 사용자가 막 보낸 메시지를 받습니다. SessionStart는 새 세션인지 resume인지 구분하는 정보를 함께 받아, 시작과 재개를 다르게 처리할 수 있습니다.

hook 스크립트의 종료 코드는 세션 흐름을 직접 제어합니다. **0은 정상 종료**, **2는 차단**, 그 외 값은 일반 오류로 다뤄집니다. PreToolUse에서 종료 코드 2로 끝나면 그 도구 호출은 막히고, stderr로 출력한 메시지가 모델에게 사유로 전달됩니다. 모델은 차단 사유를 보고 다음 시도를 다르게 만들 수 있습니다. PostToolUse에서 종료 코드 2가 되면 도구 결과가 모델에게 전달되지 않고 차단 사유로 대체되어, 후속 추론이 잘못된 결과를 누적하지 않게 막아 줍니다.

hook은 `settings.json`의 `hooks` 객체에 등록합니다. 이벤트 이름을 키로 두고, 그 아래에 매처와 명령 목록을 배열로 적습니다. 매처는 PreToolUse와 PostToolUse에서 `tool_name`을 기준으로 특정 도구만 잡을 수 있게 해 줍니다.

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          { "type": "command", "command": "python .claude/hooks/block_dangerous.py" }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          { "type": "command", "command": "bash .claude/hooks/format.sh" }
        ]
      }
    ],
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          { "type": "command", "command": "bash .claude/hooks/session_info.sh" }
        ]
      }
    ]
  }
}
```

위 설정의 PreToolUse 블록은 `Bash` 도구 호출에만 매치되어 `block_dangerous.py`를 부릅니다. PostToolUse 블록의 `Edit|Write`는 정규식으로 두 도구 모두를 잡습니다. SessionStart 블록의 빈 매처는 매처 없이 모든 SessionStart에 동작합니다.

여러 hook이 같은 이벤트에 등록되면, 배열에 적힌 순서대로 차례로 호출됩니다. 한 hook이 종료 코드 2로 차단을 결정하면 뒤 hook은 호출되지 않습니다. 같은 이벤트의 같은 매처에 hook 두 개를 등록해 두면 둘 다 돌게 되므로, 가벼운 검사 hook과 무거운 검사 hook을 분리해 실행 비용을 통제할 수 있습니다.

디버깅에 유용한 신호는 stderr와 환경 변수입니다. hook 스크립트가 stderr로 출력한 내용은 Claude Code가 사용자나 모델에게 사유로 보여 줍니다. 따라서 hook이 차단을 결정한 이유나 진행 상황을 stderr에 적어 두면, 어떤 패턴이 매치되었는지 알기 쉽습니다. 또한 hook 실행 환경에는 `CLAUDE_PROJECT_DIR` 같은 환경 변수가 들어와 있어, 작업 디렉토리 기준 경로를 안전하게 계산할 수 있습니다.

마지막으로 hook을 만들 때는 한 가지 원칙을 지킵니다. **빠르게, 결정적으로** 끝납니다. hook은 모든 도구 호출과 메시지 흐름에 끼어들기 때문에, 한 hook이 느려지면 세션 전체가 느려집니다. 외부 API를 부르거나 큰 명령을 돌려야 한다면, 결과를 캐시에 보관하고 다음번에는 캐시만 보는 식으로 hook 자체는 짧게 끝나도록 설계합니다.

정리하면, Claude Code hook은 세션 흐름의 정해진 시점마다 끼어들 수 있는 외부 스크립트 자리이고, 핵심 PreToolUse·PostToolUse·UserPromptSubmit·Stop·SessionStart·PreCompact 여섯에 보조 Notification·SubagentStop·SessionEnd를 합쳐 아홉 가지가 정의되어 있습니다. `settings.json`의 `hooks` 객체에 이벤트별로 매처와 명령을 등록하고, 종료 코드 0과 2로 흐름을 통제합니다.

다음 단원인 4.1.2에서는 가장 자주 쓰이는 PreToolUse hook을 사용해 위험 작업을 실제로 차단하는 스크립트를 작성해 봅니다.
