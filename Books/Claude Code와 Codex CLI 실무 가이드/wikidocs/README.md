# Claude Code와 Codex CLI 실무 가이드: 하네스 엔지니어링 쿡북

AI 코딩 에이전트인 Claude Code와 Codex CLI를 실무에서 안전하고 효율적으로 운용하기 위한 설정·확장·운영 레시피 모음입니다. 하네스 엔지니어링의 다섯 축(외부 환경, 통제, 피드백, 오케스트레이션, 신뢰성)을 두 도구의 실제 설정 파일과 명령으로 풀어 냅니다.

## 구성

- **Phase 1. 시작하기** — Claude Code와 Codex CLI의 하네스 구조 차이를 설명하고, 두 도구를 각각 설치한 뒤 첫 세션을 안전한 모드로 실행할 수 있다.
- **Phase 2. 프로젝트 컨텍스트 설계** — 프로젝트 지시문 파일과 설정 파일을 두 도구의 관례에 맞춰 작성하고, 어떤 정보를 어느 파일에 두어야 하는지 판단할 수 있다.
- **Phase 3. 권한과 샌드박스** — 두 도구의 권한 모델과 샌드박스 모드를 설명하고, 프로젝트 위험도에 맞춘 최소 권한 설정을 작성할 수 있다.
- **Phase 4. Hook과 자동화** — Claude Code의 6가지 hook 종류를 이해하고, 실무에서 자주 쓰이는 정책 주입·자동 포맷·환경 점검 hook을 작성할 수 있다.
- **Phase 5. 도구 확장과 재사용** — MCP 서버를 두 도구에 각각 연결하고, 슬래시 커맨드·스킬로 자주 쓰는 작업을 재사용 가능한 형태로 묶을 수 있다.
- **Phase 6. 에이전트 오케스트레이션** — Claude Code subagent와 Plan mode, Codex CLI의 작업 분해·profile을 활용해 큰 작업을 안전하게 나누어 실행할 수 있다.
- **Phase 7. 피드백 루프** — 테스트·린트·CI 결과를 모델에게 다시 흘려보내는 피드백 루프를 두 도구에서 동일하게 동작하도록 구성할 수 있다.
- **Phase 8. 운영과 보안** — 팀에서 두 도구를 운영할 때 필요한 비용 관리·공유 설정·보안 점검·트러블슈팅을 체크리스트로 수행할 수 있다.
