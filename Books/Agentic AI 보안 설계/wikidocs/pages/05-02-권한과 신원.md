# 5.2 권한과 신원

이 섹션의 토픽과 학습 목표는 다음과 같습니다.

- **5.2.1 Capability 기반 권한: 읽기와 쓰기는 다른 권한** — Capability 기반 권한이 도구별·액션별로 스코프를 세분화하고 읽기 전용 기본값에 명시적 승격을 두는 설계임을 설명할 수 있다
- **5.2.2 단기(JIT) 크리덴셜과 시크릿 볼트** — 장기 API 키를 심는 대신 작업 시점에 최소 범위 토큰을 발급하는 JIT 크리덴셜과 시크릿 볼트가 ASI03의 해법인 이유를 설명할 수 있다
- **5.2.3 에이전트 신원과 행위 귀속** — 사람 계정 위임 대신 에이전트 고유 신원을 발급해 행위 귀속을 보장하는 설계를 설명할 수 있다
