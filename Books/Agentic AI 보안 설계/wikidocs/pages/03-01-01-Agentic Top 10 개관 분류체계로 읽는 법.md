# 3.1.1 Agentic Top 10 개관: 분류체계로 읽는 법

앞 Phase까지 오면서 에이전트가 왜 위험한지에 대한 조각들을 하나씩 모았습니다. 2.2.1에서 다룬 과도한 에이전시가 그 출발점이었고, 2.2.2에서는 프롬프트에 하지 말라고 적는 모델 계층 방어만으로는 부족하다는 결론까지 왔습니다. 그런데 조각을 아무리 모아도, 그것들이 서로 어떻게 다르고 어디에 놓이는지를 알려 주는 지도가 없으면 막상 내 시스템을 점검할 때 무엇부터 봐야 할지 막막합니다. 위협을 하나씩 아는 것과 빠짐없이 훑는 것은 다른 일입니다.

이 막막함을 풀어 주는 도구가 분류체계입니다. 여기서 **분류체계(taxonomy)**란 흩어진 대상을 정해진 기준에 따라 칸으로 나눠 정리한 목록을 말합니다. 생물을 계와 문, 강으로 나누는 분류표를 떠올리면 됩니다. 위협을 분류체계로 정리하면 세 가지 이점이 생깁니다. 첫째, 빠뜨림을 줄입니다. 열 개의 칸이 있으면 열 개를 순서대로 확인하게 되므로, 아는 위협만 챙기고 모르는 위협은 지나치는 실수가 줄어듭니다. 둘째, 공통 언어가 생깁니다. 같은 코드로 같은 위협을 부르면, 서로 다른 팀이나 문서가 같은 문제를 이야기하고 있는지 헷갈리지 않습니다. 셋째, 대응을 붙일 자리가 생깁니다. 위협마다 칸이 정해져 있으면 그 칸에 방어책과 점검 항목을 하나씩 매달 수 있습니다.

에이전트 보안을 이렇게 칸으로 정리해 놓은 것이 **OWASP Top 10 for Agentic Applications**입니다. 이름을 풀면, 에이전트로 동작하는 애플리케이션에서 특히 자주 나타나고 위험한 위협 열 가지를 골라 정리한 목록이라는 뜻입니다. 2026년을 대상으로 정리된 판이 2025년 12월에 공개되었고, 이 책은 이 목록을 에이전트 보안 공부의 중심 축으로 삼습니다. 중심 축이라는 말은, 앞으로 배울 개별 공격과 방어 기법을 모두 이 열 개의 칸 어딘가에 걸어 두고 익힌다는 뜻입니다.

이 목록의 위치를 이해하려면 그 이전 목록과의 관계를 짚어야 합니다. 2.1에서 훑은 **OWASP Top 10 for LLM Applications**는 LLM을 쓰는 애플리케이션 일반의 위협을 정리한 목록이었습니다. 프롬프트 인젝션, 민감정보 유출, 부적절한 출력 처리처럼, 대체로 텍스트를 넣으면 텍스트가 나오는 시스템에서 생기는 문제들입니다. 그중 LLM06 과도한 에이전시가 에이전트 쪽으로 넘어가는 다리였고, 이 부분은 2.2.1에서 이미 자세히 다뤘습니다.

Agentic Top 10은 그 LLM 목록을 대체하는 것이 아니라 그 위에 얹히는 목록입니다. 둘의 차이는 대상이 무엇을 하느냐에서 갈립니다. LLM 목록은 대체로 말을 만들어 내는 시스템을 다룹니다. 반면 Agentic 목록은 그 말을 근거로 스스로 도구를 부르고, 여러 단계를 이어 실행하며, 기억을 쌓고, 다른 에이전트와 주고받는 시스템을 다룹니다. 잘못된 문장 하나가 곧바로 파일 삭제나 송금, 외부 전송 같은 실제 행동으로 이어질 수 있다는 점이 결정적인 차이입니다. 그래서 텍스트 목록만으로는 이 행동 계층의 위협을 담아낼 수 없어, 에이전트 전용 목록이 따로 필요해진 것입니다.

두 목록의 관계를 한 그림으로 정리하면 다음과 같습니다.

```
LLM Top 10   : 텍스트를 만드는 계층의 위협 (LLM01~10)
                 └─ LLM06 과도한 에이전시 = 다리
Agentic Top 10: 행동하는 계층의 위협   (ASI01~10)
                 도구 호출·다단계 실행·기억·에이전트 간 통신 위에서 생김
관계: 대체가 아니라 그 위에 쌓는 확장
```

목록을 읽으려면 항목에 붙은 코드부터 이해해야 합니다. 열 개의 항목에는 각각 ASI01부터 ASI10까지 코드가 붙습니다. 여기서 **ASI**는 에이전트 보안 항목을 가리키는 접두어이고, 뒤의 두 자리 숫자는 목록 안에서의 순번입니다. LLM 목록의 항목이 LLM01, LLM02처럼 불렸던 것과 같은 방식입니다. 앞의 글자가 이것이 에이전트 계층의 위협임을 알려 주고, 뒤의 번호가 그 안에서 몇 번째 항목인지를 가리킵니다. 이렇게 코드를 붙여 두는 이유는 앞서 말한 공통 언어 때문입니다. 위협의 이름은 문서마다 조금씩 다르게 번역되거나 줄여 불리기 쉽습니다. 목표 탈취를 어떤 문서는 목표 하이재킹이라 적고 다른 문서는 목표 변조라 적을 수 있습니다. 이때 ASI01이라는 코드 하나만 함께 적어 두면, 표현이 달라도 같은 항목을 가리킨다는 사실이 흔들리지 않습니다. 번호가 낮다고 더 위험하다는 순위를 뜻하지는 않으며, 관련된 위협을 묶어 놓은 목록의 자리 번호로 읽으면 됩니다.

열 개 항목의 코드와 이름, 핵심 위협을 한자리에 모으면 다음과 같습니다. 각 항목의 자세한 내용은 이어지는 단원에서 하나씩 풀어 가고, 여기서는 전체 지형을 눈에 익히는 것이 목적입니다.

```
ASI01 Agent Goal Hijack        악성 콘텐츠가 에이전트의 목표 자체를 바꿔치기
ASI02 Tool Misuse              정상 도구를 위험하게 사용 (파괴적 명령 등)
ASI03 Identity & Privilege     물려받은 고권한 토큰의 재사용·전파
ASI04 Agentic Supply Chain     오염된 도구·플러그인·템플릿 유입
ASI05 Unexpected Code Exec     에이전트가 만든 코드·명령의 무방비 실행
ASI06 Memory & Context Poison  메모리·RAG·요약본을 오염시켜 미래 결정 조작
ASI07 Insecure Inter-Agent     에이전트 간 메시지의 인증·검증 부재
ASI08 Cascading Failures       한 에이전트의 오류가 연쇄로 증폭 전파
ASI09 Human-Agent Trust        사용자의 과잉 신뢰를 악용한 조작
ASI10 Rogue Agents             오염된 에이전트가 정상인 척 지속 활동
```

이 목록을 남의 이야기가 아니라 내 문제로 바꾸는 방법이 항목별 해당 여부 표를 만드는 것입니다. 목록을 눈으로만 읽으면 열 개 모두가 막연히 위험해 보이지만, 실제로 내 에이전트에 다 해당하지는 않습니다. 외부 콘텐츠를 전혀 읽지 않는 에이전트라면 목표 탈취의 위험은 낮고, 다른 에이전트와 통신하지 않는 단일 에이전트라면 에이전트 간 통신 항목은 지금은 비껴갑니다. 이 구별을 눈대중이 아니라 표로 고정하면 무엇을 먼저 막을지가 분명해집니다.

표를 만드는 절차는 단순합니다. 왼쪽에 ASI01부터 ASI10까지 열 줄을 적고, 각 줄마다 세 가지를 판단해 채웁니다. 이 위협이 내 시스템 구조상 성립할 수 있는가, 성립한다면 지금 어떤 방어가 되어 있는가, 그리고 그 위험을 어느 순위로 다룰 것인가입니다.

```
ASI 코드   해당 여부   현재 방어 상태        우선순위
--------   --------   ------------------   --------
ASI01      해당       입력 필터만 있음      높음
ASI02      해당       삭제 도구 승인 게이트  중간
ASI03      해당       장기 API 키 심어 둠    높음
...        ...        ...                  ...
ASI07      비해당      단일 에이전트라 무관   -
```

이 표를 한 번 채우고 나면, 목록이 추상적인 위협 열거에서 내 시스템의 점검표로 바뀝니다. 아직 각 항목의 정확한 판단 기준은 배우지 않았으므로 지금 표를 완성할 필요는 없습니다. 다만 이런 표를 목표로 두고 이어지는 단원을 읽으면, 각 항목을 배울 때마다 표의 한 줄을 채워 나갈 수 있습니다.

정리하면, OWASP Top 10 for Agentic Applications는 에이전트에서 특히 위험한 위협 열 가지를 ASI01부터 ASI10까지 코드로 정리한 분류체계이고, 텍스트 계층을 다루던 LLM Top 10을 대체하는 것이 아니라 행동 계층의 위협을 담아 그 위에 얹은 확장입니다. 위협을 이렇게 칸으로 나누면 빠뜨림을 줄이고 공통 언어와 대응 자리를 얻으며, 항목별 해당 여부 표를 만들어 두면 이 목록이 내 시스템의 점검표로 바뀝니다.

3.1.2에서는 이 목록의 첫 항목인 ASI01 Agent Goal Hijack, 곧 악성 콘텐츠가 에이전트의 목표 자체를 바꿔치기하는 목표 탈취를 다룹니다.

이 단원을 마치면 OWASP Top 10 for Agentic Applications가 무엇이며, 그것이 LLM Top 10과 어떻게 다르고, 이 목록을 에이전트 보안 학습의 중심 축이자 내 프로젝트의 점검표로 삼는 법을 설명할 수 있습니다.
