# 5 방어 설계 패턴

실행 격리·권한/신원·흐름 통제·입출력/데이터 네 묶음의 방어 패턴을 코드·인프라 관점에서 이해하고, 3장에서 진단한 공격이 더 이상 통하지 않도록 설계를 구성할 수 있다.

이 Phase는 다음 섹션으로 구성됩니다.

- 5.1 실행 격리
  - 5.1.1 샌드박싱: 컨테이너, microVM, WASM
  - 5.1.2 Egress 통제: 외부 전송을 구조적으로 막기
  - 5.1.3 세션·테넌트 격리
- 5.2 권한과 신원
  - 5.2.1 Capability 기반 권한: 읽기와 쓰기는 다른 권한
  - 5.2.2 단기(JIT) 크리덴셜과 시크릿 볼트
  - 5.2.3 에이전트 신원과 행위 귀속
- 5.3 흐름 통제
  - 5.3.1 HITL 승인 게이트와 액션 분류표
  - 5.3.2 Dual-LLM 패턴: 특권과 격리의 분리
  - 5.3.3 CaMeL: 제어 흐름과 데이터 흐름의 분리
  - 5.3.4 Plan-then-Execute: 읽기 전에 계획 확정
- 5.4 입출력과 데이터
  - 5.4.1 가드레일: 입력 필터와 출력 검증, 그리고 한계
  - 5.4.2 데이터 등급 분류와 DLP
