# 4.1 MCP 보안

이 섹션의 토픽과 학습 목표는 다음과 같습니다.

- **4.1.1 MCP란 무엇인가: 도구 연결 표준** — MCP(Model Context Protocol)가 모델과 도구를 잇는 표준임을 설명하고 왜 이 배관이 공격면이 되는지 서술할 수 있다
- **4.1.2 MCP 공격 벡터 1: 서버·도구 기만** — Tool Poisoning, Rogue Server, rug pull이 서버·도구를 신뢰한 데서 비롯되는 기만 공격임을 설명할 수 있다
- **4.1.3 MCP 공격 벡터 2: 위임·네트워크 남용** — Confused Deputy, Token Passthrough, SSRF가 위임과 네트워크 접근을 남용하는 MCP 공격임을 설명할 수 있다
- **4.1.4 MCP 방어: OAuth 2.1, 서버 검증·핀닝, 게이트웨이** — MCP 5대 공격에 대응하는 OAuth 2.1 인증·서버 검증·핀닝·게이트웨이/레지스트리 방어를 설명할 수 있다
