# 4.2.4 MMP의 역할 변화 — AppsFlyer, Adjust, Singular

앞 단원 4.2.3에서는 Meta가 SKAN 환경에 맞춰 도메인당 8개 우선순위 이벤트만 보고하는 Aggregate Event Measurement(AEM) 체계를 도입한 과정을 살펴보았습니다. 매체 한 곳이 자체 측정 체계를 새로 짠 셈이지만, 게임 회사가 실제로 들여다보는 화면은 Meta 한 곳만의 리포트가 아닙니다. 광고는 보통 Meta, Google, TikTok, Unity, AppLovin처럼 여러 매체에 걸쳐 동시에 집행되고, 각 매체가 보내 주는 신호의 형식과 지연이 모두 다릅니다. 이 신호들을 같은 잣대로 묶어 한 화면에 정리해 주는 사업자가 1.1.2에서 짧게 언급한 **MMP**(Mobile Measurement Partner)입니다. 이번 단원에서는 IDFA 시대에 굳어진 MMP의 자리가 SKAN 시대로 넘어오면서 어떻게 달라졌는지를 정리합니다.

가장 자주 거론되는 MMP는 **AppsFlyer**, **Adjust**, **Singular** 세 곳입니다. 셋은 모두 게임 앱에 심는 SDK와, 광고 네트워크가 직접 호출하는 서버 측 연동을 동시에 제공합니다. 게임 회사가 어느 한 곳을 표준 측정 도구로 정해 두면, 광고 네트워크들이 그 표준에 맞춰 클릭과 노출 정보를 보내 주고, 앱 안에서 일어나는 설치, 첫 실행, 결제 같은 사건도 같은 SDK가 같은 형식으로 받아 모읍니다. 결과는 캠페인, 크리에이티브, 매체 단위의 통합 리포트로 정리됩니다. 세 회사의 강점은 조금씩 다릅니다. AppsFlyer는 가장 큰 시장 점유율과 광범위한 파트너 연동을, Adjust는 가벼운 구현과 사기 방지 분석을, Singular는 광고비 데이터와 측정 데이터를 한쪽에서 묶어 보여 주는 기능을 강조해 왔습니다. 어느 쪽을 쓰든 게임 회사가 노리는 것은 매체별로 다른 리포트를 매번 환산하지 않고 같은 좌표에서 비교하는 일입니다.

IDFA가 자유롭게 쓰이던 시기의 MMP는 본질적으로 **데이터 게이트웨이**에 가까웠습니다. 게이트웨이라는 말은 여러 통로에서 들어오는 신호가 한 번 거쳐 나가는 관문이라는 뜻입니다. 광고 네트워크가 클릭을 보고하면, MMP는 그 클릭에 담긴 IDFA와 앱이 보낸 설치 이벤트의 IDFA를 맞대 보고, 가장 최근에 클릭이 일어난 매체에 그 설치를 귀속시켰습니다. 4.1.1에서 살펴본 결정론적 어트리뷰션이 곧 MMP의 기본 동작이었던 셈입니다.

이 위에 두 번째 역할인 **사기 방지**가 얹혔습니다. 사기 방지는 봇이 만든 가짜 클릭, 이미 설치된 기기를 새 설치인 것처럼 꾸민 리퍼럴, 한 사람이 여러 기기로 보상만 받는 패턴 같은 비정상 신호를 걸러 내는 일입니다. MMP는 IP, 클릭과 설치 사이의 시간 간격, 디바이스 핑거프린트의 분포 같은 단서를 모아 의심 트래픽을 표시하고, 정산에서 제외하도록 도와주었습니다. 이 시기의 MMP는 사실에 가까운 데이터를 모아 정리해 넘기는 정직한 중개자였습니다.

SKAN 도입 이후로는 이 그림이 흔들립니다. 옵트인하지 않은 사용자에 대해서는 IDFA를 받지 못하므로 클릭과 설치를 직접 잇는 일이 불가능해졌고, 4.2.1에서 본 SKAN 포스트백이 매체별로 시간차를 두고 들어옵니다. 4.2.2에서 정리한 결정론과 확률론의 혼합도 MMP가 구현 책임을 떠안는 영역이 되었습니다. 한쪽에서는 옵트인 사용자에 대한 결정론적 매칭을 그대로 굴리고, 다른 한쪽에서는 옵트아웃 사용자에 대해 캠페인 메타와 시간대, 디바이스 종류 같은 간접 단서로 가능성 높은 매체를 추정하는 모델을 돌립니다. 두 경로의 결과를 가중 평균해 한 줄의 어트리뷰션으로 보여 주는 일까지가 한 패키지로 묶입니다. MMP의 일은 사실 데이터를 옮기는 데서 끝나지 않고, 빠진 자리를 채우는 추정 결과까지 만들어 내는 쪽으로 옮겨 갔습니다.

이 새 역할을 설명할 때 자주 등장하는 도구가 **결측 보정**입니다. 결측 보정은 식별자가 빠진 자리를 비워 두지 않고, 들어온 신호와 과거 데이터를 근거로 가장 그럴듯한 값을 메우는 작업입니다. 예를 들어 SKAN 포스트백에는 정확한 사용자 ID가 들어 있지 않지만, 캠페인 ID와 conversion value, 발신 시각이 함께 들어 있습니다. MMP는 이 단서들을 자기 쪽에 쌓인 광범위한 캠페인 패턴과 맞대어, 어느 사용자 코호트에 속할 가능성이 높은지를 확률로 매깁니다. 이 확률을 LTV 예측에 다시 넣어, 어느 매체가 결과적으로 더 가치 있는 사용자를 데려왔는지 추정합니다. 이 과정에서 만들어지는 숫자는 사실이 아니라 모델의 출력입니다. 운영 화면에 그대로 띄워 의사결정을 내리려면 그 숫자가 어느 정도의 오차로 흔들릴 수 있는지까지 함께 표시되어야 합니다.

또 하나의 핵심 도구가 **데이터 클린룸**입니다. 데이터 클린룸은 서로 다른 회사가 가진 사용자 단위 데이터를 한곳으로 합치되, 어느 쪽도 상대 회사의 원천 데이터를 직접 보지 못하도록 격리해 둔 환경을 가리킵니다. 한 회사가 가진 광고 노출 로그와 다른 회사가 가진 결제 로그를 같은 키로 맞대 보고 싶어도, 사용자 ID를 그대로 주고받는 일이 프라이버시 정책에 어긋나는 시기가 되었기 때문에 등장한 구조입니다. 클린룸 안에서는 두 데이터가 암호화된 키로 매칭되고, 결과는 충분히 큰 집계 단위로만 밖으로 나옵니다. MMP는 자체 클린룸을 제공하거나, 매체가 운영하는 클린룸과 자기 SDK 데이터를 연결해 주는 partner 통합을 점점 더 많이 맡습니다. partner 통합이라는 말은 단순히 API를 한 줄 연결한다는 뜻이 아니라, 매체별로 다른 입력 형식과 출력 권한, 보안 요구를 표준화된 한 흐름에 끼워 넣는 작업 전체를 가리킵니다.

전체 흐름을 한 그림으로 보면 MMP가 어디에 끼어들어 무엇을 만드는지가 분명해집니다.

```
[광고 매체들]                        [게임 앱]
   |                                    |
   | 캠페인 메타, 노출, SKAN 포스트백        | 결정론적 이벤트(옵트인), 결제, 진행도
   v                                    v
              ┌──────── MMP ────────┐
              | 결정론적 매칭        |
              | 확률론적 추정        |
              | 사기 필터            |
              | 결측 보정 모델        |
              | 클린룸/partner 통합   |
              └──────────┬──────────┘
                         |
                  통합 어트리뷰션 + 신뢰구간
                         v
                 [퍼블리셔 운영팀]
```

이 새 위치는 게임 회사 입장에서 새로운 고민을 만듭니다. MMP가 만들어 주는 숫자가 단순한 사실 정리가 아니라 추정 결과라면, 그 추정의 가정과 한계는 결국 한 사업자의 선택입니다. 어느 매체에 가중치를 더 줄지, 어떤 시간 윈도를 어트리뷰션 기준으로 삼을지, 어떤 사기 패턴을 강하게 거를지 같은 결정이 모두 모델 안에 녹아 들어갑니다. 한 MMP의 결과만 보고 예산을 옮기다가 그 MMP의 모델 가정 변경 한 번에 매체별 성과 순위가 뒤집히는 일도 드물지 않습니다.

그래서 큰 게임사일수록 **인하우스 측정**을 함께 굴리는 흐름이 강해졌습니다. 인하우스 측정이라는 말은 회사 내부 데이터 팀이 자체 데이터 웨어하우스에 광고비, 노출, 인앱 이벤트, LTV 신호를 모아, 자신만의 어트리뷰션과 incrementality 모델을 따로 운영하는 체계를 뜻합니다. 균형은 한쪽으로 치우치기 어렵습니다. MMP만 의존하면 모델 변경에 휘둘리고, 인하우스만 굴리면 매체별 SDK 연동과 사기 방지에 들어가는 인력 비용이 빠르게 커집니다. 현실적인 선택은 두 체계를 같은 화면에서 비교 운영하는 쪽입니다. MMP가 만든 매체 단위 어트리뷰션을 일상 의사결정의 기본선으로 두고, 인하우스 모델은 핵심 캠페인의 incrementality 검증과 LTV 예측 같은 좁은 영역에 투입합니다. 두 결과가 크게 어긋나는 매체가 나오면, 그 매체에 한해 클린룸 안에서 추가 매칭을 돌리거나 holdout 실험을 따로 설계해 검증합니다.

정리하면, MMP는 IDFA 시대의 데이터 게이트웨이에서 SKAN 시대의 모델링 사업자로 위치를 옮겼습니다. 어트리뷰션 통합과 사기 방지라는 전통 역할은 그대로 남았지만, 결측 보정과 클린룸 기반 partner 통합이 핵심 가치로 부상했고, 그만큼 게임 회사는 한 MMP의 결과를 사실로 받아들이는 대신 인하우스 측정과의 균형을 의식적으로 설계하는 단계로 들어섰습니다.

다음 단원 4.3.1에서는 이 결측 보정 안쪽으로 들어가, ML 기반 어트리뷰션 추정이 어떤 입력을 받아 어떤 출력을 만드는지 그 구조를 풀어 봅니다.

이 단원을 마치면 MMP의 역할이 데이터 게이트웨이에서 모델링 사업자로 어떻게 변했는지를, 그 변화가 게임 회사의 측정 체계 운영에 어떤 균형을 요구하는지와 함께 설명할 수 있습니다.
