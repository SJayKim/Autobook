# 1.2.1 대규모 언어 모델의 구조

1.1.3에서 Foundation Model이 무엇인지, 그리고 GPT, LLaMA, Claude, Gemini 같은 모델이 저마다 다른 크기와 라이선스로 제공된다는 점을 살펴보았습니다. 이 단원에서는 한 걸음 더 들어가, 이런 모델들이 실제로 어떤 내부 구조 위에서 동작하는지를 다룹니다. LLMOps를 수행하려면 모델을 "블랙박스"로만 다루기보다, 입력이 어떤 경로를 거쳐 출력으로 바뀌는지 큰 흐름을 파악하고 있어야 합니다. 모델의 구조를 이해하면 파라미터 수가 왜 중요한지, 추론 비용이 왜 커지는지, 최적화 기법이 어디에 적용되는지를 자연스럽게 파악할 수 있습니다.

오늘날 거의 모든 대규모 언어 모델은 **Transformer**라는 아키텍처를 기반으로 합니다. Transformer는 2017년에 제안된 신경망 구조로, 그 이전에 널리 쓰이던 순환 신경망(RNN)의 한계를 극복하기 위해 만들어졌습니다. RNN은 문장을 앞에서부터 한 단어씩 순서대로 처리해야 했기 때문에, 문장이 길어질수록 앞쪽 단어의 정보가 점점 희미해지는 문제가 있었습니다. 또한 단어를 하나씩 차례로 처리하는 구조이므로 병렬 계산이 어려워, 학습 속도도 느렸습니다. Transformer는 이 두 가지 문제를 한꺼번에 해결합니다. 문장 전체를 한 번에 입력받아 모든 단어 사이의 관계를 동시에 계산하므로, 긴 문장에서도 멀리 떨어진 단어의 관계를 놓치지 않으며, GPU를 활용한 병렬 처리에도 유리합니다.

Transformer 아키텍처의 원래 설계는 **인코더**와 **디코더** 두 부분으로 구성되어 있습니다. 인코더는 입력 문장을 읽어 내부 표현으로 변환하는 역할을 하고, 디코더는 그 내부 표현을 바탕으로 출력 문장을 한 단어씩 생성하는 역할을 합니다. 번역 같은 작업에서는 인코더가 원문을 이해하고, 디코더가 번역문을 만들어 내는 식입니다. 그런데 GPT 계열처럼 텍스트를 생성하는 대규모 언어 모델 대부분은 인코더 없이 **디코더만** 사용합니다. 이 구조를 **Decoder-only 구조**라고 부릅니다. Decoder-only 구조에 대해서는 뒤에서 더 자세히 다루겠습니다.

Transformer를 구성하는 핵심 요소는 크게 네 가지입니다. 첫째, 입력 단어들 사이의 관계를 파악하는 **Self-Attention** 메커니즘입니다. 둘째, Self-Attention을 여러 관점에서 동시에 수행하는 **Multi-Head Attention**입니다. 셋째, Attention의 결과를 더 풍부한 표현으로 변환하는 **Feed-Forward Network**입니다. 넷째, 단어의 순서 정보를 부여하는 **위치 인코딩(Positional Encoding)**입니다. 이 네 요소가 하나의 층(layer)을 이루고, 이런 층이 수십 개에서 수백 개까지 쌓여 하나의 Transformer 모델이 됩니다. 아래에서 각 요소를 하나씩 살펴보겠습니다.

**Self-Attention**은 "문장 안의 각 단어가 다른 모든 단어와 얼마나 관련 있는지"를 계산하는 메커니즘입니다. 왜 이런 계산이 필요할까요. "은행에서 돈을 인출했다"라는 문장을 생각해 보겠습니다. 여기서 "은행"이 금융 기관을 뜻하는지, 강가의 둑을 뜻하는지는 "돈"이나 "인출"과 같은 주변 단어를 봐야 알 수 있습니다. Self-Attention은 바로 이 과정을 수학적으로 수행합니다. 문장 안의 모든 단어가 서로를 참조하여, 각 단어의 의미를 문맥에 맞게 조정하는 것입니다.

Self-Attention의 동작을 단계별로 살펴보겠습니다. 입력으로 문장 속 각 단어가 숫자 벡터로 변환되어 들어옵니다. 벡터란 여러 개의 숫자를 한 묶음으로 표현한 것으로, 단어의 의미를 숫자 공간에 표현한 것이라고 생각하면 됩니다.

첫 번째 단계에서, 각 단어 벡터로부터 세 종류의 벡터를 만듭니다. **Query(질의)**, **Key(키)**, **Value(값)**입니다. 이 이름은 검색 시스템에서 빌려온 비유입니다. Query는 "나는 어떤 정보가 필요한가"를 나타내고, Key는 "나는 어떤 정보를 갖고 있는가"를 나타내며, Value는 "실제로 전달할 정보의 내용"을 담고 있습니다. 세 벡터는 원래 단어 벡터에 각각 다른 가중치 행렬을 곱하여 만듭니다. 가중치 행렬은 학습 과정에서 데이터를 보며 자동으로 조정되는 숫자 표입니다.

두 번째 단계에서, 한 단어의 Query와 문장 안 모든 단어의 Key를 비교합니다. 비교 방법은 두 벡터의 **내적(dot product)**을 구하는 것입니다. 내적이란 두 벡터의 대응하는 숫자를 각각 곱한 뒤 모두 더하는 연산입니다. 두 벡터가 비슷한 방향을 가리키면 내적 값이 크고, 관련이 적으면 작습니다. 그래서 내적 결과가 클수록 "이 두 단어는 서로 밀접하게 관련되어 있다"는 뜻이 됩니다.

세 번째 단계에서, 내적 결과를 Key 벡터 차원 수의 제곱근으로 나눕니다. 이 과정을 **스케일링(scaling)**이라고 합니다. 벡터 차원이 크면 내적 값도 자연히 커져서, 이후 단계에서 특정 단어 하나에만 점수가 몰리는 현상이 생길 수 있습니다. 제곱근으로 나누면 점수의 분포가 적당한 범위로 조정되어, 여러 단어에 골고루 주의를 기울일 수 있게 됩니다.

네 번째 단계에서, 스케일링된 점수에 **소프트맥스(softmax)** 함수를 적용합니다. 소프트맥스는 여러 숫자를 0과 1 사이 값으로 변환하면서, 모든 값의 합이 1이 되도록 만드는 함수입니다. 이렇게 하면 각 점수가 "이 단어에 얼마나 주의를 기울일 것인가"를 나타내는 비율, 즉 **어텐션 가중치(attention weight)**가 됩니다.

다섯 번째 단계에서, 어텐션 가중치를 각 단어의 Value 벡터에 곱한 뒤, 모두 더합니다. 관련도가 높은 단어의 Value는 많이 반영되고, 관련도가 낮은 단어의 Value는 적게 반영됩니다. 그 결과, 해당 단어의 새로운 벡터 표현이 만들어집니다. 이 새 벡터에는 문맥 정보가 녹아들어 있습니다.

이 전체 과정을 수식으로 표현하면 다음과 같습니다.

```
Attention(Q, K, V) = softmax(Q * K^T / sqrt(d_k)) * V
```

여기서 Q는 Query 행렬, K는 Key 행렬, V는 Value 행렬이고, d_k는 Key 벡터의 차원 수입니다. K^T는 K의 전치(행과 열을 뒤집은 것)이며, sqrt는 제곱근입니다. Q * K^T로 모든 단어 쌍 사이의 관련도를 한꺼번에 계산하고, d_k의 제곱근으로 스케일링한 뒤, softmax로 가중치를 구하고, 마지막으로 V에 곱하여 문맥이 반영된 표현을 얻는 것입니다.

아래 다이어그램은 Self-Attention의 흐름을 보여줍니다.

```
입력 단어 벡터
     |
     +---> [가중치 W_Q] ---> Query (Q)
     |
     +---> [가중치 W_K] ---> Key   (K)
     |
     +---> [가중치 W_V] ---> Value (V)

           Q * K^T
              |
        sqrt(d_k)로 나눔
              |
         softmax 적용
              |
       어텐션 가중치 * V
              |
        문맥 반영 벡터 (출력)
```

Self-Attention은 문장 속 모든 단어 쌍을 동시에 계산하므로, "나"와 "은행" 사이의 관계, "나"와 "돈" 사이의 관계, "은행"과 "돈" 사이의 관계가 한 번의 연산으로 모두 구해집니다. RNN처럼 단어를 하나씩 순서대로 처리할 필요가 없기 때문에 병렬 처리가 가능하고, 문장이 길어도 먼 거리의 단어 관계를 직접 참조할 수 있습니다.

그런데 Self-Attention을 한 번만 수행하면, 한 가지 관점에서만 단어 관계를 파악하게 됩니다. 자연어에서는 단어 사이에 여러 종류의 관계가 동시에 존재합니다. 예를 들어, "그 학생이 수학 시험에서 높은 점수를 받았다"라는 문장에서 "학생"과 "받았다"는 주어-서술어 관계이고, "시험"과 "점수"는 의미적 연관 관계이며, "높은"과 "점수"는 수식 관계입니다. 하나의 Attention으로는 이 다양한 관계를 모두 포착하기 어렵습니다.

이 문제를 해결하는 것이 **Multi-Head Attention**입니다. Multi-Head Attention은 Self-Attention을 여러 개의 독립적인 "헤드(head)"로 나누어 동시에 수행하는 방법입니다. 각 헤드는 자체적인 Query, Key, Value 가중치 행렬을 갖고 있어, 서로 다른 관점에서 단어 관계를 파악합니다. 한 헤드는 문법적 관계에 주목할 수 있고, 다른 헤드는 의미적 유사성에 주목할 수 있으며, 또 다른 헤드는 인접한 단어 간의 국소적 패턴에 주목할 수 있습니다.

Multi-Head Attention의 동작 과정은 다음과 같습니다. 먼저, 입력 벡터의 차원을 헤드 수만큼 분할합니다. 예를 들어 입력 벡터가 512차원이고 헤드가 8개라면, 각 헤드는 64차원 공간에서 Self-Attention을 수행합니다. 각 헤드가 독립적으로 Attention을 계산한 뒤, 8개 헤드의 출력을 이어 붙여 다시 512차원 벡터로 합칩니다. 마지막으로, 합쳐진 벡터에 가중치 행렬을 한 번 더 곱하여 최종 출력을 만듭니다.

```
입력 (512차원)
     |
     +---> Head 1 (64차원) ---> Attention 출력 1
     |
     +---> Head 2 (64차원) ---> Attention 출력 2
     |
     +---> ...
     |
     +---> Head 8 (64차원) ---> Attention 출력 8
                                      |
                              이어 붙임 (Concat)
                                      |
                              가중치 곱 (W_O)
                                      |
                              최종 출력 (512차원)
```

차원을 나누어 계산하므로, 전체 계산량은 헤드가 하나일 때와 비슷합니다. 그러면서도 여러 관점의 관계 정보를 동시에 포착할 수 있어, 모델의 표현력이 크게 향상됩니다.

Multi-Head Attention을 거친 벡터는 다음으로 **Feed-Forward Network(FFN)**를 통과합니다. FFN은 각 단어 위치에 대해 독립적으로 적용되는 간단한 신경망입니다. Attention이 단어 사이의 관계를 파악하는 역할이라면, FFN은 각 단어의 표현을 더 풍부하게 변환하는 역할을 합니다.

FFN의 구조를 살펴보겠습니다. FFN은 두 개의 선형 변환과 그 사이의 활성화 함수로 이루어져 있습니다. 선형 변환이란 벡터에 가중치 행렬을 곱하고 편향(bias)을 더하는 연산입니다. 첫 번째 선형 변환은 입력 차원을 더 큰 차원으로 확장합니다. 예를 들어 512차원 벡터를 2048차원으로 늘립니다. 이렇게 높은 차원으로 올리면, 벡터가 더 넓은 공간에서 다양한 특징을 표현할 수 있게 됩니다. 확장된 벡터에 **ReLU**나 **GELU** 같은 활성화 함수를 적용합니다. 활성화 함수는 값에 비선형 변환을 가하는 함수로, 신경망이 단순한 직선 관계만이 아니라 복잡한 패턴을 학습할 수 있게 해 줍니다. 마지막으로, 두 번째 선형 변환이 차원을 다시 원래 크기로 줄입니다. 2048차원을 다시 512차원으로 되돌리는 것입니다.

```
FFN(x) = Linear_2( GELU( Linear_1(x) ) )

입력 (512차원) --> 확장 (2048차원) --> 활성화 (GELU) --> 축소 (512차원)
```

이 확장-축소 과정을 통해 모델은 각 단어의 표현에 비선형적이고 복잡한 변환을 가할 수 있습니다. Attention만으로는 단어 간 관계 정보를 섞는 것에 그치지만, FFN을 거치면 그 정보가 더 정교한 표현으로 가공됩니다. Transformer 모델에서 전체 파라미터의 상당 부분이 이 FFN 층에 집중되어 있습니다. 이 점은 뒤에서 파라미터 수를 논의할 때 다시 살펴보겠습니다.

지금까지 설명한 Self-Attention, Multi-Head Attention, FFN은 모두 단어의 의미와 관계를 다루지만, 한 가지 빠진 정보가 있습니다. 바로 단어의 **순서**입니다. Self-Attention은 모든 단어 쌍을 동시에 비교하므로, "고양이가 쥐를 쫓는다"와 "쥐가 고양이를 쫓는다"를 구분하지 못합니다. 단어가 문장에서 몇 번째에 위치하는지에 대한 정보가 없기 때문입니다.

이 문제를 해결하기 위해 **위치 인코딩(Positional Encoding)**을 사용합니다. 위치 인코딩은 각 위치에 고유한 숫자 패턴을 만들어, 단어 벡터에 더해 주는 방식입니다. 첫 번째 위치에는 첫 번째 위치만의 패턴이, 열 번째 위치에는 열 번째 위치만의 패턴이 있어, 모델이 "이 단어는 문장에서 몇 번째에 있다"는 정보를 알 수 있게 됩니다.

원래 Transformer 논문에서는 사인(sin)과 코사인(cos) 함수를 사용하여 위치 인코딩을 만들었습니다. 이 방식은 학습 없이 수학 공식으로 위치 벡터를 생성하므로, 학습 데이터에서 본 적 없는 더 긴 문장에도 자연스럽게 확장할 수 있다는 장점이 있었습니다. 최근의 대규모 언어 모델에서는 **RoPE(Rotary Position Embedding)**처럼 학습 가능한 위치 인코딩을 사용하는 경우가 많습니다. RoPE는 벡터를 회전시키는 방식으로 위치 정보를 부여하여, 두 단어 사이의 상대적 거리 정보를 더 효과적으로 표현합니다. 어떤 방식을 쓰든, 위치 인코딩은 Transformer가 단어 순서를 인식하게 만드는 필수 장치입니다.

```
단어 임베딩:    [0.2, -0.5, 0.8, ...]   ("고양이"의 의미)
위치 인코딩:  + [0.0,  0.1, 0.0, ...]   (1번째 위치의 패턴)
                -------------------------
최종 입력:      [0.2, -0.4, 0.8, ...]   (의미 + 위치 정보)
```

이렇게 단어의 의미 벡터에 위치 인코딩을 더한 벡터가 Transformer의 실제 입력이 됩니다. 이후 Self-Attention과 FFN을 거치면서 의미와 순서 정보가 함께 처리됩니다.

이제 네 가지 핵심 요소를 모두 살펴보았으니, 현재 대규모 언어 모델에서 가장 널리 쓰이는 **Decoder-only 구조**를 정리하겠습니다. 1.1.3에서 살펴본 GPT, LLaMA 같은 Foundation Model은 대부분 이 구조를 채택하고 있습니다.

Decoder-only 구조에서 모델은 주어진 텍스트의 다음 단어를 예측하는 방식으로 학습합니다. "오늘 날씨가"라는 입력이 들어오면, 다음에 올 단어로 "좋습니다"를 예측하는 식입니다. 이때 중요한 규칙이 하나 있습니다. 각 단어는 자기보다 앞에 있는 단어만 참조할 수 있고, 뒤에 올 단어는 참조할 수 없습니다. "오늘"을 처리할 때는 "오늘"만 볼 수 있고, "날씨가"를 처리할 때는 "오늘"과 "날씨가"만 볼 수 있습니다. 이 제약을 **인과적 마스킹(causal masking)**이라고 합니다.

인과적 마스킹은 Self-Attention 단계에서 구현됩니다. Query와 Key의 내적을 계산한 뒤, 미래 위치에 해당하는 점수를 매우 큰 음수 값으로 설정합니다. 그러면 softmax를 통과한 뒤 해당 가중치가 사실상 0이 되어, 미래 단어의 정보가 전혀 반영되지 않습니다. 이렇게 하면 모델은 왼쪽에서 오른쪽으로만 정보를 읽는 것처럼 동작합니다.

```
         오늘  날씨가  좋습니다
오늘     [O]   [X]     [X]
날씨가   [O]   [O]     [X]
좋습니다 [O]   [O]     [O]

O = 참조 가능, X = 마스킹 (참조 불가)
```

Decoder-only 구조의 한 층(layer)은 다음과 같이 구성됩니다. 먼저 입력에 **층 정규화(Layer Normalization)**를 적용하여 값의 범위를 안정시킵니다. 다음으로 인과적 마스킹이 적용된 Multi-Head Attention을 수행합니다. Attention의 출력에 원래 입력을 더하는 **잔차 연결(residual connection)**을 적용합니다. 잔차 연결이란 변환을 거치기 전의 입력을 변환 결과에 그대로 더하는 것으로, 층이 깊어져도 정보가 소실되지 않게 돕는 장치입니다. 다시 층 정규화를 거친 뒤 FFN을 통과하고, 여기서도 잔차 연결을 적용합니다.

```
입력
 |
 +--> Layer Norm --> Multi-Head Attention (causal mask) --+
 |                                                        |
 +----------------------------(잔차 연결)----------------+
                                                          |
 +--> Layer Norm --> Feed-Forward Network ----------------+
 |                                                        |
 +----------------------------(잔차 연결)----------------+
                                                          |
                                                        출력
```

이 구조가 하나의 Transformer 층입니다. GPT-3는 이런 층을 96개 쌓았고, LLaMA 2 70B는 80개를 쌓았습니다. 층을 더 많이 쌓을수록 모델은 더 복잡한 언어 패턴을 학습할 수 있지만, 그만큼 파라미터 수도 늘어납니다.

**파라미터**란 모델이 학습 과정에서 조정하는 숫자 하나하나를 뜻합니다. Query, Key, Value를 만드는 가중치 행렬, FFN의 가중치 행렬, 편향 등이 모두 파라미터에 해당합니다. 파라미터 수가 많을수록 모델이 더 많은 지식과 패턴을 저장할 수 있는 용량이 커지지만, 저장 공간, 계산 비용, 메모리 사용량도 함께 증가합니다.

모델의 파라미터 수는 주로 세 가지 설계 값이 결정합니다. 첫째, 입력 벡터의 차원 수(보통 d_model이라고 표기합니다)입니다. 차원이 클수록 각 단어를 더 풍부하게 표현할 수 있지만, 가중치 행렬의 크기도 그만큼 커집니다. 둘째, Transformer 층의 개수입니다. 층이 많아지면 그만큼 가중치 행렬 세트가 반복됩니다. 셋째, FFN의 확장 차원입니다. FFN에서 내부 차원을 d_model의 4배로 설정하는 것이 일반적이어서, FFN 한 층의 파라미터만으로도 상당한 양을 차지합니다.

실제 모델의 파라미터 수를 비교하면, 설계 값이 커질수록 모델 규모가 급격히 늘어나는 것을 확인할 수 있습니다.

```
모델              파라미터 수    층 수    d_model    메모리(FP16 기준)
-----------------------------------------------------------------
GPT-2             1.5억         48       1600       약 0.3 GB
GPT-3             1750억        96       12288      약 350 GB
LLaMA 2  7B       70억          32       4096       약 14 GB
LLaMA 2 70B      700억          80       8192       약 140 GB
```

GPT-2에서 GPT-3로 가면서 층 수는 2배, d_model은 약 8배 늘었고, 파라미터 수는 약 1,000배 이상 증가했습니다. 이렇게 파라미터 수가 늘어나면 모델이 더 정교한 언어 능력을 보여 주는 경향이 있지만, 모델을 메모리에 올리고 추론을 실행하는 비용도 비례하여 커집니다. LLMOps에서 모델 서빙, 추론 최적화, 모델 경량화를 다루는 이유가 바로 여기에 있습니다. 파라미터 수에 따라 필요한 GPU 메모리, 응답 지연 시간, 운영 비용이 직접적으로 달라지기 때문입니다.

정리하면, 대규모 언어 모델은 Transformer 아키텍처를 기반으로 하며, 그 핵심 구성요소는 Self-Attention, Multi-Head Attention, Feed-Forward Network, 위치 인코딩입니다. Self-Attention은 문장 속 모든 단어 관계를 동시에 계산하고, Multi-Head Attention은 이를 여러 관점에서 수행하며, FFN은 각 단어의 표현을 풍부하게 변환하고, 위치 인코딩은 단어 순서 정보를 부여합니다. 오늘날 GPT 계열로 대표되는 대규모 언어 모델은 Decoder-only 구조를 채택하여, 인과적 마스킹 아래에서 다음 단어를 예측하는 방식으로 동작합니다. 모델의 파라미터 수는 벡터 차원, 층 수, FFN 확장 차원에 의해 결정되며, 파라미터가 많을수록 표현력이 커지는 대신 운영 비용도 함께 증가합니다.

다음 단원인 1.2.2에서는 토큰화와 어휘를 다룹니다. 텍스트가 모델에 입력되기 전에 어떤 단위로 분리되는지, 그리고 토큰화 방식에 따라 모델 성능과 운영 비용이 어떻게 달라지는지를 살펴봅니다.

이 단원을 마치면 Transformer 아키텍처의 핵심 구성요소인 Self-Attention, Multi-Head Attention, Feed-Forward Network, 위치 인코딩의 역할과 동작 방식을 설명할 수 있습니다.
