# 7.2.1 데이터 큐레이션과 품질 관리

7.1에서 파인튜닝의 기법들을 살펴보았습니다. LoRA, QLoRA, PEFT 프레임워크를 통해 모델을 효율적으로 조정하는 방법은 갖추었지만, 정작 학습에 넣을 데이터의 품질이 낮으면 기법이 아무리 좋아도 결과가 좋지 않습니다. 잘 정제된 소량의 데이터로 파인튜닝한 모델이, 품질을 신경 쓰지 않은 대량의 데이터로 학습한 모델보다 나은 성능을 보이는 사례가 반복적으로 보고됩니다. 파인튜닝의 성패를 가르는 것은 기법보다 데이터라고 해도 과언이 아닙니다. 이 단원에서는 파인튜닝용 데이터를 체계적으로 수집, 정제, 관리하는 과정인 데이터 큐레이션을 다룹니다.

**데이터 큐레이션(data curation)**이란 학습에 사용할 데이터를 수집하고, 정제하고, 구조화하여 목적에 맞는 고품질 데이터셋으로 만드는 전 과정을 뜻합니다. 미술관에서 큐레이터가 작품을 선별하고 배치하여 관람객에게 의미 있는 경험을 제공하는 것처럼, 데이터 큐레이션은 원시 데이터 중에서 학습에 유용한 것만 골라내고, 형식을 맞추고, 오류를 제거하여 모델에게 효과적인 학습 경험을 제공하는 것입니다.

데이터 큐레이션이 중요한 이유를 구체적으로 짚어 보겠습니다. 첫째, 파인튜닝은 사전 학습과 달리 비교적 적은 데이터로 진행됩니다. 사전 학습에서는 수조 개의 토큰을 사용하지만, 파인튜닝에서는 수천에서 수만 건의 데이터를 사용하는 것이 일반적입니다. 데이터 수가 적을수록 개별 데이터의 품질이 전체 결과에 미치는 영향이 큽니다. 잘못된 데이터 한 건이 전체 학습 방향을 왜곡할 수 있습니다. 둘째, 파인튜닝 데이터는 모델이 사용자에게 보여줄 응답의 스타일과 패턴을 직접 결정합니다. 학습 데이터에 오타가 많으면 모델도 오타를 생성하고, 학습 데이터가 특정 주제에 편중되면 모델도 그 주제에만 강해집니다. 셋째, 품질이 낮은 데이터는 학습 비용을 낭비합니다. GPU 시간, 전기, 엔지니어의 시간이 모두 비용인데, 쓸모없는 데이터를 학습하는 데 이 자원을 쓰는 것은 손실입니다.

파인튜닝에 사용하는 학습 데이터는 특정 포맷을 따릅니다. 가장 널리 사용되는 형태는 **Instruction-Input-Output 포맷**입니다. 이 포맷은 세 가지 필드로 구성됩니다.

**Instruction** 필드는 모델에게 수행할 작업을 지시하는 문장입니다. "다음 문장을 영어로 번역하세요", "아래 텍스트를 세 줄로 요약하세요", "주어진 코드의 버그를 찾아 수정하세요"처럼, 모델이 무엇을 해야 하는지를 명확히 기술합니다.

**Input** 필드는 작업의 대상이 되는 입력 데이터입니다. 번역할 원문, 요약할 텍스트, 버그를 찾을 코드 등이 여기에 들어갑니다. 모든 작업에 입력이 필요한 것은 아닙니다. "Python의 리스트와 튜플의 차이를 설명하세요"처럼 지시 자체에 모든 정보가 담겨 있으면 Input 필드는 비워둡니다.

**Output** 필드는 해당 지시와 입력에 대한 기대 응답입니다. 모델이 학습을 통해 이 응답을 생성할 수 있도록 되는 것이 파인튜닝의 목표입니다.

구체적인 예시를 보겠습니다.

```
예시 1: 번역 작업
{
  "instruction": "다음 문장을 영어로 번역하세요.",
  "input": "오늘 날씨가 매우 좋습니다.",
  "output": "The weather is very nice today."
}

예시 2: 질의응답 작업
{
  "instruction": "주어진 맥락을 바탕으로 질문에 답하세요.",
  "input": "맥락: 대한민국의 수도는 서울이며, 인구는 약 950만 명이다.\n질문: 대한민국의 수도는 어디인가?",
  "output": "대한민국의 수도는 서울입니다."
}

예시 3: 입력 없는 작업
{
  "instruction": "HTTP 상태 코드 404의 의미를 설명하세요.",
  "input": "",
  "output": "HTTP 404는 클라이언트가 요청한 리소스를 서버에서 찾을 수 없음을 나타내는 상태 코드입니다."
}
```

이 포맷이 표준이 된 이유는 인스트럭션 튜닝(instruction tuning)의 연구에서 비롯됩니다. 7.1.1에서 다룬 것처럼, 인스트럭션 튜닝은 모델이 자연어 지시를 이해하고 따르도록 학습시키는 방법입니다. 지시-입력-출력의 세 필드가 분리되어 있으면, 모델이 "무엇을 해야 하는가"(지시)와 "무엇에 대해 해야 하는가"(입력)를 구분하여 학습할 수 있습니다. 실제 사용 시에도 사용자는 지시와 입력을 함께 보내고, 모델은 출력을 생성하는 구조가 되므로, 학습 데이터의 구조와 실제 사용 패턴이 일치합니다.

이제 데이터의 **품질 지표**를 살펴보겠습니다. 데이터 품질을 체계적으로 관리하려면, "좋은 데이터란 무엇인가"를 측정 가능한 기준으로 정의해야 합니다. 핵심적인 품질 지표는 세 가지입니다.

첫 번째는 **다양성(diversity)**입니다. 학습 데이터가 다양한 주제, 표현 방식, 난이도, 맥락을 포함하는지를 측정합니다. 데이터가 특정 유형에 편중되면 모델도 그 유형에만 강해지고, 다른 유형에는 취약해집니다. 예를 들어, 고객 서비스 챗봇을 파인튜닝할 때 환불 문의만 수천 건 있고 배송 문의나 제품 문의는 수십 건뿐이라면, 모델은 환불 관련 질문에는 잘 답하지만 다른 유형의 질문에는 성능이 떨어집니다. 다양성을 측정하는 방법으로는, 데이터를 주제별로 분류한 뒤 각 주제의 비율을 확인하거나, 텍스트 임베딩을 구한 뒤 클러스터링하여 클러스터 분포를 보는 방법이 있습니다.

두 번째는 **정확성(accuracy)**입니다. Output 필드에 담긴 응답이 사실에 부합하고, 지시를 올바르게 수행한 결과인지를 측정합니다. 번역 작업이면 번역이 정확한지, 요약 작업이면 핵심 내용이 빠짐없이 포함되었는지, 코드 생성이면 코드가 실제로 동작하는지를 확인합니다. 정확하지 않은 데이터로 학습하면, 모델은 틀린 답변을 자연스럽게 생성하는 법을 배웁니다. 정확성 검증은 자동화가 어려운 경우가 많아, 도메인 전문가의 검토가 필요합니다.

세 번째는 **일관성(consistency)**입니다. 같은 유형의 질문에 대해 응답 형식과 어조가 일정한지를 측정합니다. 어떤 데이터에서는 격식체로 답하고 다른 데이터에서는 구어체로 답하면, 모델이 일관된 톤을 학습하기 어렵습니다. 마찬가지로, 같은 유형의 질문인데 한 데이터에서는 간결하게, 다른 데이터에서는 장황하게 답하면, 모델의 응답 길이도 불안정해집니다. 일관성은 어노테이션 가이드라인을 통해 사전에 확보하고, 작성된 데이터를 사후에 검토하여 유지합니다.

```
데이터 품질 지표 정리

+-----------+---------------------------+------------------------------+
| 지표      | 측정 대상                 | 부족할 때 발생하는 문제      |
+-----------+---------------------------+------------------------------+
| 다양성    | 주제, 표현, 난이도 분포   | 특정 유형에만 강하고         |
|           |                           | 나머지에 취약                |
+-----------+---------------------------+------------------------------+
| 정확성    | 응답의 사실성,            | 모델이 틀린 답변을           |
|           | 지시 수행 결과의 올바름   | 자연스럽게 생성              |
+-----------+---------------------------+------------------------------+
| 일관성    | 응답 형식, 어조,          | 모델의 응답 톤과 길이가      |
|           | 길이의 통일성             | 불안정                       |
+-----------+---------------------------+------------------------------+
```

품질 지표를 갖추었다면, 데이터를 실제로 정제하는 과정이 필요합니다. 정제의 핵심 단계 중 하나가 **필터링과 중복 제거**입니다.

데이터 필터링은 학습에 적합하지 않은 데이터를 걸러내는 과정입니다. 걸러내야 할 대상은 여러 가지입니다. 형식이 잘못된 데이터가 있습니다. JSON 구조가 깨졌거나, 필수 필드가 비어 있거나, 인코딩 오류로 깨진 문자가 포함된 경우입니다. 품질이 낮은 데이터가 있습니다. Output이 지나치게 짧아 의미 있는 응답이라 보기 어려운 경우, 지시와 무관한 응답이 담긴 경우, 의미 없는 반복 텍스트가 포함된 경우입니다. 유해하거나 부적절한 데이터가 있습니다. 혐오 표현, 개인정보, 저작권 침해 소지가 있는 콘텐츠가 포함된 경우입니다.

중복 제거는 동일하거나 거의 동일한 데이터를 제거하는 과정입니다. 완전히 같은 데이터가 여러 건 존재하는 경우(정확 중복)는 단순 비교로 제거할 수 있습니다. 문제는 표현만 약간 다르고 내용은 사실상 같은 데이터(유사 중복)입니다. "Python에서 리스트를 정렬하는 방법"과 "Python 리스트 정렬 방법을 알려주세요"는 다른 문장이지만 의미는 같습니다. 유사 중복을 탐지하려면 텍스트 유사도를 계산해야 합니다. 3.1.1에서 다룬 텍스트 임베딩을 활용하면, 두 문장의 임베딩 사이 코사인 유사도를 구하여 일정 임계값(예: 0.95) 이상이면 중복으로 판정할 수 있습니다. 중복 데이터가 많으면 모델이 특정 패턴을 과도하게 학습하여 다양성이 떨어집니다.

다음으로 **데이터 어노테이션 가이드라인**을 다루겠습니다. 어노테이션이란 원시 데이터에 레이블이나 응답을 부여하는 작업을 뜻합니다. 파인튜닝 맥락에서는 주로 Instruction에 대한 Output을 작성하는 것이 어노테이션에 해당합니다. 어노테이션의 품질이 곧 학습 데이터의 품질이므로, 어노테이터가 따라야 할 규칙을 문서화한 것이 어노테이션 가이드라인입니다.

좋은 어노테이션 가이드라인에는 다음 항목이 포함되어야 합니다. 응답의 어조와 형식 규칙이 있어야 합니다. 격식체를 사용할지 구어체를 사용할지, 응답의 최소/최대 길이는 얼마인지, 목록 형식과 서술 형식 중 어떤 것을 기본으로 할지를 정의합니다. 정확성 기준이 있어야 합니다. 확인되지 않은 정보는 포함하지 않는다, 코드 예시는 반드시 실행 가능해야 한다 등의 규칙을 명시합니다. 경계 사례 처리 방침이 있어야 합니다. 질문이 모호할 때, 답변하기 어려운 질문일 때, 여러 해석이 가능할 때 어떻게 처리할지를 안내합니다. 금지 사항이 있어야 합니다. 개인정보 포함 금지, 저작권 침해 금지, 유해 콘텐츠 금지 등을 명시합니다.

가이드라인을 작성한 후에는 어노테이터 간 일치도(inter-annotator agreement)를 측정하여, 가이드라인이 충분히 명확한지 검증합니다. 같은 지시에 대해 여러 어노테이터가 작성한 응답이 크게 다르면, 가이드라인이 모호하다는 뜻이므로 보완이 필요합니다.

마지막으로 **데이터셋 크기와 파인튜닝 성능의 관계**를 짚겠습니다. "데이터가 많을수록 성능이 좋다"는 일반적인 직관은 맞지만, 파인튜닝에서는 반드시 그렇지 않습니다. 사전 학습에서는 데이터 양과 성능이 거의 비례 관계를 보이지만, 파인튜닝에서는 일정 수준을 넘으면 성능 향상이 둔화됩니다. 이 현상이 발생하는 이유는, 파인튜닝이 모델에 새로운 지식을 가르치는 것이 아니라 이미 가진 지식을 특정 형식과 패턴으로 표출하도록 조정하는 과정이기 때문입니다. 패턴 조정에 필요한 정보는 비교적 적은 양의 데이터로도 전달할 수 있습니다.

실험적으로, 수백 건에서 수천 건의 고품질 데이터만으로도 의미 있는 파인튜닝 결과를 얻을 수 있다는 보고가 많습니다. 반면 수만 건의 저품질 데이터는 오히려 성능을 떨어뜨릴 수 있습니다. 핵심은 양보다 질이라는 것입니다. 데이터셋 크기를 결정할 때는 다음 원칙을 참고합니다. 최소 수백 건의 고품질 데이터로 시작하여 성능을 측정합니다. 성능이 목표에 미달하면 데이터를 추가하되, 추가 데이터의 품질을 기존과 동일하게 유지합니다. 성능 향상이 더 이상 관찰되지 않으면 데이터 추가를 중단하고, 다른 요소(모델 크기, 하이퍼파라미터, 학습 전략)를 조정합니다.

정리하면, 데이터 큐레이션은 파인튜닝에 사용할 학습 데이터를 수집, 정제, 구조화하는 전 과정이며, 파인튜닝의 성패를 좌우하는 핵심 요소입니다. 학습 데이터는 Instruction-Input-Output 포맷으로 구성하고, 다양성, 정확성, 일관성의 세 가지 품질 지표로 관리합니다. 형식 검증, 품질 필터링, 중복 제거를 통해 데이터를 정제하고, 어노테이션 가이드라인으로 작성 품질을 통제합니다. 데이터셋 크기는 양보다 질이 중요하며, 수백에서 수천 건의 고품질 데이터로도 효과적인 파인튜닝이 가능합니다.

다음 단원인 7.2.2에서는 합성 데이터 생성을 다룹니다. 고품질 데이터를 사람이 직접 작성하는 비용이 높을 때, LLM을 활용하여 학습 데이터를 자동으로 만들어 내는 방법입니다.

이 단원을 마치면 파인튜닝용 데이터셋의 품질 기준을 수립하고 큐레이션 파이프라인을 설계할 수 있습니다.
