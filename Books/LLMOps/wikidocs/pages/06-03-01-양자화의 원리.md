# 6.3.1 양자화의 원리

6.2절에서 GPU 추론 최적화 기법, 배칭 전략, KV Cache 관리를 다루었습니다. 이 기법들은 모두 모델의 가중치를 그대로 둔 채 추론 과정을 효율화하는 방식이었습니다. 그런데 모델의 가중치 자체가 차지하는 메모리를 줄일 수 있다면 어떨까요. 가중치가 작아지면 HBM에서 SRAM으로 옮기는 데이터량이 줄어들어 메모리 바운드 문제가 완화되고, GPU 메모리에 여유가 생겨 더 큰 배치를 구성할 수 있으며, 모델이 더 작은 GPU에서도 동작할 수 있게 됩니다. 이런 목적으로 가중치의 정밀도를 낮추는 기법이 **양자화(Quantization)**입니다.

양자화라는 용어가 다소 추상적으로 느껴질 수 있습니다. 원래 양자화는 물리학과 신호 처리에서 연속적인 값을 이산적인(불연속적인) 값으로 변환하는 것을 뜻합니다. 딥러닝에서의 양자화도 같은 원리입니다. 모델 가중치는 원래 매우 정밀한 숫자(소수점 이하 7자리 이상)로 표현되어 있는데, 이를 덜 정밀한 숫자(소수점 이하 3자리 정도)나 정수로 바꾸는 것입니다. 정밀도를 낮추면 숫자 하나를 표현하는 데 필요한 비트 수가 줄어들고, 결과적으로 모델의 전체 크기가 작아집니다.

양자화를 구체적으로 이해하려면, 컴퓨터가 숫자를 표현하는 방식부터 알아야 합니다. 컴퓨터에서 소수점이 있는 숫자를 표현하는 가장 일반적인 방식이 **부동소수점(floating point)** 형식입니다. "부동"이라는 말은 소수점의 위치가 고정되어 있지 않고 떠다닌다는 뜻입니다. 부동소수점 숫자는 부호 비트, 지수부(exponent), 가수부(mantissa)의 세 부분으로 이루어집니다. 부호 비트는 양수인지 음수인지를 나타내고, 지수부는 숫자의 크기 범위(매우 크거나 매우 작은)를 결정하며, 가수부는 유효 숫자의 정밀도를 결정합니다.

```
부동소수점 숫자의 구조

|부호(1비트)|   지수부    |         가수부          |
     +/-     크기 범위 결정    정밀도 결정

예시: -6.75를 이진수로 표현하면
부호: 1 (음수)
지수: 이진수 110.11 -> 1.1011 x 2^2 -> 지수부 = 2
가수: 1011...
```

LLM 학습에서 가장 흔히 사용되는 정밀도는 **FP32(32비트 부동소수점)**입니다. FP32는 부호 1비트, 지수 8비트, 가수 23비트로 구성됩니다. 숫자 하나를 표현하는 데 4바이트(32비트)를 사용합니다. 매우 넓은 범위의 숫자를 높은 정밀도로 표현할 수 있어 학습 시 수치 안정성이 뛰어납니다. 그러나 70억 개의 파라미터를 FP32로 저장하면 7,000,000,000 x 4바이트 = 28GB가 필요합니다.

**FP16(16비트 부동소수점)**은 부호 1비트, 지수 5비트, 가수 10비트로 구성됩니다. 숫자 하나에 2바이트(16비트)만 사용하므로, FP32 대비 메모리가 절반으로 줄어듭니다. 70억 파라미터 모델이 14GB로 작아집니다. 표현 가능한 숫자 범위가 FP32보다 좁고 정밀도도 떨어지지만, 추론에는 대부분 충분합니다. 다만 FP16은 표현 가능한 최대값이 약 65,504로, 학습 중 그래디언트가 이 범위를 초과하면 오버플로가 발생하는 문제가 있습니다.

**BF16(Brain Float 16)**은 Google이 제안한 16비트 형식으로, 부호 1비트, 지수 8비트, 가수 7비트로 구성됩니다. FP16과 같은 16비트이지만, 지수부가 FP32와 동일한 8비트이므로 표현 범위가 FP32와 같습니다. 대신 가수부가 7비트로 줄어 정밀도가 FP16보다 낮습니다. 학습 시 오버플로 위험이 적으면서도 메모리를 절반으로 줄이는 이점이 있어, 최근 LLM 학습에서 널리 사용됩니다.

```
부동소수점 형식 비교

형식    전체 비트  부호  지수  가수   파라미터당 크기  7B 모델 크기
------  --------  ----  ----  ----   --------------  -----------
FP32    32비트    1     8     23     4바이트          28 GB
FP16    16비트    1     5     10     2바이트          14 GB
BF16    16비트    1     8     7      2바이트          14 GB
```

이제 양자화의 핵심인 **정수 양자화**를 살펴보겠습니다. 부동소수점을 더 낮은 비트의 부동소수점으로 바꾸는 것도 일종의 양자화이지만, 더 극적인 크기 절감은 부동소수점을 정수(integer)로 변환할 때 이루어집니다.

**INT8(8비트 정수)**은 -128에서 127까지의 정수만 표현합니다. 숫자 하나에 1바이트만 필요합니다. FP16 대비 메모리가 다시 절반으로 줄어 70억 파라미터 모델이 약 7GB가 됩니다. **INT4(4비트 정수)**는 -8에서 7까지 16개의 값만 표현합니다. 숫자 하나에 0.5바이트(4비트)를 사용하여, 70억 파라미터 모델이 약 3.5GB까지 작아집니다.

부동소수점 가중치를 정수로 변환하는 과정을 구체적으로 설명하겠습니다. 가중치 그룹에서 최솟값과 최대값을 찾고, 이 범위를 정수의 범위에 대응시키는 **스케일 팩터(scale factor)**를 계산합니다. 각 가중치를 스케일 팩터로 나누고 반올림하면 정수 값이 됩니다. 추론 시에는 정수 값에 스케일 팩터를 다시 곱하여 원래 범위의 값으로 복원합니다.

```
양자화 예시 (FP16 -> INT8)

원래 가중치 (FP16): [0.12, -0.45, 0.89, -0.23, 0.67]

1. 범위 파악: 최소 = -0.45, 최대 = 0.89
2. 스케일 팩터 계산: scale = 0.89 / 127 = 0.00701
3. 양자화: 각 값을 scale로 나누고 반올림
   0.12 / 0.00701 = 17.1  -> 17
  -0.45 / 0.00701 = -64.2 -> -64
   0.89 / 0.00701 = 127.0 -> 127
  -0.23 / 0.00701 = -32.8 -> -33
   0.67 / 0.00701 = 95.6  -> 96

양자화된 값 (INT8): [17, -64, 127, -33, 96]

4. 역양자화 (추론 시 복원):
   17 x 0.00701 = 0.119   (원래 0.12, 오차 0.001)
  -64 x 0.00701 = -0.449  (원래 -0.45, 오차 0.001)
```

이 예시에서 볼 수 있듯이, 양자화와 역양자화 과정에서 반올림으로 인한 작은 오차가 발생합니다. 이 오차가 **양자화 오류(quantization error)**입니다. 개별 가중치의 오차는 작지만, 수십억 개의 가중치에서 오차가 누적되면 모델의 출력 품질에 영향을 미칠 수 있습니다.

양자화에는 **대칭 양자화(symmetric quantization)**와 **비대칭 양자화(asymmetric quantization)** 두 가지 방식이 있습니다.

대칭 양자화는 0을 중심으로 양쪽 대칭인 범위에 가중치를 매핑합니다. INT8의 경우 -127에서 127까지의 범위를 사용합니다. 가중치의 절대값 중 최대값을 찾아 스케일 팩터를 계산합니다. 0에 가까운 값은 양자화 후에도 0에 가깝게 유지됩니다. 구현이 단순하고 연산 속도가 빠르다는 장점이 있습니다. 하지만 가중치 분포가 비대칭인 경우(예를 들어, 대부분의 가중치가 양수인 경우) 범위의 절반이 낭비될 수 있습니다.

비대칭 양자화는 실제 가중치의 최솟값과 최대값을 각각 정수 범위의 최솟값과 최대값에 대응시킵니다. INT8의 경우 가중치의 최솟값을 -128에, 최대값을 127에 매핑합니다. 0이 정확히 0으로 매핑되지 않을 수 있으므로, 스케일 팩터 외에 **제로 포인트(zero point)**라는 추가 파라미터가 필요합니다. 비대칭 분포의 가중치를 더 정확하게 표현할 수 있지만, 제로 포인트 연산이 추가되어 대칭 양자화보다 계산이 복잡합니다.

```
대칭 양자화 vs 비대칭 양자화

대칭 양자화:
  가중치 범위: [-0.5, 0.9]
  매핑: |최대 절대값| = 0.9
  scale = 0.9 / 127
  INT8 범위:  -127 ......... 0 ......... 127
  가중치 매핑: -0.5 ......... 0 ......... 0.9
                        ^
              0이 정확히 0에 대응

비대칭 양자화:
  가중치 범위: [-0.5, 0.9]
  매핑: -0.5 -> -128, 0.9 -> 127
  scale = (0.9 - (-0.5)) / (127 - (-128)) = 1.4 / 255
  zero_point = round(-128 - (-0.5) / scale)
  INT8 범위:  -128 ......... 0 ......... 127
  가중치 매핑: -0.5 .................... 0.9
                          ^
              0이 정확히 0에 대응하지 않을 수 있음
```

양자화가 모델 성능에 미치는 영향을 세 가지 측면에서 살펴보겠습니다.

첫째, **메모리 절감**입니다. 앞서 계산한 것처럼, FP16에서 INT8로 양자화하면 모델 크기가 절반으로, INT4로 양자화하면 4분의 1로 줄어듭니다. 이는 더 작은 GPU에서 모델을 실행할 수 있게 해 주고, 같은 GPU에서 더 많은 요청을 동시에 처리할 수 있게 합니다.

둘째, **추론 속도 향상**입니다. 6.2.1에서 설명한 메모리 바운드 상황에서, HBM에서 읽어야 할 데이터가 줄어들면 데이터 전송 시간이 단축됩니다. INT8 가중치는 FP16 대비 절반의 데이터만 전송하면 되므로, 이론적으로 메모리 바운드 구간에서 약 2배의 속도 향상을 기대할 수 있습니다. 실제로는 양자화/역양자화 오버헤드, 하드웨어 특성 등에 따라 실측 향상 폭이 달라집니다.

셋째, **출력 품질 변화**입니다. 양자화는 가중치의 정밀도를 떨어뜨리므로, 모델의 출력이 원본과 달라질 수 있습니다. 일반적으로 FP32에서 FP16이나 BF16으로 변환할 때는 품질 저하가 거의 없습니다. INT8 양자화에서는 대부분의 작업에서 품질이 유사하게 유지되지만, 수학 추론이나 코드 생성처럼 정밀한 계산이 필요한 작업에서는 미세한 차이가 관찰될 수 있습니다. INT4 양자화에서는 일부 작업에서 눈에 띄는 품질 저하가 발생할 수 있으며, 이를 보상하기 위한 다양한 기법이 연구되고 있습니다.

```
양자화 수준별 영향 비교

정밀도   메모리 (7B)   속도 향상*   품질 영향
------   ----------    ----------   ---------
FP32     28 GB         기준선       기준선
FP16     14 GB         ~2x          거의 동일
BF16     14 GB         ~2x          거의 동일
INT8     7 GB          ~2x-3x       대부분 유사
INT4     3.5 GB        ~3x-4x       작업에 따라 저하 가능

* FP32 대비 추론 속도 향상 (메모리 바운드 구간 기준, 하드웨어별 상이)
```

마지막으로 양자화의 적용 시점에 따른 분류를 살펴보겠습니다. **Post-Training Quantization(PTQ, 학습 후 양자화)**는 이미 학습이 완료된 모델에 양자화를 적용하는 방식입니다. 학습 과정을 다시 수행할 필요 없이, 가중치를 낮은 정밀도로 변환하기만 하면 됩니다. 적용이 간편하고 빠르다는 장점이 있습니다. 소량의 보정 데이터(calibration data)를 사용하여, 양자화 오류를 최소화하는 스케일 팩터를 찾는 과정이 포함되기도 합니다. 6.3.2에서 다룰 GPTQ, AWQ 등이 PTQ에 해당합니다.

**Quantization-Aware Training(QAT, 양자화 인식 학습)**은 학습 과정에서부터 양자화를 고려하는 방식입니다. 학습 중에 가중치를 의도적으로 양자화했다가 역양자화하는 과정을 시뮬레이션합니다. 모델이 학습 단계에서 양자화 오류에 적응하도록 유도하여, PTQ보다 품질 저하가 적습니다. 하지만 학습 과정 전체를 다시 수행해야 하므로 시간과 비용이 많이 듭니다. 대규모 언어 모델에서는 학습 비용이 매우 높기 때문에, 실무에서는 PTQ가 훨씬 많이 사용됩니다.

```
PTQ vs QAT

Post-Training Quantization (PTQ)
  학습 완료된 모델 --> 양자화 적용 --> 양자화된 모델
  장점: 빠르고 간편, 학습 불필요
  단점: 공격적인 양자화(INT4)에서 품질 저하 가능

Quantization-Aware Training (QAT)
  학습 중 양자화 시뮬레이션 포함 --> 양자화에 강인한 모델
  장점: 같은 비트 수에서 PTQ보다 높은 품질
  단점: 학습 비용 큼, LLM에서는 비현실적인 경우 많음
```

정리하면, 양자화는 모델 가중치의 정밀도를 낮추어 메모리 사용량과 데이터 전송량을 줄이는 기법입니다. FP32, FP16, BF16은 부동소수점 형식으로 정밀도와 메모리 사이의 균형이 다르고, INT8과 INT4는 정수 형식으로 더 극적인 크기 절감을 제공합니다. 대칭 양자화는 구현이 단순하고, 비대칭 양자화는 비대칭 분포에 더 정확합니다. PTQ는 학습 없이 빠르게 적용할 수 있고, QAT는 학습 과정에서 양자화 오류에 적응하여 더 높은 품질을 유지합니다.

다음 단원인 6.3.2에서는 실무에서 널리 쓰이는 양자화 방식인 GPTQ, AWQ, GGUF를 비교합니다. 각 방식의 원리, 적용 방법, 그리고 서빙 프레임워크에서의 지원 현황을 살펴봅니다.

이 단원을 마치면 양자화의 기본 원리를 설명하고, 부동소수점과 정수 정밀도의 특성을 비교하며, PTQ와 QAT의 차이를 구분할 수 있습니다.
