# 1.1.1 이 책이 다루는 것: 두 에이전트를 해부하는 이유

2026년 상반기, 개인 컴퓨터나 작은 서버에 늘 켜져 있으면서 메신저로 지시를 받는 새로운 종류의 AI 소프트웨어가 빠르게 퍼졌습니다. 브라우저에 질문을 넣고 창을 닫으면 끝나는 챗봇과 달리, 이 소프트웨어는 꺼지지 않는 프로세스로 사용자의 머신에 상주합니다. 시키지 않아도 주기적으로 할 일을 점검하고, 파일을 만지고, 결과를 WhatsApp이나 Telegram으로 보고합니다. 이 책은 이런 소프트웨어를 **상주형 개인 AI 에이전트**라고 부릅니다. 한 사람을 위해 항상 떠 있고, 대화가 끝나도 기억이 남고, 필요한 능력을 스스로 늘려 가는 에이전트라는 뜻입니다.

이 장르를 연 대표 시스템이 이 책의 두 주인공인 **OpenClaw**와 **Hermes Agent**입니다. OpenClaw는 2025년 11월 한 개발자의 개인 실험으로 출발한 TypeScript 기반 프로젝트입니다. 2026년 1월 말 지금 이름으로 다시 공개되면서 48시간 만에 10만 스타를 모았고, 2026년 7월 현재 저장소에는 38만이 넘는 스타가 표시됩니다. Hermes Agent는 AI 연구조직 Nous Research가 2026년 2월 말 공개한 Python 기반 시스템입니다. OpenClaw의 파생본이 아닌 독자 구현이며, 경험에서 스스로 스킬을 만들어 내는 학습 루프를 대표 기능으로 내세웁니다. 둘 다 MIT 라이선스 오픈소스라서 내부 구조를 코드와 공식 문서 수준까지 확인할 수 있습니다. 같은 문제에 두 팀이 서로 다른 답을 내놓았기 때문에, 나란히 놓고 뜯어 보면 한쪽만 볼 때는 드러나지 않는 설계의 선택지가 보입니다.

해부라고 했지만 겨누는 대상은 모델 자체가 아닙니다. 두 시스템 모두 뇌에 해당하는 LLM은 외부 API로 갈아 끼울 수 있게 두고, 실행과 기억과 채널 연결이라는 몸을 사용자의 머신에 둡니다. 이 책이 뜯어 보는 것은 그 몸입니다. 게이트웨이, 세션, 메모리, 스킬, 보안 장치가 그 부품들입니다.

해부의 도구는 두 개의 질문입니다. 첫 번째 질문은 '왜 이렇게 설계했는가'입니다. OpenClaw는 모든 메신저 연결을 단일 상주 프로세스 하나에 몰아넣고, 기억을 평문 마크다운 파일로 남깁니다. Hermes Agent는 에이전트 루프를 단일 코어로 통일하고 플랫폼별 차이를 얇은 어댑터로 밀어냅니다. 이런 선택 뒤에는 세션 충돌 방지, 프롬프트 캐시 비용, 감사 가능성 같은 구체적인 이유가 있습니다. 이 책은 구성요소마다 역할, 공식적으로 밝힌 설계 이유, 그 밑에 깔린 소프트웨어 개념을 차례로 짚습니다. 두 번째 질문은 '실무에서 무엇을 챙길 것인가'입니다. 상주형 에이전트는 API 키와 셸 접근 권한을 한 몸에 모은 채 24시간 돌아갑니다. 실제로 OpenClaw는 2026년 상반기에 대규모 노출 사고와 여러 건의 취약점 공개를 겪었습니다. 그래서 각 설계를 설명할 때 배포 형태, 운영 비용, 보안 기본값처럼 도입하는 사람이 확인할 항목을 함께 다룹니다.

전체는 여덟 개 Phase로 짜여 있습니다.

```
Phase 1      상주형 개인 AI 에이전트라는 장르   (공통 기초)
Phase 2-4    OpenClaw     : 뼈대 -> 기억·능동성·확장 -> 보안과 운영
Phase 5-7    Hermes Agent : 뼈대 -> 기억·자가 개선 -> 보안과 운영
Phase 8      비교, 그리고 이식 가능한 설계
```

Phase 1은 상주형 개인 AI 에이전트라는 장르 자체를 정리해 이후 장에서 쓸 공용 어휘를 만듭니다. Phase 2부터 4까지는 OpenClaw 차례입니다. 게이트웨이와 실행 모델이라는 뼈대에서 시작해, 기억과 능동성과 확장 장치를 거쳐, 보안 사고와 운영 노하우로 마무리합니다. Phase 5부터 7까지는 같은 순서로 Hermes Agent를 다룹니다. 단일 코어와 어댑터라는 뼈대, 기억과 자가 개선, 보안과 운영 순서입니다. 마지막 Phase 8은 두 시스템을 비교하고, 어느 쪽을 쓰지 않더라도 자기 시스템에 옮겨 심을 수 있는 설계 패턴을 추립니다.

수치와 버전에는 기준 시점이 필요합니다. 두 프로젝트의 움직임이 매우 빠르기 때문입니다. OpenClaw는 2026년 7월 14일에 v2026.7.1을, Hermes Agent는 2026년 7월 20일에 v0.19.0을 냈습니다. 스타 수 같은 지표는 측정 시점에 따라 몇 배씩 달라지기도 합니다. 그래서 이 책의 모든 수치, 버전, 기능 설명은 2026년 7월 21일을 기준으로 삼습니다. 시점에 따라 편차가 큰 수치는 본문에 측정 시점을 함께 적습니다. 다만 책의 무게 중심은 숫자보다 설계 원리에 있습니다. 버전이 올라가도 게이트웨이를 하나로 두는 이유나 기억을 두 층으로 나누는 이유는 그대로 남기 때문입니다.

이 책은 두 시스템을 직접 설치해 부려 보려는 사람과, 자기 서비스에 에이전트 구조를 설계하려는 개발자를 함께 겨냥합니다. 전제 지식은 LLM API를 호출해 보고 터미널을 다뤄 본 경험 정도면 충분합니다. 게이트웨이나 프롬프트 캐시 같은 용어는 처음 나올 때마다 배경부터 설명합니다. 읽는 방법은 처음부터 순서대로 읽는 것이 기본입니다. 한쪽 시스템만 급하다면 Phase 1을 마친 뒤 해당 Phase 묶음으로 건너뛰어도 됩니다. 다만 Phase 8의 비교는 양쪽을 모두 안다는 전제 위에서 진행됩니다. 각 단원이 하나의 초점만 다루므로, 나중에 특정 구성요소만 찾아 읽는 참고서로 써도 됩니다.

정리하면, 이 책은 상주형 개인 AI 에이전트라는 장르를 연 두 시스템, OpenClaw와 Hermes Agent를 사례로 삼습니다. 그리고 '왜 이렇게 설계했는가'와 '실무에서 무엇을 챙길 것인가'라는 두 질문으로 그 내부를 해부합니다. 전체 구성은 공통 기초인 Phase 1에서 시작해 OpenClaw의 Phase 2에서 4, Hermes Agent의 Phase 5에서 7을 지나 비교인 Phase 8로 끝납니다. 모든 수치와 버전은 2026년 7월 21일 기준이며, 독자에게는 LLM API 호출 경험 정도만 전제합니다.

1.1.2에서는 첫 질문의 출발점으로, 챗봇과 상주형 에이전트를 가르는 네 가지 차이를 다룹니다.

이 단원을 마치면 이 책이 두 에이전트를 해부하는 두 질문과 여덟 개 Phase의 전체 구성, 그리고 수치의 기준 시점을 설명할 수 있습니다.
