# 7.2.3 비용 구조와 캐시 무효화 함정

7.2.1에서 설치 명령 한 줄이 실제로 무엇을 하는지 살펴봤습니다. 그렇게 설치를 마치고 나면 자연스럽게 떠오르는 질문이 있습니다. 이 에이전트를 상시 켜 두면 매달 얼마가 나가는가입니다. 이 단원에서는 Hermes를 운영할 때의 비용이 어디에서 나오는지 그 구조를 먼저 밝히고, 그 안에서 가장 큰 절감 레버가 왜 모델 라우팅과 캐시 유지인지를 실측 사례와 함께 설명합니다.

먼저 큰 그림부터 잡겠습니다. Hermes의 소프트웨어 자체는 무료입니다. 오픈소스이므로 설치본을 쓰는 데 라이선스 비용이 붙지 않습니다. 그러면 돈은 어디서 나갈까요. 두 군데입니다. 하나는 판단을 맡는 뇌를 빌려 쓰는 값인 **LLM API** 요금이고, 다른 하나는 에이전트라는 몸을 24시간 재워 두는 서버 임대료인 **호스팅** 비용입니다. 즉 비용은 LLM API와 호스팅의 합으로만 이루어집니다. 이 구조를 머리에 넣으면 어디를 손봐야 청구서가 줄어드는지가 명확해집니다.

이 두 항목의 크기 감각을 실측 보고로 잡아 보겠습니다. 실사용자들이 남긴 기록을 보면 월 API 지출은 대략 30달러에서 65달러 사이, 호스팅까지 포함한 총액은 15달러에서 50달러 남짓으로 모입니다. 수치의 폭이 넓은 것은 어떤 모델을 얼마나 부리느냐에 따라 달라지기 때문입니다. 호스팅 쪽을 보면 상당수 사용자가 월 5달러에서 20달러 수준의 저사양 VPS 위에서 돌립니다. 라즈베리파이 4나 맥 미니 같은 소형 기기에서 상시 운영한 사례도 흔합니다. 다시 말해 몸을 재워 두는 비용은 이미 충분히 저렴하며, 청구서를 좌우하는 쪽은 대체로 뇌를 쓰는 값, 곧 API 요금입니다.

그렇다면 API 요금을 줄이는 가장 큰 레버는 무엇일까요. 바로 **모델 라우팅**입니다. 모델 라우팅이란 어떤 작업에 어떤 모델을 배정할지 정하는 결정을 말합니다. 같은 일을 시켜도 최상위 모델은 비싸고, 한 단계 낮은 모델이나 다른 사업자의 모델은 훨씬 쌉니다. 그래서 라우팅을 조금만 바꿔도 절감 폭이 큽니다. 한 사용자는 여러 모델을 섞어 부르는 조합으로 갈아탄 뒤 토큰 지출을 약 90퍼센트 줄였습니다. 5일에 130달러 나가던 지출이 같은 기간 10달러 수준으로 내려간 기록입니다. 이 수치는 특정 시점의 개인 보고이므로 그대로 재현되리라 단정하긴 어렵지만, 라우팅이라는 레버가 얼마나 크게 움직이는지는 분명히 보여 줍니다. Hermes가 배경 리뷰 같은 가벼운 작업을 저렴한 모델로 따로 보내도록 설정할 수 있는 것도 같은 맥락입니다.

여기서 반드시 짚어야 할 함정이 하나 있습니다. 절감을 노려 모델을 자주 갈아 끼우다가 오히려 요금이 튀는 경우입니다. 이는 6.1.3에서 다룬 prefix cache의 경제학과 직결됩니다. 그 원리를 실무 경고로 다시 꺼내면 이렇습니다. LLM API는 프롬프트 앞부분이 이전과 똑같으면 그 부분을 다시 계산하지 않고 캐시에서 꺼내 훨씬 싸게 처리합니다. 그런데 **캐시 무효화**가 일어나면 이 할인이 사라집니다. 대화 중에 모델을 전환하거나 자격증명을 회전하면 캐시의 전제가 깨지고, 그다음 턴은 앞부분까지 통째로 다시 계산해 전액이 재과금됩니다. 정리하면 라우팅 변경은 아끼는 레버지만, 한 대화 안에서 지나치게 자주 바꾸면 캐시가 깨져 절감분을 도로 까먹습니다. 바꿀 때 바꾸되 필요 이상으로 흔들지 않는 것이 쌉니다.

마지막으로 비용을 눈으로 확인하고 다듬는 실무 도구를 소개합니다. 세 가지 명령을 기억하면 됩니다. 긴 대화가 쌓여 매 턴 입력 비용이 불어날 때는 /compress로 지난 대화를 요약해 컨텍스트를 줄입니다. 지금까지 무엇에 얼마를 썼는지 파악하려면 /usage로 소비 내역을 조회하고, 최근 30일가량의 소비 흐름과 패턴을 보려면 /insights를 씁니다.

```
비용 = LLM API 요금  +  호스팅(서버 임대료)
        │  (청구서를 좌우)      │  (VPS 월 $5~20, 이미 저렴)
        │                      │
   레버: 모델 라우팅        확인: /usage · /insights
   함정: 잦은 모델 전환·자격증명 회전 → 캐시 무효화 → 다음 턴 전액 재과금
   완화: /compress 로 컨텍스트 축소
```

정리하면, Hermes의 비용은 무료 소프트웨어가 아니라 LLM API와 호스팅의 합에서 나오며, 호스팅은 저사양 VPS로 이미 저렴하기에 청구서를 좌우하는 쪽은 API 요금입니다. 그 API 요금을 줄이는 가장 큰 레버는 모델 라우팅이고, 실측 사례에서 라우팅 조합 변경만으로 큰 폭의 절감이 확인됩니다. 다만 모델 전환과 자격증명 회전은 prefix cache를 무효화해 재과금을 부르므로 자주 흔들지 않아야 하며, /compress와 /usage, /insights로 소비를 다듬고 들여다볼 수 있습니다.

7.2.4에서는 이렇게 비용을 관리하며 상시 운영하는 Hermes를 실제로 어떻게 부리는지, 특히 코딩 에이전트들을 지휘하는 지휘자로 쓰는 활용 패턴을 살펴봅니다.

이 단원을 마치면 Hermes의 비용이 LLM API와 호스팅으로 이루어진 구조를 설명하고, 모델 라우팅과 캐시 유지가 왜 가장 큰 절감 레버인지 실측 사례와 함께 설명할 수 있습니다.
