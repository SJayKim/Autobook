# 6.3.2 Nous Portal: 구독 게이트웨이와 단기 자격증명

6.3.1에서는 Hermes가 여러 모델 공급자를 하나의 인터페이스로 다루고, 대화 도중에도 모델을 갈아 끼운다는 점을 보았습니다. 그런데 실제로 모델을 쓰려면 공급자마다 계정을 만들고 결제를 등록하고 접속에 필요한 열쇠를 받아 와야 합니다. 이 준비 과정이 번거롭기도 하지만, 더 큰 문제는 그 열쇠를 안전하게 보관하는 일입니다. 이 단원에서 다루는 Nous Portal은 바로 그 두 가지, 즉 모델을 모아 주는 편의와 열쇠를 안전하게 다루는 방식을 함께 푸는 장치입니다.

먼저 배경으로 열쇠 이야기를 해야 합니다. 외부 모델을 프로그램에서 부르려면 보통 **API 키**가 필요합니다. API 키란 서비스가 발급하는 긴 문자열입니다. 요청을 보낼 때마다 이걸 함께 실어 보내면, 서비스가 나를 알아보고 사용료를 내 계정에 매깁니다. 일종의 장수명 비밀번호인 셈입니다. 문제는 이 비밀번호가 한 번 만들면 스스로 지우기 전까지 계속 유효하다는 점입니다. 게다가 프로그램이 자동으로 쓰려면 파일 어딘가에 적어 두어야 합니다. 그런데 상주형 에이전트는 자격증명과 셸 접근을 한 몸에 모은 고가치 표적입니다. 그 파일이 새어 나가면, 유출된 API 키로 누군가 내 계정을 마음껏 쓸 수 있습니다.

이 위험을 줄이는 표준 도구가 두 가지 있습니다. 하나는 **OAuth**입니다. OAuth는 비밀번호를 상대에게 직접 넘기지 않고, 브라우저에서 한 번 로그인해 특정 프로그램에 권한만 위임하는 방식입니다. 다른 서비스에 가입할 때 구글 계정으로 로그인하는 그 절차가 OAuth입니다. 비밀번호는 원래 서비스에만 있고, 프로그램은 대신 쓸 권한만 건네받습니다. 다른 하나는 **JWT**입니다. JWT는 짧은 수명을 가진 출입증이라고 생각하면 됩니다. 놀이공원에서 신분증을 맡기고 받은 하루짜리 입장 팔찌처럼, 정해진 시간이 지나면 저절로 쓸모가 없어지는 임시 증표입니다. Nous Portal은 이 둘을 엮어 앞의 유출 위험을 구조적으로 낮춥니다.

Nous Portal은 Nous Research가 운영하는 통합 구독 게이트웨이입니다. 여러 모델 공급자를 하나의 구독과 하나의 청구서 아래로 묶고, 접속에 필요한 자격증명 발급까지 대신 맡는 창구라고 보면 됩니다. 처음 붙이는 절차는 명령 한 줄로 시작합니다.

```
hermes setup --portal
```

이 명령을 실행하면 브라우저가 열리고 OAuth 로그인 화면이 뜹니다. 여기서 한 번 로그인해 Hermes에 사용 권한을 위임하면 준비가 끝납니다. API 키를 손으로 복사해 붙일 필요가 없습니다. 이후 구독 상태나 남은 크레딧이 궁금하면 hermes portal info로 확인합니다.

여기서 핵심 설계가 등장합니다. OAuth 로그인이 끝나면 Portal은 장수명 API 키를 파일에 남기지 않습니다. 대신 **refresh token**이라는 재발급용 증표 하나만 로컬 자격증명 파일에 보관합니다. refresh token은 그 자체로 모델을 부르는 열쇠가 아니라, 진짜 출입증을 새로 받아 오는 교환권입니다. Hermes는 모델에 요청을 보낼 때마다 이 refresh token으로 짧은 수명의 JWT를 새로 발급받고, 그 JWT를 요청에 실어 보냅니다. 흐름을 그리면 다음과 같습니다.

```
[브라우저 OAuth 로그인] ──▶ refresh token 저장 (로컬, 재발급 전용)
                                     │
        모델 요청이 생길 때마다      ▼
   refresh token ──▶ 단기 JWT 발급 ──▶ 요청에 실어 전송
                        (짧은 수명 뒤 자동 만료)
```

이 방식이 주는 이점을 **단기 자격증명**(short-lived credential)이라고 부릅니다. 파일에 남는 것은 그 자체로는 모델을 부를 수 없는 refresh token뿐이고, 실제로 모델에 닿는 JWT는 매번 새로 만들어져 금세 만료됩니다. 설령 요청 하나에 쓰인 JWT가 새어 나가도, 수명이 짧아 곧 무용지물이 됩니다. 장수명 API 키를 파일에 두었을 때처럼 유출 하나가 오래 살아남아 계정을 위협하는 상황을 구조적으로 줄이는 것입니다. Nous Portal은 이 설계의 목적을 장수명 API 키 노출을 없애는 데 둔다고 밝힙니다.

편의 쪽 이점도 큽니다. Portal은 구독 하나로 300개가 넘는 프런티어 모델을 부를 수 있다고 표방합니다. 서로 다른 회사가 만든 Claude, GPT 계열, Gemini, 그리고 DeepSeek이나 Qwen 같은 여러 계열을 각각 계약하지 않고 한 창구에서 씁니다. 사용료도 한 장의 청구서로 정산됩니다. 6.3.1에서 본 여러 공급자 추상화가 계정 관리와 결제 층위에서도 하나로 합쳐지는 셈입니다.

구독은 여러 티어로 나뉩니다. 무료로 시작하는 Free, 그 위로 Plus, Super, Ultra 순서로 올라가며, 상위 티어일수록 매달 포함되는 사용 크레딧이 커지는 구조입니다. 다만 각 티어의 정확한 월 요금이나 크레딧 액수는 여기서 단정하지 않겠습니다. 이 가격 정보는 공식 문서가 직접 싣지 않고 제3자가 정리한 내용을 바탕으로 하며, 요금제는 시기에 따라 바뀔 수 있기 때문입니다. 개괄만 기억하면 됩니다. 부담 없이 시험해 볼 무료 등급이 있고, 쓰는 양에 맞춰 위 등급으로 올려 갈 수 있다는 점입니다.

Nous Portal이 모으는 것은 모델만이 아닙니다. 에이전트가 자주 쓰는 외부 기능도 같은 구독 아래 **도구 게이트웨이**로 묶여 있습니다. 도구 게이트웨이란 웹검색, 이미지 생성, 음성 합성처럼 모델 바깥의 작업을 처리하는 유료 서비스들을 Portal이 대신 연결해 주는 통로입니다. 웹에서 정보를 긁어 오는 웹검색, 그림을 만들어 내는 이미지 생성이 여기 들어갑니다. 글을 소리로 바꾸는 TTS(text-to-speech, 글을 음성으로 읽어 주는 기능)도 포함됩니다. 화면 없는 서버에서 웹페이지를 실제로 조작하는 클라우드 브라우저까지 이 창구를 통해 쓸 수 있습니다. 이런 기능도 원래는 각각 별도 계정과 열쇠가 필요하지만, Portal을 쓰면 모델과 같은 방식으로 한데 묶여 관리됩니다.

정리하면, Nous Portal은 Nous Research가 운영하는 통합 구독 게이트웨이로, hermes setup --portal의 브라우저 OAuth 한 번으로 붙습니다. 파일에는 재발급용 refresh token만 남기고 요청마다 짧은 수명의 JWT를 새로 발급하는 단기 자격증명 설계로 장수명 API 키 노출을 없앱니다. 구독 하나로 300개 넘는 모델을 한 청구서에 묶고, Free/Plus/Super/Ultra 티어로 나뉘며, 웹검색과 이미지 생성, TTS, 클라우드 브라우저 같은 도구 게이트웨이까지 함께 제공합니다. 6.3.3에서는 이렇게 외부 구독으로 모델을 부르는 방식과 대비되는 쪽, 즉 DGX Spark 같은 장비부터 라즈베리파이까지 모델을 내 장비에서 직접 돌리는 로컬 구동을 다룹니다. 이 단원을 마치면 Nous Portal이 OAuth와 단기 JWT로 장수명 API 키 노출을 없애는 구조, 그리고 구독 티어와 도구 게이트웨이를 설명할 수 있습니다.
