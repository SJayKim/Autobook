# 1.3.3 첫 MCP 서버 구현

지금까지 다섯 프리미티브와 3계층 구조, 그리고 capability negotiation을 살펴보았습니다. 이 단원에서는 그 개념들을 묶어 실제로 동작하는 가장 단순한 MCP 서버를 만들어 봅니다. 목표는 두 가지입니다. 첫째, echo와 파일 읽기 두 도구를 노출하는 서버를 직접 만든다. 둘째, 그것을 Claude Desktop에 연결해 동작을 확인한다.

가장 먼저 SDK를 선택합니다. MCP는 공식 SDK를 여러 언어로 제공합니다. Python(`mcp` 패키지), TypeScript(`@modelcontextprotocol/sdk`), Java, Kotlin, C#, Rust 등이 있습니다. 이 책에서는 Python을 기본으로 사용하고, 필요할 때 TypeScript를 보조로 보여 줍니다. 설치는 한 줄입니다.

```
pip install mcp
```

Python SDK는 저수준 Server 클래스와 그 위의 고수준 추상화인 FastMCP를 함께 제공합니다. FastMCP는 데코레이터로 도구·자원·프롬프트를 등록할 수 있게 해 주는 라이브러리로, 직관적이고 빠른 시작에 적합합니다. 다음은 가장 단순한 echo 서버입니다.

```python
# my_server.py
from mcp.server.fastmcp import FastMCP

server = FastMCP("my-first-server")

@server.tool()
def echo(message: str) -> str:
    """Echo the given message back."""
    return message

if __name__ == "__main__":
    server.run()
```

이 18줄짜리 코드만으로 MCP 서버가 완성됩니다. FastMCP는 함수 시그니처에서 inputSchema를 자동 생성하고, docstring을 description으로 가져옵니다. `server.run()`은 기본적으로 stdio transport로 실행되어 표준 입출력으로 메시지를 주고받습니다.

이번에는 파일 읽기 도구를 추가합니다. 보안을 위해 사용자가 명시한 디렉토리 안에서만 동작하도록 제한합니다.

```python
from pathlib import Path
from mcp.server.fastmcp import FastMCP

server = FastMCP("my-first-server")
ALLOWED_ROOT = Path("/Users/alice/scratch").resolve()

@server.tool()
def echo(message: str) -> str:
    """Echo the given message back."""
    return message

@server.tool()
def read_file(relative_path: str) -> str:
    """Read a text file inside the allowed root."""
    target = (ALLOWED_ROOT / relative_path).resolve()
    if ALLOWED_ROOT not in target.parents and target != ALLOWED_ROOT:
        raise ValueError("Path escapes the allowed root.")
    return target.read_text(encoding="utf-8")

if __name__ == "__main__":
    server.run()
```

여기서 두 가지가 추가되었습니다. 첫째, resolve()로 절대 경로를 만들고 ALLOWED_ROOT 아래인지 확인합니다. 둘째, 검사 실패 시 예외를 던집니다. FastMCP는 도구 함수가 예외를 던지면 그 메시지를 isError=true 콘텐츠로 변환해 클라이언트에게 돌려줍니다. 호스트와 모델은 결과를 받아 보고 다음 행동을 결정합니다.

서버를 만들었으면 호스트와 연결합니다. Claude Desktop은 macOS에서는 `~/Library/Application Support/Claude/claude_desktop_config.json`, Windows에서는 `%APPDATA%\Claude\claude_desktop_config.json` 경로의 설정 파일을 읽습니다. mcpServers 키에 서버를 등록합니다.

```json
{
  "mcpServers": {
    "my-first": {
      "command": "python",
      "args": ["/Users/alice/code/my_server.py"]
    }
  }
}
```

Claude Desktop을 재시작하면 사이드바에 my-first 서버가 나타나고, 도구 메뉴에 echo와 read_file이 노출됩니다. 채팅에서 "scratch 폴더의 hello.txt를 읽어 줘"라고 입력하면 모델이 read_file 도구를 호출하고, 결과가 다시 모델에게 돌아가 자연어로 응답이 만들어집니다. 첫 호출에서 Claude Desktop은 사용자에게 도구 사용 승인 다이얼로그를 띄웁니다. 이 흐름이 MCP에서 호스트가 정책 결정을 책임지는 모습입니다.

TypeScript로 만들고 싶다면 비슷한 구조로 작성할 수 있습니다.

```typescript
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";

const server = new Server({ name: "my-first-server", version: "0.1.0" }, {
  capabilities: { tools: {} },
});

server.setRequestHandler("tools/list", async () => ({
  tools: [{
    name: "echo",
    description: "Echo the given message back.",
    inputSchema: { type: "object", properties: { message: { type: "string" } }, required: ["message"] },
  }],
}));

server.setRequestHandler("tools/call", async (req) => {
  const { name, arguments: args } = req.params;
  if (name === "echo") {
    return { content: [{ type: "text", text: args.message }] };
  }
  throw new Error("Unknown tool");
});

await server.connect(new StdioServerTransport());
```

TypeScript SDK는 FastMCP에 해당하는 자동 등록 도구가 비교적 최근에 들어왔으므로, 위처럼 핸들러를 명시적으로 등록하는 형태도 흔히 볼 수 있습니다.

서버를 만들었으면 디버깅을 위한 도구도 알아 두는 것이 좋습니다. Anthropic이 제공하는 **MCP Inspector**는 서버를 실행하고 메시지를 한눈에 보여 주는 GUI입니다.

```
npx @modelcontextprotocol/inspector python my_server.py
```

Inspector는 브라우저를 열어 도구 목록, 자원 목록, 호출 로그를 보여 줍니다. 호스트와 연결하기 전에 서버가 정상 동작하는지 확인하기 좋습니다. 메시지 형식이 의심스러울 때 Inspector로 직접 메서드를 호출해 응답을 살펴보면 빠르게 문제를 좁힐 수 있습니다.

처음 만들어 본 서버에는 한 가지 함정이 있습니다. **로그 출력**입니다. stdio transport에서는 stdout이 통신 채널이기 때문에, print()를 쓰면 메시지가 깨집니다. 로그는 반드시 stderr로 보내야 합니다. Python에서는 `import sys; print("debug", file=sys.stderr)` 또는 `logging` 모듈의 기본 핸들러(stderr로 출력)를 사용합니다. 이 작은 차이가 처음 사용자가 가장 자주 마주치는 문제 중 하나입니다.

정리하면, 가장 단순한 MCP 서버는 FastMCP 데코레이터 몇 줄과 한 줄짜리 실행 코드로 시작할 수 있고, 호스트 설정 파일에 명령어를 등록하기만 하면 곧바로 사용 가능한 도구가 됩니다. stdout을 통신 채널로 쓰는 transport 특성, capability 자동 협상, 예외의 isError 변환 같은 SDK의 도움을 이해해 두면 이후 더 복잡한 서버를 만들 때도 막힘없이 진행할 수 있습니다.

다음 단원인 1.4.1에서는 이 모든 메시지가 어떤 규약 위에서 흐르고 있는지, JSON-RPC 2.0의 구조를 살펴봅니다.
