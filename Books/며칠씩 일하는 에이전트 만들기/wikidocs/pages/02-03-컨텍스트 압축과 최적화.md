# 2.3 컨텍스트 압축과 최적화

이 섹션의 토픽과 학습 목표는 다음과 같습니다.

- **2.3.1 컨텍스트 압축(compaction)의 원리** — 대화 기록을 요약·압축해 토큰을 회수하는 compaction의 원리를 설명할 수 있다
- **2.3.2 점진적·공격적 압축 전략** — 토큰 사용량이 커질수록 점점 더 공격적으로 압축을 적용하는 전략을 설명할 수 있다
- **2.3.3 컨텍스트 오프로딩과 외부화** — 컨텍스트를 외부 저장소로 내보내고 참조만 남기는 오프로딩을 설명할 수 있다
- **2.3.4 서브에이전트를 통한 컨텍스트 격리** — 작업을 서브에이전트로 분리해 각자의 컨텍스트를 격리하는 이점을 설명할 수 있다
- **2.3.5 GEPA와 프롬프트 자동 최적화** — GEPA 같은 기법이 프롬프트를 자동으로 진화·최적화하는 원리와 컨텍스트 엔지니어링과의 관계를 설명할 수 있다
- **2.3.6 컨텍스트 엔지니어링 종합 점검** — 수집·구조화·압축·격리·최적화를 하나의 컨텍스트 전략으로 통합해 설명할 수 있다
