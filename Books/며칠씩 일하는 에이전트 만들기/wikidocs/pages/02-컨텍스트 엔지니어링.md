# 2 컨텍스트 엔지니어링

컨텍스트 윈도우를 유한 자원으로 보고, 무엇을 어떻게 담고 압축할지 설계하여 긴 루프에서도 에이전트의 컨텍스트를 안정적으로 관리할 수 있다.

이 Phase는 다음 섹션으로 구성됩니다.

- 2.1 컨텍스트라는 유한 자원
  - 2.1.1 컨텍스트 윈도우와 토큰
  - 2.1.2 컨텍스트는 왜 유한 자원인가
  - 2.1.3 컨텍스트 부패(context rot)
  - 2.1.4 프롬프트 엔지니어링을 넘어서는 컨텍스트 엔지니어링
  - 2.1.5 토큰 예산 세우기
- 2.2 컨텍스트의 구성과 수집
  - 2.2.1 컨텍스트 윈도우의 구성요소
  - 2.2.2 적시(just-in-time) 컨텍스트 로딩
  - 2.2.3 검색 증강(RAG)과 컨텍스트
  - 2.2.4 도구 선택과 컨텍스트 부담
  - 2.2.5 구조화된 컨텍스트와 형식 설계
- 2.3 컨텍스트 압축과 최적화
  - 2.3.1 컨텍스트 압축(compaction)의 원리
  - 2.3.2 점진적·공격적 압축 전략
  - 2.3.3 컨텍스트 오프로딩과 외부화
  - 2.3.4 서브에이전트를 통한 컨텍스트 격리
  - 2.3.5 GEPA와 프롬프트 자동 최적화
  - 2.3.6 컨텍스트 엔지니어링 종합 점검
