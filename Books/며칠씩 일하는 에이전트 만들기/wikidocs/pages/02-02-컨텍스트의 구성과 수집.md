# 2.2 컨텍스트의 구성과 수집

이 섹션의 토픽과 학습 목표는 다음과 같습니다.

- **2.2.1 컨텍스트 윈도우의 구성요소** — 시스템 프롬프트·도구·대화 기록·검색 자료 등 컨텍스트를 채우는 요소를 분해할 수 있다
- **2.2.2 적시(just-in-time) 컨텍스트 로딩** — 미리 채우기와 적시 로딩의 차이를 설명하고 적시 로딩의 이점을 들 수 있다
- **2.2.3 검색 증강(RAG)과 컨텍스트** — 검색 증강 생성이 컨텍스트에 외부 지식을 주입하는 방식과 한계를 설명할 수 있다
- **2.2.4 도구 선택과 컨텍스트 부담** — 도구 수가 많아질 때 컨텍스트와 선택 정확도에 미치는 영향을 설명할 수 있다
- **2.2.5 구조화된 컨텍스트와 형식 설계** — 컨텍스트를 구조화(섹션·태그·우선순위)해 모델이 잘 활용하게 만드는 법을 설명할 수 있다
