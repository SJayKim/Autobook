# 3.2 레이어드 메모리의 네 계층

이 섹션의 토픽과 학습 목표는 다음과 같습니다.

- **3.2.1 레이어드 메모리 아키텍처 개관** — 메모리를 명시 규칙·자동 학습·온디맨드·서브에이전트 네 계층으로 나누는 이유와 전체 그림을 설명할 수 있다
- **3.2.2 명시적 규칙 계층 (CLAUDE.md)** — 사람이 직접 작성하는 명시적 규칙 계층(CLAUDE.md)의 역할과 특징을 설명할 수 있다
- **3.2.3 자동 학습 계층 (MEMORY.md)** — 에이전트가 스스로 축적하는 자동 학습 계층(MEMORY.md)의 동작을 설명할 수 있다
- **3.2.4 온디맨드 메모리 계층 (Memory Tool)** — 필요할 때 API로 읽고 쓰는 온디맨드 메모리(Memory Tool)의 동작을 설명할 수 있다
- **3.2.5 서브에이전트 메모리 계층** — 각 서브에이전트가 자신만의 영속 마크다운 지식 저장소를 갖는 구조를 설명할 수 있다
- **3.2.6 계층 간 상호작용과 승급** — 네 계층이 함께 동작하며 정보가 한 계층에서 다른 계층으로 승급되는 흐름을 설명할 수 있다
