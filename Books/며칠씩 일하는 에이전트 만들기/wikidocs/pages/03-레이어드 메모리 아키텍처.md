# 3 레이어드 메모리 아키텍처

세션을 넘어 지식을 영속화하는 레이어드 메모리(명시 규칙·자동 학습·온디맨드 API·서브에이전트 메모리)를 계층별로 설계하고 운영할 수 있다.

이 Phase는 다음 섹션으로 구성됩니다.

- 3.1 메모리가 필요한 이유
  - 3.1.1 모델은 기억하지 못한다
  - 3.1.2 컨텍스트와 메모리의 구분
  - 3.1.3 메모리가 장기 실행을 가능하게 하는 방식
- 3.2 레이어드 메모리의 네 계층
  - 3.2.1 레이어드 메모리 아키텍처 개관
  - 3.2.2 명시적 규칙 계층 (CLAUDE.md)
  - 3.2.3 자동 학습 계층 (MEMORY.md)
  - 3.2.4 온디맨드 메모리 계층 (Memory Tool)
  - 3.2.5 서브에이전트 메모리 계층
  - 3.2.6 계층 간 상호작용과 승급
- 3.3 메모리 운영 패턴
  - 3.3.1 메모리 쓰기: 무엇을 언제 저장할까
  - 3.3.2 메모리 읽기: 회상과 관련성
  - 3.3.3 메모리 구조화: 스키마와 메타데이터
  - 3.3.4 메모리 통합과 망각
  - 3.3.5 메모리 대 검색(RAG): 무엇을 언제
  - 3.3.6 메모리의 함정: 오염과 낡음
