# 3.3 메모리 운영 패턴

이 섹션의 토픽과 학습 목표는 다음과 같습니다.

- **3.3.1 메모리 쓰기: 무엇을 언제 저장할까** — 저장할 가치가 있는 정보와 그렇지 않은 정보를 구분하는 기준을 설명할 수 있다
- **3.3.2 메모리 읽기: 회상과 관련성** — 저장된 메모리에서 현재 작업에 관련된 것만 골라 회상(recall)하는 방식을 설명할 수 있다
- **3.3.3 메모리 구조화: 스키마와 메타데이터** — 메모리를 일관된 스키마(타입·설명·링크)로 구조화하는 법을 설명할 수 있다
- **3.3.4 메모리 통합과 망각** — 메모리를 통합(consolidation)하고 오래되거나 틀린 메모리를 정리(pruning)하는 필요성을 설명할 수 있다
- **3.3.5 메모리 대 검색(RAG): 무엇을 언제** — 영속 메모리와 RAG/벡터 검색의 역할 차이와 선택 기준을 설명할 수 있다
- **3.3.6 메모리의 함정: 오염과 낡음** — 메모리 오염(poisoning)·낡은 메모리·과신이 일으키는 위험과 대응을 설명할 수 있다
