# Phase 3 그라운딩 브리프 — 레이어드 메모리 아키텍처

## 핵심 프레이밍 (책의 2번 트렌드)
- 세션 단위가 아니라 영속적 지식 저장이 표준이 되고 있다. 메모리는 계층화된다.
- 네 계층(읽는 시점·주체·지속성이 다름):
  1. 명시적 규칙 계층 — CLAUDE.md: 사람이 직접 쓴 규칙. 항상 로드. 프로젝트 관례·금지사항·우선순위 지침.
  2. 자동 학습 계층 — MEMORY.md: 에이전트가 세션 중 스스로 학습한 것을 자동 기록. 인덱스를 통해 로드. 사실 단위 파일.
  3. 온디맨드 계층 — Memory Tool: 필요할 때 도구(API) 호출로 읽고 쓰는 메모리. 디렉터리 기반 저장. 평소엔 컨텍스트 밖에 두었다가 적시 로딩 → 컨텍스트 절약.
  4. 서브에이전트 메모리 계층: 각 서브에이전트가 자신만의 영속 마크다운 지식 저장소를 가짐. 역할 특화 지식.
- 이 책 자체가 사례다(메타 사례): CLAUDE.md(명시 규칙) + MEMORY.md(자동 메모리) + 메모리 디렉터리(파일별 1사실)로 운영되는 Autobook 프로젝트가 그 예. 단, 본문에서는 특정 제품/프로젝트 홍보 톤이 아니라 '일반화된 패턴'으로 설명.

## 메모리가 필요한 이유
- 모델은 stateless: 추론할 때마다 백지에서 시작한다. 세션(대화 한 번)이 끝나면 그 안의 기억은 사라진다. 컨텍스트는 그 세션 동안만 유지되는 일시적 작업기억일 뿐.
- 컨텍스트 vs 메모리: 컨텍스트 = 단기·휘발(작업기억), 메모리 = 장기·영속(장기기억). 인간 비유: 지금 머릿속에 떠올린 것(컨텍스트) vs 노트에 적어 둔 것(메모리). 메모리에서 필요한 것을 골라 컨텍스트로 적재(회상)한다.
- 메모리가 장기 실행을 떠받친다: 세션 재개, 학습 누적, 반복 실수 회피, 사용자 선호 유지. 며칠짜리 작업은 여러 세션·여러 압축을 거치므로 영속 메모리 없이는 연속성이 깨진다.

## 계층 간 상호작용·승급
- 읽기 우선순위: 명시 규칙(CLAUDE.md)이 가장 강한 지침. 자동 메모리·온디맨드는 보조.
- 승급: 일시 메모 → 영속 메모리로, 자동 학습한 사실이 반복 확인되면 → 명시 규칙으로 승격. 계층 간 충돌은 명시 규칙 우선으로 해소.
- 어디에 둘까: 항상 지켜야 할 규칙은 명시 계층, 세션 중 발견한 사실은 자동 계층, 가끔만 필요한 큰 지식은 온디맨드, 역할별 전문 지식은 서브에이전트 계층.

## 메모리 운영 패턴
- 쓰기: 저장 가치 판단이 핵심. 일반화 가능한 사실·사용자 선호·피드백은 저장. 코드/깃 히스토리/문서로 이미 알 수 있는 것은 저장하지 않는다(중복·노이즈). 비결정적·일회성 정보 제외.
- 읽기(회상): 인덱스/설명(description) 필드로 관련성 판단 후 필요한 것만 회상. 과다 회상은 컨텍스트 비용. 회상한 내용은 낡았을 수 있으니 검증 후 사용.
- 구조화: 프론트매터(name·description·type 등)로 스키마화. 메모리 타입 분류(예: 사용자/피드백/프로젝트/참조). 메모리 간 링크. 인덱스 파일 1줄 요약으로 회상 효율↑.
- 통합·망각: 중복은 통합(갱신 vs 신규 구분), 오래되거나 틀린 메모리는 정리(pruning). 망각은 결함이 아니라 필수 기능 — 낡은 사실이 쌓이면 메모리 부패.
- 메모리 vs RAG: 메모리 = 에이전트가 '학습해 적어 둔 사실'(쓰기 주체가 에이전트/사용자). RAG = '외부 문서를 검색'(읽기 위주, 원본은 외부). 결합 가능: 메모리로 선호·요약을, RAG로 대량 원문을. 선택 기준: 지속·개인화는 메모리, 대량·최신 원문은 RAG.
- 함정: 잘못된 메모리가 전파되면 계속 틀린 판단. 메모리 오염(poisoning) — 악의적/오류 정보 주입. 신뢰 경계 설정, 검증 후 사용, 출처·시점 기록.

## 집필 주의
- '컨텍스트와 메모리 구분'(3.1.2)은 Phase 2의 컨텍스트 개념(2.1.1)을 전제로 자연스럽게 잇는다.
- Memory Tool(3.2.4)은 적시 로딩(2.2.2)과 직접 연결. 서브에이전트 메모리(3.2.5)는 컨텍스트 격리(2.3.4)와 연결.
- 특정 벤더 API 시그니처를 단정적으로 쓰지 말 것. '도구 호출로 디렉터리 기반 저장에 읽고 쓴다' 수준의 일반화로.
