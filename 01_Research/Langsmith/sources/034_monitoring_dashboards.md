# LangSmith — Monitor Projects with Dashboards (Prebuilt & Custom)

## 대시보드 개요

LangSmith는 두 가지 대시보드 유형을 제공합니다.

**프리빌트 대시보드(Prebuilt Dashboard)**: 추적 프로젝트마다 자동으로 생성되며, 사전 정의된 섹션으로 필수 메트릭을 즉시 확인할 수 있습니다.

**커스텀 대시보드(Custom Dashboard)**: 사용자 요구에 맞게 설정 가능한 차트 모음으로, 원하는 메트릭과 필터를 자유롭게 조합합니다.

## 프리빌트 대시보드 섹션

| 섹션 | 주요 측정 항목 |
|------|--------------|
| **Traces** | 추적 횟수, 지연시간(Latency), 오류율(Error Rate) |
| **LLM Calls** | LLM 호출 횟수 및 호출별 지연시간 |
| **Cost & Tokens** | 토큰 유형별(입력/출력) 총 비용 및 토큰 수 |
| **Tools** | 도구별 실행 횟수, 오류율, 지연 통계 (상위 5개) |
| **Run Types** | 루트 실행의 직접 자식 실행 분석 (상위 5개) |
| **Feedback Scores** | 상위 5개 피드백 유형의 집계 통계 |

## Latency 지표 — P50/P99

지연시간 백분위 메트릭은 성능의 전형적 수준과 최악의 경우를 모두 측정합니다.

- **P50**: 중앙값 지연시간 — 전형적인 응답 시간 (예: 2.75초)
- **P99**: 99번째 백분위 지연시간 — 가장 느린 1%의 응답 (예: 3.82초)

P99 지연시간이 상승하면 다운스트림 API 문제나 비효율적인 프롬프트 구조가 원인일 수 있습니다.

## 커스텀 대시보드 구성

### 차트 생성 단계

1. 추적 프로젝트 선택 및 실행 필터링
2. 드롭다운 메뉴에서 메트릭 선택 (Y축 설정)
3. 데이터 분할 방식 결정
4. 차트 유형 선택 (라인 또는 막대)

### 데이터 분할 방식

**Group By**: 실행 태그, 메타데이터, 실행 이름 또는 유형으로 자동 분할. 상위 5개 그룹으로 제한됩니다.

**Data Series**: 개별 필터로 수동 정의하여 세밀한 비교가 가능합니다. 예를 들어 모델 버전별 또는 환경별 비교 차트를 직접 구성할 수 있습니다.

### 메타데이터 활용

태그 또는 메타데이터로 데이터를 애플리케이션에 중요한 속성으로 분할할 수 있습니다. 예를 들어 `user_tier: premium` 메타데이터가 있는 트레이스만 별도 시리즈로 표시하는 식입니다.

## 차트 관리 기능

- 저장, 편집, 삭제, 복제 기능 제공
- 추적 프로젝트에서 기본 대시보드로 설정 가능

## 토큰 사용량 모니터링

통계 패널에는 전체 토큰 사용량과 실행당 중앙값 토큰 수가 표시됩니다. 토큰 소비를 모니터링하면 예상치 못한 비용 급증을 방지하고, 더 효율적인 소형 모델로의 전환 여부를 판단하는 데 도움이 됩니다.

## 오류율(Error Rate) 모니터링

오류율은 낮을수록 모델 신뢰성이 높음을 나타냅니다. 대시보드에서 시간 경과에 따른 오류율 추이를 확인하여 배포 후 회귀를 조기에 발견할 수 있습니다.
