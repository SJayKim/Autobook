# From Traces to Insights: Understanding Agent Behavior at Scale

## 핵심 문제 인식

프로덕션 환경의 에이전트는 매일 수천 개의 트레이스를 생성하지만, 하루에 100,000개의 트레이스를 기록하는 팀들이 이를 전혀 활용하지 못하는 경우가 많습니다(Dev Shah). 수동 검토는 이 규모에서 확장되지 않습니다.

## 에이전트의 고유 특성

기존 소프트웨어와 달리 에이전트는 세 가지 특성 때문에 일반적인 모니터링 방식이 통하지 않습니다.

| 특성 | 일반 소프트웨어 | 에이전트 |
|------|----------------|---------|
| 결정론적 동작 | 예 | 아니오 |
| 소규모 입력 변화에 견고함 | 예 | 아니오 |
| 제한된 입력 공간 | 예 | 아니오 |

에이전트는 비결정론적이고 무제한적인 입력을 받기 때문에 배포 전에 동작을 완전히 예측할 수 없습니다.

## Insights Agent의 차별점

### 기존 분석 도구의 한계

기존 제품 분석(클릭, 페이지뷰, 전환율 등)은 "무엇이 변했는지"는 보여주지만 "왜 변했는지"는 보여주지 않습니다. Insights Agent는 비정형 대화 데이터를 분석하여 원인을 규명합니다.

### 클러스터링 기반 패턴 발견

미리 정의하지 않은 패턴을 자동으로 발견합니다. 다층 계층 구조로 조직됩니다.
- 최상위 클러스터 (예: "코딩 도움말", "데이터 분석", "문서 작성")
- 각 클러스터의 세부 그룹화
- 개별 실행 항목

### 실패 모드 발견

부정적 피드백을 받은 트레이스만 대상으로 분석을 실행하면 에이전트가 공통적으로 실패하는 패턴을 자동으로 식별합니다. 예: "외부 API 응답 지연 시 오류 처리 미흡", "모호한 사용자 요청에 대한 명확화 질문 부재" 등

## 필터링 및 집계

- 부정적 피드백 등 특정 하위 집합에 대해서만 분석 실행 가능
- 즉시 계산된 속성으로 필터링 (예: "사용자가 답답함을 표현")
- 클러스터별 통계 집계로 이상치 탐지

## 자연어 인터페이스

핵심 혁신은 대화형 인터페이스입니다. 사용자가 분석 목표를 자연어로 설명하면 시스템이 이를 커스텀 분석 워크플로우로 자동 변환합니다. 예: "사용자가 가장 자주 실패하는 작업 유형을 찾아줘"
