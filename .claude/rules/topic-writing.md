# 단원 파일 편집 리마인더

교재 .md 파일을 작성/수정할 때 반드시 준수할 사항:

1. **톤앤매너** (`Rules/3. 책 작성 톤앤 매너.md`):
   - 합니다체 통일, 서술형 문단 기본
   - 직선 따옴표만 (' "), 곡선 따옴표 금지
   - AI체/대화체/과장 금지
   - 출처/원서 표기 금지
   - 새 용어마다 배경부터 설명 (전문가 전제 금지)

2. **문서 구조** (`Rules/4. 세부 개념 문서 작성 규칙.md`):
   - 최상단 `# a.b.c 제목` 하나만 사용
   - `##` 이하 헤더 사용 금지
   - 마무리에 "정리하면," 요약 포함
   - 다음 단원 안내 포함 (마지막 토픽 제외)

3. **내용 커버리지**:
   - curriculum.json의 learning_content 전 항목을 본문에 반영
   - learning_objectives를 본문만으로 검증 가능해야 함
   - 선수 범위 밖 개념을 끌어오지 않음

4. **초보자 친화성**:
   - 한 문단에 한 개념, 새 굵은 용어 3개 이상 동시 도입 금지
   - 추상 정의 전후 2문단 이내에 구체 사례(코드, 시나리오, 수치) 배치
   - 3단계 이상 프로세스에 시각 보조(ASCII 다이어그램, 표) 필수
   - 번역 투("~에 의해", "~것이 가능하다"), 명사화 남용("처리를 수행") 지양
   - 장문(80자 초과) 지양, 의식적으로 분리

5. **파일 경로**: `교재/{책이름}/wikidocs/pages/{pp}-{ss}-{tt}-{토픽}.md`
   - pp = phase id를 2자리 zero-pad (예: "1" → "01")
   - ss = section id의 두 번째 숫자를 2자리 zero-pad (예: "1.2" → "02")
   - tt = topic id의 세 번째 숫자를 2자리 zero-pad (예: "1.2.3" → "03")
