# 단원 리뷰

대상: **$ARGUMENTS** (토픽 ID `a.b.c` 또는 파일 경로)

## 리뷰 절차

### 1단계: 파일 및 커리큘럼 로드

1. `$ARGUMENTS`가 토픽 ID이면: curriculum.json에서 해당 토픽 정보를 찾고, `wikidocs/pages/{pp}-{ss}-{tt}-*` 패턴으로 대응하는 .md 파일을 탐색.
2. `$ARGUMENTS`가 파일 경로이면: 해당 파일을 읽고, 파일명에서 토픽 ID를 추출하여 curriculum.json에서 토픽 정보를 찾음.
3. 토픽의 `learning_objectives`, `learning_content`, `prerequisites`를 확보.

### 2단계: 10항목 체크리스트 점검

`Rules/4. 세부 개념 문서 작성 규칙.md` §7 기준 + 초보자 친화성 3항목으로 점검한다.

**1. 경로/파일명 대응**
- 파일명이 `{pp}-{ss}-{tt}-{토픽제목}.md` 형식인가? (pp=phase 2자리, ss=섹션 2자리, tt=토픽 2자리)
- 파일이 `wikidocs/pages/` 디렉토리에 있는가?
- 토픽 제목이 curriculum.json의 title과 대응하는가?

**2. learning_content 반영**
- curriculum.json의 learning_content 각 항목(키워드/개념)이 본문에서 다뤄지는가?
- 빠진 항목이 있으면 구체적으로 나열.

**3. learning_objectives 검증 가능성**
- 본문만 읽고 learning_objectives의 각 문장을 충족할 수 있는가?
- 목표 달성에 필요한 설명이 본문에 누락되지 않았는가?

**4. 선수 범위 준수**
- 본문에서 사용하는 개념이 해당 토픽의 prerequisites에 포함된 토픽 또는 본 토픽의 learning_content 범위 안에 있는가?
- 아직 소개하지 않은 개념을 예시/체크포인트에 넣지 않았는가?

**5. 톤앤매너 (`Rules/3. 책 작성 톤앤 매너.md`)**
- 합니다체 통일 여부. (해요체, 다체, 반말 섞임 없는가?)
- 직선 따옴표만 사용했는가? (곡선 ' ' " ", 전각, 화살괄호 검출)
- AI체 패턴 없는가? (과한 비유, 짧은 문장만 나열, 목록 위주 본문)
- 대화체 없는가? ("해 봅시다", "여러분" 남발)
- 출처/원서/원문 인용 표기 없는가?

**6. 배경 우선 설명 (§14)**
- 새 용어/전제가 등장할 때 전문가 가정 없이 배경부터 풀었는가?
- 정의 한 줄만 달고 넘어가는 경우가 없는가?

**7. 본문 구조**
- 최상단 `# a.b.c 제목`이 있는가?
- `##` 이하 헤더가 본문에 없는가?
- 마무리에 "정리하면," 요약이 있는가?
- 다음 단원 안내가 있는가? (마지막 토픽이 아닌 경우)

**8. 인지 부하 관리**
- 8a. 한 문단에 무관한 두 개념이 섞여 있지 않은가?
- 8b. 한 문단에 새 굵은 용어(`**term**`)가 3개 이상 충분한 설명 없이 등장하지 않는가?
- 8c. 5줄 이상 문단이 learning_content 어떤 항목과도 무관하고 구조적 역할(도입/전환/정리)도 아닌 경우가 없는가?

**9. 구체-추상 순서와 시각 보조**
- 9a. 추상적 정의/메커니즘에 구체 사례(코드, 시나리오, 수치)가 2문단 이내에 동반되는가?
- 9b. 3단계 이상 프로세스 서술 시 시각 보조(ASCII 다이어그램, 표, 코드블록)가 최소 1개 있는가?

**10. 한국어 문장 품질**
- 10a. 번역 투("~에 의해 수행되는", "~하는 것이 가능하다", "~에 대한 ~에 대한")가 3건 이상이 아닌가?
- 10b. 명사화 남용("처리를 수행한다", "확인을 진행한다")이 3건 이상이 아닌가?
- 10c. 마침표 없이 80자 초과 문장이 5건 이상이 아닌가?
